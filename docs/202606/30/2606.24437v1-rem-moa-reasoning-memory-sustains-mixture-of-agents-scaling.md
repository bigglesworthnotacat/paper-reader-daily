## 摘要
混合专家（MoA）架构通过将多个大语言模型智能体组织成层级推理流水线来改进推理时扩展。然而，现有的MoA变体在深度增加时无法维持性能提升，出现退化、早期平台期或饱和现象。我们提出ReM-MoA，一种记忆增强的MoA框架，通过两种机制维持扩展：（1）排名推理记忆，使用比较性审阅智能体持久存储和排序所有层的推理轨迹；（2）策展多样化记忆路由方案，将不同智能体暴露于成功与失败轨迹的不同组合中，在传播高质量推理的同时保持探索多样性。我们进一步引入可选的多领域审阅蒸馏流水线，通过前沿模型监督提升排序质量。在涵盖数学、形式逻辑、代码、知识和常识的五项推理基准测试中，ReM-MoA在深度和宽度扩展上均始终优于先前的MoA变体，并且其优势随深度增加而扩大，确立了结构化跨层推理记忆作为可扩展多智能体推理的关键缺失机制。

## Abstract
Mixture-of-Agents (MoA) architectures improve inference-time scaling by organizing multiple LLM agents into layered reasoning pipelines. However, existing MoA variants fail to sustain gains as depth increases, exhibiting degradation, early plateauing, or saturation. We propose ReM-MoA, a memory-augmented MoA framework that sustains scaling through two mechanisms: (1) a Ranked Reasoning Memory that persistently stores and ranks reasoning traces from all layers using a comparative Reviewer Agent, and (2) a Curated Diversified Memory Routing scheme that exposes different agents to distinct combinations of successful and failed traces, preserving exploration diversity while propagating high-quality reasoning. We further introduce an optional multi-domain Reviewer distillation pipeline that improves ranking quality through frontier-model supervision. Across five reasoning benchmarks spanning math, formal logic, code, knowledge, and commonsense, ReM-MoA consistently outperforms prior MoA variants across both depth and width scaling, and its advantage widens with depth, establishing structured cross-layer reasoning memory as a key missing mechanism for scalable multi-agent inference.

## 论文总结：ReM-MoA: 推理记忆维持混合专家规模扩展

### 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有的多智能体混合专家（Mixture-of-Agents, MoA）架构在增加推理层数（深度）和每层智能体数量（宽度）时，其性能提升会失效。
- **具体表现**：三种典型的失败模式：
    - **退化**：标准MoA架构性能在浅层达到峰值后显著下降。
    - **早期平台期**：如RMoA模型，性能在少量层数后便停滞不前。
    - **饱和**：如AttentionMoA模型，性能提升幅度随深度增加而迅速减小。
- **根本原因**：
    - 层间简单的信息拼接会传播累积的错误。
    - 缺乏有效的跨层记忆机制，导致早期产生的高质量推理轨迹无法被后续层有效利用。
    - 随着上下文积累，智能体之间的探索多样性迅速丧失，产生冗余的解空间。
- **研究意义**：本文的核心动机是解决MoA架构在扩展到更深层次时面临的性能瓶颈，提出简单连接或单层交互之外的第三种关键机制——结构化的跨层推理记忆，以实现真正可扩展的多智能体协同推理。

### 2. 方法论：核心思想与关键技术细节

**核心思想**：将MoA的扩展问题视为一个**质量-多样性联合保持问题**，通过引入结构化的跨层推理记忆来解决。

**关键技术细节**：

1.  **排序推理记忆（Ranked Reasoning Memory）**：
    - **存储**：在每一层，所有`N`个智能体生成的推理轨迹（`rl,j`）都被存入一个统一的、可跨层访问的共享记忆`M`中。
    - **排序**：每个轨迹并非简单存储，而是由一个专门的**审阅智能体（Reviewer Agent）** 在层内对所有`N`个轨迹进行比较性评估。审阅者会为每个轨迹分配一个整数分数（1-10分）和一个针对性的解释性理由。
    - **作用**：确保了所有层产生的高质量或低质量推理轨迹都能以结构化的、带有质量标签的形式被保存下来，供后续任何层访问，实现了**质量保持**。

2.  **策展多样化记忆路由（Curated Diversified Memory Routing）**：
    - **策展参考集**：从下一层开始，每个智能体在生成新轨迹时，会从记忆`M`中获得一个经过策展的参考轨迹集`Sl,j`。这个集合不是简单的全部历史，而是基于质量有意选择的组合。
    - **多样化路由函数**：在同一层内，不同的智能体接收**不同**的参考集。这些参考集来源于三种类型的路由函数：
        - **对比式**：同时包含最高分和最低分轨迹，提供正反例信号。
        - **正向强化**：仅包含最高分轨迹，强调成功模式。
        - **失败模式聚焦**：仅包含最低分轨迹，揭示常见陷阱。
    - **作用**：通过向不同智能体提供差异化的质量感知参考，既传播了高质量推理（**质量保持**），又避免了所有智能体因接收完全相同的历史信息而趋同，从而保持了探索多样性（**多样性保持**）。

3.  **可选增强：审阅蒸馏（Reviewer Distillation）**：
    - **目的**：进一步提升审阅智能体的排序质量。
    - **方法**：使用一个更强的教师模型（如GPT-5.5）对ReM-MoA在运行过程中产生的（问题，轨迹）对生成高质量的评分和理由。然后，通过LoRA（低秩适配）参数高效微调，将教师模型的能力蒸馏到一个较小的开源学生模型（同为Qwen2.5-7B-Instruct）上。
    - **优势**：蒸馏出的审阅者学会了判断模型实际产生的轨迹分布，并且因为训练数据覆盖多领域，能够跨域泛化。

1.  **ReM-MoA实现了持续扩展**：在**所有**测试的深度和宽度配置下，ReM-MoA（尤其是带蒸馏的ReM-MoA\*）的性能均**始终优于**所有基线模型。

1.  **宽度探索有限**：宽度扩展实验最多只到每层8个智能体，计算约束阻碍了对更大宽度可能性的探索。

4.  **继承偏差**：评审者本身也是LLM，可能带有已知的评估者偏差（如对自身输出的偏好、位置偏差等）。虽然对比评分和对比路由试图减轻，但无法完全消除。提案者也可能生成错误或有偏见的推理，框架会继承这些偏差。
5.  **计算成本**：每层需调用`L*N + L + 1`次LLM，对于高度依赖成本的部署场景，这可能是一个制约因素。
