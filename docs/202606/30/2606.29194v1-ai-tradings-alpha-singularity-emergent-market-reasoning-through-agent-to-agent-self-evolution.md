## 摘要
自动化Alpha挖掘保持评分函数固定，并在此基础上变化搜索算法。针对固定评分器收敛的搜索会过拟合评分器无法惩罚的任何因素，这是样本外泛化差距的主要原因。我们将评分函数视为与Alpha因子并列的搜索制品，并研究什么条件使这种联合搜索可行。密封联合搜索（SJS）是一个框架：它在一套自主发现系统中对信息流施加一组结构条件，防止联合搜索崩溃为自我确认，同时保持评估器密封。条件涵盖角色分解、类型化角色间通信、来源密封读取、版本化存储和基础局部提升。Agora对SJS进行了实证测试：五个LLM智能体类通过三个通道通信，演变出八个技能库，Alpha库基于AlphaGen算子构建。三个评估器编写报告并汇总为一份简报，保留分歧而非投票。我们在CSI 1000上运行Agora 100轮，并在一个从所有LLM输入密封的91天2026年保留集上评估。Agora实现保留Sharpe +1.87；最佳基线在有利种子下为+1.334，跨种子均值为-0.755。将Agora的两个指标预加载到冻结库消融实验中仅恢复+2.25 Sharpe差距中的+0.40，而添加没有库演进的PPO则加剧了差距。这两个指标是涌现而非设计的。注意事项：单种子运行，空头集中信号，旨在多空。

## Abstract
Automated alpha mining holds the scoring function fixed and varies the search algorithm over it. A search that converges against a fixed scorer overfits whatever the scorer cannot penalize, a primary cause of the out-of-sample generalization gap. We treat the scoring function as a search artifact alongside the alpha factors and study what conditions make this joint search admissible. Sealed Joint Search (SJS) is a framework: a set of structural conditions on information flow in an autonomous-discovery system that prevent joint search from collapsing into self-confirmation while keeping the evaluator sealed. Conditions cover role decomposition, typed inter-role communication, provenance-sealed reads, versioned stores, and substrate-local promotion. Agora tests SJS empirically: five LLM agent classes communicate via three channels, evolving eight skill libraries, with alpha libraries built on AlphaGen operators. Three evaluators write reports aggregated into one brief, carrying forward disagreement instead of voting. We run Agora for 100 rounds on CSI 1000 and evaluate on a 91-day 2026 holdout sealed from all LLM inputs. Agora achieves holdout Sharpe +1.87; best baseline +1.334 at favorable seed and -0.755 cross-seed mean. Pre-loading Agora's two metrics into a frozen-library ablation recovers only +0.40 of the +2.25 Sharpe gap, and adding PPO without library evolution worsens the gap. The two metrics emerge rather than being designed. Caveats: single-seed run, short-side concentrated signal, intended for long-short.

### 1. 论文的核心问题与整体含义（研究动机和背景）
传统自动alpha挖掘方法存在根本性缺陷：它们固定评分函数（如IC、Sharpe等），只改变搜索算法。这样，搜索算法会过拟合评分函数无法惩罚的任何模式，导致严重的样本外泛化差距（out-of-sample generalization gap）。论文认为，要突破这一瓶颈，必须将评分函数本身也视为可动态演进的搜索工件，即“联合搜索”（joint search over signals and scoring functions）。然而，联合搜索容易崩溃为自我确认（self-confirmation）——搜索算法可以同时操纵信号和评分标准，从而“骗过”自己。为此，论文提出**Sealed Joint Search (SJS)**框架，通过设计信息拓扑的结构性条件来防止这种崩溃，实现真正自主涌现的市场推理能力。

### 2. 论文提出的方法论：核心思想、关键技术细节
**核心思想**：将评分函数视为与alpha因子并列的搜索工件，二者在密封（sealed）的信息拓扑下联合优化，防止联合搜索退化。SJS不是具体算法，而是一组信息流约束条件（F1～F5）加上边界条件，确保外部评估器在运行前固定且不可访问。

**五个结构性条件**：
- **F1 分解提案（Decomposed Proposal）**：提案者和评估者必须分离角色，不共享状态。
- **F2 类型化通信（Typed Inter-role Communication）**：角色间只能通过类型化通道通信，禁止自由聊天。
- **F3 来源密封读取（Provenance-Sealed Reads）**：每条记录标记数据段来源，不同角色只能按权限读取授权范围内的记录；保持段（holdout）对提案路径完全不可见。
- **F4 持久版本化技能库（Persistent Versioned Artifact Stores）**：每个技能库（operator, metric, topic等）由唯一角色拥有，状态机 transition（内建→试验→接受→拒绝）。
- **F5 基础局部提升规则（Substrate-Local Promotion Rule）**：技能库状态转换由基础层根据内部产出自动计算，不依赖外部评估器，且基于训练段Sharpe的相关系数（|ρ|>0.3提升，<0.05降级）。

**实例化系统Agora**：
- 五个LLM角色（研究报告、alpha挖掘者、评估挖掘者、两个评估员面板各3个实例），共9个独立LLM客户端（claude-sonnet-4-6）。
- 三通道通信：A（定向建议）、B（跨轮简报，有容量限制）、C（LLM Wiki，来源过滤）。
- 八个技能库：正alpha侧4个（operator, RL-network, RL-algorithm, reward）基于AlphaGen；另一侧包括metric, topic, rubric, meta-rubric。metric库初始仅4个内建指标，进化集中于此。
- 代码提案必须通过静态限制、动态沙箱测试和RL-algorithm的烟雾训练。
- 每一轮执行固定序列：研究报告→alpha候选→alpha评估员（训练段评分）→评价挖掘者提出metric动作→测试段评分→因子评估员写报告→提交更新。

### 3. 实验设计：数据集、基准、对比方法
**数据集**：CSI 1000（中国A股动态面板），来源RiceQuant，2014-10至2026-05。分为训练段（2014-10至2019-12，1277个交易日）、测试段（2020-01至2025-12，1461个交易日）、保持段（2026-01至2026-05，91个交易日）。保持段对所有LLM输入完全密封。

**评估**：统一外部回测（10分位排序，5天再平衡，9bps单边成本）。每个方法选训练段IC前30个alpha，等权Z-score组合，报告组合Sharpe等指标。

**对比基准（7个）**：
- **B1**：遗传编程（gplearn，改编自AlphaGen参考）
- **B2**：AlphaGen纯PPO（Maskable PPO，500,000步）
- **B3**：单LLM一次性生成50个因子
- **B4**：单LLM迭代10轮，每轮反馈top-3/bottom-3
- **B5**：WorldQuant Alpha101（前30个OHLCV子集）
- **B6**：冻结库消融（Agora的结构但库进化被禁用，5轮）

**内部消融**：B6+增强内建指标（预加载两个Agora发现的metric）；B6+PPO中继（冻结库但启用PPO训练）；以及20轮B6+增强。

### 6. 论文的主要结论与发现
- Agora在完全密封的91天保持段上取得组合Sharpe +1.87，最佳基线AlphaGen-PPO（单种子+1.334，跨种子均值-0.755）。
- 绝大多数基线在保持段上表现不佳，甚至为负，凸显固定评分函数+搜索的脆弱性。
- 冻结库消融（+0.019 Sharpe）与Agora的差距+2.25，其中静态预加载两个metric仅贡献+0.40，剩下+1.85归因于metric库的持续进化及其与PPO中继的交互。
- 两个被提升的metric（单调性得分v1、超额回撤惩罚v1）是涌现产物：非单agent设计，而是通过多轮积累证据由基础层自动决定提升。
- PPO中继在无库进化时反而有害（-0.80），说明搜索与评分不对称升级会恶化泛化。

### 7. 优点
- **方法论创新**：首次系统化提出“搜索信息拓扑”的概念，将评分函数与信号联合搜索置于密封约束下，为自主发现提供新范式。
- **实验设计严谨**：多维度消融、滚动窗口、成本敏感性、统计检验都涵盖；仔细密封了所有泄露路径（5条路径审计）。
- **涌现证据**：展示了LLM多agent系统可以涌现出标准量化指标，而无需人工设计或单agent预定义目标。
- **开源承诺**：代码、基准配置、审计日志、快照将全部开源，促进可复现性。
- **问题意识先进**：指出固定评分函数的根本局限，并提供了理论框架+实证解决方案。

### 8. 不足与局限
- **单种子风险**：Agora仅运行一次，种子方差未知，而B2跨种子变异性极大（4.4 Sharpe swing）。结论可能不稳健。
- **消融轮次不对等**：B6及其变体仅5轮，Agora 100轮，部分+1.85差距可能来自轮次而非进化。20轮B6+aug结果提示轮次数不能解释全部差距，但仍需对齐轮次。
- **测试段信息泄露**：评价挖掘者通过4分位数分类反馈借鉴了测试段信息，并非完全密封；仅保持段严格OOS。滚动窗口2020-2025同样有软泄露。
- **统计显著性有限**：91天窗口下NW HAC检验多数对比未达显著（仅对B1、B7显著），置信区间宽。需要更长保持段或更多种子。
- **F1（角色分解）未直接消融**：缺少单Agent+进化的消融基线，无法证明分解的必要性。
- **泛化场景局限**：仅在中国A股CSI 1000上测试，未在其他市场（如美股）或非金融领域（如定理证明、编程）验证SJS框架的可迁移性。
- **进化范围窄**：8个库中仅metric库有实质性进化，其他7个保持内建，可能未充分利用LLM能力。
- **部署限制**：信号空头集中，长期多端收益弱于基准，建议仅多空或多空增强策略，限制实际应用场景。
