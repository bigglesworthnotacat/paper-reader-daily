## 摘要
LLM智能体在数字环境中遵循实用的执行循环：它们对结构化状态进行推理，调用工具，检查反馈，并修正动作。将这个循环扩展到物理机器人是困难的，因为物理执行是连续的、依赖于具体具身、具有不确定性，并受到安全限制。现有的具身AI系统在操作、空间理解、导航和类人控制方面取得了进展，但这些能力通常仍然是专门的模块或松耦合的决策循环。在这项工作中，我们介绍了HoloAgent-0，一个用于真实世界机器人部署的统一具身智能体框架。Embodied AgentOS将语言指令转换为可执行的技能图，调度机器人资源，监控执行，并根据运行时反馈触发澄清或重新规划。HoloAgent-0通过三个耦合层组织异构的机器人模型和控制器：用于闭环执行的Embodied AgentOS、用于物理世界定位的3D空间记忆以及用于机器人动作的具身技能。我们在真实硬件上部署了HoloAgent-0，并评估了其空间记忆、长视野导航以及跨运动生成、物体搜索、跨机器人协调和移动操作的闭环执行。

## Abstract
LLM agents follow a practical execution loop in digital environments: they reason over structured states, invoke tools, inspect feedback, and revise actions. Extending this loop to physical robots is difficult because physical execution is continuous, embodiment-dependent, uncertain, and constrained by safety. Existing embodied-AI systems have advanced manipulation, spatial understanding, navigation, and humanoid control, but these capabilities often remain specialized modules or loosely coupled decision loops. In this work, we introduce HoloAgent-0, a unified embodied agent framework for real-world robot deployment. Embodied AgentOS converts language instructions into executable skill graphs, schedules robot resources, monitors execution, and triggers clarification or re-planning from runtime feedback. HoloAgent-0 organizes heterogeneous robot models and controllers through three coupled layers: Embodied AgentOS for closed-loop execution, 3D spatial memory for physical world grounding, and embodied skills for robot action. We deploy HoloAgent-0 on real hardware and evaluate its spatial memory, long-horizon navigation, and closed-loop execution across motion generation, object search, cross-robot coordination, and mobile manipulation.

## 二、论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 将长期任务执行视为**系统组织问题**，提出**Embodied AgentOS**作为闭环运行时，将自然语言指令转化为可执行技能图，调度资源、监视执行、利用运行时反馈进行重新规划。
- 框架由三层耦合组成：
  1. **Embodied AgentOS层**：负责任务规划、调度、监控和失败恢复。
  2. **记忆层（Memory Layer）**：提供持久空间定位（spatial grounding）和时间执行历史。
  3. **技能层（Skill Layer）**：将机器人能力暴露为类型化、可监控的技能调用。

### 关键技术细节

#### 1. 设计原则
- **闭环优先**：重复的观察-检索-行动-监控循环。
- **记忆中心**：持久空间和时间记忆作为规划的主要上下文。
- **类型化技能接口**：机器人能力通过命令参数和状态事件暴露，分离高层推理和底层控制。
- **默认可观察**：记录命令/状态事件、检索的记忆、状态转换和技能结果，支持调试和验证。

#### 2. AgentOS层
- 将用户指令解析为技能图，包含节点顺序、前置条件和恢复依赖。
- 调度器分发技能调用，消费状态反馈，触发澄清或重新规划。

#### 3. 技能层
- **类型化技能调用**：每个技能声明命令名称、类型参数、前置条件、目标引用、预期效果。例如 `move_to(room=kitchen)`。
- **运行时状态接口**：后端发布执行跟踪（进度、成功/失败模式、置信度、延迟、可恢复性）。
- **具体技能家族**：
  - **交互与感知技能**：语音（ASR/TTS）、开集检测、定位、验证。
  - **导航探索（HoloNavi）**：支持语言目标、物体目标、边界点目标。内部包含层次化物体导航、在线验证循环（快速特征匹配 + 慢速VLM验证）、主动空间探索。
  - **操作（HoloBrain）**：VLA后端，执行 pick/place/open 等，报告物体未找到、抓取失败等状态。
  - **全身运动（HoloMotion）**：支持参考跟踪、速度控制、姿态调整、恢复技能，提供平衡状态、接触风险等信息。
  - **跨具身协调**：通过共享记忆和状态事件，不同机器人可协作完成同一任务。

#### 4. 记忆层
- **空间记忆**：
  - **几何记忆**：坐标帧、机器人位姿、稠密几何、可通行性。支持LiDAR后端（FAST-LIVO）和纯视觉后端（GeoFlow-SLAM++，多相机扩展）。

  - **层次化多模态场景图（HMSG）**：楼层、房间、视角、物体四层。通过层次边（空间包含）和拓扑边（视角连接、物体可见性）组织。用于粗到细的检索和VLM验证。
- **时间记忆**：
  - 目标和计划状态：解析的用户意图、活跃技能图、待决子目标。
  - 执行和恢复轨迹：每个技能调用的参数、上下文、状态、验证结果、失败模式、重试。
  - 结果和经验摘要：任务段结束时存储最终状态、恢复策略等。

#### 5. 记忆更新
- 三种事件触发更新：新的传感器观察（冲突时刷新几何/语义）、技能结果（标记物体移动/缺失）、用户显式反馈。

## 五、论文的主要结论与发现

- HoloAgent-Nav在模拟导航中达到**82.6% SR**和**42.8% SPL**，超越所有对比方法（如MSGNav 74.1%/33.4%，FSR-VLN慢推理 80.8%/41.0%），表明反馈驱动的执行提升了路径效率和成功率。

## 六、优点

1. **统一的闭环框架**：将空间记忆、技能调度、监控和恢复集成在一个运行时中，使物理技能可组合、可观察、可验证。

3. **持久3D空间记忆**：结合几何、语义、层次化场景图，支持语言查询和粗到细检索，对环境变化具有适应性（动态更新）。
4. **跨平台兼容性**：已部署在两种人形机器人、一种轮式机器人上，并通过共享接口支持跨机器人协调。
5. **性能领先**：在导航基准上取得最优结果，真实机器人表现极佳（>97%成功率）。
