## 摘要
大型语言模型（LLM）的快速集成推动了多智能体系统（MAS）的发展，其中专门的智能体协作执行复杂工作流。在这些环境中，有效的编排需要鲁棒的路由机制，以将任务高效分配给最合适的智能体。然而，现有路由器从根本上依赖于未经验证的代理，从文本自我描述到静态替代表示，来衡量智能体的能力。这种对非经验数据的依赖在智能体的预期配置与其实际操作能力之间造成了关键差距，引入了严重的安全漏洞。恶意智能体可以轻易歪曲自身能力或隐藏后门，从而逃避标准外部分析和静态表示学习技术。在这项工作中，我们介绍了ANTAP（自动非文本智能体选择器），这是一种评估驱动的路由架构，它摒弃间接代理，转而采用主动能力测试。通过动态查询智能体以经验性地确定其真实能力，ANTAP将性能提炼为共享语义空间内的固定行为算子。在推理时，路由通过纯非文本代数投影进行，建立了一个“语言防火墙”，使得基于元数据的攻击无法表达。在我们的实验中，ANTAP针对基于描述注入攻击的ASR接近零，而基于描述的路由基线为67.3%及以上。针对自适应嵌入攻击，ANTAP的ASR比基于嵌入的基线降低了20%，同时通过设计保持了对描述操纵的韧性。

## Abstract
The rapid integration of Large Language Models (LLMs) has driven the evolution of Multi-Agent Systems (MAS), where specialized agents collaborate to execute complex workflows. Effective orchestration in these environments requires robust routing mechanisms to efficiently allocate tasks to the most suitable agent. However, existing routers fundamentally rely on unverified proxies, ranging from textual self-descriptions to static surrogate representations, to gauge an agent's competence. This reliance on non-empirical data creates a critical gap between an agent's projected profile and its actual operational capabilities, introducing severe security vulnerabilities. Malicious agents can easily misrepresent their proficiencies or harbor covert backdoors that evade both standard external analysis and static representation-learning techniques. In this work, we introduce ANTAP (Automatic Non-Textual Agent Picker), an evaluation-driven routing architecture that discards indirect proxies in favor of active capability testing. By dynamically querying agents to ascertain their true competencies empirically, ANTAP distills performance into fixed behavioral operators within a shared semantic space. At inference time, routing is performed via a purely non-textual algebraic projection, establishing a "linguistic firewall" that renders metadata-based attacks inexpressible. In our experiments, ANTAP achieves near-zero ASR against description-based injection attacks, compared to 67.3\% and above for the description-based router baseline. Against adaptive embedding attacks, ANTAP achieves substantially lower ASR than the embedding-based baseline, with a 20\% reduction, while remaining resilient to description manipulation by design.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：多智能体系统（MAS）中，现有路由机制普遍依赖智能体的**文本描述**（如自称的能力描述）或**静态嵌入表示**来评估其是否适合处理某个查询。这种基于“间接代理”的方式存在严重安全漏洞：
  - 恶意智能体可通过在描述中嵌入提示注入指令来劫持路由器（Router Hijacking）。
  - 潜伏智能体（Sleeper Agent）可在正常行为下隐藏后门，仅在特定触发词下激活恶意行为，绕过静态分析。
  - 自适应攻击可通过对查询嵌入进行离散优化，绕过基于嵌入的路由器。
- **动机**：需要一个**从架构层面消除文本/元数据依赖**的安全路由方案，使得基于元数据的攻击在路由决策中变得“不可表达”。
- **整体含义**：ANTAP（Automatic Non-Textual Agent Picker）通过**主动经验性测试智能体真实能力**，将其性能蒸馏为**固定几何算子**，在推理时仅进行代数投影，从而构建“语言防火墙”。

## 2. 论文提出的方法论

### 2.1 核心思想
将路由选择从**解析文本**转化为**几何投影**：每个智能体被表示为一个固定线性算子 \( W^ \in \mathbb{R}^{1 \times d} \)，将查询嵌入 \( z \in \mathbb{R}^d \) 直接映射为分数 \( s = W^ z \)，分数越高表示该智能体越可能成功完成查询。

### 2.2 关键技术与公式

- **离线注册阶段**（一次完成，安全环境中执行）：
  - 使用 \( N \) 个可信基准查询 \( B = \{q_1, \dots, q_N\} \)，对每个智能体 \( a_i \) 执行任务，记录其是否成功（成功 = 答案正确 + 无非法工具调用，编码为 +1；否则为 -1）。
  - 构建性能矩阵 \( Y \in \mathbb{R}^{K \times N} \)（K 为智能体数量）。
  - 将查询嵌入矩阵 \( Q \in \mathbb{R}^{N \times d} \)（使用固定句子编码器 all-mpnet-base-v2 得到 d=768）。
  - 求解 Tikhonov 正则化最小二乘问题，得到每个智能体的行为算子：
    \[
    W^ = Y (Q Q^\top + \lambda I)^{-1} Q
    \]
    其中 \( \lambda \) 为正则化参数（默认 1.0）。
  - 伪逆 \( Q^\dagger_\lambda = (Q Q^\top + \lambda I)^{-1} Q \) 可预计算并缓存，新增智能体仅需 \( o(N) \) 评价 + 一次矩阵乘法。

- **在线路由阶段**（推理时）：
  - 对用户查询 \( q_{\text{new}} \)，计算嵌入 \( z = E(q_{\text{new}}) \in \mathbb{R}^d \)。
  - 计算所有智能体的分数向量 \( s = W^* z \)（\( W^* \in \mathbb{R}^{K \times d} \)）。
  - 选择 \( a^* = \arg\max_i s_i \)。
  - 整个过程中**不读取任何文本描述或元数据**。

- **隐式安全训练**：论文将“非法工具调用”也编码为 -1，因此潜伏智能体在触发词下的恶意行为会被直接学习为负信号，无需独立安全分类器。

## 6. 主要结论与发现

- **安全优势**：
  - 对描述注入攻击，ANTAP 的 ASR 为 **0.2%–2.4%**（MMLU），而 AutoGen 高达 75.3%，EmbedLLM 为 5.8%–21.3%。
  - 对 5 种潜伏智能体攻击，ANTAP 的 ASR 仅 **0.3%–2.4%**，基线为 73%–78%。
  - 对自适应攻击，ANTAP 的 ASR 保持在 **55%–63%** 之间，几乎不随触发词长度增加而变化；而 EmbedLLM 在 3 个 token 后即飙升至 86%+。
- **性能优势**：
  - ANTAP 在线延迟约 **27 ms**，远低于 AutoGen 的 428 ms。
  - 路由准确率（ACC）高于基线（如 MMLU 上 65.1% vs AutoGen 12.3%）。
- **结构性防御**：由于推理时仅处理数值，文本描述中的恶意指令**在数学上无法影响决策**，攻击面被消除。
- **隐式安全对齐**：离线校准中对非法工具调用的双极性编码使算子自动学习到安全相关的负信号。

## 7. 优点

1. **创新性高**：首次提出通过几何投影而非文本解析进行路由，从根本上消除文本攻击面。
2. **架构级安全**：防御不是基于启发式过滤或模型微调，而是**数学上无法表达**攻击。
3. **高效可扩展**：离线注册复杂度 \( O(N) \)，在线路由仅需一次矩阵向量乘法（\( O(Kd) \)），与描述长度无关。
4. **负责任的实验**：覆盖三种独立攻击向量，包含消融、自适应攻击、敏感性分析；代码已开源。
5. **隐式安全训练**：通过严格标签定义（非法工具调用 = -1），无需额外安全分类器即可压制潜伏智能体。

## 8. 不足与局限

1. **信任假设**：依赖于**可信的离线校准环境**和**未受污染的基准数据**；如果攻击者能够污染基准或校准流程，防御可能失效。
2. **双极性编码的粗糙性**：仅用 ±1 编码成功/失败，可能丢失细粒度差异（如部分成功、延迟差异等），未来可扩展为连续或多维评分。
3. **线性近似局限**：将智能体行为近似为线性算子，可能无法完全捕捉复杂任务中的非线性交互；虽然在路由粒度上足够，但理论保证有限。
4. **AgentHarm 校准样本过少**：仅 32 条，可能影响泛化性；论文虽展示了较好的结果，但未证明在更大规模有害查询库上的稳定性。
5. **实验范围有限**：
   - 未在真实生产级 MAS（如多轮对话、动态工具集）上验证。
   - 未与更多当代路由器（如 RouteLLM、MasRouter）进行直接对比。
   - 仅测试了 19 个智能体的场景，更大规模（如 50+）的扩展性未充分评估。
6. **对嵌入层的攻击**：论文指出假设嵌入模型可信，但若攻击者能操纵嵌入提取函数，则仍可能绕过防御。
