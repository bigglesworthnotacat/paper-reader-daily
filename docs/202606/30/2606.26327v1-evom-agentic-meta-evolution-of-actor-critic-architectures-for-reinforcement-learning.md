## 摘要
在演员-评论家强化学习中，网络架构通常需要手动设计。自动化设计面临挑战，因为每个候选架构在评估前必须经过训练，且设计空间是开放的。为了解决这些问题，我们引入了EVOM，一个用于发现高性能演员-评论家架构的智能元演化框架。我们将架构搜索构建为双层优化：内循环通过低保真度的近端策略优化（PPO）训练权重，外循环通过迭代优化架构程序驱动元演化。关键在于，外循环由一个基于LLM的设计智能体驱动，该智能体纯粹作为架构设计师，与策略执行和环境控制完全解耦。实验表明，EVOM优于手动设计的基线、LLM引导的随机搜索以及最先进的LLM引导的程序化策略搜索方法MLES，在Ant-v4和HalfCheetah-v4上表现出更优性能。消融研究验证了元演化循环和LLM设计智能体对最终性能都不可或缺。

## Abstract
In actor-critic reinforcement learning, network architectures are typically manually designed. Automating this design is challenging because each candidate must be trained before evaluation, and the design space is open-ended. To address these challenges, we introduce EVOM, an agentic meta-evolution framework for discovering high-performance actor-critic architectures. We frame architecture search as a bi-level optimization: an inner loop trains weights via the low-fidelity proximal policy optimization (PPO), while an outer loop drives meta-evolution by iteratively refining architecture programs. Crucially, this outer loop is powered by an LLM-based design agent that operates purely as an architecture designer, completely decoupled from policy execution and environment control. Experiments reveal that EVOM outperforms the manually designed baseline, an LLM-guided random search, and the state-of-the-art LLM-guided programmatic policy search method MLES, delivering superior performance on Ant-v4 and HalfCheetah-v4. Ablation studies validate that both the meta-evolution loop and the LLM Design Agent are indispensable for final performance.

## 2. 方法论
### 2.1 核心思想
- 将架构搜索建模为**双层优化**：
  - **内环（Inner Loop）**：使用低保真度的PPO（Proximal Policy Optimization）训练给定架构的权重，在较短预算（100k步）下获取快速评价信号。
  - **外环（Outer Loop）**：基于LLM的设计智能体（LLM Design Agent）通过元演化迭代优化架构程序，包括初始化、变异和交叉操作。
- LLM设计智能体**仅作为架构设计师**，与策略执行和环境控制完全解耦，不接触梯度、参数值或环境状态。

### 2.2 关键技术细节
- **架构表示**：每个候选架构 \( p \) 以可执行的Python程序表示，包含生成标签、结构化设计描述和完整代码，必须定义 `PolicyNet` 和 `ValueNet` 类。
- **演化循环**：
  1. **种群初始化**：LLM根据环境描述生成 \( N \) 个初始架构程序。
  2. **繁殖**：每代将 \( N/2 \) 个通过变异（修改单一父代架构的组件，如激活函数、容量、归一化等）生成，另 \( N/2 \) 通过交叉（融合两个父代架构的设计特征）生成。
  3. **评估**：每个子代架构使用PPO在低保真预算 \( B_{\text{low}} = 100k \) 步下训练，并在 \( K=3 \) 个确定性评估回合上取平均回报作为适应度。
  4. **排名与选择**：将当前精英种群与子代合并，按评估信号排序，保留前 \( N \) 个作为下一代精英。
- **最终评价**：演化结束后，选择最优架构从头开始进行全预算（\( B_{\text{full}} = 5M \) 步）训练和评估。
- **无效架构处理**：编译失败、输出非有限值或训练崩溃的架构被赋予惩罚适应度 -1000，确保其难以存活。

## 6. 主要结论与发现
- EVOM在Ant-v4和HalfCheetah-v4上均显著优于手动基线、随机搜索和MLES，获得更高平均回报和更低方差（尤其在HalfCheetah-v4上比随机搜索更稳定）。

- 最终发现的架构具有重复出现的设计模式：残差块、归一化（BN/LN）、SiLU激活、Dropout、更大宽度等，而手动基线仅为简单MLP。

- 元演化循环和LLM设计智能体两者缺一不可：去除演化（随机搜索）或不用LLM（如固定算子）都会导致性能下降。
