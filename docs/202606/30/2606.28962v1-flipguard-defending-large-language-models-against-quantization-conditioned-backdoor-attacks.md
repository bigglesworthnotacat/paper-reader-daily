## 摘要
模型量化对于大型语言模型（LLMs）的高效部署至关重要，但引入了一个关键漏洞：量化条件后门（QCB）攻击。在此类攻击中，恶意行为在全精度模型中处于休眠状态，仅在特定量化失真后激活，从而绕过标准安全审计。为缓解这一问题，我们提出了FlipGuard，一种主动防御框架，在量化前选择性地扰动模型权重。通过破坏对手在权重模式与量化边界之间的精确对齐，FlipGuard在不访问训练数据或触发样本的情况下抑制后门激活。我们还提出了防御效能比（DER），这是一个统一指标，用于联合评估安全增益、效用保持和计算成本。在七个大型语言模型（包括StarCoder和LLaMA系列模型）以及三种量化方案（INT8、FP4、NF4）上的大量实验表明，FlipGuard能有效消除三种场景下的QCB，即易受攻击的代码生成、内容注入和过度拒绝，在实现高安全性的同时性能退化可忽略不计。

## Abstract
Model quantization is essential for the efficient deployment of Large Language Models (LLMs), but introduces a critical vulnerability: Quantization-Conditioned Backdoor (QCB) attacks. In these attacks, malicious behaviors remain dormant in full-precision models and activate only after specific quantization distortions, bypassing standard security audits. To mitigate this, we introduce FlipGuard, a proactive defense framework that selectively perturbs model weights prior to quantization. By breaking the adversary's precise alignment between weight patterns and quantization boundaries, FlipGuard suppresses backdoor activation without requiring access to training data or trigger samples. We further propose the Defense Effectiveness Ratio (DER), a unified metric to jointly evaluate security gains, utility preservation, and computational cost. Extensive experiments across seven LLMs (including StarCoder and LLaMA-family models) and three quantization schemes (INT8, FP4, NF4) demonstrate that FlipGuard effectively neutralizes QCBs across three scenarios, i.e., vulnerable code generation, content injection, and over-refusal, achieving high security with negligible performance degradation.

## 1. 核心问题与整体含义（研究动机与背景）
- **核心问题**：大型语言模型（LLM）部署中广泛使用量化技术（INT8、FP4、NF4）以降低计算和存储开销，但量化过程引入了一个新型安全威胁——**量化条件后门（Quantization-Conditioned Backdoor, QCB）攻击**。攻击者通过精确调整权重使其在量化时被映射到预设的恶意值，后门仅在模型被量化后激活，而在全精度模型中保持良性，从而绕过标准安全审计。
- **整体含义**：QCB攻击构成了严重的供应链安全风险，因为用户通常从第三方平台（如Hugging Face）下载全精度模型并本地量化。现有防御（如样本级检测、模型级修改）大多假设后门行为在量化前后一致，无法检测这种“潜伏型”攻击。论文旨在设计一种无需训练数据或触发样本、轻量且模型无关的防御框架，以消除QCB威胁。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：破坏攻击者预设的权重与量化边界之间的精确对齐。QCB依赖权重量化时的“恰好”落入攻击者指定的离散值区间。FlipGuard通过在量化前对高量化误差的权重施加微小扰动，使其跨过决策边界进入相邻区间，从而改变量化结果，阻止后门激活。扰动幅度极小（仅需偏移小于一个量化步长），对模型正常功能影响可忽略。
- **关键技术细节（算法流程）**：
  1. **分块量化误差计算**：将全精度权重张量按块（block）划分，对每个块独立计算缩放因子（Scale = max(|w|)/max(|A|)，A为量化字母表），然后量化并计算每个权重的缩放值到最近量化值的误差。
  2. **选择高误差权重**：对误差降序排序，选取前k%的权重（k为微调比例，如10%）。
  3. **确定调整方向**：对于每个选中的缩放值ws，找到其所在区间相邻的两个量化值（α_a < α_b），计算中点m = (α_a+α_b)/2。若ws在左半区间，则调整至m+ε（移到右区间）；若在右半区间，则调整至m-ε。ε为小常数（如1e-5），确保新值严格属于目标区间并会被映射到相邻量化值。
  4. **恢复全精度**：将调整后的缩放值乘以缩放因子，得到最终全精度权重。
  5. **无数据要求**：整个过程仅依赖权重本身和量化配置，无需任何训练数据或触发器。
- **公式**：更新规则为式(2)：当α_a < w_s < m时，w_s ← m + ε；当m < w_s < α_b时，w_s ← m - ε。
