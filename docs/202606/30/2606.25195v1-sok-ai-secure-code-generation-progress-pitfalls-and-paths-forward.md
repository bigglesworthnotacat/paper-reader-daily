## 摘要
日益增长的AI代码生成系统使用引发了一个核心安全问题：如今的模型和编码代理在实际生成安全代码方面能做到什么？它们仍在哪里失败？什么能推动该领域向前发展？现有工作探索了用于安全代码生成的提示、微调、强化学习和代理工作流，但该领域仍缺乏对这些技术如何提升安全性以及为何持续存在重大失败的系统性理解。在这篇SoK中，我们系统化了AI安全代码生成的进展、陷阱与未来方向。我们引入了一个三层框架，分别衡量模型对安全编码原则的自然语言理解、生成过程中对这些原则的代码级执行，以及两者之间的知识-执行差距。我们在涵盖孤立函数级安全和完整Web应用安全的基准上，跨模型和编码代理实例化该框架。结果表明，安全编码原则理解是代码级结果的统计显著性预测因子，包括功能正确性、安全性以及功能-安全联合正确性。然而，仍存在显著的知识-执行差距：模型能识别相关安全原则，但仍无法将其转化为既安全又功能正确的代码。这些发现提供了以原则为中心的对AI安全代码生成现状的阐述，并通过原则引导的生成、评估、基准测试和代理工作流，识别出具体的前进道路。

## Abstract
The increasing use of AI systems for code generation raises a central security question: what can today's models and coding agents actually do to produce secure code, where do they still fail, and what would move the field forward? Existing work has explored prompting, fine-tuning, reinforcement learning, and agentic workflows for secure code generation, but the field still lacks a systematic understanding of how these techniques improve security and why substantial failures persist. In this SoK, we systematize the progress, pitfalls, and paths forward for AI secure code generation. We introduce a three-level framework that measures models' natural-language understanding of secure coding principles, their code-level actuation of those principles during generation, and the knowledge--actuation gaps between the two. We instantiate this framework across models and coding agents on benchmarks covering both isolated function-level security and full web-application security. Our results show that secure-coding-principle understanding is a statistically strong predictor of code-level outcomes, including functional correctness, security, and joint functional-security correctness. Yet substantial knowledge--actuation gaps remain: models can recognize relevant security principles but still fail to translate them into secure and functional code. These findings offer a principle-centered account of where AI secure code generation stands today and identify concrete paths forward through principle-guided generation, evaluation, benchmarking, and agentic workflows.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将安全代码生成分解为三层测量，使用安全编码原则（SCP，如OWASP、CERT规则）作为桥梁，连接模型的自然语言理解、代码生成行为及两者之间的转化缺陷。
- **三层框架（KAUGE）**：
  - **Layer 1（知识层）**：测量模型对SCP的自然语言理解能力。构建了包含6382道固定答案题的NLU基准，涵盖4类推理维度（陈述性、因果性、程序性、上下文性）和9种NLP任务格式（WiC、QNLI、MRPC、MNLI、COPA、CosmosQA、ReClor、BoolQ、MultiRC）。
  - **Layer 2（执行层）**：测量代码级生成结果是否同时满足功能正确性和安全性。使用**可执行基准**（CWEval、BaxBench）的通过率指标：pass@k、secure-pass@k、secure@pass k。
  - **Layer 3（差距层）**：诊断知识与执行之间的差距。通过构造**漏洞-SCP防御公式映射**（DNF：每个漏洞对应一组充分防御策略的析取范式）和**SCP合规性法官**（两个LLM法官+裁决），将功能正确的代码划分为4类：原则成功、另有安全措施、合规但脆弱、执行失败。
- **关键技术细节**：
  - NLU基准：从456条来源规则（OWASP 218条、CERT C 157条、CERT C++ 81条）自动生成，经语言模型验证和人工抽样校验。
  - DNF映射：每个漏洞映射到一个或多个充分防御策略的OR-of-ANDs公式；使用GPT-5-nano和Claude Haiku 4.5双法官加裁决器构建。
  - SCP合规性法官：双法官共识（F1 84.0%）优于单个模型（76.7–72.3）；对代码判断是否实现了映射中的每个SCP。

### 6. 论文的主要结论与发现

1. **安全编码原则理解普遍较高**：所有系统在Layer 1得分远高于随机（86%-96%），但**因果推理**在CERT C/C++上是最弱维度（p=0.008），安全微调未能实质改善。
2. **知识是执行的重要预测因子**：10个基础模型上，higher Layer 1 knowledge与higher CWEval secure-pass@k显著相关（ρ=0.71, p=0.025），BaxBench方向一致（ρ=0.63, p=0.056）。
3. **知识-执行差距显著**：执行层得分系统低于知识层（CWEval差距47.9点，BaxBench达72.9点）。差距源于功能正确性瓶颈（Web应用级尤其）和原则实现不精确。
4. **干预效果差异大**：
   - 安全感知提示在接近正确的任务上有效，RCI可能过度修正导致功能损坏。
   - 安全微调（SafeCoder）在Web应用级有帮助，但在函数级不明显；PurpCodeRL整体下降。
   - 代理脚手架（Codex Scaffold）提升最大，因为改善了软件构建能力，而非新增安全知识。
5. **四种诊断模式揭示隐藏问题**：
   - **原则成功**（安全且合规）—期望情况。
   - **另有安全措施**（安全但不合规）—覆盖局限。
   - **合规但脆弱**（合规但漏洞仍可被利用）—原则过于抽象，未指定正确实现边界。
   - **执行失败**（可被利用且无合规证据）—模型未应用原则。
6. **抽象原则不足以指导安全实现**：合规但脆弱的案例（如CRLF注入）表明，原则必须与具体接口绑定才能有效。
