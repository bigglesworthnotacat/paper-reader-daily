## 摘要
智能体语言模型极大地扩展了人工智能的应用，然而关于如何为通用型智能体策划训练数据的公开知识仍然很少。现有的开源工作，如SWE-Smith、SERA和Nemotron-Terminal，通常针对单一的基准测试，留下了如何训练能够跨多种智能体任务泛化的模型这一问题。OpenThoughts-Agent (OT-Agent) 项目通过一个完全开放的数据策划管道来解决这一差距，用于训练智能体模型。我们进行了超过100次受控消融实验，系统地研究了管道的每个阶段，获得了关于任务来源和多样性重要性的见解。然后，我们从管道中组装了一个包含10万条示例的训练集，并在此数据集上微调了Qwen3-32B，在七个智能体基准测试上取得了44.8%的平均准确率，比现有最强的开源数据智能体模型（Nemotron-Terminal-32B，40.9%）提高了3.9个百分点。此外，我们的训练数据表现出强大的缩放特性，在计算资源受控的比较中，每个训练集大小上均优于其他开源数据集。我们在openthoughts.ai公开发布了我们的训练集、数据管道、实验数据和模型，以支持未来关于智能体模型训练的开源研究。

## Abstract
Agentic language models dramatically expand the applications of AI yet little is publicly known about how to curate training data for broadly capable agents. Existing open efforts such as SWE-Smith, SERA, and Nemotron-Terminal typically target a single benchmark, leaving open the question of how to train models that generalize across diverse agentic tasks. The OpenThoughts-Agent (OT-Agent) project addresses this gap with a fully open data curation pipeline for training agentic models. We conduct more than 100 controlled ablation experiments to systematically investigate each stage of the pipeline, yielding insights on the importance of task sources and diversity. We then assemble a training set of 100K examples from our pipeline and fine-tune Qwen3-32B on this dataset, which yields an average accuracy of 44.8% across seven agentic benchmarks and a 3.9 percentage point improvement over the strongest existing open data agentic model (Nemotron-Terminal-32B, 40.9%). Moreover, our training data exhibits strong scaling properties, outperforming alternative open datasets at every training set size in compute-controlled comparisons. We publicly release our training sets, data pipeline, experimental data, and models at openthoughts.ai to support future open research on agentic model training.

### 2. 方法论：核心思想、关键技术细节

- **核心思想**：构建一个 **六阶段 SFT 数据策划管线**，并通过超过 100 次受控消融实验优化每个阶段；同时研究 RL 数据的来源影响。
- **六阶段 SFT 管线**（图 2）：
    1. **任务来源（Sourcing Tasks）**：测试 95 种任务生成策略（包括合成问题、人工编写问题等），涵盖不同领域（如 SWE、Linux、加密货币等）。最优策略：SWE-Smith、StackExchange SuperUser、StackExchange Tezos、IssueTasks。
    2. **混合任务（Mixing Tasks）**：从 top-k 任务来源中均匀采样任务描述。最优：Top-4 或 Top-8 混合（随机洗牌），平衡各基准性能。
    3. **任务增强（Task Augmentation）**：尝试用 LLM 添加约束、提高难度等。结论：所有增强策略均未优于原始任务描述。
    4. **任务过滤（Filtering Tasks）**：使用 LLM（GPT-5）对任务难度评分，保留需要更多 token 解决的任务。结果：相比随机采样提升约 3 pp。
    5. **教师模型（Teacher Model）**：用不同模型（GLM-4.7-AWQ、Kimi K2.5、GLM-5、GPT-5.3-Codex 等）生成轨迹。最佳教师：GLM-4.7-AWQ（量化版本），并非最强模型（GPT-5.3-Codex 最差）。
    6. **轨迹过滤（Filtering Agent Rollouts）**：过滤至少 5 轮交互的轨迹（保留长轨迹）。效果最好，计算量控制后仍有效。
- **最终 SFT 数据集（OpenThoughts-Agent-v2）**：100K 条轨迹，来自 Top-4 任务来源，Tezos 部分使用合成增强（从 997 个唯一任务扩展到 21K 多表面形式），教师为 GLM-4.7-AWQ，保留 ≥5 轮轨迹。
- **RL 实验**：使用 RLOO 算法（基于 SkyRL 框架），从冷启动 SFT 检查点开始训练。测试 8 个 RL 数据源，其中 pymethods2test（竞赛编程问题重写为 Python 函数合约）最优。

- **实验数量**：超过 **100 次受控消融实验**（SFT 管线每个阶段独立消融，包括 95 种任务来源、多种混合/过滤/增强策略、6 种教师模型、3 种轨迹过滤等）。RL 部分进行了 8 个数据源比较，并进行了运行间可重复性测试（3 次近似复制，ID/OOD 差异约 1.6/2.0 pp）。

### 6. 主要结论与发现

1. **任务来源是数据管线中最关键的因素**，不同来源导致下游准确率相差可达 30 pp（SWE-Bench）。
2. **混合 Top-4 到 Top-8 的任务来源优于单一来源**，避免过度专业化。
3. **任务描述增强（LLM 改写）并不带来收益**，原始描述最佳。
4. **任务过滤中使用 LLM 难度信号（GPT-5 响应长度）提升约 3 pp**。
5. **教师模型的选择：强≠优**，GLM-4.7-AWQ 优于更强的 GPT-5.3-Codex。
6. **保留长轨迹（≥5 轮）显著提升性能**，且效果独立于计算量。
7. **数据缩放**：简单上采样（重复轨迹）很快饱和，而**合成任务增强（扩充任务来源多样性）**持续提升性能至 100K 规模。
8. **RL 数据来源同样关键**，pymethods2test（竞赛编程问题）最佳，促进政策探索（更多思考、工具调用），而其他来源（如 llm-verifier-freelancer）导致政策压缩，效果较差。
9. **SFT + RL 组合优于单独 SFT 或 RL**：8B 上平均 27.9%（vs SFT only 27.4% 和 RL only 10.2%），且基于中等强度 SFT 的 RL 效果最好。

### 7. 优点

- **完全开源**：数据、管道、模型全部公开，易复现。
- **系统消融**：六阶段 SFT 管线独立消融，实验设计严谨，控制变量。
- **计算量公平比较**：轨迹过滤和缩放实验进行了 token 预算匹配，排除额外训练计算量混淆。
- **跨基准评估**：涵盖 7 个不同领域的代理基准（SWE、终端、工具调用、医疗、金融、通用助手），衡量泛化能力。
- **SFT 与 RL 结合**：首次在同一框架下研究两者交互，并证明可组合提升。
- **丰富的开源基础**：建立在 Llama-Factory、SkyRL、Harbor 等成熟开源生态系统上。

### 8. 不足与局限

- **RL 实验仅限制在 8B 规模**：未验证同样的 RL 配方是否适用于 32B 规模（计算限制）。
- **基础模型选择未消融**：所有实验基于 Qwen3 家族（Qwen3-8B 和 Qwen3-32B），未分离预训练贡献；迁移到 Qwen3.5 或其他模型的效果未知。
- **数据集规模有限**：最大训练集 100K 轨迹，未探索百万级规模下的缩放趋势。
- **评估基准可能不全面**：虽然覆盖 7 个基准，但真实世界代理任务边界更广，可能存在未覆盖的能力维度。
- **RL 可重复性验证有限**：仅进行 3 次近复制运行，且不同数据源间的噪声预估尚不完整。
- **教师模型与基础模型不匹配**：教师 GLM-4.7-AWQ 可能引入了特定偏差，影响泛化。
- **部分基线结果不可完全复现**（如 OpenSWE-32B 报告 62.4% 但在同一子集上仅 44.0%），提示方法差异。
