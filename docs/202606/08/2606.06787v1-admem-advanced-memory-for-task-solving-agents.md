## 摘要
大型语言模型（LLMs）作为工具使用智能体展现出潜力，但在需要记忆、组织和重用知识的长期任务中仍受限。先前的记忆方法试图解决这一问题，但主要侧重于存储事实信息。近期关于程序性记忆的工作改进了任务重用，但往往简化为重放过去的成功案例，而未处理失败情况或在线可扩展性。我们引入了一个统一且自动化的记忆框架，将语义记忆、情景记忆和程序性记忆整合在一个双层设计中，结合了短期和长期存储。一个由行动者、记忆和评论者智能体组成的多智能体架构实现了自动记忆生成、奖励注释和自适应检索。长期记忆通过基于奖励的评估、合并和修剪进行管理，确保了可扩展性和持续改进。在多个环境中的实验表明，与现有基线相比，我们的方法在长回合多步任务上提高了鲁棒性和成功率。这项工作突显了全面、自适应记忆对于推进基于LLM的智能体的重要性。

## Abstract
Large Language Models (LLMs) show promise as tool-using agents but remain limited in long-horizon tasks that require remembering, organizing, and reusing knowledge. Prior memory approaches aim to resolve the situation, but mainly focus on storing factual information. Recent work on procedural memory improves task reuse, yet often reduces to replaying past successes without addressing failure cases or online scalability. We introduce a unified and automatic memory framework that integrates semantic, episodic, and procedural memory in a bi-level design combining short-term and long-term stores. A multi-agent architecture with actor, memory, and critic agents enables automatic memory generation, reward annotation, and adaptive retrieval. Long-term memory is managed through reward-based evaluation, merging, and pruning, ensuring scalability and continual improvement. Experiments across various environments show that our approach improves robustness and success on long multi-turn tasks compared to existing baselines. This work highlights the importance of comprehensive, adaptive memory for advancing LLM-based agents.

## 1. 研究动机与核心问题
- **背景**：大型语言模型（LLMs）作为工具使用智能体展现强大能力，但在需要记忆、组织和重用知识的长期多步任务中表现受限。
- **现有记忆方法的局限**：
  - 多数方案仅侧重存储**事实性信息**（语义记忆），忽视了程序性经验。
  - 近期出现的程序记忆（如 Agent Workflow Memory）只是重放过去成功案例，未处理失败场景，缺乏在线扩展能力。
  - 现有系统缺乏统一的记忆架构，缺少基于奖励的动态管理、评估与剪枝机制，无法支持智能体从交互中持续改进。
- **核心目标**：设计一个**统一、自动、自适应的记忆框架**，融合语义记忆、情景记忆与程序记忆，支持智能体在长生命周期的任务流中自我演化。

## 2. 方法论
### 2.1 整体架构
- **双层记忆系统**：短期记忆（STM） + 长期记忆（LTM）。
  - 短期记忆：维持当前任务上下文，采用类似**程序栈的压缩结构**（子目标/计划），任务结束后清除并编码入长期记忆。
  - 长期记忆：跨任务持久存储，包含三类记忆：
    - 语义记忆（事实与知识）
    - 情景记忆（历史事件摘要）
    - 程序记忆（从行动-结果中提取的决策指导）

- **多智能体协作**：由三个 LLM 智能体组成：
  - **行动者（Actor）**：与环境交互，使用记忆执行动作，并为每步动作设定预期结果。
  - **评论者（Critic）**：缓存动作上下文与结果，比较实际观察与预期，生成程序记忆条目（含成功标志、反思、替代方案）并给出奖励 \(r_t \in \{0,1\}\)。
  - **记忆智能体（Memory Agent）**：管理长期记忆存储、检索与维护（剪枝、合并）。

### 2.2 程序记忆的生成与奖励机制
- 行动者在执行动作时明确描述**预期结果**。
- 评论者收到观察后，将动作、上下文、预期和实际结果打包，生成程序记忆。
- 通过 **“添加子目标”与“结束子目标”** 等规划工具，实现类似搜索树的探索，并形成有层次的过程记录。
- 每次评论者产出奖励时，同时把奖励传递给记忆存储，用于更新所调用记忆条目的有效性评估。

### 2.3 长期记忆的检索与管理
- **检索策略**：对程序记忆，结合**任务相似度**与**记忆有效性评分**进行检索。
  - 有效性参数 \(v_m \in [0,1]\)，通过 EM 算法在线估计。
  - 模型假设：每次决策的最终奖励是各个被检索记忆的“有帮助”状态（服从伯努利分布）与无记忆帮助状态的逻辑或，其中 \(l_t^{(i)} \sim B(v_{m^{(i)}} \psi(c_t)^\top \psi(c^{(i)}))\)。
  - 据此更新每个 \(v_m\)，并贪心地选择 top-n 高 \(v_m \cdot \text{similarity}\) 的记忆条目。
- **记忆管理**：根据检索频率进行**剪枝**（低频条目删除）和**合并**（高共现条目合并），保证存储规模可控，并提升检索效率。

### 2.4 上下文压缩（短期记忆管理）
- 利用类似程序调用栈的结构：将当前子目标、计划以及压缩过的外部回合摘要作为短期记忆，使模型只关注当前步骤所需信息，极大节省上下文窗口。

## 6. 主要结论与发现
- AdMem 在绝大多数测试域上**显著提升任务完成率和进度**，特别是在具有可迁移知识的域，且未造成非迁移域的明显性能下降。
- 短期规划提升局部任务执行效率，而**长期程序记忆是实现跨任务自我演化的核心**，随着 epoch 增加，性能持续提升。
- 基于奖励的检索和管理机制使得记忆条目能够被有效评估和筛选，避免了过去方法仅记录成功而产生的噪声。
- 框架体现了全面、自适应记忆对于 LLM 智能体在长生命周期任务中的重要价值。

## 7. 优点
- **理论-实践桥梁**：将认知科学中的三种记忆分类与双层记忆设计实际落地到多智能体 LLM 系统中。
- **自适应程序记忆**：不仅记录成功，也利用预期差异处理失败案例，并通过奖励模型动态评估记忆有用性。
- **在线可扩展管理**：基于检索频率的剪枝/合并，以及 EM 估计的检索策略，使系统能持续运行而不臃肿。
- **上下文压缩巧妙**：借鉴程序栈思想，使短期记忆既紧凑又可管理，支持多级子目标规划。
- **并行化设计**：行动者、评论者、记忆智能体可多线程运行，记忆编码与维护不显著增加延迟。
