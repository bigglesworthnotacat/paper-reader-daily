## 摘要
我们研究了在非结构化知识库上执行现实客户服务工作流的工具使用AI代理的编排机制。我们认为，声明式代理——在系统提示中附加自然语言技能文件的AI代理——是一种有效的编排范式。具体来说，我们比较了（i）一个DeclarativeAgent，它在推理时读取三个特定领域的技能文件并决定自己的控制流，（ii）一个基于具有显式阶段的程序化状态机的ImperativeAgent，以及（iii）一个模仿$τ$-Knowledge基准代理的无脚手架基线代理。我们的ImperativeAgent受递归语言模型和基于图的编排框架中的外部化控制推理的启发。我们将这三个代理形式化为去中心化部分可观察马尔可夫决策过程中的策略类，并分析其信息论和结构特性；然后我们在五个语言模型和两种检索机制上实证检验了预测的差异。我们的结果表明，检索质量是AI代理的主要瓶颈：当证据不完整或偏斜时，所有代理的性能都会大幅下降，而技能文件无法恢复失去的性能。然而，在高质量检索下，声明式技能持续提高程序性任务的准确性并减少编排错误，而命令式状态机的脆弱性并不能可靠地提高任务成功或合规性。

## Abstract
We study orchestration mechanisms for tool-using AI agents in realistic customer-service workflows over an unstructured knowledge base. We argue that declarative agents -- AI agents equipped with natural-language skill files appended to the system prompt -- are an effective orchestration paradigm. Concretely, we compare (i) a DeclarativeAgent that reads three domain-specific skill files at inference time and decides its own control flow, (ii) an ImperativeAgent based on a programmatic state machine with explicit phases, and (iii) an unscaffolded baseline agent modeled after the $τ$-Knowledge benchmark agent. Our ImperativeAgent is motivated by externalised-control inference as in Recursive Language Models and graph-based orchestration frameworks. We formalise the three agents as policy classes within a decentralised partially-observable Markov decision process and analyse their information-theoretic and structural properties; we then test the predicted differences empirically on five language models and two retrieval regimes. Our results show that retrieval quality is a dominant bottleneck for AI agents: when evidence is incomplete or skewed, all agents degrade substantially, and skill files cannot recover lost performance. Under high-quality retrieval, however, declarative skills consistently improve accuracy on procedural tasks and reduce orchestration errors, while the imperative state machine's brittleness does not reliably improve task success or compliance.

- **整体含义**：论文认为声明式代理（DeclarativeAgent）是一种有效且低成本的编排范式，其优势来自将过程性知识转化为可组合的自然语言技能文件，而非硬编码的确定性控制流。研究还揭示**检索质量**是影响工具型 AI 代理性能的首要瓶颈，任何编排方式都无法弥补劣质检索带来的信息损失。

### 2. 论文提出的方法论
- **核心思想**：定义并对比三类策略类（policy classes）：
  - **基线策略 (πB)**：使用标准系统提示，无额外控制流。
  - **声明式策略 (πD)**：在系统提示中追加三份领域特定的 Markdown 技能文件（银行流程、客户交互、知识发现），由模型在推理时自行选择和编排工具调用序列。
  - **命令式策略 (πI)**：基于程序化有限状态机，具有 GREETING → TRIAGE → VERIFICATION → PLANNING → EXECUTION → CONFIRMATION → COMPLETE（及 ADVISORY、ESCALATE 分支）等显式阶段；每阶段注入阶段特定指令并限制可用动作集，状态转移由代码行确定而非模型自由决定。
- **形式化**：将客户服务交互建模为有限时域、两主体（AI 代理与用户模拟器）的去中心化部分可观察马尔可夫决策过程（Dec-POMDP）。世界状态 S 包含数据库状态和对话历史，奖励基于最终数据库状态与标准答案匹配，并完成所有必选写工具调用。
- **理论分析**：通过信息论和策略类容量给出四个命题：
  - 命题1：当模型存在过程性能力差距时，技能文件能降低条件熵，从而提高动作选择准确性。
  - 命题2：命令式策略由于动作空间受限，其行为集合是基线策略的真子集，理论上存在性能上限损失。
  - 命题3：命令式策略通过重试预算限制可能降低平均 LLM 调用次数，从而控制成本。
  - 命题4：检索噪声会通过数据信息处理不等式消解技能文件带来的信息增益，声明式优势在噪声下将消失。

### 6. 论文的主要结论与发现
- **RQ1（技能文件作为过程先验）**：在黄金检索下，DeclarativeAgent 在四/五个模型上优于基线，提升幅度与模型内在程序能力差距大致正相关；对能力较强的 Gemini-Flash-Lite 基本持平。
- **RQ2（命令式作为合规执行者）**：ImperativeAgent 的任务成功率普遍低于基线，且其未授权写入率、超重试率并未降低，甚至在嵌入检索下写前验证比例最高，显示确定性门控在实践中并不必然改善合规性，反而引进脆弱性。
- **RQ3（检索作为瓶颈）**：所有代理在嵌入检索下 Pass@1 大幅下降，声明式优势在大多数模型上消失或减弱，确认检索质量是主导瓶颈，技能文件无法补偿错误的观测信息。
- **写操作精度**：DeclarativeAgent 在两种检索下均严格优于基线和命令式代理，命令式代理写精度损失约20个百分点。
- **成本与维护**：声明式代理仅增加少量静态技能文件的 token 成本，可利用提示缓存消除延迟；技能文件易于由领域专家编辑和版本化，适合实际部署。
