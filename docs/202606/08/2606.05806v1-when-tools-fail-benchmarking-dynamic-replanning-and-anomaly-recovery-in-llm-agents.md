## 摘要
现有基准在理想化的“快乐路径”上评估大语言模型（LLM）的工具集成推理（TIR），很大程度上忽视了现实世界中的工具失效。我们引入了ToolMaze，一个针对TIR智能体中动态路径发现与错误恢复的基准。为了将系统性重规划与盲目试错区分开，ToolMaze采用二维设计：基于有向无环图（DAG）的拓扑复杂度，以及一个关于工具扰动的2×2分类法（显式/隐式，暂时/永久）。评估表明，扰动几乎降低了所有模型的性能，其中在隐式语义失效下下降最为剧烈。由于对损坏输出的系统性过度信任，扰动恢复率（PRR）在这些场景下暴跌约37%，而复杂拓扑则将智能体困在徒劳的试错循环中。至关重要的是，智能体的容错能力随模型规模提升的速度比基本任务执行慢3.66倍，这凸显了动态重规划是一个模型缩放或提示工程未能解决的独特瓶颈。数据和代码可在https://github.com/Zhudongsheng75/ToolMaze获取。

## Abstract
Existing benchmarks evaluate Tool-Integrated Reasoning (TIR) in LLMs on idealized ''happy paths'', largely overlooking real-world tool failures. We introduce ToolMaze, a benchmark for dynamic path discovery and error recovery in TIR agents. To separate systematic replanning from blind trial-and-error, ToolMaze adopts a two-dimensional design: DAG-based topological complexity and a $2 \times 2$ taxonomy of tool perturbations (explicit/implicit, transient/permanent). Evaluations show that perturbations degrade performance across nearly all models, with the sharpest drops under implicit semantic failures. Driven by systemic over-trust in corrupted outputs, Perturbation Recovery Rate (PRR) plummets by around 37\% in these scenarios, while complex topologies trap agents in futile trial-and-error loops. Crucially, agentic fault-tolerance improves with model scale $3.66\times$ slower than basic task execution, highlighting dynamic replanning as a distinct bottleneck unaddressed by model scaling or prompting. Data and code are available at https://github.com/Zhudongsheng75/ToolMaze.

- **整体含义**：论文提出，需要将评估范式从静态、单一路径执行转变为**动态路径发现与异常恢复**，并指出动态重规划能力是当前 LLM 独立于普通任务执行的**独特瓶颈**，无法通过简单的模型缩放或提示工程弥补。

## 方法论
### 核心思想
- 构造一个名为 **ToolMaze** 的二维评估框架，将每个评估实例定义为 **拓扑复杂度（C 轴）** 与 **扰动模式（P 轴）** 的交叉点，以此系统性地考察智能体的容错与重规划能力。

### 关键技术细节
1. **工具语料库（Tool Corpus）**  
   - 包含 270 个人工设计的模拟真实 API 的工具，分为三类：  
     - `Source`（数据获取）、`Processor`（数据转换）、`Action`（带副作用的终点操作）。  
   - 工具还标记了 6 个应用领域（金融、旅行、办公等），以保证生成任务的语义连贯性。

2. **C × P 评估矩阵**  
   - **C 轴（拓扑复杂度）**：基于有向无环图（DAG）定义 4 个层级。  
     - C1：线性单一路径，无替代方案。  
     - C2：1 对 N 替代路径，允许直接替换单个工具。  
     - C3：多对多多路径，呈现恢复路径的组合爆炸。  
     - C4：集成多分支，组合多个 C2/C3 模式，要求在多分支节点上并行推理。  
   - **P 轴（扰动模式）**：  
     - 两正交维度交叉：**显式/隐式** × **暂时/永久**。  
     - P1：显式暂时（如 503 临时不可用）；P2：显式永久（如 404 已删除）；P3：隐式暂时（如返回格式正确但语义荒谬的数据）；P4：隐式永久。  
   - 扰动注入位置是预先指定的工具节点，而非随机，确保可公平度量恢复行为。

3. **任务生成管道**  
   - 采用“工具优先”范式，先构造 DAG 再生成自然语言查询，以保证解空间的完整性：  
     a. **DAG 组装与验证**：通过 LLM 架构师根据目标 C 层级采样工具，构建有向无环图，并校核数据流一致性。  
     b. **解空间枚举**：按功能等价性聚类工具，穷举所有有效的拓扑顺序，得到基础真值解集 S，并指定最短路径为无扰动默认路径。  
     c. **任务自然化**：将 DAG 转化为上下文丰富的用户请求，再通过反向验证确保重构的工具依赖关系与原始 DAG 一致。

4. **运行时扰动引擎**  
   - 确定性注入：每次（任务，P 模式）组合返回相同的预定义合成响应，排除随机性。  
   - 故障激活规则：对于多路径任务，扰动关联到一个替代组，智能体调用该组内任一工具即触发故障，且每个组只激活一次，防止同时扰动多个替代项。

5. **评估指标**  
   - **任务成功率（TSR）**：所有实例中成功完成任务的比例。  
   - **扰动恢复率（PRR）**：在暴露于扰动的条件下，成功激活有效恢复策略（重试/切换路径/明智中止）的概率。  
     \[
     \text{PRR}_m = \frac{\sum_{\tau \in T_m} I_{\text{recov}}(\tau) \cdot I_{\text{pert}}(\tau)}{\sum_{\tau \in T_m} I_{\text{pert}}(\tau)}
     \]
   - **恢复成本（RC）**：衡量恢复效率，惩罚多余的工具调用。定义为：
     \[
     C_{\text{rec}}(\tau) = 1 - \frac{c^*(\tau)}{\max\{c(\tau), c^*(\tau)\}} \cdot I_{\text{succ}}(\tau)
     \]
     模式级 RC 为所有轨迹的均值。RC 趋近 0 表示最优恢复，趋近 1 表示大量浪费步骤或彻底恢复失败。

## 主要结论与发现
1. **扰动普遍导致性能下降**：几乎所有模型从无扰动到有扰动时 TSR 和 PRR 均显著降低，RC 升高，表明“快乐路径”能力与故障恢复能力是**解耦的**。
2. **隐式语义失效破坏最严重**：当工具返回结构合法但语义错误的数据时（P3/P4），智能体因**过度信任**而很少主动检测异常，导致 PRR 较显式错误平均降低约 37%，成为核心瓶颈。
3. **拓扑复杂度的影响呈倒 U 形**：C2 因有替代路径，性能甚至略优于线性 C1；但到 C3/C4，组合爆炸使性能恶化，智能体陷入大量徒劳调用。
4. **故障感知提示只能部分缓解**：加入扰动提示普遍提升性能（最高 +20.8%），但仍远未恢复到无扰动水平，说明当前智能体缺乏内源的异常意识。
5. **容错能力扩展远慢于任务执行能力**：在开源模型上，参数规模每增大一个数量级，TSR 提升约 17.85 个百分点，而 PRR 仅提升 4.88 个百分点，**容错增长速率约为任务的 1/3.66**。这证明动态重规划并非通用能力的自然副产品，而是需要专门训练的独立能力。
6. **PRR 与 RC 呈负相关**：越难恢复的扰动（如 P4），智能体不仅成功率低，且额外开销急剧增加，反映了低效的试错行为。

## 优点
- **系统性的二维评估矩阵**：首次将拓扑复杂度与扰动分类正交化，精确定义了可恢复路径的真值空间，能严格区分系统性重规划与随机替换。
- **完善的指标设计**：TSR、PRR 和 RC 三者结合，既能衡量最终成功，又能剥离恢复能力与效率。
- **工具优先的任务合成范式**：先构造 DAG 再生成自然语言，既保证了语义合理，又实现了解决方案的穷举枚举。
- **大规模且公平的实验**：覆盖多种前沿模型和提示策略，扰动实施确定且一致，案例研究丰富。
- **揭示关键洞察**：明确指出动态重规划是独立于模型规模的核心能力，为后续研究指明了方向。

## 不足与局限
- **受控拓扑环境**：任务 DAG 为程序化生成，虽通过反向验证保证语义自然，但与完全开放的真实网络环境仍有差距，可能无法完全代表真实世界中无预设结构的故障场景。
- **故障分类有限**：仅覆盖基本的二元维度，未包含级联故障、对抗攻击等更复杂的复合故障。
- **任务规模与世界模型**：基准规模为 400 基础任务，工具 270 个，领域虽多但模拟覆盖度仍无法穷尽真实 API 生态。
- **依赖 LLM 生成任务**：任务质量受生成模型能力影响，尽管有验证步骤，可能仍存在某些隐藏的偏差或不自然之处。
- **评估仅限工具调用推理**：尚未将多轮对话、长期记忆或更复杂的多智能体协作纳入考虑。
