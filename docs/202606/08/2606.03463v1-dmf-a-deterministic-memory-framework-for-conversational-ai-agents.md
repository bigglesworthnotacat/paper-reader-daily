## 摘要
对话式AI智能体需要既具可扩展性，又在长交互时间范围内保持语义连贯的记忆系统。现有方法在写入时主要依赖基于大语言模型（LLM）的摘要，这引入了非确定性、不断攀升的token成本以及剪枝决策的不透明性。我们提出确定性记忆框架（DMF），这是一种CPU优先的方法，用基于经典NLP分析、向量几何和数学评分的完全确定性流水线取代生成式记忆压缩。DMF为每次对话交互分配一个生存分数$Ω$，该分数由确定性内容信号、对话线索和结构化来源计算得出，并通过逻辑投影组合。交互计数衰减律（记为$Ω_{\mathrm{eff}}(Δn)$）控制着相关性如何随着新对话轮次的加入而演变，其中$Δn$是较新交互的数量而非墙上时钟时间，从而保持了完全的确定性。我们给出了DMF的数学表述、其结构化召回流水线、剪枝决策流程以及评估方案。实验在基于LoCoMo和LongMemEval数据集专门构建的基准上进行。我们将DMF与Mem0（一种流行的AI智能体记忆层）进行了比较。DMF在准备记忆上下文时使用了零个token，在整个对话过程中使用的token数量减少了5倍至242倍，同时取得了可比的准确率。这些结果表明，从记忆管理循环中消除LLM调用是可能的，能将token成本降至几乎为零，并为对话式AI智能体实现确定性记忆系统。

## Abstract
Conversational AI agents require memory systems that are both scalable and semantically coherent across long interaction horizons. Existing approaches rely predominantly on large language model (LLM)-based summarisation at write time, which introduces non-determinism, escalating token costs, and opacity in pruning decisions. We present the Deterministic Memory Framework (DMF), a CPU-first approach that replaces generative memory compression with a fully deterministic pipeline grounded in classical NLP analysis, vector geometry, and mathematical scoring. DMF assigns each conversational interaction a Survival Score $Ω$ computed from deterministic content signals, conversational cues, and structured provenance, combined through a logistic projection. An interaction-count decay law, denoted as $Ω_{\mathrm{eff}}(Δn)$, governs how relevance evolves as new turns arrive, where $Δn$ is the number of newer interactions rather than wall-clock time, preserving full determinism. We present the mathematical formulation of DMF, its structured recall pipeline, the pruning decision procedure, and the evaluation protocol. Experiments are conducted on a purpose-built benchmark using the LoCoMo and LongMemEval datasets. We compare DMF against Mem0, a popular memory layer for AI agents. DMF achieves comparable accuracy while using zero tokens to prepare the memory context and 5x to 242x fewer tokens over the entire conversation. These results show that it is possible to eliminate LLM calls from the memory-management loop, reducing token costs to nearly zero and enabling deterministic memory systems for conversational AI agents.

### 1. 论文的核心问题与整体含义

*   **核心问题**：现有对话式AI智能体的记忆管理，主要依赖在写入时由大语言模型（LLM）进行生成式摘要。这种方法存在三个重大缺陷：
    *   **非确定性**：对同一段历史对话，多次调用LLM可能生成不同的摘要，导致记忆状态不可复现，系统难以调试。

    *   **决策不透明与语义漂移**：无法追溯为何某个事实被保留或丢弃，且一旦摘要形成，其语义便被固化，无法随模型或上下文演变而更新。
*   **整体含义**：本文提出了一种从根本上颠覆现有范式的方案，即用一个完全确定性的、基于CPU的数学流水线，取代LLM生成式记忆压缩，从而在保证记忆连贯性和准确性的同时，将记忆管理的token成本降至零。

### 2. 论文提出的方法论

DMF的核心思想是通过经典NLP分析、向量几何和数学评分机制，构建一个端到端的、无LLM调用的记忆管理流水线。

*   **核心思想：确定性评分与衰减**
    *   系统为每次对话交互计算一个**生存分数（Survival Score, Ω）**，并基于`交互计数`（而非墙上时钟时间）对其进行指数衰减，最终得分`Ω_eff`决定了记忆条目的去留。整个过程完全由算法和配置参数决定，无任何随机性。

*   **关键技术细节**
    *   **特征提取流水线（无LLM）**：
        *   **信息密度 (`ID`)**：使用spaCy分析词性，计算承载语义的词汇（名词、动词、形容词等）占比。
        *   **情感强度 (`|S|`)**：使用VADER情感分析工具，取情感复合得分的绝对值。
        *   **命名实体数量 (`E_norm`)**：使用spaCy识别并统计命名实体，并进行饱和归一化处理。
        *   **语义发散度 (`D`)**：计算当前对话嵌入向量与近期对话滑动窗口内平均向量的余弦距离，用于识别话题漂移。
        *   **对话/语用信号**：通过规则适配器提取确定性信号，如约束、偏好、纠正、当前状态等。
    *   **生存分数 (Ω) 计算**：
        *   通过一个线性聚合公式将上述内容信号、操作通道（对话信号）和来源通道（如用户纠正、偏好更新等元数据）相加，得到总预激活值 `z_total`。
        *   然后通过一个以`x0`为中心的**逻辑Sigmoid函数**将其投影到`(0, 1)`区间，得到`Ω`。
    *   **基于交互计数的衰减模型**：
        *   定义`Δn`为“比当前条目更新的交互数量”，以此作为衰减轴，确保记忆状态完全由消息序列决定。
        *   引入**分数依赖的惯性机制**：高质量条目（高`Ω`）的衰减速率更慢。有效衰减率`λ_eff = λ(1 - ηΩ)`。
        *   最终有效分数公式：`Ω_eff(Δn) = Ω · exp(-λ · (1 - ηΩ) · Δn)`。条目根据`Ω_eff`被划分为`HEALTHY`、`UNSTABLE`和`CRITICAL`三个等级。
    *   **剪枝机制**：
        *   **周期性硬清除**：移除所有`Ω_eff`低于绝对阈值`Ω_kill`的条目。
        *   **Token预算压力剪枝**：当活跃记忆的token数超限时，从非`HEALTHY`条目中计算复合剪枝分数 `S_prune = Ω_eff + B_ret - P_sup`。其中`B_ret`是保留加分（对约束、偏好等操作信号），`P_sup`是新话题取代旧话题的惩罚分。按此分数从低到高移除，直至满足预算。
    *   **长期记忆与召回**：
        *   被清除的条目以其**原始文本记录**形式存入长期记忆（LTM）作为权威来源，保证存储的稳定性。
        *   可同时生成确定性的摘要卡片作为辅助检索工具，但卡片必须链接回原始记录。
        *   查询时进行**结构化召回**：包含查询理解、多通道候选生成（语义/词法搜索原始记录和卡片）、基于可回答性的重排序以及证据组装，最终将原始支持性对话渲染进提示词中。

*   **零Token记忆管理是可行的**：DMF在准备记忆上下文时使用了**零个Token**，在整个对话过程中，其内部记忆管理的Token消耗为0。

*   **方法创新性**：第一个提出并实现完全确定性、零LLM调用的对话记忆管理框架，从根本上解决了非确定性和高成本问题。
*   **数学完备性**：框架设计严谨，从评分、衰减到剪枝均有清晰的数学定义，确保了系统的可复现性、可调试性和可审计性。
*   **设计巧思**：“交互计数”衰减轴解决了墙上时钟衰减带来的不确定性；“分数依赖惯性”实现了不同质量记忆的差异化留存；“原始文本为唯一权威来源”的设计保证了信息在模型升级或流水线变更时不会失真。

### 8. 不足与局限

*   **确定性规则的扩展性挑战**：特征提取（尤其是语用信号如“偏好”、“纠正”）高度依赖语言特定的规则适配器。目前仅支持英语，向多语言扩展需要大量手工编写规则的工程工作，可能难以覆盖所有语言的复杂表达。
*   **评分参数的泛化性**：生存分数`Ω`的权重（如`α, β, γ`）是静态配置的，虽然可通过配置文件调整，但其最优值可能在不同领域（如技术客服 vs. 情感陪伴）的对话中有很大差异，缺乏自适应校准能力。
*   **评估偏差风险**：使用LLM-as-Judge以及设置宽松的评估标准（如列表答案部分匹配即算正确，日期误差14天内算正确）可能会高估绝对准确率，但其在统一标准下对比DMF与Mem0的结果仍具说服力。
*   **复杂推理能力上限**：DMF本身不进行推理，只负责筛选和组织证据，最终推理能力仍受限于下游LLM。论文未讨论当需要高度抽象或整合海量历史信息形成新见解时，DMF的海量原始证据提供方式是否会撑爆LLM的上下文窗口或使LLM迷失重点。
