## 摘要
LLM智能体越来越多地在决策时检索外部精心设计的技能——程序性指令——以提高在长程交互任务上的表现。现有的技能库通常被视为与模型无关，在不同能力和行为差异显著的骨干模型间重复使用相同的技能表述。然而，我们在多个模型规模上的对照实验表明，技能的有效性强烈依赖于模型：有利于某个骨干模型的技能可能损害另一个。受此观察启发，我们提出MASA（模型感知技能对齐），一个在无需修改智能体权重的情况下将技能适配到每个目标骨干模型的框架。MASA分为两个阶段：（1）一个分层技能进化流水线，利用爬山法和UCB驱动的树搜索迭代重写通用和任务特定技能，由环境反馈和模型能力概况指导；（2）一个轻量级的模型条件技能重写器，在进化轨迹上训练，以单次前向传播复现适配过程。在三个交互环境和四个骨干模型上的实验表明，MASA持续取得最佳综合性能，相比最强基线提升高达25.8分。学习到的重写器进一步泛化到未见任务和环境而无需额外搜索，以极低的推理成本持续超越更大规模的教师LLM。

## Abstract
LLM agents increasingly retrieve externally curated skills-procedural instructions retrieved at decision time-to improve performance on long-horizon interactive tasks. Existing skill libraries are typically treated as model-agnostic, reusing the same skill formulations across backbones with substantially different capacities and behaviors. However, our controlled experiments across multiple model scales show that skill effectiveness is strongly model-dependent: a skill that benefits one backbone can harm another. Motivated by this observation, we propose MASA Model-Aware Skill Alignment, a framework that adapts skills to each target backbone without modifying agent weights. MASA operates in two stages: (1) a hierarchical skill evolution pipeline that iteratively rewrites general and task-specific skills using hill climbing and UCB-driven tree search, guided by environment feedback and model capability profiles; and (2) a lightweight model-conditioned skill rewriter trained on evolution trajectories to reproduce the adaptation in a single forward pass. Experiments across three interactive environments and four backbones show that MASA consistently achieves the best overall performance, with gains of up to 25.8 points over the strongest baseline. The learned rewriter further generalizes to unseen tasks and environments without additional search, consistently outperforming a much larger teacher LLM at a fraction of the inference cost.

### 1. 核心问题与整体含义

本论文指出了一个LLM智能体（LLM Agents）研究领域中被广泛忽视的关键问题：**“一刀切”的技能库假设是脆弱的**。

*   **研究背景**：当前LLM智能体普遍采用“检索增强式技能”范式，即在决策时从外部库中检索预定义的技能（程序性指令）来提升任务表现。
*   **核心矛盾**：现有的技能库设计通常是“模型无关”（Model-Agnostic）的，同一个技能库被不加区分地用于不同规模、不同能力甚至不同家族的骨干模型。
*   **关键发现**：作者通过初步实验（图1）证明，技能的有效性**高度依赖模型**。对于某个模型（如Qwen3-8B）最优的技能粒度（Granularity），对另一个模型（如Qwen3-32B）可能并非最优，甚至会**显著损害**其性能。这表明，当技能形式与模型不匹配时，检索到的技能不仅没有帮助，反而会分散模型的注意力，干扰其固有的推理模式。
*   **整体含义**：因此，技能设计必须从“模型无关”转向“模型感知”（Model-Aware），需要为特定的骨干模型定制化地适配技能，以放大其优势，解锁其在通用技能下无法展现的能力。

### 2. 方法论：MASA框架

针对上述问题，论文提出了**MASA**（Model-Aware Skill Alignment，模型感知技能对齐）框架。其核心思想是在不修改智能体模型权重的前提下，将技能库的**表述形式**对齐到目标模型的特性和能力上。

MASA包含两个互补的阶段：

*   **阶段一：分层模型条件技能进化（Hierarchical Model-Conditioned Skill Evolution）**
    *   **定义**：这是一种由搜索驱动的、借助更强“教师”LLM（Teacher LLM）的技能改写过程。该过程通过环境反馈（Episodic Reward）来判断改写是否成功。
    *   **两级技能库结构**：
        *   **通用技能**：跨任务的策略原则（如“在执行操作后总是检查状态”）。
        *   **任务特定技能**：面向特定任务类型的领域程序（如ALFWorld中“加热物体”的具体步骤）。
    *   **两阶段优化策略**：
        1.  **通用技能优化**：采用**迭代爬山法（Iterative Hill Climbing）**。每次迭代收集失败轨迹，由教师LLM结合失败归因、历史最佳技能集和**模型卡片**（Model Card，包含骨干模型架构、能力优缺点等结构化信息）来重写通用技能集，并通过在全任务集上的评估来决定是否接受。
        2.  **任务特定技能优化**：采用**UCB驱动的树搜索（UCB-Driven Tree Search）**。为每种任务类型独立构建一棵搜索树，每个节点代表一套候选技能。通过平衡探索与利用的UCB1算法选择要展开的节点，再由教师LLM进行重写、评估和反向传播。
*   **阶段二：轻量级模型条件技能重写器（Model-Conditioned Skill Rewriter）**
    *   **动机**：进化搜索虽强，但推理成本高昂且依赖环境交互。重写器的目标是**学习并固化**这种搜索策略，实现零开销的快速适配。
    *   **训练**：以一个小型骨干模型（论文用Qwen3-4B）为基础，使用进化流水线中产生的（初始技能 -> 最优技能）轨迹数据进行监督微调（SFT）。训练样本经过精心构造，包含不同质量、来自不同模型、甚至带噪声的技能，增强了重写器的鲁棒性和泛化能力。
    *   **推理**：部署时，给定目标模型卡片、输入技能集和任务描述，重写器在单次前向传播中即可产出适配后的最优技能，无需任何环境交互。
