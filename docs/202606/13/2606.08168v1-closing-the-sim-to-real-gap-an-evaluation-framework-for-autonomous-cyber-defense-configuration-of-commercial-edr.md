## 摘要
领先的商业终端检测与响应（EDR）产品已从操作员配置的规则集转变为多组件系统，其中自主AI组件与操作员部署的策略并存，甚至越来越多地取而代之。使用商业EDR作为加固手段的自主防御代理不再调校被动工具，而是操控一个能做出供应商特定决策的黑箱自主系统。我们提出了首个针对硬化商业EDR的自主防御代理评估框架，并在活动目录游戏（GOAD）实验环境中实例化，采用Horizon3.ai的NodeZero作为自主渗透测试方，Microsoft Defender XDR作为EDR。我们运行了基于两个大语言模型（LLM）骨干（Claude Sonnet 4.6与Cisco Foundation-Sec-8B）的防御代理基准测试样本。我们总结出三条无法通过模拟或开源EDR评估揭示的经验教训：（i）商业EDR遥测是为安全运营中心（SOC）分析师工作流而非科学基准测试设计的；（ii）按策略归因以区分防御代理行为与自主EDR行为的重要性；（iii）EDR的自主行为在评估窗口期内会发生变化。这些发现共同突显了企业防御中仿真与现实的差距，并推动在配有黑箱自主工具的环境中制定评估方法以基准测试自主防御代理。

## Abstract
Leading commercial endpoint detection and response (EDR) products have shifted from operator-configured rule sets to multi-component systems where autonomous AI components operate alongside, and increasingly in place of, operator-deployed policies. Autonomous defense agents using commercial EDR as their hardening tool are no longer tuning a passive tool, but a black-box autonomous system capable of making vendor-specific decisions. We present the first evaluation framework for autonomous defense agents hardening commercial EDR. We instantiate it in a Game of Active Directory (GOAD) lab with Horizon3.ai's NodeZero as the autonomous pentester and Microsoft Defender XDR as the EDR. We run a sample benchmark of defense agents with two large language model (LLM) backbones (Claude Sonnet 4.6 and Cisco Foundation-Sec-8B). We report three lessons learned that neither simulation nor open-source-EDR evaluation can surface: (i) commercial EDR telemetry is engineered for Security Operations Center (SOC) analyst workflows rather than scientific benchmarking; (ii) the importance of per-policy attribution to separate defense agent actions from autonomous EDR actions; and (iii) the EDR's autonomous behavior varies during the evaluation window. Together, these findings highlight a sim-to-real gap for enterprise defense and motivate evaluation methodology for benchmarking autonomous defense agents in environments with black-box, autonomous tools.

### 6. 论文的主要结论与发现

- **经验教训一：商业 EDR 遥测不是为科学基准测试设计的。**
  - ASR 规则事件被每小时每进程节流，且报告时间戳为首次出现时间，导致日志无法准确匹配攻击动作。
  - 许多 ASR 规则触发在良性系统进程上（如 `svchost.exe`），而非攻击者工具，若不经过程序级归因会误判策略效果。
  - 有约 13% 的 NodeZero 检测到的 EDR 干预并未出现在相应遥测中。
- **经验教训二：按策略归因是将防御代理行动与 EDR 自主行为分离的关键。**
  - 在所有后基线测试中，操作员部署的 ASR 规则产生的阻断仅占 0.7%–1.8%（Claude/Cisco），而 EDR 内置 AIR 生成的 `ContainedUser*` 阻断占 28.6%–100%，说明 EDR 自主层输出了主导地位。
  - 若只汇总总阻断数，将完全抹除代理的实际贡献，也无法发现 ASR 规则大多打在错误进程上的事实。
- **经验教训三：EDR 的自主动态在评估窗口内会变化。**
  - AIR 的阻断量和行为类型在无任何人工变更的情况下，9 天内自行增长了 8.9 倍，且行动词汇扩展。
  - AIR 仅针对单一用户账号（`robb.stark`）有效，对其他后续横向移动账号无阻断，显现出上下文选择性。
  - 这意味着基线特性难以固定，防御代理的效果测量结果与 EDR 内部状态和时间高度相关。
- **整体结论**：在商用 EDR 环境中，防御代理面对的硬化工具不再是被动的规则界面，而是一个自主的、不断演化的多智能体系统。正确评估这类代理需要颗粒度的策略归因、攻击者侧验证，以及将 EDR 自主组件视为实验中的一个动态变量来管理。

### 7. 优点

- **现实相关性极强**：直接集成商业 EDR 和商业渗透测试工具，首次研究真实黑箱环境下的自主防御代理评估，填补了模拟/开源 EDR 与现实之间的空白。
- **框架设计合理**：闭环实验控制器模块化，Proposer 可插拔，支持不同 LLM 和代理架构，具有较好的扩展性。
- **多维遥测关联**：融合攻击侧遥测、Defender Advanced Hunting 和 Windows 本地事件日志，实现了细颗粒度的攻防事件对齐和策略归因。
- **发现具有洞察力**：三条教训从工程实践角度深刻揭示了 EDR 日志本质、归因需求以及自主行为的时间漂移，这些是实验室模拟中无法观察到的。
- **透明承认限制**：明确说明单 EDR、单拓扑、手动清理环节、AIR 不可控噪声等限制，并指出未来解决的开放问题，非常客观。

### 8. 不足与局限

- **实验覆盖面窄**：只测试了一个 EDR 厂商（Microsoft Defender XDR）、一个实验室拓扑（GOAD）、一个攻击平台（NodeZero），外部效度尚不确定。
- **防御代理动作空间有限**：仅使用了少数 ASR 规则、LSA 保护等，未触及更复杂的配置项（例如，排除列表、AIR 自动化级别调整），也排除了依赖较高许可或硬件的策略。
- **自动清理不完善**：用户遏制（user containment）无法通过 API 解除，需手动干预；AIR 状态清理虽已部分实现，但仍有残留影响。
- **成本与可重复性**：商业工具（NodeZero、Microsoft 365 E5 许可）和云实验室费用较高，限制了研究者在资源有限情况下的复现。
- **评估指标侧重定性**：虽然提供了大量遥测表格，但对防御代理的“性能”并未给出单一量化分数，更多围绕框架能力与挑战展开讨论。
- **噪声控制**：EDR 自主行为波动、渗透测试路径不完全确定性等无法完全控制，可能引入结果偏差，文中有考虑但对小样本实验仍是一个局限。
