## 摘要
技能文档是用结构化自然语言指导大语言模型（LLM）智能体的说明书，对现代智能体框架至关重要，然而LLM在编写实际可用的技能方面表现挣扎。在SkillsBench上，人类撰写的技能将通过率提升了16.2个百分点，而LLM撰写的技能没有带来可衡量的提升。我们提出了SkillAxe，一个完全无监督的框架，使LLM能够迭代地诊断和完善自己的技能。SkillAxe将技能质量分解为四个可解释的维度（质量影响、触发精度、带故障归因的指令合规性以及解决路径覆盖），生成结构化的改进简报，无需真实标签、测试套件或环境奖励。在SkillsBench上，SkillAxe将LLM技能的通过率相对提升了28%，并弥合了与人类技能差距的47–67%。我们在SpreadsheetBench的野外观测中验证了该方法作为持续改进引擎的有效性：SkillAxe构建的技能库从过去的智能体轨迹中学习，仅使用22个技能便将通过率从16.0%提高到了52.0%。

## Abstract
Skill documents, structured natural-language instructions that guide Large Language Model (LLM) agents, are critical to modern agent frameworks, yet LLMs struggle to write skills that actually work. On SkillsBench, human-authored skills improve pass rates by 16.2 percentage points, while LLM-authored skills provide no measurable gain. We introduce SkillAxe, a fully unsupervised framework that enables LLMs to iteratively diagnose and refine their own skills. SkillAxe decomposes skill quality into four interpretable dimensions (quality impact, trigger precision, instruction compliance with fault attribution, and solution-path coverage), producing structured improvement briefs that require no ground-truth labels, test suites, or environment rewards. On SkillsBench, SkillAxe improves pass rates by 28\% relative over unimproved LLM skills and closes 47--67\% of the gap to human-authored skills. We validate the approach as a continuous improvement engine in the wild on SpreadsheetBench, where a SkillAxe-built skill library learns from past agent trajectories and raises pass rate from 16.0\% to 52.0\% using only 22 skills.

### 2. 论文提出的方法论

SkillAxe 的核心思想是：**将技能本身隐含的行为约束转化为可评估的诊断信号，通过对有无技能的智能体输出进行配对比较，定位技能缺陷并提供可操作的改写建议**。整个过程不依赖外界监督，仅使用任务指令 q、无技能的响应 r₀、有技能的响应 rₛ 以及技能文本 s。

#### 2.1 四维诊断指标

1. **质量影响（Quality Impact）**：  
   - 先通过 LLM 评判器比较 rₛ 与 r₀ 的偏好方向 d ∈ {−1, 0, +1}，再估计改善幅度 m ∈ [0, 1]，得出质量分数 Quality(s, τ) = d·m ∈ [−1, 1]。  
   - 它作为外层评估信号，用于判断技能是否产生了正面、负面或零影响，为后续细化提供方向（小幅退化可局部改写，灾难性失败则需彻底重写）。

2. **触发精度（Trigger Precision）**：  
   - 从技能描述中抽取正向触发短语 P⁺（应激活）和负向触发短语 P⁻（不应激活），通过依存分析处理否定逻辑，并用上下文感知的嵌入向量（结合短语本身及其在技能描述中的语境）构造激活区域和排斥区域。  
   - 计算三个几何量：  
     - **覆盖广度（Coverage Breadth）**：正向短语之间的平均余弦距离，衡量触发是否过于狭窄。  
     - **负向特异性（Negative Specificity）**：每个负向短语到最近正向短语的平均距离，衡量排斥区域的分离程度。  
     - **边界锐度（Boundary Sharpness）**：正负短语间的最小距离，识别最危险的模糊边界。  
   - 这些指标联合诊断触发过窄、排斥不足和决策边界模糊等三类失败模式。

3. **指令合规性与故障归因（Instruction Compliance with Fault Attribution）**：  
   - 将技能自动分解为可评估的规则列表 {Rᵢ}，并赋予严重性权重 wᵢ（1/2/3）。  
   - 对每条规则，LLM 评判器在给定任务指令和实际输出文件的基础上，评估：相关性 ρᵢ、遵守度 aᵢ、规则质量 gᵢ（具体性和可操作性），以及技能故障归因 fᵢ（未遵守时，多大概率是技能规则本身的问题）。  
   - 定义原始合规度 Compliance 和经过故障调整的技能信用 ci = aᵢ + (1 − aᵢ)(1 − fᵢ)，最终得到技能质量评分 SkillScore(s, τ)。  
   - 这一区分保证了正确但被智能体忽略的规则不会被错误修改，避免“修坏”原本有效的技能。

4. **解决路径覆盖（Solution‑Path Coverage）**：  
   - 对任务指令 q，LLM 枚举若干可行的解决路径 {Pⱼ}，再计算技能文本各块（chunk）嵌入与每条路径描述嵌入的最大余弦相似度的均值。  
   - 低覆盖表明技能只支持极窄的操作策略，智能体即使正确触发并意图执行，也可能因缺少适配的引导而失败。

#### 2.2 改进流程

1. **单轮迭代**：将四个诊断指标综合为一个结构化改进简报，送入 LLM 重写器，生成更新后的技能文本。循环执行直至收敛（实验中仅使用一次迭代即获得显著提升）。  
2. **持续构建技能库（SpreadsheetBench 场景）**：利用嵌入路由（嵌入 3 区决策：FORK/IMPROVE/SKIP）对训练任务聚类，通过两阶段改进（先基于触发和路径覆盖的诊断，再基于来自智能体轨迹的质量影响与合规性诊断）不断压缩并强化技能库，最终形成一个 22 技能的精简高激活库。

### 6. 论文的主要结论与发现

- **SkillAxe 可显著且可靠地提升 LLM 自生成的技能**：在 SkillsBench 上相对提升 28%，缩小与人类技能差距的 47–67%。

### 7. 优点

- **完全无监督，无需 ground‑truth 标签、测试套件或环境奖励**，仅利用任务指令和智能体输出即可诊断和改进技能，具有很强的实际部署可行性。  
- **四维可解释的诊断**将技能失败原因拆解为触发问题、指令模糊、智能体不遵守、路径覆盖窄等可操作类别，使改进过程透明、定向。  
- **故障归因机制**精细区分了“技能规则本身有问题”与“智能体未遵循规则”，避免错误修改正确规则，这是与前人工作的重要区别。  
- **持续改进引擎的设计**展示了从历史轨迹中不断积累和压缩技能库的能力，对生产环境中长期运行的智能体系统具有直接价值。  
- **实验设计双轨**：受控基准（SkillsBench）+ 野外连续学习（SpreadsheetBench），并辅以双评价协议，增强了结论的鲁棒性。

### 8. 不足与局限

- **结构对齐盲区**：规则级别的精炼无法检测到技能内部逻辑自洽但整体策略完全错误的情况，可能使错误策略被固化。  
- **技能间交互未建模**：当前方法独立评估每个技能，无法捕捉多技能同时激活时的冲突或相互干扰。  
- **对 LLM 评判器的依赖**：合规性评估和公平评分均依赖 LLM（Claude / GPT），这些评判器可能存在位置偏差、校准偏差或系统性偏好，一定程度上影响诊断准确性。  
- **迭代深度有限**：实验仅使用一次迭代改进，剩余与人类技能的差距可能需更复杂的在线执行反馈和多轮迭代，但论文未深入探索。  
- **SkillsBench 重复次数不足**：每任务仅 2 次运行（k=2），导致置信区间较宽，统计显著性不够强；而对于 SpreadsheetBench 的连续学习场景，训练/测试划分固定（200/50），也未进行多次随机分割的鲁棒性检验。  
- **泛化到其他领域的证据尚缺**：虽然 SpreadsheetBench 提供了一定泛化性，但整体仍限于特定领域的工具调用任务，其方法在开放式对话、代码生成等多类实际应用中的表现有待验证。
