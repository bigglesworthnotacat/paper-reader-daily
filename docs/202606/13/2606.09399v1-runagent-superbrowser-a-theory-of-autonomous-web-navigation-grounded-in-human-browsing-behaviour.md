## 摘要
我们提出了 SUPERBROWSER，一个基于单一指导假设设计的自主网络导航代理：网络代理应以人类浏览的方式进行浏览。阅读页面的人不会保留他们看到的每个像素；他们会查看几个候选目标，决定其中一个，并仅记住保持目标所需的内容。我们将这一感知-认知-行动三元组操作化为三个耦合机制。首先，视觉优先的边界框管道在每个截图上标记候选交互区域，并将它们异步预取输入语言模型，使得“眼睛”先于“手”。其次，三角色大脑——一个进行分类和路由的协调器，一个每隔几步评估进展的规划器，以及一个发出每一步动作的执行器——将战略推理与操作推理分离。第三，结构化的账本仅存储人类会存储的内容：目标、最后三个动作、一组少量事实和死胡同，以及少量检查点；一个六阶段的驱逐循环系统地从实时上下文中丢弃过时的截图、状态块和推理轨迹。动作执行是一个三层点击级联（Chrome DevTools 协议到 Puppeteer 再到脚本化），带有人性化的贝塞尔运动，加上一个识别尖括号的边界框捕捉器，解决了“大标签旁的小箭头”的歧义。在 Mind2Web Hard 基准测试（66 个任务）中，SUPERBROWSER 取得了 89.47% 的成功率，总排名第三，并以大幅优势领先于所有已发布的开源/研究浏览器代理基线。我们认为，这种增益并非来自任何单一技巧，而是来自整个系统中认知契约的一致应用。

## Abstract
We present SUPERBROWSER, an autonomous web-navigation agent designed against a single guiding hypothesis: a web agent should browse the way a person browses. A human reading a page does not retain every pixel they have seen; they look at a few candidate targets, decide on one, and remember only what is needed to keep the goal alive. We operationalize this perception-cognition-action triad as three coupled mechanisms. First, a vision-first bounding-box pipeline labels candidate interactive regions on every screenshot and feeds them, asynchronously prefetched, to the language model so that the "eye" precedes the "hand". Second, a three-role brain -- an Orchestrator that classifies and routes, a Planner that evaluates progress every few steps, and a Worker that emits per-step actions -- separates strategic from operational reasoning. Third, a structured Ledger stores only what a person would: the goal, the last three actions, a small set of facts and dead-ends, and a handful of checkpoints; a six-phase eviction loop systematically discards stale screenshots, state blobs, and reasoning traces from the live context. Action execution is a three-tier click cascade (Chrome DevTools Protocol to Puppeteer to scripted) with humanized Bezier motion, plus a chevron-aware bounding-box snapper that resolves the "small arrow beside a large label" ambiguity. On the Mind2Web Hard benchmark (66 tasks), SUPERBROWSER attains 89.47% success, placing third overall and ahead of every published open/research browser-agent baseline by a large margin. We argue that the gain comes not from any single trick but from the consistent application of a cognitive contract throughout the system.

## 1. 核心问题与整体含义

- **研究背景**：当前基于 LLM 的自主网页代理常采用「朴素积累」策略——将每次交互的截图、元素列表、推理链全部追加到上下文，导致长程任务中提示词爆增、注意力漂移、缓存命中率暴跌，可靠性急剧下降。
- **核心假设**：一个网页代理应按人类浏览的方式运作——人类不会记住页面上每个像素，而是扫视少数候选区域，做出决策，仅保留维持目标所必需的信息。
- **整体含义**：论文提出一套系统化的「认知契约」，将人类浏览的感知–认知–行动三元组可操作化，并严格约束记忆和组织架构，从而在复杂、多步网络任务中实现稳定、高效的自主导航。

## 2. 方法论

论文将人类浏览的感知–认知–行动三元组映射为三大耦合机制，并辅以结构化的记忆和拟人化执行系统。

### 2.1 感知：视觉优先的边界框管道
- **视觉标注**：每次截图送入多模态视觉模型，输出交互候选区域的边界框 `[V0], [V1]...`，格式为归一化坐标。
- **异步预取**：变异动作后立即调度背景视觉调用，使下一 LLM 推理时可直接使用缓存结果，消除约 900 ms 延迟。
- **DOM 富化**：桥接层为每个边界框补充 DOM 索引、`aria-expanded` 状态、活动标记、`aria-labelledby` 文本等信息。
- **复合行拆分**：当单个边界框同时包含标签文字和扩展箭头（如「United States ▼」）时，自动拆分为两个候选框。
- **跨回合感知锚定**：将上轮边界框以虚线彩框覆盖到当前截图，让视觉模型在比较后保留稳定元素，减少标签漂移。
- **子目标感知强调**：若已知当前子目标，视觉模型仅对服务于该子目标的元素标记 `intent_relevant=true`，帮助 Worker 聚焦。
- **页面类型感知分级**：复杂页面（如过滤表单）检测后自动升级至更大的视觉模型，避免闪存级模型截断。
- **验证触发的补全扫描**：当动作验证器报告关键标签缺失时，放宽边界框上限进行二次覆盖扫描，补救遗漏。

### 2.2 认知：三角色大脑
- **Orchestrator（协调器）**：策略层，通过确定性动词分类器（基于正则模式匹配）将用户任务路由至搜索 Worker（只读聚合）或浏览器 Worker（事务/视觉任务），并维护按域学习的经验。
- **Planner（规划器）**：战术层，每 N 步（默认 4）或 Worker 报告可能完成时被调用，评估进度并输出观察、挑战、是否完成、下一步计划和最终答案。
- **Worker（执行器）**：操作层，仅看到当前页的 `[Vn]` 边界框视图和精简后的账本，发出具体动作序列（最多 5 个动作后重新评估）。

### 2.3 记忆：结构化账本与六阶段驱逐循环
- **结构化账本**：包含目标、计划、当前子目标、最近 3 步动作队、事实词典、死胡同列表、检查点列表和临时笔记。账本作为权威状态，即使上下文被裁剪也不会丢失关键信息。
- **六阶段驱逐**：每次 LLM 调用前执行，依次进行：
  1. 截图回溯补丁：仅保留最近 2 张截图，旧截图替换为文本描述。
  2. 失败折叠：仅保留最新一次失败详情，旧失败压缩为一行死胡同记录。
  3. 状态块驱逐：清除过时的 `[SESSION STATE]` 块，除非与当前页面相同的“卡滞”信号。
  4. 思考块剥离：删除旧助理消息中的推理/思考内容，保留工具调用/结果配对。
  5. 旧消息剔除：若消息总数超过 30，将该窗口外的消息体替换为 `[archived]`。
  6. 元素列表折叠：仅保留最新 `[ELEMENTS]` 块，旧块提示可重新列出。
- **角色切片视图**：Orchestrator 看到完整账本，Worker 只看到当前子目标、最近 3 步、最相关 5 条事实和当前 URL 的死胡同——实现注意力门控。

### 2.4 行动：三层点击级联与拟人化运动
- **边界框捕捉器**（带尖括号决断）：采用中心点扫描 → 网格扫描复合评分，结合面积交叠、标签匹配和尖括号权重（对 `aria-expanded`、`▼▶◀▲` 等扩展控件放大权重），解决了“大标签旁的小箭头点击无效”的长期故障模式。
- **三层点击级联**：
  - 第 1 层：通过 Chrome DevTools Protocol 调度真实 `mousePressed`/`mouseReleased`，保留 `isTrusted=true`。
  - 第 2 层：回退至 Puppeteer 的 `page.click`。
  - 第 3 层：仅当显式允许时，使用 `el.click()`（`isTrusted=false`，可能触发反爬）。
- **动作人性化**：光标沿贝塞尔曲线移动，添加高斯抖动、正弦/逻辑斯谛速度曲线、末端减速摆动、点击前犹豫；打字添加字符间延迟、语法停顿和 2% 概率的模拟错字校正；拖动使用 S 形速度轮廓，长距离有 5–15% 概率的过冲修正。
- **过程性点击**：通过 DOM 缓存和页面指纹（DOM 哈希、文本哈希、iframe 签名）判断页面是否自上次视觉调用以来未变，若未变则走便宜的「程序性路径」直接点击，无需重新截图，显著减少视觉调用。

### 2.5 直接导航恢复路径
- 当 Worker 能识别目标网站的 URL 规律时，允许直接通过 `browser navigate` 跳转到指定状态 URL，绕过繁琐的 UI 过滤链，但受域名锁定、挑战守卫、回归检测等严格保护，防止误用。

## 6. 主要结论与发现

- **主要结果**：SUPERBROWSER 在 Mind2Web Hard 上达到 89.47% 成功率，排名第三，并大幅领先最佳开源/研究基线（8.1%，SeeAct），超过 80 个百分点。
- **认知契约有效性**：成功不是单一技巧所致，而是源于系统性地贯彻人类浏览认知原则（感知–认知–行动、有限工作记忆、结构化回忆）。
- **记忆管理关键**：六阶段驱逐循环使实时上下文令牌保持近似恒定，缓存命中率保持在 80% 以上，避免线性增长带来的性能坍塌。
- **工具经济性差异**：中美前沿模型在成功率相当时，工具调用次数存在明显差距（部分中国模型调用次数多出约 1/3），并倾向于重复使用昂贵的视觉路径而非便宜的过程性点击，提示训练后分布（Post-training）对代理工具使用纪律的影响。
- **设计预言验证**：通过系统设计印证了三个可证伪预测：上下文增长受限、失败被记为负性路标、小交互子元素优于大文本标签。

## 7. 优点

- **认知启发的系统设计**：将认知科学理论（工作记忆、信息觅食理论、扫描路径等）转化为具体的系统组件，方法论清晰，且有理论深度。
- **模块化且模型无关**：记忆、分工、动作执行均可插拔，能搭配不同 LLM 和视觉模型，易于复用。
- **精细的工程细节**：异步视觉预取、过程性点击、复合行拆分、尖箭头决断、三层点击级联等人性化动作充分考虑了实际网站交互的细微之处。
- **严格的上下文管理**：六阶段驱逐循环与账本重注入在工程上实现了“稳定遗忘”，避免了上下文爆炸，极大地提升了长程稳健性。
- **开放的评估与可复现性**：承诺开放源代码和评估套件，使用公开基准。

## 8. 不足与局限

- **消融实验缺失**：未实施计划中的单组件消融，无法量化各模块（如记忆驱逐、尖箭头决断、人性化运动）对最终性能的独立贡献。

- **依赖视觉 API**：冷启动页面（视觉模型检测不到交互框）时回退至 DOM 枚举，成功下降约 15 个百分点，对非拉丁文字、极简 UI 等场景可能薄弱。
- **成本分析缺失**：未提供算力或经济成本（如推理 tokens 总开销、美元成本），对实用部署的指导意义有限。
- **未经验证的预测**：论文提出第四个预测（任务中断与恢复几乎无成本）但未实际测试。
- **模型选择对经济性的影响**：发现不同模型的工具调用效率差异很大，但其归因（Post-training）仅止于假设，缺乏严格证明。
- **直接导航的潜在风险**：虽然加了守卫，但 Worker 误猜 URL 可能导致幻觉结果的风险依然存在。
