## 摘要
本文探讨了智能体人工智能工具在网络安全领域的价值。我们评估了一个基于通用生成式人工智能大语言模型（GenAI）的代理，在驱动其运行的三种不同Ollama托管开源通用模型下的效能。通过精确率、召回率、误报数量以及基于这些指标间相互作用计算得出的综合得分，我们将每个代理的性能与现有的、经过验证的静态应用安全测试（SAST）工具Bandit的基线表现进行对比。我们的发现驳斥了以下观点：在现实条件下，基于现代开源GenAI大语言模型的代理目前适合静态应用安全测试扫描这一专门任务。

## Abstract
This paper explores the value of agentic AI tools for cybersecurity purposes. We evaluate the efficacy of a general-purpose GenAI Large Language Model- (GenAI-) based agent when powered by three different Ollama-hosted general-purpose open source models. We assess each agent's performance using precision, recall, false positive count, and a calculated composite score based upon the interplay of the captured metrics, against the baseline performance of an existing, vetted Static Application Security Testing (SAST) tool, Bandit. Our findings refute the notion that a modern open-source GenAI LLM-based agent is currently suitable for the specialized task of SAST scanning under realistic conditions.

- **关键技术设置**：
  - **代理架构**：Snitch是Python程序，递归遍历目录，提取Python文件，调用本地Ollama部署的LLM进行分析。

- **评估指标**：
  - **召回率**：针对Bandit检测到的中/高严重度问题，计算LLM代理找到的比例（完全匹配1分，部分匹配0.5分）。

### 6. 主要结论与发现

- **性能全面落后**：无论使用哪种LLM，代理的召回率均未超过0.25，而误报率高达40%～90%，综合得分均为负值。qwen2.5表现相对最好，但依然不及格。
- **幻觉问题突出**：模型频繁“伪造”漏洞所在文件和行号，甚至错误地将标准库常量或开发环境配置视为安全漏洞（如将`.env`使用视为硬编码密钥）。
- **解释性与可靠性缺失**：Bandit输出可追溯、可置信度分级；LLM常以极高置信度给出错误结果，且缺乏可解释的决策依据，加剧了开发者的不信任。
- **效率低下**：代理扫描耗时比Bandit高出两个数量级以上，若应用于实际CI/CD流程，延迟和成本均不可接受。
- **局限应用前景**：代理偶有发现Bandit遗漏的新问题，但不足以支撑其作为独立SAST工具。目前更适合作为传统工具的补充而非替代。
