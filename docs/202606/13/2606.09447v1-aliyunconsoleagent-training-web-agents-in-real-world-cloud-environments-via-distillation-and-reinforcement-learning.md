## 摘要
我们提出了AliyunConsoleAgent，一个用于真实云控制台中自动化文档验证的Web智能体框架。主流云平台涵盖数百种产品，功能迭代迅速，导致控制台界面常常与对应的文档不一致。验证文档化流程是否准确反映当前控制台并能够端到端执行，估计每年需要400万次重复检查，但人工覆盖率低于1%。虽然基于前沿专有模型构建的智能体系统取得了较高的成功率，但其高昂的成本和数据隐私限制阻碍了大规模部署。我们提出了一种两阶段训练范式：首先在蒸馏的前沿模型轨迹上进行监督微调，随后在真实云环境中使用分组相对策略优化和双通道结果奖励模型进行强化学习。为了支持大规模强化学习训练，我们构建了一个高确定性回放系统，该系统具有基于Terraform的资源预置和LLM驱动的按需置备功能，有效地将环境噪声与训练信号隔离开来。我们进一步引入了基于后端审计日志的基于规则的奖励评估协议，提供了客观且抗奖励黑客行为的结果判断。我们的模型从机械遵循指令发展到具有云控制台和产品特定理解的自主决策。在一个具有278个挑战性任务的基准测试中，最好的前沿模型仅达到65.34%的成功率，而AliyunConsoleAgent-32B实现了63.52%的平均成功率——比基模型提升了20.24个百分点，将与最好的前沿专有模型的差距缩小到1.82个百分点（自助法95%置信区间[-1.27, 7.39]），同时推理成本降低了92%。

## Abstract
We present AliyunConsoleAgent, a web agent framework for automated documentation verification in real-world cloud consoles. Major cloud platforms encompass hundreds of products with rapid feature iteration, causing console UIs to frequently diverge from their corresponding documentation. Verifying that documented procedures accurately reflect the current console and can be executed end-to-end demands an estimated 4 million recurring inspections annually, yet manual coverage remains below 1%. While agent systems built on frontier proprietary models achieve high success rates, their prohibitive cost and data privacy constraints preclude large-scale deployment. We propose a two-stage training paradigm: supervised fine-tuning (SFT) on distilled frontier-model trajectories, followed by reinforcement learning using Group Relative Policy Optimization (GRPO) and a dual-channel outcome reward model in real cloud environments. To support large-scale RL training, we construct a high-determinism rollout system featuring Terraform-based resource pre-provisioning and LLM-driven on-demand provisioning, which effectively isolates environment noise from the training signal. We further introduce a rule-based reward evaluation protocol grounded in backend audit logs, providing objective, reward-hacking-resistant outcome judgment. Our model evolves from mechanical instruction following to autonomous decision-making with cloud console and product-specific understanding. Experiments on a challenging 278-task benchmark where the best frontier model achieves only 65.34% demonstrate that AliyunConsoleAgent-32B achieves a 63.52% mean success rate -- a 20.24 percentage-point improvement over the base model, narrowing the gap to the best frontier proprietary model to 1.82 pp (bootstrap 95% CI [-1.27, 7.39]) -- at 92% lower inference cost.

- **核心问题**：云平台控制台界面因快速迭代与文档之间缺乏自动同步，导致“结构性文档漂移”（documentation drift）。用户按照文档操作时，步骤常与当前界面不匹配。据估计，要保持文档时效性，每年需进行约400万次重复人工检查，但当前人工覆盖率不足1%。
- **整体含义**：用自动化 Web 智能体执行端到端文档验证是紧迫的行业需求。基于前沿闭源模型的系统虽然成功率高，但推理成本极高且存在数据隐私合规风险（需将含敏感云元数据的上下文传至外部服务）。因此，需要训练一个可以本地部署、成本可控的 32B 轻量模型，使其在真实云控制台环境中逼近前沿模型的性能。

### 2. 方法论

#### 核心思想
采用 **SFT + GRPO** 的两阶段训练范式，让模型从“机械指令跟随”进化为“自主决策”。同时，通过构建高确定性回放环境和基于审计日志的奖励机制，解决真实云环境中的资源依赖和环境噪声问题。

#### 关键技术细节

- **问题形式化**：将智能体-控制台交互建模为部分可观测马尔可夫决策过程（POMDP）。状态同时包含可见前端 UI 和隐藏的后端资源配置（如 VPC、IAM 权限）。引入“按需置备动作”（\(a_{env}\)）来稳定动态转移。观测 \(z_t\) 包含：Set-of-Mark 标注截图 \(I_t\)、提取的 DOM 文本 \(E_t\)、动作历史 \(H_t\)。

- **WebAgent 框架**：遵循 ReAct 范式（观察—推理—动作循环）。动作为四选一：GUI 交互（通过 Playwright + 无头 Chromium 执行）、`task_complete`（声明成功）、`task_unachievable`（无法继续）、\(a_{env}\)（触发外部 ResourceCoder 按需置备资源）。按文档子步骤分配交互预算（每子步 40 步）。

- **高确定性回放（Rollout）环境**（核心贡献）：
  - 四层架构：账户池管理、沙箱执行（容器化 + StatefulSet）、资源 META 离线置备、运行时按需置备。
  - **资源 META** 五元组：\( \langle D, K, C_{\text{create}}, C_{\text{verify}}, C_{\text{destroy}} \rangle \)，包含依赖描述、关键属性配置（如 ECS 实例规格、可用区）、Terraform 创建/验证/销毁脚本。通过 LLM 驱动的流水线离线生成，并支持自纠正。
  - **运行时 ResourceCoder**：当智能体发现环境阻塞（如空列表）时触发，动态写 Terraform/CLI 脚本置备缺失资源，并将资源报告注入上下文；成功后该模板会被持久化复用。
  - 实现了“置备—执行—恢复—销毁”闭环，将空账户并行执行的成功率从 33.81% 提升至 84.39%，同时大幅缩短整体任务时间，为 GRPO 提供了干净的奖励信号。

- **SFT 阶段**：
  - 数据来源：① 前沿模型成功轨迹的**蒸馏**（经任务级和步骤级双重过滤）；② 模型**自主自探索**（从产品入口自由执行 CRUD 任务，覆盖长尾 UI 状态）。
  - 训练格式：强制生成显式 Thought 轨迹（评估进度、分析 UI、确定子目标）。
  - 在 Qwen3-VL-32B-Instruct 上进行全参数微调，~160K 单步样本，1 epoch，64 GPU，DeepSpeed ZeRO-2。

- **GRPO 阶段**：
  - **双通道结果奖励模型（ORM）**：
    - 通道 1：基于 **ActionTrail 审计日志**的规则验证，提供无假阳性的二元奖励。
    - 通道 2：**LLM-as-Judge 集成**（两个强通用 LLM），需共识才接受奖励；否则标记为无效（`-1`），轨迹不参与训练。集成准确率达 96.7%（vs 单模型 91.9%）。
    - 奖励函数：\( r_i = 1.0 \)（ActionTrail 确认成功 或 双法官共识成功），\( 0.0 \)（共识失败），\( -1.0 \)（否则，排除）。
  - **两层优势归一化**：先按任务组归一化（消除任务难度差异），再全局跨任务归一化；当同一任务所有 rollout 结果相同时，优势为零，任务被跳过。
  - **策略更新**：使用 GRPO 的裁剪代理目标 + KL 正则化。KL 惩罚采用均方误差形式（log-prob 差值平方），提供更稳定梯度。超参：`ε=0.2`，`β=0.1`，组大小 `G=16`（1.25 倍过采样），学习率 `5e-7`。回放执行与梯度计算完全解耦，独立缩放。

### 6. 主要结论与发现

- **性能逼近前沿**：AliyunConsoleAgent-32B (SFT+GRPO) 在 278 任务基准上达到 63.52% 平均成功率，与最佳前沿模型 Gemini 3 Pro Preview 的 65.34% 仅差 1.82 个百分点（自助法 95% CI 包含零，即差异不显著），同时推理成本降低 92%。
- **阶段递增收益**：SFT 较基模型提升 13.61 pp，GRPO 在 SFT 基础上再提升 6.63 pp（统计显著），且 GRPO 在标准任务和困难任务上均带来进一步增益。
- **能力涌现**：GRPO 使模型获得“前置条件构建”（如先开启后再执行可验证的关闭操作）和“自适应计划调整”（当文档指定的 UI 路径不存在时，自主切换到其他有效路径）等类人推理能力，且无一致退化。
- **失败主要来源**：资源置备缺口（确定性失败）、UI 交互非确定性（间歇失败）、智能体决策错误（能力上限），为后续优化指明方向。
- **生产可行性**：该模型已在阿里云的部分文档验证流程中灰度测试，可大幅节省成本并满足隐私要求，已帮助在实际审计中确认 4399 个缺陷。

### 7. 优点

- **真实环境下的 RL 训练**：在真实云控制台而非沙箱模拟中开展强化学习，并设计了完善的高确定性回放基础设施，成功隔离资源状态依赖所带来的环境噪声，为训练提供干净信号。
- **抗奖励黑客的评估与奖励设计**：基于 ActionTrail 审计日志的规则奖励机制，从后端状态变化验证成功，远比视觉或 LLM 评判客观；双通道 ORM 中集成 LLM-as-Judge 时强制共识并排除分歧样本，有效抑制奖励黑客。
- **成本效益显著**：以 92% 更低的推理成本（0.56 元/任务 vs 7.0 元/任务）实现与前沿模型统计无差异的性能，并支持私有化部署，解决隐私合规问题。
- **完整的工程闭环与开源**：提供了从资源 META 生成、按需置备、训练到评估的完整流水线，并开放评估基准、数据、环境代码等，促进复现与研究。
- **对云文档验证的实际影响**：模型已在生产环境辅助发现数千个确认缺陷，证明了该方法在真实问题中的价值。

### 8. 不足与局限

- **资源置备维护成本**：Rollout 环境依赖由前沿模型生成的静态 Terraform 模板，随云平台 API 和文档更新会迅速过时。持续用前沿 API 重新生成模板成本高，目前缺乏本地化的低成本模板维护方案。
- **视觉定位依赖 DOM 解析**：Set-of-Mark 强依赖前端 DOM 提取与标注逻辑，面对复杂且频繁变化的控制台前端，解析逻辑脆弱且维护负担重；未来需向纯视觉交互迁移。
- **高昂的基础设施费用**：大规模执行实时文档验证时，某些依赖高价资源（如 GPU 实例）的文档若以同样频率测试，经济上不可持续，需动态调整测试频率。
- **部分失败仍未解决**：资源置备缺口仍然是失败最大来源，尤其是一些需要长时间数据生成（如备份需 24 小时）的场景；UI 交互中的竞态条件、弹出干扰等非确定性失败也尚未根除。
- **GRPO 训练稳定性**：训练过程中观察到非单调的性能振荡，虽通过验证集选 checkpoint 缓解，但需进一步稳定训练动力。
- **泛化边界**：文中评估集中在阿里云内部产品和文档，对于跨云平台、跨语言文档的泛化能力未经测试，可能存在平台锁定。
