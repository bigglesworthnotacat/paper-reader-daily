## 摘要
提出了一个协议驱动的框架，将 SBOM 和 AIBOM 工件与确定性环境捕获和结构化运行时遥测绑定。利用声明的工件、观察到的激活条件和强制执行的执行策略计算可利用性。从组合的静态和运行时证据生成 CSAF VEX 安全公告，并对其进行加密签名和通过确定性重放进行验证。评估使用了约 10,000 个组件项，跨越 50 到 5000 个组件的合成自主 AI 工作负载，融合了 OSV、GitHub Advisory、KEV 和 EPSS 数据集。

## Abstract
A protocol driven framework is presented that binds SBOM and AIBOM artefacts to deterministic environment capture and structured runtime telemetry. Exploitability is computed from declared artefacts, observed activation conditions, and enforced execution policies. CSAF VEX advisories are generated from combined static and runtime evidence, cryptographically signed, and validated through deterministic replay. Evaluation uses approximately 10000 component entries across synthetic Agentic AI workloads 50 to 5000 components, incorporating OSV, GitHub Advisory, KEV, and EPSS datasets.

-   **整体含义**：论文旨在将安全咨询生成从“静态枚举式”转变为“执行边界型”评估，通过结合人工智能物料清单（AIBOM）和运行时证据，为Agentic AI的供应链安全提供一种可复现、上下文感知的自动化方案。

### 2. 论文提出的方法论

-   **核心思想**：将漏洞的可利用性评估定义为一个三条件合取问题：**存在性（Presence）、激活状态（Activation）和可行性（Feasibility）**。一个漏洞只有在三个条件同时满足时，才被判定为“受影响（Affected）”，否则为“不受影响（Not Affected）”。
-   **关键技术细节与算法流程**：
    -   论文提出了一个基于三大协议的**六阶段流水线框架**：
        1.  **确定性环境捕获**：使用模型上下文协议（MCP）捕获完整的执行环境快照，包括容器镜像摘要、模型哈希、提示模板版本、工具端点定义和安全策略等，生成加密签名的执行信封。
        2.  **运行时遥测收集**：使用Agent2Agent（A2A）协议，通过边车代理（sidecar agents）安全地收集运行时行为数据，如系统调用、网络连接、动态库加载、工具调用和权限状态。
        3.  **漏洞关联**：将捕获的SBOM/AIBOM工件（通过SPDX pURLs和CPE标识符）与来自OSV.dev、GitHub Advisory等结构化漏洞情报进行确定性匹配。
        4.  **可利用性推断**：结合运行时遥测和执行策略进行上下文过滤。例如，若一个漏洞需要外发网络连接，但遥测显示无出口流量或策略禁止出口，则该漏洞被判定为“Not Affected”。最终遵循CSAF VEX分类（Affected, Not Affected, Fixed）。
        5.  **安全咨询生成**：根据推断结果，通过AGNTCY治理层生成包含CVE ID、组件哈希、AIBOM引用、运行时证据哈希和EPSS评分等的CSAF 2.0 VEX文档，并进行加密签名。
        6.  **可复现性验证**：在一个独立的审计节点中，使用相同的密封执行信封重放整个工作负载，并通过确定性哈希比对生成的咨询结果，确保评估的可复现性。
