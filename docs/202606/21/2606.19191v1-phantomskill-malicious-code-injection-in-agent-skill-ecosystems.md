## 摘要
Agent技能使得基于LLM的编码代理能够从第三方包获取领域特定能力，但它们也引入了新的供应链攻击面。我们提出了PhantomSkill，一种将恶意行为隐藏在技能的辅助资源而非文本描述中的攻击框架。其核心技术VulMask将公开的恶意脚本重写为形态似漏洞的实现，其恶意行为仅在攻击者控制的触发条件下激活。这种设计将可见信号从明显的恶意意图转变为看起来普通的非安全代码。在代表性的宿主技能、攻击目标、编码代理、生成模型和自动审查者中，VulMask保留了良性功能，同时与公开的恶意脚本相比降低了对警告和恶意软件级别的检测。我们的结果表明，技能生态系统需要资源级审查、运行时遏制，以及将代理技能中可被利用的漏洞视为潜在恶意载荷的安全策略。

## Abstract
Agent skills allow LLM-based coding agents to acquire domain-specific capabilities from third-party packages, but they also introduce a new supply-chain attack surface. We present PhantomSkill, an attack framework that hides malicious behavior in a skill's auxiliary resources rather than in its textual description. Its core technique, VulMask, rewrites overt malicious scripts into vulnerability-shaped implementations whose malicious behavior is activated only under attacker-controlled trigger conditions. This design shifts the visible signal from explicit malicious intent to ordinary-looking insecure code. Across representative host skills, attack goals, coding agents, generation models, and automated reviewers, VulMask preserves benign utility while reducing warning and malware-level detection compared with overt malicious scripts. Our results show that skill ecosystems require resource-level vetting, execution-time containment, and security policies that treat exploitable vulnerabilities in agent skills as potential malicious payloads.

- **研究背景**：基于大语言模型（LLM）的编码代理（如Claude Code、Cursor）通过加载第三方“技能包”（Agent Skills）来扩展领域能力，每个技能包含 SKILL.md 说明文件和脚本等辅助资源。这种模块化生态在提升效率的同时带来了新的软件供应链攻击面。

### 2. 论文提出的方法论

- **核心思想**：审查者对显式恶意代码（如窃取凭证、执行系统命令）高度敏感，而对一般的“不安全编码”或“可被利用的漏洞”容忍度较高。VulMask 正是利用这一“检测偏差”，将恶意脚本**改写为漏洞形态**，使其看起来像普通的不安全实现，而非蓄意攻击代码。
- **技术流程**（VulMask）：
    1. **选择漏洞类别**：根据攻击意图（凭证窃取、命令执行等）选取一组 CWE 类型的弱点，如信息泄露（CWE-522）、命令注入（CWE-94）、代码注入等。
    2. **载荷重写**：由一个 LLM 驱动生成器 \( g \) 将原始恶意脚本 \( p^\star \) 转化为掩码后的载荷 \( p_m = g(p^\star, W) \)，保留恶意的副作用，但将其实现方式重塑为触发式漏洞。
    3. **嵌入宿主技能**：不替换技能原有的辅助资源，而是将 \( p_m \) 嵌入到宿主资源 \( r_h \) 中，得到 \( r_m = \text{Embed}(r_h, p_m) \)，保持原有脚本名、路径和正常输入输出行为。
    4. **有效性验证**：在隔离环境中验证，确保无触发器时宿主技能功能完好，存在攻击者控制的触发条件 \( \theta \) 时才触发恶意行为。
- **威胁模型**：攻击者仅控制分发的技能包内容，无法控制受害者端 LLM、代理框架或项目上下文。防御者可审查技能的所有组件（包括辅助资源），但问题的关键在于**审查者是否能识别出漏洞形态代码中的真实恶意意图**。

### 6. 论文的主要结论与发现

- **VulMask 有效降低检测**：在 GPT‑5.5 代理中，VulMask 攻击成功率 58.8%，警告率仅 11.4%，而明显恶意脚本的警告率为 25.4%；与 Crazy‑Ivan 相比，警告率从 27.2% 降至 11.4%。
- **防御存在明显缺口**：现代编码代理往往能识别直接恶意代码，但难以区分“看起来像漏洞”的恶意载荷与普通不安全实现。Cisco 静态分析、LLM 审计工具等均将大量 VulMask 载荷标记为“漏洞”而非“恶意软件”。
- **跨目标与跨模型通用性**：方法在凭证窃取、命令执行、代理操纵和破坏四大目标上均有效，且可使用不同生成模型（如 Opus‑4.7、GLM-4.7）产生攻击载荷，说明攻击技术不依赖单一生成器。
- **资源层审查的必要性**：仅扫描 SKILL.md 不足以保证安全；必须对辅助资源（脚本、模板等）进行细粒度审查，并将“可被利用的漏洞”提升到恶意载荷的优先级处理。

### 7. 优点

- **视角新颖**：将攻击面从常规的提示注入或明显恶意脚本转移到“辅助资源内的漏洞形态代码”，精准利用了安全审查中的检测偏差。
- **技术设计精巧**：VulMask 方法保留宿主技能的正常功能，并在无触发器时完全良性，提高隐蔽性和实用性。
- **评估扎实**：覆盖多个攻击类别、多个 LLM 代理、多个自动审查器，且包含消融实验和逐技能分析，结论具有较强说服力。
