## 摘要
我们引入了GRACE-DS，这是一个在数据科学领域中具有守卫奖励引导的代理纠正环境，用于对基于LLM的AutoML代理进行部署前评估。GRACE-DS是一套可在隔离环境中应用的评估指标，针对特定组织的表格化机器学习任务。它让代理经历现实工作流阶段，从规划和数据检查到特征工程、模型开发、验证、代码修复直至最终提交，同时隐藏的可执行验证器不仅衡量最终的预测性能，还衡量泄漏避免、可重现性、协议有效性、纠正行为以及奖励对齐。最强的结构化制度，即灵活的迭代交互（我们的方法），相比单次生成、非结构化交互和基于重启的基线，取得了更高的端到端归一化隐藏测试质量，同时提高了协议有效完成率。经过超过7,000个事件单元的验证，这些结果确立了GRACE-DS作为一个稳健的平台，能够评估基于LLM的AutoML代理在生产类条件下并符合组织特定要求执行机器学习工作流的能力。

## Abstract
We introduce GRACE-DS, a Guarded Reward-guided Agent Correction Environment in Data Science for pre-deployment evaluation of LLM-powered AutoML agents. GRACE-DS is a set of evaluation metrics in an isolated environment that can be applied to tabular ML tasks specific to a particular organization. It exposes agents to realistic workflow stages, from planning and data inspection through feature engineering, model development, validation, and code repair to final submission, while hidden executable validators measure not only final predictive performance but also leakage avoidance, reproducibility, protocol validity, correction behavior, and reward alignment. The strongest structured regime, flexible iterative interaction (our approach), achieves higher end-to-end normalized hidden-test quality than single-shot generation, unstructured interaction, and restart-based baselines, while also improving protocol-valid completion. Validated across more than 7,000 episodes, these results establish GRACE-DS as a robust platform for assessing the capacity of LLM-based AutoML agents to execute machine learning workflows under production-like conditions and in accordance with organization-specific requirements.

### 6. 论文的主要结论与发现

*   **结构化迭代至关重要**：**灵活迭代制度**在所有评估中表现最优，端到端归一化隐藏测试质量达到**0.754**，显著优于单次生成（0.536）、非结构化代理（0.527）和重启基线（0.672-0.686）。其优势不仅在于更好的模型，更在于将失败轨迹转化为有效提交的**可靠性提升**。
*   **环境提供的奖励是有效的进度信号，但不能替代最终测试**：过程奖励与任务质量在任务内呈正相关，但奖励最大化探针实验表明，代理可以为了讨好检查清单而轻微提高奖励，却导致隐藏测试质量下降。这证明了分离过程反馈与最终仲裁的必要性。
*   **揭示了代理的弱点**：非结构化和固定调度的制度会在最终失败和关键方法论错误（如数据泄露）上损失10-23个百分点的协议有效质量，而GRACE-DS能够清晰诊断出这些在只看最终分数的基准测试中会被掩盖的失败模式。
*   **环境对不同规模的模型均有效，但影响不同**：结构化环境对所有模型都有益，但作为“能力均衡器”，它对在单次生成中表现较弱的中等规模模型提升最大（最高+0.342）。对于顶尖模型（如Gemini-3.1 Pro），由于其单次生成已接近极限，提升空间较小。
*   **质量-效率前沿**：在质量-时间-成本分析中，灵活迭代在质量和时间上均优于调用次数匹配的重启基线，但消耗了更多的令牌。单次生成是最高效（但质量最低）的选择，而灵活迭代则定义了高质量前沿。

### 7. 优点

*   **部署导向的评估**：将评估重点从“分数多高”转向“过程是否可信”，捕捉了目标泄漏、不可复现代码等对生产部署至关重要的失败模式。
*   **设计精巧的隔离与反馈机制**：通过隐藏验证器和抽象反馈，在引导代理的同时避免了代理投机取巧地迎合检查器，确保了评估的客观性。
*   **全面的可解释性**：不只是一个分数，其分解奖励、终端路径分解、验证指标增长轨迹等工具，让每个制度为何得到该分数变得**可审计**和**可诊断**。
*   **架构无关且可本地化**：GRACE-DS可与任何LLM代理架构配合使用，且允许企业用自己的内部`task.json`文件在本地运行完全相同的评估，无需暴露敏感数据。

### 8. 不足与局限

*   **范围局限**：当前框架**仅针对表格化监督学习任务**，未扩展至深度学习、多模态、强化学习或开放式科学发现等更广泛的机器学习领域。
*   **部署认证的不完整性**：论文明确指出，通过GRACE-DS**不等于**模型已生产就绪。它不涵盖监控、概率校准、对分布漂移的鲁棒性、延迟约束、可解释性审查等其他MLOps关键环节。
*   **成本与效率权衡**：虽然质量最高，但灵活迭代范式的令牌消耗（约43k）是单次生成（约2.4k）的18倍，对于令牌预算受限的场景可能代价高昂。尽管有一个节省成本的“精简反馈”版本，但会牺牲少量质量。
*   **潜在的偏差风险**：尽管数据集多样化，但任何基准测试集都可能存在偏差。所有实验均基于当前的LLM API状态，模型行为、速度和成本可能随提供商更新而变化，影响实验的可复现性。代理的表现也高度依赖于LLM提供商的性能。
