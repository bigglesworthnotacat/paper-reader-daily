## 摘要
视觉语言模型正成为复杂多模态任务的通用接口。然而，部署仍面临三个差距：处理密集视频帧和长提示时视觉语言模型通常会产生高延迟和高成本，智能体框架在部署后保持静态，以及标准视频问答基准不测试智能体能否在工具使用的工作空间内利用视觉证据。我们提出VisualClaw，一个基于两个原则的自进化多模态智能体。首先，混合编码通过级联门控过滤信息量较低的流式帧，并通过冷热top-k注入压缩文本技能库，从而降低部署成本。其次，技能进化让智能体从失败中学习：检索的记忆作为直接上下文或引导证据调节进化器，生成有助于未来问题的技能库更新。在基于两个视觉语言模型的4个视频问答基准测试中，相比全帧上传，VisualClaw将每个问题的API成本平均降低-98%，比离线均匀8帧基线降低-25.9%，同时在大多数设置中提升准确率，例如在EgoSchema上使用Gemini 3 Flash平均提升+3.85%，峰值提升+15.80%。为弥补差距，我们精心构建了VisualClawArena，一个包含200个场景的多模态智能基准，通过严格的五阶段流水线构建；模型必须使用工作空间内的视觉证据、文档、动态更新和可执行检查。在VisualClawArena上，与无进化基线相比，采用计算机使用智能体后端的同一框架将Codex (GPT-5.5)的宏观准确率提高+2.9%，将Claude Code (Sonnet 4.6)提高+3.2%，且与均匀采样基线相比成本降低-9.5%。这些特性使VisualClaw天然适合边缘应用，其中级联将1小时流式会话从约3600次API上传减少到仅5-20次调用，而自进化使其成为完美的个性化助手。

## Abstract
Vision language models are serving as general-purpose interfaces for complex multimodal tasks. However, deployment still faces three gaps: VLMs typically incur high latency and cost when processing dense video frames and long prompts, the agent scaffold remains static after deployment, and standard video-QA benchmarks do not test whether agents can use visual evidence inside tool-using workspaces. We present VisualClaw, a self-evolving multimodal agent built around two principles. First, hybrid encoding reduces deployment cost by filtering less informative streaming frames with a cascaded gate and compressing the text skill bank through hot/cold top-k injection. Second, skill evolution lets the agent learn from failures: retrieved memories condition an evolver as direct concatenated context or as guided evidence, producing skill-bank updates that help future questions. Across 4 video-QA benchmarks with 2 VLMs, VisualClaw cuts per-question API cost by an average -98% versus full-frame upload and by -25.9% over the offline uniform 8 frame baseline, while boosting accuracy in most settings, e.g., an average +3.85% and a peak +15.80% on EgoSchema with Gemini 3 Flash. To address the gap, we curate VisualClawArena, a 200-scenario multimodal agentic benchmark built through a strict five-stage pipeline; models must use video evidence, documents, dynamic updates, and executable checks inside a workspace. On VisualClawArena, the same framework with computer-use agent backends improves macro accuracy by +2.9% for Codex (GPT-5.5) and +3.2% for Claude Code (Sonnet 4.6) over no-evolution baselines, with a -9.5% cost reduction compared to the uniform-sampled baseline. These properties make VisualClaw a natural fit for edge applications, where the cascade reduces a 1-hour streaming session from ~3,600 API uploads down to only 5-20 calls and the self-evolution makes it a perfect personalized assistant.

### 1. 论文的核心问题与整体含义

论文聚焦于视觉语言模型在现实世界部署时面临的三大核心鸿沟：

*   **成本与延迟瓶颈**：云端的视觉语言模型在处理密集视频帧和长文本提示时，会产生高昂的API调用成本和响应延迟，难以满足实时流式视频问答的需求。
*   **静态智能体框架**：现有智能体在部署后，其技能库和记忆库保持固定，无法像人类一样从自身失败中持续学习与进化，缺乏个性化和适应性。
*   **评估基准脱节**：主流的视频问答基准仅评估模型的离线“一次性”答题能力，未能测试模型在一个需要综合使用视觉证据、工具操作和文件编辑的真实工作空间中的表现。

因此，本文的核心目标是**构建一个高效、可自进化的多模态智能体框架VisualClaw**，以极低的成本实现对流式视频的实时感知、持续演化和个性化问题解决，并为此提出了一个全新的面向工具使用场景的多模态智能体基准。

### 2. 论文提出的方法论

VisualClaw的核心思想是“混合编码”与“自进化”，它在三个时间尺度上运行：

*   **每帧操作：级联视觉编码门控（边缘端）**
    *   **核心思想**：在边缘设备上（无需GPU），通过一个轻量级流水线，实时判断每一帧是否足够重要以至于需要上传至云端视觉语言模型，从而过滤掉冗余帧。
    *   **技术细节**：
        1.  **感知哈希**：对当前帧与近期帧进行汉明距离比对，丢弃几乎完全相同的重复帧。
        2.  **轻量编码器**：使用`128维`的CPU特征编码器（基于HSV直方图、亮度、边缘密度等），将图像帧转化为低维向量。
        3.  **自适应变化门控**：将当前帧向量与滚动参考向量进行余弦相似度比较，根据动态衰减的阈值，将帧分类为“重要”(MAJOR)、“次要”(MINOR)或“跳过”(SKIP)。只有“重要”帧才会上传作为关键帧。

*   **每问题操作：技能库的热/冷混合注入（云端）**
    *   **核心思想**：随着智能体不断进化，其技能库会不断扩大，为每个问题注入全部技能会导致提示词过长、成本激增且信号稀释。因此，采用分层注入策略。
    *   **技术细节**：
        *   **热技能**: 根据当前问题的文本嵌入，从技能库`S`中检索出相关性最高的`top-k`个技能，并将其完整文本完整注入系统提示。
        *   **冷技能**: 将剩余的技能压缩为一个仅包含名称和一句话描述的目录，使模型可按需获取。
        *   此方法将每次请求的提示成本增长与技能库规模`|S|`解耦，固定为常数`k`。

*   **每会话操作：记忆引导的元进化（离线）**
    *   **核心思想**：当积累了一定数量的失败案例后，智能体利用一个强大的语言模型进化器，通过检索过去的成功经验来更新技能库，从而在未来避免同类错误。
    *   **技术细节**：
        1.  **记忆检索**：将回答正确的问题作为成功案例存入记忆库`Mv`，并使用其嵌入进行高置信度检索。
        2.  **进化器激发**：当失败案例`|Dfail|`达到阈值`Nevo`时，启动进化器`E`。
        3.  **两种融合模式**：
            *   **拼接模式 (Cat.)**：直接将检索到的成功记忆作为额外的上下文，拼接到技能更新的提示中。
            *   **引导模式 (Guide)**：在提示中加入引导指令，要求进化器将检索到的记忆抽象为可复用的技能，并避免过度拟合具体场景。
        4.  **技能库维护**：通过“名称去重”和“效用追踪与剪枝”两个过滤器，保证技能库的质量和紧凑性。
