## 摘要
当前的自主机器人系统能够编写可执行的代码即策略程序，观察反馈，并在多次尝试中调整行为，但它们仍然主要以任务为驱动：可复用的技能仅在接收到明确指令后才获得。我们研究了玩趣式自主机器人学习，其中具身编码智能体在下游任务到来之前，将自主玩耍作为一个持续的技能学习阶段。我们引入了机器人智能体团队（RATs），专为玩耍期间的技能获取而设计。在玩耍过程中，RATs提出新颖且可学习的探索性任务，规划并执行机器人代码策略，验证中间进展，诊断失败，利用密集的步骤级反馈进行重试，并将成功执行的结果提炼到一个持久的代码技能库中。在测试时，智能体从这个冻结的技能库中重用相关技能，以帮助解决新任务。在LIBERO-PRO和MolmoSpaces上的实验表明，玩耍中学到的技能相比无玩耍和随机玩耍的基线，在留出的下游任务上分别提升了20.6和17.0个百分点（相对于CaP-Agent0）。此外，所学的技能可以通过简单地检索到上下文中，即插即用到其他推理时的代码即策略智能体中，分别在RoboSuite和现实世界迁移上提升了8.9和8.8分，而无需对底层模型进行微调。

## Abstract
Current agentic robot systems can write executable Code-as-Policy programs, observe feedback, and revise behavior across multiple attempts, but they remain largely task-driven: reusable skills are acquired only after explicit instructions. We study Playful Agentic Robot Learning, where an embodied coding agent uses self-directed play as a continual skill-learning stage before downstream tasks arrive. We introduce RATs, Robotics Agent Teams designed for play-time skill acquisition. During play, RATs proposes novel yet learnable exploratory tasks, plans and executes robot-code policies, verifies intermediate progress, diagnoses failures, retries with dense, step-level feedback, and distills successful executions into a persistent code skill library. At test time, the agent reuses relevant skills from this frozen library to help solve new tasks. Experiments in LIBERO-PRO and MolmoSpaces show that play-learned skills improve held-out downstream tasks over no-play and random-play baselines, with 20.6 and 17.0 percentage-point gains over CaP-Agent0 on LIBERO-PRO and MolmoSpaces, respectively. Moreover, the learned skills can be plugged into other inference-time Code-as-Policy agents by simply retrieving them into the context, improving RoboSuite and real-world transfer by 8.9 and 8.8 points, respectively, without finetuning the underlying model.

### 论文方法论

*   **核心思想**：RATs（机器人智能体团队）将机器人的“玩耍”视为一个**显式的技能获取过程**，而非随机的动作探索。其核心目标是，在无外部任务指令的玩耍阶段，通过内在动机自主提出、练习并蒸馏出可复用的代码技能，构建一个持久的技能库，并在测试时利用该库解决新任务。
*   **系统架构与关键技术流程**：RATs是一个多智能体系统，由三个核心团队协作完成，其算法流程（详见论文算法1）可概括为：
    1.  **任务提出者团队**：
        *   **候选任务生成**：根据当前场景、已有的技能库和失败记忆，利用大语言模型生成一组探索性的候选任务。
        *   **“金发姑娘”驱动任务选择**：依据“金发姑娘原则”（Goldilocks Principle）选择一个既不过于简单也不过于困难的任务。该选择通过最大化一个兼顾**新颖性**和**可学习性**的得分来实现。
        *   **新颖性**项鼓励智能体尝试罕见的目标-技能组合，而**能力前沿**项则偏爱成功率约50%的任务，即处于智能体当前能力边界的任务。
    2.  **执行团队**：
        *   **规划智能体**：为选定的任务制定一系列步骤，并为每个步骤匹配相关技能。
        *   **执行智能体（策略编写器）**：将规划步骤转化为可执行的Python机器人控制代码。
        *   **验证智能体**：在多个层面提供反馈，包括：
            *   **规划验证器**：检查计划是否与场景物理一致。
            *   **质量检查器**：静态筛查代码中的安全或格式错误。
            *   **目标验证器**：评估最终任务是否成功。
            *   **步骤级验证器**：对每个执行步骤的成功与否给出局部判定。
            *   **失败诊断器**：分析失败模式并提出修正建议。
        *   **编排机制**：整个执行过程是一个**“编写-执行-验证-诊断”的循环**。如果任务失败，诊断器会反馈给策略编写器进行重试。这个循环允许系统保留工作正常的子步骤，仅修正出错的代码或规划，甚至在遇到顽固瓶颈时，会启动一个“子智能体”来专门练习该子动作并返回可复用的辅助程序。
    3.  **记忆管理团队**：
        *   **技能库更新**：在成功执行任务后，系统会将成功代码中的关键行为抽取为带有文档字符串、参数化的独立函数，作为**实验性技能**存入技能库。
        *   **失败记忆库更新**：将失败经历提炼为简洁的教训（例如：出错的前提条件或修正方法），供未来规划时参考。
        *   **技能生命周期管理**：技能被标记为“实验”、“已验证”或“已弃用”，系统会基于后续使用的成功率进行动态调整。团队还定期进行记忆整理，合并重复技能和清理冗余教训。
