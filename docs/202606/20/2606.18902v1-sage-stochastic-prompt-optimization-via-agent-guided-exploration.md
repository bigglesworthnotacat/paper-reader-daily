## 摘要
上下文工程已成为无需参数更新即可改进 AI 系统的主要手段。最近的研究表明，文本梯度不能作为真正的梯度使用，这促使将自动提示优化（APO）视为黑盒搜索。我们引入了 SPO（随机提示优化），一个在提示空间进行随机搜索的框架，并比较了三种复杂度递增的策略：错误知情随机搜索、带有进化算子的遗传算法，以及 SAGE（通过智能体引导探索的 SPO），一种包含诊断代码执行的多智能体流水线。在三个基准测试中，没有单一策略占据主导地位；有效性取决于景观结构与错误类型的相互作用。我们进一步在持续优化范式下将 SAGE 部署于心理健康聊天机器人，在该范式中，它将八个单独嘈杂的 A/B 测试周期复合为次日留存率上统计稳健的增益。我们认为，将定性诊断与定量验证相结合，正是使得智能体优化对于开放式任务导向对话有效的原因。

## Abstract
Context engineering has emerged as a primary lever for improving AI systems without parameter updates. Recent work showing that textual gradients do not function as real gradients motivates treating automatic prompt optimization (APO) as black-box search. We introduce SPO (Stochastic Prompt Optimization), a framework for stochastic search over prompt space, and compare three strategies of increasing sophistication: error-informed random search, a genetic algorithm with evolutionary operators, and SAGE (SPO via Agent-Guided Exploration), a multi-agent pipeline with diagnostic code execution. Across three benchmarks, no single strategy dominates; effectiveness depends on the interaction of landscape structure with error type. We further deploy SAGE on a mental-health chatbot under a continuous optimization paradigm, where it compounds eight cycles of individually-noisy A/B tests into a statistically robust gain in next-day retention. We argue that coupling qualitative diagnosis with quantitative validation is what makes agentic optimization effective for open-ended task-oriented dialogue.

### 2. 方法论：核心思想与技术细节
#### 2.1 SPO 框架（共享骨架）
- 目标：最大化经验回报 \(\hat{R}(p)\)，每次迭代依据公式 \(p_{k+1} = \arg\max_{p \in \{p_k\} \cup S(p_k,D)} \hat{R}(p)\) 更新提示，**仅接受严格提升**（随机爬山法）。
- 搜索空间特点：离散自然语言空间，语义相似提示性能相近，具有**丰富的局部结构**。

#### 2.2 三种策略
- **SPO-RS（残差搜索）**
  - 给定当前最佳提示及其错误摘要，由高温（\(\tau=0.9\)）元模型独立生成 Q 个候选。
  - 本质是**错误感知的随机扰动**，保留跨迭代反馈。
- **SPO-GA（遗传算法）**
  - 维护大小为 P 的精英种群，采用 LLM 作为进化算子：
    - **交叉**：组合两父代提示及错误摘要，生成新提示。
    - **变异**：对锦标赛选出的父代提示做针对性改进。
  - 交叉比例 \(\rho=0.3\)，温度降低至 0.7，牺牲随机性换取针对性。
- **SAGE（智能体引导探索）**
  - 多智能体流水线（分析器、编排器、调查员、生成器），每个智能体均可**执行诊断代码**（文件读写、脚本运行、代码搜索）。
  - 具体流程：
    1. **分析器**：运行分析脚本，从所有顶级提示的评估结果中提出 H 个证据支持的假设。
    2. **编排器**：筛选出 I 个最具价值的调查方向。
    3. **调查员**（并行）：通过精确计数和推理轨迹分析验证假设，产出可落地的修复方案。
    4. **生成器**：每个调查方向生成一个完整的修订提示。
  - 关键优势：将搜索从随机扰动转变为**假设驱动的诊断式优化**。

### 6. 主要结论与发现
- **无单一最优策略**：SAGE 在需要深层因果分析的 Formula 任务领先；SPO-GA 在 AppWorld 的程序化错误上占优；而在光滑的 FiNER 景观中，简单策略（SPO-RS）泛化更好。
- **景观结构×错误类型决定策略效果**：Ruggedness（崎岖度）本身不决定最优方法，**级联推理错误**呼唤 SAGE 的根因诊断，**程序化、逐样本错误**只需更简单的搜索。
- **持续优化范式有效**：在 Ash 聊天机器人上，通过 8 轮个体噪声较大但复合后置信区间离开零的 A/B 测试，实现次日留存率的统计显著提升，证明了“定性诊断 + 定量验证”耦合的价值。
- **SAGE 分析深度优势**：诊断代码执行使智能体能够精准定位错误模式，避免盲目提示膨胀，并在适当任务上生成更短的提示。

### 7. 优点
- **分析深度与代码执行突破**：首次将代码执行用于诊断错误模式而非仅仅构建程序化制品，提升了错误分析的粒度。
- **景观分析视角**：引入地统计学中的半变异函数，定量刻画提示空间的崎岖度，增强了理论解释力。
- **工程实用性**：提出持续的、在线 A/B 测试的优化范式，并在真实心理健康服务中取得可测量增益。
- **框架统一且可控**：三种策略共享爬山算法外壳，隔离了搜索策略的差异，使消融研究结果可靠。

### 8. 不足与局限
- **评估函数依赖**：SPO 依赖自动评价函数，无明确 ground truth 的任务需借助 LLM 评分或人工评价，引入额外噪声。
- **分布偏移**：持续优化假设多轮用户行为平稳，现实中用户群体和聊天机器人共演化可能导致优化效果衰减。

- **实验覆盖有限**：仅三个学术基准任务，部署为单一领域（心理健康），结论的广泛性待更多验证。
- **算力透明度不足**：未报告任何计算资源消耗，无法评估方法的实际成本效益。
- **部署统计偏倚**：每轮选择 Q 个候选中的最佳者可能轻微上偏衡量到的增量，且绝对留存率可能受应用同期演化影响，解释时需谨慎。
