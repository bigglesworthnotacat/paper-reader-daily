## 摘要
生产环境中的LLM智能体越来越依赖实时搜索，然而原生的搜索接地将检索策略、提供者选择、证据注入、成本、延迟和生成行为捆绑在单一模型提供者边界之后。这种耦合使得接地难以检查、调整、重用或移植，并可能引发搜索导致的冗余，破坏严格的输出合约。我们提出了解耦搜索接地（DSG），一种供应商无关的边界，通过兼容MCP的网关将接地移出推理模型，暴露提供者路由、源感知上下文渲染、配置的回退、检索深度控制、以及精确和语义缓存作为一级控制。在SimpleQA、FreshQA和HotpotQA上的五个前沿模型上，原生搜索在对时效性敏感的FreshQA上领先，但当需要控制时DSG展现出更强的能力：在SimpleQA上，它以低91%的搜索成本几乎匹配原生准确率（86.1%对比87.7%），保留简洁的答案合约，并以低68%的延迟达到99.4%的热缓存命中率。作为大规模智能体工作负载的共享生产接地层部署，具有可互换的模型，DSG在电子商务查询理解（QIU）工作负载上匹配或略微超过原生搜索准确率，同时将搜索成本降低超过98%。实时接地最好被当作一个可优化的接口边界，而不是一个固定的模型特性。

## Abstract
Production LLM agents increasingly depend on real-time search, yet native search grounding bundles retrieval policy, provider choice, evidence injection, cost, latency, and generation behavior behind a single model-provider boundary. This coupling makes grounding hard to inspect, tune, reuse, or port, and can trigger Search-Induced Verbosity that breaks strict output contracts. We present Decoupled Search Grounding (DSG), a vendor-agnostic boundary that moves grounding outside the reasoning model through an MCP-compatible gateway, exposing provider routing, source-aware context rendering, configured fallback, retrieval-depth control, and exact plus semantic caching as first-class controls. Across five frontier models on SimpleQA, FreshQA, and HotpotQA, native search leads on recency-sensitive FreshQA, but DSG exposes a stronger frontier when control matters: on SimpleQA it nearly matches native accuracy (86.1% vs. 87.7%) at 91% lower search cost, preserves concise answer contracts, and reaches a 99.4% warm-cache hit rate with 68% lower latency. Deployed as a shared production grounding layer for large-scale agentic workloads with interchangeable models, DSG matches or slightly exceeds native-search accuracy on an e-commerce query-understanding (QIU) workload while cutting search cost by over 98%. Real-time grounding is best treated as an optimizable interface boundary, not a fixed model feature.

## 1. 论文的核心问题与整体含义

- **研究动机**：生产环境中，LLM 智能体越来越多地依赖实时搜索来获取事实证据，但主流的集成方式——“原生搜索接地”——将检索策略、供应商选择、成本、延迟、上下文注入和生成行为全部封存在模型 API 内部。这种紧耦合导致：

  - 搜索可能引发“搜索诱导冗长”（Search‑Induced Verbosity），破坏下游服务所要求的严格输出格式（如只返回最终实体、JSON、布尔标记等）。
- **整体含义**：论文主张实时搜索接地不应被视为某个模型自带的固定功能，而应被当作一个**可优化的系统接口边界**去设计。作者提出一种供应商无关的解耦搜索接地架构（DSG），将搜索从推理中剥离，从而暴露一级控制手段，同时保持推理模型可互换，并在大规模真实业务中验证其效果。

## 2. 论文提出的方法论

- **核心思想**：通过**兼容 MCP 的网关**，在智能体应用与可互换的推理模型之间插入一个结构化的搜索工具层。该网关统一管控搜索行为，而不是由模型 API 内部隐式执行搜索。
- **关键技术细节**：
  - **源感知工具格式化**：将搜索暴露为显式的可配置工具 Schema。LLM 发出搜索工具调用时，网关按统一接口执行检索，并将不同供应商的异构结果规范化成一致的内部对象，附带标题、URL、内容段落。这使得模型能根据明确来源引用证据，同时也便于日志、调试和回退。
  - **供应商抽象与回退**：网关包含供应商注册表，收纳多个通用搜索提供商（如 Serper、BrightData、Firecrawl、Exa 等）。通过 YAML 适配器即可快速接入新供应商。若某一供应商超时或出错，网关自动按配置的回退链尝试下一个供应商，确保接地上下文不中断。
  - **搜索智能层与缓存**：引入**精确缓存**和**语义缓存**两级复用机制。令 $q$ 为查询，$\bar{q}$ 为标准化形式，$C=\{(q_i,R_i)\}$ 为语义缓存，$e(\cdot)$ 为嵌入函数，$p_1,\dots,p_k$ 为回退链。网关决策为：
    \[
    D(q)=
    \begin{cases}
    R_e, & \text{精确命中 }E(\bar{q})\\
    R_{j^*}, & \text{语义相似度 } s_{j^*} \geq \tau\\
    S(p,q), & \text{执行回退链}
    \end{cases}
    \]
    其中 $s_{j^*}=\cos(e(q),e(q_{j^*}))$，$\tau$ 为可配置阈值。
    - 缓存条目按供应商和领域划分，并设置不同生存时间（TTL），保证时效性与复用平衡。
    - 共享网关实现跨应用缓存复用，热缓存命中率随流量扩大而提升。
  - **执行路径**：见论文中的算法 1，整体流程为：标准化查询→检查精确缓存→若未命中则计算嵌入，检索最近语义缓存→相似度达到阈值则复用，否则按回退链执行在线搜索，规范化结果并存入两级缓存，最终渲染为工具上下文返回模型。
- **为何解耦能缓解搜索诱导冗长**：原生搜索将检索到的文本直接混入模型上下文流，容易诱导模型产生解释性长文；而在 DSG 中，搜索证据作为明确的工具返回对象被注入，帮助模型保持原有的简洁应答风格和输出合约。

## 8. 不足与局限

- **依赖第三方 API**：DSG 的稳定性和性价比直接受第三方搜索供应商影响，如其定价、速率限制、索引变更等，报告的成本仅为特定时间点快照。
- **多跳推理增益有限**：在 HotpotQA 上，搜索解耦提升幅度不大，且部分配置表现不稳健，说明单纯解耦尚不足以完全应对复杂的多步推理，可能需要更紧密的迭代检索与推理协调机制（如规划、反思等）。
- **评判者偏差风险**：原文对公共基准主要采用 LLM‑as‑a‑judge（GPT‑4.1），虽然通过多裁判验证和人工抽检降低偏差，但仍非人工黄金标准。
- **缺乏对模型调用成本的分析**：文中详细记录了搜索供应商费用，但未量化模型推理 API 自身的费用与延迟，完整的总体拥有成本（TCO）比较尚不完整。
- **场景覆盖偏向短答案**：实验设计以短实体、事实或意图分类为主，对于需要长文本生成且必须保持格式与事实验证的任务，效果有待验证。
- **搜索诱导冗长的机理未完全揭示**：论文主要展示现象及 DSG 的缓解效果，但未从模型内部机制上解释原生搜索为何降低指令遵循度，后续需要供应商侧遥测或更细粒度的归因研究。
