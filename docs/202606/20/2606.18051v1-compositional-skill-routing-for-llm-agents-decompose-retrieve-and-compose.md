## 摘要
LLM智能体越来越依赖外部技能——可复用的工具规范——但现实任务往往需要组合多个技能，而不仅仅是选择一个技能。我们将此形式化为组合技能路由问题：给定一个复杂的用户查询和大型技能库，将查询分解为原子子任务，为每个子任务检索合适的技能，并组合成一个可执行计划。我们提出了SkillWeaver，一个结合LLM任务分解器、采用FAISS索引的双编码器技能检索器以及依赖感知的DAG规划器的分解-检索-组合框架。为了支持评估，我们引入了CompSkillBench，一个包含300个组合查询的基准，涵盖来自公共MCP生态系统的2,209个真实MCP服务器技能，横跨24个功能类别。我们的实验表明，任务分解质量是主要瓶颈：标准LLM分解在步骤级别仅达到34.2%的类别召回率。为解决这一问题，我们提出了迭代技能感知分解（SAD），这是一个检索增强的反馈循环，能够迭代地将分解与可用技能对齐。SAD在单次迭代中将分解准确率从51.0%提升至67.7%（+32.7%，Wilcoxon p < 10^-6）；依赖感知条件分析证实，正确的粒度是有效检索的前提（当DA=1时，CatR@1从34%提升至41%）。SkillWeaver将上下文窗口消耗降低了超过99%，迁移实验证实了其泛化能力（即使目标类别不在检索池中，相对DA增益仍达+35.6%）。

## Abstract
LLM agents increasingly rely on external skills -- reusable tool specifications -- but real-world tasks often require composing multiple skills, not just selecting one. We formalize this as the Compositional Skill Routing problem: given a complex user query and a large skill library, decompose the query into atomic sub-tasks, retrieve the appropriate skill for each sub-task, and compose an executable plan. We present SkillWeaver, a decompose-retrieve-compose framework combining an LLM task decomposer, a bi-encoder skill retriever with FAISS indexing, and a dependency-aware DAG planner. To support evaluation, we introduce CompSkillBench, a benchmark of 300 compositional queries over 2,209 real MCP server skills spanning 24 functional categories, sourced from the public MCP ecosystem. Our experiments reveal that task decomposition quality is the primary bottleneck: standard LLM decomposition reaches only 34.2% category recall at the step level. To address this, we propose Iterative Skill-Aware Decomposition (SAD), a retrieval-augmented feedback loop that iteratively aligns decomposition with available skills. SAD improves decomposition accuracy from 51.0% to 67.7% (+32.7%, Wilcoxon p < 10^-6) in a single iteration; DA-conditioned analysis confirms that correct granularity is the prerequisite for effective retrieval (CatR@1 rises from 34% to 41% when DA=1). SkillWeaver reduces context window consumption by over 99%, and transfer experiments confirm generalization (+35.6% relative DA gain even when target categories are absent from the retrieval pool).

- **核心问题**：将“组合技能路由”形式化为一个优化问题——给定复杂用户查询和大型技能库，需将其分解为原子子任务，为每个子任务检索合适技能，并组合成一个考虑依赖关系的可执行计划。
- **核心贡献**：提出SKILL WEAVER框架及配套基准COMP SKILL BENCH，并创新性地引入技能感知迭代分解（SAD）方法，通过检索反馈显著提升任务分解粒度准确性，从而整体改善组合路由质量。

### 2. 方法论
- **框架整体流程（SKILL WEAVER）**：
  - **阶段1：分解（Decompose）**：利用指令微调LLM（主要为Qwen2.5-7B-Instruct）将复杂查询拆解为有序的原子子任务列表，每个子任务对应单一技能。
  - **阶段2：检索（Retrieve）**：对每个子任务，使用双编码器（all-MiniLM-L6-v2，384维）在技能库中检索top-m候选。编码对象为技能元数据（名称、描述）或扩展至正文片段，通过FAISS进行高效内积搜索。
  - **阶段3：组合（Compose）**：基于检索结果，通过兼顾子任务-技能相关性与步骤间兼容性的目标函数（加权余弦相似度与兼容性分数，α=0.5）选取最终技能，并利用语言标记和输入输出重叠检测步骤依赖，构建有向无环图（DAG）作为执行计划。

- **技能感知分解（SAD）**：
  - **核心思想**：标准LLM分解产生的描述常与技能库元数据存在词汇鸿沟。SAD通过检索增强的反馈循环，将首轮分解得到的候选技能信息作为提示，重新让LLM进行分解，促使分解粒度与可用技能词汇对齐。
  - **迭代流程**（算法1）：  
    (1) 初始分解 \(D^{(0)}\)；  
    (2) 对每个子任务检索top-H候选技能，汇集为提示集\(H^{(i)}\)；  
    (3) 若连续两轮提示集相似度（Jaccard）> τ 则收敛；否则，将\(H^{(i)}\)注入提示，生成下一轮分解\(D^{(i+1)}\)。  
    实际使用单轮（Pass-1 + Pass-2）既已收敛。
  - **优化目标形式**：框架整体优化目标为最大化加权子任务-技能相关性和步骤间兼容性总和（公式1），SKILL WEAVER通过级联流水线近似该目标。

### 3. 实验设计
- **基准数据集（COMP SKILL BENCH）**：
  - 从公开MCP生态系统（Model Context Protocol）整理出2,209个真实技能，映射至24个功能类别（如开发者工具、金融、集成等）。
  - 生成300个组合查询，分为简单（2技能）、中等（3技能）、困难（4-5技能）三个难度等级，均配有真实子任务描述、技能ID、所需类别和顺序。
  - 另构建200个人类风格查询（由独立LLM生成，无技能名直接重叠）用于泛化性评估。
- **评估指标**：
  - 分解准确率（DA）：预测步骤数与真值精确匹配的比例。
  - 宽松DA（DA±1）：允许±1步误差。
  - 步骤级指标：类别召回率@k（CatR@k，正确类别技能出现在top-k中的步骤比例），技能召回率@k（Skill R@k）。
  - 链级指标：链精确匹配，链类别匹配（Chain cat）。
- **对比方法**：
  - **Vanilla**：无提示的LLM分解+检索。
  - **+SAD**：单轮技能感知分解（H=15提示）。
  - **LLM-Direct**：提供100个技能名，直接让强模型qwen-max选择工具（上限估计）。
  - **ReAct‑style**：迭代思考‑行动‑观察的Agent。
  - 跨模型测试：Qwen2.5-7B/14B，qwen-max。
  - 迁移实验：类别级留出（2个类别移除）、技能级留出（20%随机移除）。
  - 消融实验：限定步数的oracle基线、DA=1条件下的检索表现。
- **资源与算力**：
  - 单张NVIDIA V100-SXM2-16GB GPU，7B模型约占用15GB显存。

### 5. 主要结论与发现
- **分解粒度为第一瓶颈**：Vanilla LLM的DA仅51.0%，导致CatR@1低至34.2%；当分解步数与真值匹配（DA=1）时，CatR@1跳升至41.2%，CatR@10达81.6%。
- **SAD有效提升分解准确率**：单轮SAD将DA从51.0%提升至67.7%（相对提升+32.7%），DA±1从71.3%升至84.3%。受益于粒度修正，有75个查询的分解错误被纠正，其CatR@1从23.6%跃升至37.0%。
- **SAD是粒度校正器而非词汇对齐器**：在DA原本就正确的查询上，SAD带来的每步检索增益几乎为0，说明其作用完全来自于修正分解步数。
- **上下文窗口大幅压缩**：SKILL WEAVER将执行LLM的上下文消耗降低了>99%（从约88.4万token降至约1,160 token）。
- **泛化能力**：在类别级技能留出实验中，SAD的相对DA增益仍达+35.6%；随机技能留出下为+23.2%，表明方法学到的是一种结构化的词汇引导，而非记忆特定技能映射。
- **重排序可进一步填补检索差距**：一个基于LLM的listwise重排序试点将SAD后的CatR@1再提升10.3%（相对），证实检索@1与@10之间的差距可通过学习式重排序部分解决。

### 6. 优点
- **问题新颖且贴近实际**：首次形式化组合技能路由，契合LLM Agent多工具协同的迫切需求。

- **基准规模大、覆盖全面**：基于2,209个真实MCP技能和300个组合查询，基准真实性强，为后续研究提供了扎实的评估基础。

- **泛化与实用性并重**：上下文窗口的大幅削减、跨模型验证及泛化实验证明了方法的实用潜力和鲁棒性。

### 7. 不足与局限
- **查询生成模板化**：基准查询源于动词短语模板，虽经改写和人类风格评估验证，仍可能残留系统性模式，对完全开放域的适应性需进一步检验。
- **评估局限于检索匹配**：主要衡量技能类别是否被正确检索，而非端到端执行成功率。虽然附录提供了小型执行试点（链完成率76.7%），但组合阶段自身的独立评估（依赖兼容性真值）尚未进行。
- **算力与模型覆盖面有限**：主实验仅使用Qwen2.5-7B单一模型，虽做了14B和qwen-max点验，但缺乏GPT-4、Claude等主流模型的系统对比。检索编码器仅用了all-MiniLM-L6-v2一种，虽进行了BGE抽查，但未全面考察不同编码器的影响。
- **延迟开销**：SAD需要两次LLM推理，分解延迟近乎翻倍，影响实时应用。
- **子任务-技能映射假设简单**：假设每个子任务恰好需要一个技能，未考虑一对多或多对多的灵活映射。
- **难样本数量偏少**：困难子集仅50个查询，统计效力可能不足。
