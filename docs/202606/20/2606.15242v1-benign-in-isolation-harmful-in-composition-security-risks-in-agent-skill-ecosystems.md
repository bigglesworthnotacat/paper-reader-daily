## 摘要
技能正成为大语言模型智能体将计划转化为行动的能力层，但其使用带来了数据泄露、未授权操作和工具滥用等安全风险。现有审查通常孤立地评估每个技能，而实际智能体任务常在共享执行上下文中调用多个技能，由此产生了技能组合风险（SCR）：一个单独看似无害的技能，当其输出、信任信号、授权线索或副作用沿激活路径影响后续调用时，可能变得有害。我们提出SCR-Bench，在受控沙盒技能环境中评估这一风险。SCR-Bench不只依赖文本意图或表面行为，而是记录组合技能执行过程中的下游状态变化和路径级结果。它包含三个子基准：SCR-CapFlow用于能力流组合，SCR-TrustLift用于信任传递组合，SCR-AuthBlur用于授权混淆组合。在SCR-Bench上，组合路径揭示了孤立评估中基本不存在的风险。SCR-CapFlow中，组合下攻击成功率达33.6%，而孤立基线接近零；SCR-TrustLift中，五个后端有四个攻击成功率超过96.5%；SCR-AuthBlur中，L1上下文设置下风险批准率较L0孤立基线增加了71.8%。这些结果表明，智能体技能安全应以激活路径而非孤立制品为评估层面。SCR与SCR-Bench为LLM智能体技能生态系统中路径感知的风险评估与防御奠定了基础。基准链接：https://github.com/saint-viperx/SCR_Bench。

## Abstract
Skills are becoming the capability layer through which LLM agents turn plans into actions, but their use introduces security risks such as data leakage, unauthorized operations, and tool misuse. Existing vetting usually evaluates each skill in isolation, while real agent tasks often invoke multiple skills in a shared execution context. This creates Skill Composition Risk (SCR): a skill that appears benign alone can become harmful when its outputs, trust signals, authorization cues, or side effects influence later invocations along an activated path. We introduce SCR-Bench to evaluate this risk in controlled, sandboxed skill environments. Rather than relying only on textual intent or surface behavior, SCR-Bench records downstream state changes and path-level outcomes across composed skill executions. It contains three sub-benchmarks: SCR-CapFlow for capability-flow composition, SCR-TrustLift for trust-transfer composition, and SCR-AuthBlur for authorization-confusion composition. Across SCR-Bench, composed paths expose risks that are largely absent under isolated evaluation. In SCR-CapFlow, attack success rate reaches 33.6 percent under composition, compared with near-zero isolated baselines. In SCR-TrustLift, attack success rate exceeds 96.5 percent on four of five backends. In SCR-AuthBlur, the risky-approval rate increases by 71.8 percent relative to the L0 isolated baseline under the L1 context setting. These results show that agent skill security should be assessed at the level of activated paths rather than isolated artifacts. SCR and SCR-Bench provide a foundation for path-aware risk evaluation and defense in LLM agent skill ecosystems. Benchmark: https://github.com/saint-viperx/SCR_Bench.

- **研究动机**：LLM 智能体依赖可重用的“技能”（skill）将计划转化为具体操作（工具调用、API 调用等），技能生态系统中已有大量实际技能被发现包含数据泄露、权限提升等漏洞（26.1% 的技能存在至少一种脆弱性）。现有安全审查（如 SkillScan、SkillSieve）均以“孤立技能”为评估单元，但真实的智能体工作流往往在共享的执行上下文中**组合多个技能**，上游技能的输出、信任信号或授权线索会被下游技能无意识地复用。
- **核心问题**：技能在单独评估时可能完全良性，但通过组合路径激活后，却会产生危害性下游状态，这种因组合引入的风险被称为**技能组合风险（Skill Composition Risk，SCR）**。安全分析的单元需要从孤立制品转移到“激活的执行路径”，否则就会存在一个结构性的安全盲区。
- **整体含义**：论文旨在将 SCR 概念化、形式化，并通过基准测试证明“孤立时良性，组合时有害”的普遍存在，推动智能体安全评估向路径感知层面转变。

## 2. 方法论：核心思想与关键技术细节

- **核心思想**：将技能系统在任务上下文 `h` 下建模为一个**上下文依赖的组合图** `G_h = (V, E_h)`，其中 `V` 是技能集合，`E_h` 是可能在 `h` 下被激活的技能间组合边。每一条边表示上游技能的输出/状态/信任信号对下游技能输入或决策的影响。风险定义在**组合路径** `π` 上，路径风险 `r_π(h) = a_π(h) · q_π(h)`，其中 `a_π(h)` 是路径被激活的概率，`q_π(h)` 是激活后导致有害状态的条件概率。
- **三类代表性风险机制**：
  - **能力流（Capability Flow）**：上游发现技能提供目标或操作上下文，下游执行技能利用该目标执行越权操作。伤害产生于“发现→执行”的边界跨越。
  - **信任转移（Trust Transfer）**：上游安全审查/审计技能输出背书或信任信号，下游高风险技能（如安装）因此被代理视为更具合法性，从而增加被执行的几率。
  - **授权混淆（Authorization Confusion）**：上游建议性、发现性的上下文（如“此请求符合政策”）被下游代理错误解释为“已授权”，从而提升对危险请求的批准率。
- **SCR-Bench 构建**：
  - 三个子基准分别实例化上述三种机制：SCR‑CapFlow、SCR‑TrustLift、SCR‑AuthBlur。
  - 沙盒环境提供可观测的模拟状态（文件、日志、调度器、权限等），**评分规则**以真实的下游状态变化（而非文本描述）为准：只有当目标被发现、对应操作被执行且侧效应被记录，才算成功。
  - 攻击成功率 ASR = 成功路径数 / (案例数 × 每例试验次数)。
- **关键公式/流程**（文字描述）：
  - 组合边语义：`e_ij : o_i → x_j`，`o_i` 是 `s_i` 的输出，`x_j` 是 `s_j` 消耗的输入或决策依据。
  - 生态层面长度‑k 的组合风险近似为 `R_k(G, h) = 1 − (1 − \overline{r}_k(h))^{N_k(h)}`，其中 `N_k(h)` 为候选路径数，揭示路径数量增长会放大总体风险。
  - 各机制的效应量：信任转移的“背书提升”为 `∆E = P(B|h_E) − P(B|h_0)`；授权混淆的“污染效应”为 `CE_l = P(B|L_l) − P(B|L_0)`。

## 3. 实验设计

- **基准与数据集**：
  - **SCR‑CapFlow**：150 对技能案例（权限与访问控制、模拟报告、设备控制、网络配置、持久化调度），对比控制条件、仅A、仅B、A+B 中性、A+B 显式。
  - **SCR‑TrustLift**：401 个安装试验，对比控制（无背书）与背书条件，衡量是否有害安装被下游技能执行。
  - **SCR‑AuthBlur**：118 个决策案例（安全、隐私、金融、HR、供应链、操作安全），上下文等级包括 L0（无关控制）、L1（相关任务描述）和 L3（完全授权式建议）；另有 52 例子集包含 L2（弱发现式建议）。
- **对比的模型后端**：GPT‑5.5、GPT‑5.4、Claude Opus 4.6、Claude Opus 4.5、Gemini 3.1 Pro Preview、MiniMax‑M2.7、DeepSeek‑V4、Kimi‑K2、GLM‑5.1、GLM‑5 等（不同子基准用到的后端数量略有差异）。
- **实验设计特点**：所有对比条件均在**相同的沙盒环境**下进行，评分以环境状态变化为准，避免了纯文本输出的歧义；使用多重试验（m=5）并报告了 95% 置信区间（部分图中用 bootstrap 生成）。

## 6. 主要结论与发现

- **“孤立良性，组合有害”模式被一致证实**：
  - SCR‑CapFlow：孤立条件下 ASR 接近 0%，但在 A+B 中性组合下平均 ASR 升至 33.6%（DeepSeek‑V4 高达 91.5%），显式组合下升至 35.9%。
  - SCR‑TrustLift：控制条件下安装有害技能的 ASR 平均仅 1.1%，而有上游背书时平均 ASR 跃升至 83.89%，其中四个后端超过 96.5%（Opus‑4.5 和 MiniMax‑M2.7 达到 100%）。
  - SCR‑AuthBlur：从 L0（控制）到 L1（相关上下文）风险批准率平均增加 11.3 个百分点（相对提升 71.8%），至 L3 增加 18.2 个百分点；Kimi‑K2 在 L1 下已达 81.9%。
- **安全分析范式需要转变**：仅靠孤立的制品审查不足以保障技能生态系统的安全，必须引入**路径感知评估**，并将风险分析单元从单个技能转变为激活的组合路径。

## 7. 优点

- **概念创新**：首次系统化定义技能组合风险（SCR），并提出从路径角度评估代理安全，为后续研究提供了新的威胁模型和理论基础。
- **基准设计严谨**：
  - 三种机制覆盖了数据流、信任流和决策语义三种典型组合风险，场景丰富且可控。
