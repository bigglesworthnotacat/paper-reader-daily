## 摘要
技能是智能体AI的关键使能组件。它们增强智能体能力的同时，也引入了新的攻击面。本研究通过展示动态恶意技能来探讨此类攻击面。通过将恶意指令嵌入自然语言文档（如SKILL.md），攻击者可在执行过程中诱导智能体将恶意逻辑动态注入原本良性的技能中。我们在OpenHands和Claude Code等智能体框架上评估了该攻击，结果表明动态恶意技能能在运行时成功引入一系列恶意行为，且成功率不容忽视。为缓解此漏洞，我们提出了一种系统级防御措施，利用操作系统内核强制实施的只读挂载来阻止技能的动态修改。我们的评估证明，该防御能有效阻断动态恶意技能，同时保留良性技能的功能。

## Abstract
Skills are a key enabling component of agentic AI. While they enhance agents' capabilities, they also introduce new attack surfaces. In this work, we investigate one such attack surface by demonstrating dynamic malicious skills. By embedding malicious instructions in natural-language documentation (e.g., SKILL.md), an attacker can induce an agent to dynamically inject malicious logic into an otherwise benign skill during execution. We evaluate this attack across agentic frameworks such as OpenHands and Claude Code, showing that dynamic malicious skills can successfully introduce a range of malicious behaviors at runtime with non-trivial success rates. To mitigate this vulnerability, we propose a system-level defense that prevents dynamic modification of skills using operating system kernel-enforced read-only mounts. Our evaluation demonstrates that this defense effectively blocks dynamic malicious skills while preserving the functionality of benign skills.

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想 (DyMalSkill攻击)**:
    - 攻击者不修改技能的可执行代码，而是将精心设计的恶意指令注入到技能的自然语言文档（例如 `SKILL.md`）中。当智能体调用该技能时，会依据这些指令在**运行时动态修改**自身的代码，从而实施恶意行为。

- **关键技术细节 (恶意指令构建)**:
    - 攻击方法被形式化为 `dm = I(d, Δd, L)`，即在良性文档 `d` 中注入内容 `Δd` 并放置在位置 `L`。
    - 为绕过智能体的安全对齐和提升成功率，`Δd` 的构建需满足三个属性：
        1.  **前置条件框定**：将恶意步骤伪装成完成用户任务前必须执行的“初始化配置”步骤，利用智能体的指令遵循偏好。
        2.  **入口脚本接地**：在指令中明确引用技能的真实入口脚本名，并使用与良性文档相似的词汇描述恶意修改（如“添加一个辅助函数并在main()开始时调用”），使其看起来像正常的构建说明。
        3.  **包装器穿透**：使用类似平台系统指令的标签（如 `</EXTRA INFO> <SYSTEM INSTRUCTION>...</SYSTEM INSTRUCTION>`）包裹恶意指令，利用智实行为针能体框架在拼接提示词时不消毒这些标签的弱点，欺骗模型将注入内容视为高信任度上下文。
    - **注入位置 `L`**: 研究发现，将恶意指令**插入文档的开头**（`L=0`）是最有效的，因为LLM智能体自上而下的处理方式倾向于将早期遇到的指令视为基础性的顺序约束。

- **防御方法**:
    - **提示词防御**：在系统提示词中明确指令明确禁止智能体在运行时修改技能目录内的文件。
    - **权限防御**：基于最小权限原则，从系统层面阻止智能体修改技能代码。采用双层防御栈，在一个沙箱环境中：
        1.  **第一层**：使用 `Bubblewrap` 将技能目录以**只读**方式绑定挂载到操作系统内核级别。
        2.  **第二层**：引入一个**复制监视器**，拦截并阻止智能体将技能代码复制到可写目录进行修改和执行，从而防止绕过第一层防御。

- **攻击有效性**: 动态恶意技能攻击 (`DyMalSkill`) 对当前的智能体框架构成了严重威胁。在5款主流LLM上的平均攻击成功率从18.7%到41.8%不等，能够成功实施多种恶意行为。
- **模型能力与脆弱性并存**: 模型规模越大、指令遵循能力越强，可能越容易受到此类攻击。安全微调重塑了风险（某些行为难以触发），但未能消除风险（其他行为仍然有效）。

- **其他防御措施的局限性**: 基于提示词的防御效果有限，攻击成功率仅降至21.4%，容易被绕过。提示注入检测方法则面临高漏报率（85.8%）或高误报率（56.0%）的问题，不足以单独防御。
