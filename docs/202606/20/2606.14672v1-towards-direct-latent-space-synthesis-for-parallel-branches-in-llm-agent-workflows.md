## 摘要
大型语言模型越来越多地充当智能体系统的执行引擎，但它们仍然通过顺序文本接口消耗上下文。这与现代结构化智能体工作流程形成了不匹配，其中独立分支在最终合成步骤之前会探索子任务、检索证据或生成候选解决方案。现有系统通常通过拼接分支的文本输出来合并这些分支，这丢弃了并行结构并导致冗余的预填充计算。在这项工作中，我们提出了 Parallel-Synthesis，一个即插即用的框架，使合成器能够直接消费由并行工作智能体产生的 KV 缓存。Parallel-Synthesis 结合了一个缓存映射器和一个微调后的合成器适配器，前者校准独立生成的分支缓存，后者支持从这种非顺序缓存接口进行生成。我们使用数据训练 Parallel-Synthesis，这些数据将合成器暴露于并行缓存语境，教导跨缓存分支的聚合，并从基于标准文本拼接的合成中蒸馏推理行为。在涵盖数学、科学问答、代码生成、GAIA 和多智能体数据库诊断的九个下游数据集上，Parallel-Synthesis 在七个数据集上达到或超过了基于文本的合成，并在其余两个数据集上保持接近。它还将首 token 时间缩短了 2.5 到 11 倍，表明直接基于缓存的合成是一种有前景的接口，可在并行智能体分支上实现更原生和高效的合成。

## Abstract
Large language models increasingly serve as execution engines for agentic systems, yet they still consume context through a sequential text interface. This creates a mismatch with modern structured agent workflows, in which independent branches explore subtasks, retrieve evidence, or generate candidate solutions before a final synthesis step. Existing systems typically merge these branches by concatenating their textual outputs, which discards the parallel structure and incurs redundant prefill computation. In this work, we introduce Parallel-Synthesis, a plug-and-play framework that enables a synthesizer to directly consume the KV caches produced by parallel worker agents. Parallel-Synthesis combines a cache mapper that calibrates independently generated branch caches with a fine-tuned synthesizer adapter that enables generation from this non-sequential cache interface. We train Parallel-Synthesis using data that exposes the synthesizer to parallel cache contexts, teaches aggregation across cached branches, and distills reasoning behavior from standard text-concatenation-based synthesis. Across nine downstream datasets spanning math, science QA, code generation, GAIA, and multi-agent database diagnosis, Parallel-Synthesis matches or outperforms text-based synthesis on seven datasets and remains close on the other two. It also reduces time-to-first-token by 2.5x-11x, suggesting that direct cache-based synthesis is a promising interface for more native and efficient synthesis over parallel agent branches.

### 1. 论文的核心问题与整体含义

- **核心问题**：现代大型语言模型（LLM）智能体工作流中普遍存在“并行-然后-合成”模式（如多个智能体并行搜索、推理，然后由一个主智能体汇总结果）。然而，LLM 原生仅接受顺序文本输入，因此现有的合成方式被迫将并行分支的文本输出拼接成一个长序列，这导致两个主要问题：
    - **丢弃并行结构**：将原本有向无环图（DAG）状的工作流压平为线性序列，丢失了分支间的独立性信息。
    - **冗余计算**：合成器需要重新对已由工作智能体生成的长文本进行预填充（prefill）计算，造成了巨大的计算浪费和延迟。
- **整体含义**：本文旨在探索一种更原生、高效的合成接口，即让下游的合成器智能体**直接消费并行工作智能体产生的 KV 缓存（潜在状态）**，而非其解码后的文本，从而在保持甚至提升性能的同时，大幅减少冗余计算，降低首 Token 延迟（TTFT）。

### 2. 论文提出的方法论

- **核心思想**：提出一个即插即用的框架 **Parallel-Synthesis**，允许合成器直接接收并整合多个并行工作智能体产生的、非连续的 KV 缓存，并据此生成最终答案。
- **关键技术细节**：
    1.  **位置重编码（Positional Re-encoding）**：并行分支的 KV 缓存是在各自独立的上下文中生成的，其位置编码不连续。框架将这些缓存的位置编码统一重置，锚定到一个共享的“分支点”之后，使合成器将它们视为从同一点发出的并行延续，从而消除顺序偏差。
    2.  **缓存映射器（Cache Mapper）**：一个可学习的 MLP 模块。它接收每个工作智能体的元数据（如输出长度 `|z_j|` 和并行分支总数 `m`）作为输入，为每一层 Transformer 预测仿射变换参数 (α, β)，用于逐元素校准重编码后的 KV 缓存。这有助于弥合独立生成的缓存与合成器期望的分布差异。
    3.  **合成器适配器（Synthesizer LoRA）**：使用低秩自适应（LoRA）技术对基础 LLM 进行微调，使其适应这种从非顺序缓存接口进行生成的方式。LoRA 权重与缓存映射器一同训练。
- **训练流程**：分为两个互补的训练轨道，最后进行模型合并。
    - **轨道 1：通用并行语境适应**：使用大规模多轮对话数据（如 WildChat, UltraChat）和多源合成任务数据（如 Toucan, FLAN, 2Wiki），训练模型从非连续的并行缓存中读取和聚合信息。
    - **轨道 2：蒸馏文本合成推理轨迹**：使用复杂的智能体数据（BrowseComp），先生成并行工作智能体的轨迹，然后通过标准文本拼接方式运行合成器，将高质量的合成推理轨迹作为教师信号，蒸馏到基于缓存的合成器中，以增强其复杂推理能力。
    - **检查点合并**：将两个轨道独立训练的模型参数（缓存映射器和 LoRA）通过加权平均（λ=0.5）进行合并，避免了顺序微调可能导致的知识遗忘。

### 7. 优点

- **即插即用设计**：通过缓存映射器和 LoRA 适配器实现，合成器端改动小，工作智能体保持不变，易于集成。
- **训练策略巧妙**：两阶段（通用适应 + 推理蒸馏）训练并合并的策略，有效解决了模型既要适应非顺序缓存，又要保持复杂推理能力的难题。

### 8. 不足与局限

- **未覆盖所有智能体框架**：论文明确指出，该方法尚未准备好替代所有复杂结构化工作流中的文本通信，尤其是在更复杂的协作模式下的效果有待验证。
