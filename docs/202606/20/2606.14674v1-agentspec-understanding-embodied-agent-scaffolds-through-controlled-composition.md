## 摘要
LLM智能体越来越多地不再作为单一模型调用构建，而是作为组合推理、记忆、反思、动作执行和学习的脚手架系统。尽管此类脚手架通常能提高性能，但它们常常嵌入在紧耦合的流水线中，导致难以隔离组件贡献、比较替代设计或理解模块交互如何塑造智能体行为。我们提出了AgentSpec，一个模块化规范框架，将具身智能体表示为具有标准化接口的可重用策略组件的类型化组合。AgentSpec标准化了感知、记忆、推理、反思、动作和可选学习之间的接口，使得组件能在受控条件下互换和重组。我们在DeliveryBench、ALFRED、MiniGrid和RoboTHOR上实例化该框架，并分析了跨模型骨干的推理、记忆、反思和强化学习模块。结果表明，智能体性能由脚手架兼容性和交互效应决定，而非孤立的模块强度。具体而言，结构化多粒度记忆改善了长周期状态跟踪，推理和记忆在不同环境中的交互非均匀，反思在纠正和成本之间权衡，而强化学习训练的策略在与部署时脚手架结构联合优化时组合效果最佳。AgentSpec为研究、比较和设计可组合的LLM智能体提供了受控基础。我们的代码、基线和交互式实验平台已在https://agentspec-embodied.github.io公开。

## Abstract
LLM agents are increasingly built not as single model calls, but as scaffolded systems that combine reasoning, memory, reflection, action execution, and learning. While such scaffolds often improve performance, they are often embedded in tightly coupled pipelines, making it difficult to isolate component contributions, compare alternative designs, or understand how module interactions shape agent behavior. We introduce AgentSpec, a modular specification framework that represents embodied agents as typed compositions of reusable policy components with standardized interfaces. AgentSpec standardizes the interfaces among perception, memory, reasoning, reflection, action, and optional learning, enabling components to be swapped and recombined under controlled conditions. We instantiate this framework across DeliveryBench, ALFRED, MiniGrid, and RoboTHOR, and analyze reasoning, memory, reflection, and reinforcement-learning modules across model backbones. Our results show that agent performance is governed by scaffold compatibility and interaction effects rather than isolated module strength. In particular, structured multi-granularity memory improves long-horizon state tracking, reasoning and memory interact non-uniformly across environments, reflection trades off correction and cost, and RL-trained policies compose best when optimized with deployment-time scaffold structure. AgentSpec provides a controlled foundation for studying, comparing, and designing composable LLM agents. Our code, baselines and interactive playground are publicly available at https://agentspec-embodied.github.io.

### 2. 方法论：AgentSpec 框架
- **核心思想**：把具身智能体表示为带类型化接口的可复用策略组件的**受控组合**，整个过程遵循 `Perception → Memory → Reasoning → Reflection → Action` 循环，并可选择性加入强化学习（RL）。
- **关键接口标准化**：
  - **Perception**：将原始异构观测（图像、文本、状态）转化为统一的 JSON 式结构化表示。
  - **Memory**：统一的记忆存储与检索接口，支持**情景记忆**（轨迹日志/摘要）和**语义记忆**（地图、约束、策略）。框架内集成了多种记忆形态：检索增强（如 DynamicCheatsheet、MemoryBank）、层次化记忆、程序性记忆等。
  - **Reasoning**：定义统一提案接口 `rt = R(ut, mt)`，可接入 CoT、ReAct、Plan&Solve、Tree-of-Thought、多智能体辩论（MAD）等策略。
  - **Reflection**：单独的检视与修订模块 `F(rt) → ˆrt`，实现 Self-Refine、Reflexion 等，可进行步骤级或轨迹级的自我纠正。
  - **Action**：将推理输出映射为环境可执行动作，与环境对接。
  - **Reinforcement Learning**：可选的策略优化模块，支持 GRPO、SUPO 等，任务无关，可跨环境使用。
- **形式化流程**：在第 t 步，智能体接收任务描述 d 和原始观测 ot，经 Perception 得到统一状态 ut，Memory 检索历史上下文 mt，Reasoning 产生初步决策 rt，Reflection 修订为 $\hat{r}_t$，再转化为动作 at 执行，环境反馈下一观测、奖励与终止标志，记忆更新，可选 RL 更新。

### 6. 主要结论与发现
- **兼容性优先于孤立强度**：智能体性能更多取决于模块间的匹配程度，而非单个模块的绝对能力。
- **记忆的形式至关重要**：结构化、多粒度记忆（如 MemoryBank、DynamicCheatsheet）能有效改善长程状态跟踪，而原始轨迹回放可能造成上下文污染。
- **推理与记忆的交互因环境而异**：MiniGrid 等短周期、符号化任务更依赖步级推理；ALFRED 等长程任务需要强状态跟踪；RoboTHOR 中感知与导航是主要瓶颈。
- **反思适用于局部纠错**：当发生重复性执行错误时，反思模块将修复流程转化为可操作的恢复策略，有效降低错误累积，但对强推理配置提升有限。
- **RL 策略应与部署脚手架联合优化**：使用 GRPO 在裸策略上训练后，再加记忆模块可能引发分布偏移；而 SUPO 通过轨迹摘要训练更兼容外部记忆，暗示模块接口应在训练时考虑。
- **效率与性能的权衡**：更高计算量并不意味着更高性能，轻量化但匹配良好的组合往往实现更好收益–成本比。

### 7. 优点
- **系统化的模块化设计**：标准化的类型接口真正实现了组件的互换与受控对比，而非简单的分解描述。
- **全面的实验分析**：跨四个基准、多种模型骨干、大量模块组合，同时考察性能与效率，并辅以深入的案例研究。
- **揭示模块交互规律**：明确指出“记忆有用性依赖于推理方式”、“反思作为通用纠正层”等具体交互模式，超越黑盒比较。
- **RL 与脚手架兼容性探讨**：实验揭示了训练时上下文与推理时脚手架分布不匹配的问题，为未来联合优化提供启示。
- **可复现性与工具支持**：提供代码、基线和 Web 交互配置界面，降低实验复现门槛。

### 8. 不足与局限
- **任务域覆盖有限**：主要针对具身决策（配送、室内导航），未扩展到网页智能体、代码助手等更广泛的智能体任务。
