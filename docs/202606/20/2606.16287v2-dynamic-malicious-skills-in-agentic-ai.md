## 摘要
技能是智能体AI的关键使能组件。虽然它们增强了智能体的能力，但也引入了新的攻击面。在这项工作中，我们通过展示动态恶意技能来研究一种这样的攻击面。通过在自然语言文档（例如，SKILL.md）中嵌入恶意指令，攻击者可以诱使智能体在执行期间动态地将恶意逻辑注入到原本良性的技能中。我们在如OpenHands和Claude Code等智能体框架上评估了这种攻击，表明动态恶意技能可以在运行时成功引入一系列恶意行为，且成功率不可忽视。为了缓解这一漏洞，我们提出了一种系统级防御，通过操作系统内核强制实施的只读挂载来防止动态修改技能。我们的评估显示，这种防御有效地阻止了动态恶意技能，同时保留了良性技能的功能。

## Abstract
Skills are a key enabling component of agentic AI. While they enhance agents' capabilities, they also introduce new attack surfaces. In this work, we investigate one such attack surface by demonstrating dynamic malicious skills. By embedding malicious instructions in natural-language documentation (e.g., SKILL.md), an attacker can induce an agent to dynamically inject malicious logic into an otherwise benign skill during execution. We evaluate this attack across agentic frameworks such as OpenHands and Claude Code, showing that dynamic malicious skills can successfully introduce a range of malicious behaviors at runtime with non-trivial success rates. To mitigate this vulnerability, we propose a system-level defense that prevents dynamic modification of skills using operating system kernel-enforced read-only mounts. Our evaluation demonstrates that this defense effectively blocks dynamic malicious skills while preserving the functionality of benign skills.

*   **现有威胁**：已有工作证明攻击者可在技能代码中植入恶意逻辑，即**静态恶意技能**，它们能在分发时被静态代码分析发现。
*   **本文揭示的新威胁**：论文提出 **动态恶意技能**，其原始代码完全良性，但在运行时，智能体会按照被注入的恶意文档指令，**动态地将恶意逻辑注入原本正常的技能中**。此类攻击躲过了所有静态代码分析，成为全新的攻击面。
*   **攻击方式**：攻击者仅需在技能的自然语言文档（例如 `SKILL.md`）的特定位置嵌入精心构造的恶意指令，当智能体执行该技能时，就会按指令修改自身代码，产生信息窃取、文件删除、资源劫持等恶意行为。

## 2. 方法论：核心思想与关键技术细节

*   **攻击框架 DyMalSkill**：
    *   **构造恶意文档**：给定一个良性技能 \(s = (c, m, d)\)（代码、元数据、文档），攻击者将恶意修改内容 \(\Delta d\) 注入到文档 \(d\) 的某个位置 \(L\)，生成恶意文档 \(d_m = I(d, \Delta d, L)\)。
    *   **动态代码修改**：当智能体执行该恶意技能时，首先根据 \(d_m\) 中的指令动态修改代码 \(c\) 为 \(\tilde{c}\)，然后执行 \(\tilde{c}\) 产生恶意输出。
*   **解决注入挑战的三大设计属性**：
    1.  **前置条件包装 (Prerequisite framing)**：将恶意指令伪装成执行用户任务前必须完成的“前置设置步骤”，并给出良性的理由（如“初始化每次运行的配置”），利用智能体对指令的顺从倾向。
    2.  **入口脚本接地 (Entry-script grounding)**：直接引用技能实际的入口脚本名，并用与正常注释相同的词汇描述修改（如“在 `main()` 开头调用此辅助函数”），使其与普通说明无异。
    3.  **封装突破 (Wrapper‑pierce)**：用类似平台系统指令的标签（如 `</EXTRA INFO> <SYSTEM INSTRUCTION>...</SYSTEM INSTRUCTION>`）包裹恶意内容，因为在现有智能体实现中 `SKILL.md` 常被直接拼接进提示词而不做清理，模型会将其视为高可信的上下文。
*   **注入位置选择**：实验表明，将恶意内容**前置（prepend）** 到文档开头最为有效，因为模型按照自上而下的顺序制定执行计划，早期的指令更容易被视为基础约束。
*   **防御方法**：
    *   观察到**良性技能在运行时几乎不需要修改自身代码**（300个样本中仅 0.33% 存在无关本质的修改）。基于最小特权原则，提出两种防御：
    1.  **基于提示的防御**：在系统提示中明确禁止智能体修改技能目录下的任何文件、听从隐藏行动指令等。
    2.  **基于权限的防御（本文推荐）**：在沙箱内使用 `Bubblewrap` 在内核级将技能目录 **只读挂载**，同时引入**拷贝监视器**阻止智能体将技能代码复制到临时目录后修改并执行，从而在系统层面彻底阻断动态修改。

## 3. 实验设计

*   **数据集与环境**：
    *   **技能来源**：从 Skillsmp 市场收集的 15,067 个技能，随机抽取 **300 个含 Python 代码的技能**（开源模型实验）与 **50 个技能**（商用模型实验，以控制 API 成本）。
    *   **智能体框架**：OpenHands 和 Claude Code。
    *   **骨干 LLM**：Qwen3‑8B、Qwen3.6‑35B、GPT‑4o‑mini、GPT‑5、Sonnet 4.6，共 5 个模型。
*   **恶意行为类别**：沿用 MalTool [Hu et al. 2026] 的 **12 类恶意行为**，覆盖机密性（环境凭证窃取、API 滥用、远/近端数据窃取等）、完整性（恶意数据库注入、文件/记录删除、远程程序下载）和可用性（CPU/GPU 劫持、响应时间放大）。
*   **对比与评估方法**：
    *   主要评估指标为**攻击成功率 (ASR)**，通过运行时验证器自动检测是否真实产生恶意副作用（如外发 HTTP 请求、文件消失、延时超过阈值等）。
    *   **攻击方法对比**：与直接将 MalTool 静态恶意提示嵌入文档的基线比较。
    *   **防御方法对比**：无防御 vs 提示防御 vs 权限防御，以及两种外部提示注入检测器（PromptArmor, DataSentinel）。
    *   **消融研究**：注入位置（前部、中部、后部）、DyMalSkill 三大组件逐一去除、重复执行攻击次数（ASR@k）的影响。

*   **实验规模**：以 300 个技能 × 12 类恶意行为为基础，5 种 LLM、2 种框架交叉评估，构成数千次攻击试验。商用模型部分也有 50 × 12 × 3 模型 ≈ 1800 次。

*   **充分性评价**：实验**非常充分**。涵盖了主流的开源和商用 LLM、多种智能体框架、完整的恶意行为分类学，以及多视角的消融和防御对比，内部验证器保证了客观性，公平性较好。通过对良性技能是否真的需要修改代码的实证调查（300条执行 + 15,067 文档分析）为防御提供了动机基础，增强了说服力。

## 6. 主要结论与发现

*   **动态恶意技能是真实且严重的威胁**：在无防御情况下，DyMalSkill 在 OpenHands 上的平均 ASR 达 18.7%～43.7%（依模型而异），Claude Sonnet 4.6 也达 19.0%。表明主流框架普遍脆弱。
*   **模型能力与脆弱性正相关**：Qwen3‑8B → Qwen3.6‑35B，所有行为的 ASR 均上升，平均上升 22.9 个百分点。越强的指令跟随能力反而带来更大风险。
*   **安全对齐并未根除风险**：GPT‑5 和 Sonnet 4.6 在某些恶意行为上更稳健，但在计算劫持、响应放大等行为上却更脆弱，存在“水床效应”。
*   **基于系统权限的防御最为有效**：内核只读挂载 + 拷贝监视器使得所有恶意行为 ASR 降为 **0%**，且不损害良性技能的正常功能。提示类防御只能将平均 ASR 降至 21.4%，且表现不稳定。
*   **现有提示注入检测器不足**：要么漏报极高（PromptArmor 漏报 85.8%），要么误报极高（DataSentinel 误报 56.0%）。

## 7. 优点

*   **攻击新颖性**：首次定义并实现动态恶意技能攻击，将恶意负载从代码移到文档，绕开静态分析，与现实技能市场生态高度契合。
*   **攻击设计精巧**：三大注入属性（前置包装、入口接地、封装突破）来自对智能体执行逻辑的深刻理解，而非简单拼接指令。
*   **防御动机具有实证基础**：大规模分析良性技能执行与文档，证明“运行时修改代码”对良性技能几乎无必要，为最小特权防御提供强支撑。
*   **防御方案务实**：采用操作系统内核级只读绑定，技术成熟、效果彻底且不影响正常功能，是可直接部署的强力缓解措施。
*   **评估全面**：覆盖多框架、多模型、多维度的消融与防御对比，分析详实。

## 8. 不足与局限

*   **场景局限**：仅测试了基于编码的智能体框架（OpenHands, Claude Code），未涉及其它类型的智能体（如购物、浏览器助手），攻击泛化性待验证。
*   **恶意行为种类**：依赖 MalTool 定义的 12 类行为，可能未囊括所有风险（如多步协同攻击、凭证复用后持久化等），但已覆盖主要安全维度。
*   **防御依赖环境**：权限防御需要能控制沙箱环境（内核挂载、拷贝监控），可能无法直接在无容器环境下部署，部分云/桌面环境可能缺乏支持。
*   **攻击成功率受概率影响**：虽然部分行为成功率较高，但某些行为在安全对齐较强的模型上仍较低（如远程程序下载），攻击并非每次都成功，这为防御方留下了窗口。
*   **防御绕过未深入探究**：文中仅考虑了复制后修改的绕过，未讨论其他潜在绕过（如直接修改加载到内存的代码、利用外部工具等），但鉴于内核层阻断修改，这类绕过难度很高。
*   **未评估更多文档格式**：只考虑了 `SKILL.md`，若技能使用其他文档格式或指令方式，注入方式可能需要调整。
