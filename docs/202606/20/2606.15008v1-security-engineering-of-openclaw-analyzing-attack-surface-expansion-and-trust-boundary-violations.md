## 摘要
智能体大型语言模型系统现已能够执行操作，而不仅仅是生成文本。当模型输出触发特权操作（如 shell 命令、浏览器自动化或外部工具调用）时，安全问题便从单纯的对齐转向系统配置和结构设计。我们分析了 OpenClaw，这是一个自托管的多智能体系统，其中 LLM 输出可以执行命令并与工具和服务交互。我们测量了入侵概率、边界失效、权限漂移，以及这些指标如何随着攻击者能力的增强而变化。使用单个智能体时，入侵概率为 0.24。当系统执行操作，若有七个智能体且任一智能体提出该操作，入侵概率升至 0.86。模型本身并未改变，增加源于输出的聚合。提示注入将不稳定性传播至整个系统。攻击面熵从 0.42 增加到 0.71，表明可利用路径的分布更广。平均权限漂移从 0.03 增加到 0.21，表明非预期的权限提升。正向的权限提升曲率 0.08 表明随着攻击者能力增强，权限增长更快。包括策略门控和执行过滤在内的防御控制将入侵概率降低 0.10，边界失效降低 0.10，权限漂移降低 0.02，均在 p < 0.0001 下具有统计显著性。系统依然敏感，但缓解措施的影响是可测量的。注入缓解成功率在不同模型间存在差异：GPT-5.2 为 0.37，Llama-4-Maverick 为 0.35，DeepSeek-R1 为 0.31。当执行可由任一智能体触发时，最脆弱的智能体决定了系统的暴露程度。缓解措施将任务效用从 0.93 略微降至 0.89，并将中位延迟从 420 毫秒增至 468 毫秒。

## Abstract
Agentic large language model (LLM) systems can now execute actions, not only produce text. When model outputs trigger privileged operations such as shell commands, browser automation, or external tool calls, the security problem shifts from alignment alone to system configuration and structural design. We analyze OpenClaw, a self-hosted multi-agent system in which LLM outputs can execute commands and interact with tools and services. We measure compromise probability, boundary failures, privilege drift, and how these metrics change as attacker capability increases. With one agent, the compromise probability is 0.24. With seven agents, when the system executes an action, the compromise rises to 0.86 if any single agent proposes it. The models do not change; the increase comes from output aggregation. Prompt injection propagates instability across the system. Attack surface entropy increases from 0.42 to 0.71, indicating a broader distribution of exploit paths. The mean privilege drift increases from 0.03 to 0.21, indicating unintended authority gain. Positive escalation curvature of 0.08 indicates that privilege grows faster as attacker capability increases. Defensive controls, including policy gating and execution filtering, reduce compromise probability by 0.10, boundary failures by 0.10, and privilege drift by 0.02, all statistically significant at p < 0.0001. The system remains sensitive, but the mitigation impact is measurable. Injection mitigation success differs across models: 0.37 for GPT-5.2, 0.35 for Llama-4-Maverick, and 0.31 for DeepSeek-R1. When execution can be triggered by any single agent, the most vulnerable agent determines system exposure. Mitigations slightly reduce task utility from 0.93 to 0.89 and increase median latency from 420 ms to 468 ms.

### 关键技术形式化  
- **攻击面量化**：  
  \[
  A = \sum_{i=1}^n P_i \cdot I_i \cdot \gamma_i \cdot \lambda_i
  \]
  其中 \(P_i\) 为利用成功概率，\(I_i\) 为影响，\(\gamma_i\) 为权限放大比，\(\lambda_i\) 为持久性。所有因子归一化至 \([0,1]\)，保证不同暴露向量可比。  
- **攻击面熵**：  
  \[
  H_A = -\sum_{i=1}^n p_i\log p_i,\quad p_i = \frac{P_i I_i \gamma_i \lambda_i}{\sum_j P_j I_j \gamma_j \lambda_j}
  \]
  反映风险在暴露向量间的分布集中度。  
- **权限漂移**：以可访问凭据集合 \((C_j)\) 和可执行动作集合 \((A_j)\) 定义状态权限：  
  \[
  Priv(S_j) = \alpha|C_j| + \beta \log(|A_j|+1)
  \]
  累积权限漂移 \(CPD = \sum \Delta Priv_j\)，归一化后比较跨配置影响。  
- **累积边界不稳定性（CBI）**：加权各信任边界的失效概率：  
  \[
  CBI = \sum_j P(B_{fail,j}) \cdot Impact(B_j)
  \]  
- **注入放大因子（IAF）**：  
  \[
  IAF = \frac{\Delta Priv_{injected}}{\Delta Priv_{benign}}
  \]
  量化提示注入导致的额外权限提升。  
- **权限升迁曲率**：\(\kappa_A = d^2 A / d\|Attacker\|^2\)，正曲率表示超线性增长。  
- **防御对策形式化**：将缓解措施定义为硬化算子 \(S^{(H)} = H(S)\)，在相同攻击下比较 \(R_{mitigated}\) 与 \(R_{baseline}\)，并接受满足 \(\Delta A<0, \Delta CPD<0, \Delta IAF\le 0\) 以及各边界失效概率均下降的缓解方案。同时要求任务效用、延迟、能耗等满足约束。

- **运维权衡**：防御使任务效用从 0.93 降至 0.89，中位延迟从 420 ms 升至 468 ms，处于可接受范围。

## 7. 论文优点  
- **架构级安全分析**：超越模型层面的对齐评估，聚焦推理-执行耦合、聚合语义和信任边界，给出系统性风险解析。  
- **量化指标全面**：引入攻击面熵、CBI、权限漂移、升迁曲率等，多维度刻画风险浓度、不稳定性和非线性增长。  
- **反事实评估设计**：配对实验有效分离防御的因果效应，避免混杂，结论可靠性高。  
- **运维约束考量**：同时评估安全和效用/延迟，体现防御的可行性。  
- **多模型、多聚合策略对比**：展现模型异质性对系统安全的影响，以及聚合规则对风险的控制作用。

## 8. 不足与局限  
- **攻击者模型简化**：能力向量虽参数化，但未覆盖完全自适应的、动态策略调整的对手，可能低估真实攻击。  
- **权限测量抽象化**：权限漂移基于可访问凭据和工具数量的对数模型，可能遗漏某些具体执行路径的复杂权限继承关系。  
- **实验范围**：仅在 OpenClaw 这一种自托管架构上进行评估，缺乏对其他多智能体框架的迁移验证；云部署或沙盒环境中的隔离机制可能改变放大曲线。  
- **聚合策略种类有限**：只比较了任意代理和多数共识，未探讨加权投票、动态信任评分等更复杂策略。  
- **供应给风险估算**：依赖于概率模型，而非实际生态系统的注入与安装数据。  
- **因果关系深入不足**：虽量化了边界不稳定的中介作用，但缺乏正式的结构因果模型，推理链条仍有一定假设成分。  
- **语义层防御未涉足**：防御集中在执行侧硬化和策略门控，未探讨模型自身的对齐或提示净化等上层方法。
