## 摘要
在交互环境中，LLM代理对其提示高度敏感，但提示工程仍是一个手动、特定于任务的过程。我们引入了一种针对LLM代理的自动提示优化框架，该框架将观察-动作管道分解为目标条件描述代理和动作选择代理，并通过环境回报引导的LLM驱动的进化循环迭代优化每个模块的提示。我们提出了一个行为分析器，将情节结果归因于特定的提示组件，以及一个变异器，在通过环境回滚验证之前对提示提出有针对性的修订。我们在BALROG基准测试的所有五个BabyAI任务上进行了评估，将我们的管道与BALROG的RobustCoTAgent在普通和引导提示初始化下进行了比较。优化在不同任务和条件下持续提高性能，无需更新模型权重。在PutNext任务中，一个RobustCoTAgent成功率为0%的多步骤协调任务，使用相同的底层LLM，我们的框架实现了高达72.5%的成功率。这些结果表明，多代理框架与自动提示优化相结合，可以在不需要微调或大量人工监督的情况下增强LLM。

## Abstract
LLM agents in interactive environments are highly sensitive to their prompts, yet prompt engineering remains a manual, task-specific process. We introduce an automated prompt optimization framework for LLM agents that decomposes the observation-to-action pipeline into a goal-conditioned descriptor agent and an action selection agent, and iteratively refines each module's prompt through an LLM-driven evolutionary loop guided by environment returns. We propose a behavior analyzer to attribute episode outcomes to specific prompt components, and a mutator to propose targeted revisions to the prompt, before validating them through environment rollouts. We evaluate on all five BabyAI tasks in the BALROG benchmark, comparing our pipeline against BALROG's RobustCoTAgent under both plain and guided prompt initializations. Optimization improves performance consistently across tasks and conditions, without requiring updates to the model weights. On PutNext, a multi-step coordination task where the RobustCoTAgent achieves 0% success, our framework reaches up to 72.5% success rate using the same underlying LLM with optimized prompts. These results suggest that a multi-agent framework, combined with automatic prompt optimization, enhances LLMs without the need for fine-tuning or extensive human supervision.

### 2. 方法论
- **代理架构分解（SPA）**：将原始的单一代理拆分为两个子代理：
  - **描述代理**（$A_{des}$）：接收原始文本观察和目标，输出与目标相关的浓缩场景描述。
  - **动作选择代理**（$A_{act}$）：接收任务、描述和历史计划，输出当前动作并更新计划。
  - 两者互相独立，描述代理不推理动作，动作代理不直接解析原始观察，仅通过描述感知。
- **RAPOA 优化框架**：
  - **进化循环**：外层循环固定预算（20轮），内层循环基于一小批轨迹（$l=6$）进行分析和变异。
  - **行为分析器（BA）**：LLM 驱动的批评模块，接收轨迹、当前提示和历史变异记录，按失败严重程度排序输出归因列表，指向需修改的子代理（描述代理或动作代理）及修改建议。
  - **变异器（Mutator）**：LLM 驱动的提示编辑器，根据归因生成精确提示增删改，禁止跨模块引用，保持最小改动。
  - **两阶段评估门控**：
    - 优化阶段：候选提示在固定种子（$S_{opt}$）上运行，要求平均奖励较当前提升超阈值 $\delta$；
    - 选择阶段：候选提示在独立固定种子（$S_{select}$）上再次与当前最佳比较，通过后方可更新。
  - **两种选择压力**：
    - 高选择压力（HSP）：要求在全部 20 组优化种子上均超越；
    - 低选择压力（LSP）：仅要求在被分析的 6 组轨迹种子上超越，以允许更敏感但高噪的信号通过。
- **公式化**：将代理系统建模为有向无环图，优化目标为 $\arg\max_P J(\pi_P)$，其中 $J$ 为期望折扣回报。

### 6. 主要结论与发现
- **优化提升普遍显著**：全部任务和架构经优化后成功率均有提升，尤其在 hard 任务 PutNext 上，从基线接近 0% 提升至最高 72.5%（SPA guided，LSP）。
- **多代理架构（SPA）优于单代理**：无论初始化提示质量，SPA 在不携带滚动历史的情况下，性能优于或匹配使用 16 步历史的 RobustCoTAgent，说明将感知和决策解耦有助于提示优化。
- **行为归因与门控机制关键**：BA 能有效定位错误来源（主要在动作代理），门控可阻止劣化突变；无门控的 always-accept 模式会因累积无过滤突变而导致性能崩溃。
- **受阈值和选择压力调节**：较低的选择压力（LSP）在奖励极度稀疏的困难任务上更可能接受有效突变，但过低的 δ 可能因噪声引入退化；需平衡。
- **跨任务泛化潜力**：针对复杂任务（如 PutNext）优化的提示对其他任务也有正向迁移，因为其中蕴含了良好的探索和导航规则。

### 7. 优点
- **无需微调**：仅通过提示优化即实现大幅性能提升，适用于闭源模型或算力受限场景。
- **架构分解创新**：将感知与决策分离，降低提示工程复杂度，同时提供清晰的归因粒度。
- **环境驱动、在线优化**：利用环境反馈而非静态数据集，减少过拟合风险，优化信号更真实。
- **可解释的优化过程**：BA 和 Mutator 产出可读的提示修改，便于理解和审计。
- **详尽的消融和分析**：对阈值、选择压力、模块约束、跨任务迁移等做了多维度分析，支撑结论可靠。
- **代码开源**：提供完整实现，促进复现和扩展。

### 8. 不足与局限
- **领域单一**：仅在 BabyAI 网格世界任务上验证，未扩展到更具真实物理或竞争性游戏环境。
- **LLM 单一**：仅使用 GPT‑OSS‑20B，未测试其他模型或模型规模，结论普适性待验证。
- **优化器设计简单**：仅采用单个弹性突变和单一群体，无交叉、多样性保持等进化算法先进机制，可能早熟。
- **环境种子固定**：优化阶段虽随机化种子，但整体优化种子与 selection 种子设定固定，可能存在微弱过拟合风险。

- **计算开销较大**：每轮需大量 episode rollout，对于超大规模环境不经济。
