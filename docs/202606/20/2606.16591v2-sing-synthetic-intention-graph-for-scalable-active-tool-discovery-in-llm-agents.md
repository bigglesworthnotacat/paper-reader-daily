## 摘要
大语言模型（LLM）智能体越来越依赖管理上下文、工具和多轮执行的智能体框架，使工具成为在现实数字环境中行动的核心接口。随着框架连接的工具生态系统扩展到成百上千个API、服务和任务特定技能，穷举式工具模式注入变得昂贵，并强加了封闭世界假设，将智能体限制在预定义的静态清单中。检索增强的工具选择提供了一种自然替代方案，但现有的一次性检索方法常常无法将孤立的工具描述与智能体的真实任务意图对齐，尤其是在长程任务中，所需能力通过分解、观察和新产生的子目标逐步显现。我们提出SING，一个意图感知的主动工具发现框架，它构建连接用户意图、工具能力和工具协作模式的意图-工具图，并根据变化的任务状态动态检索工具。利用包含7,471个工具的统一语料库，我们在三个真实世界工具使用基准上评估SING。与基线相比，SING将Global Recall@5最多提高59.8%，下游成功率最多提高28.9%，同时将全语料工具模式暴露减少99.8%，证明意图感知的图结构能够在大型智能体生态系统中实现更准确且上下文高效的工具发现。

## Abstract
Large language model (LLM) agents increasingly rely on agent harnesses that manage context, tools, and multi-turn execution, making tools a central interface for acting in realistic digital environments. As harness-connected tool ecosystems expand to hundreds or thousands of APIs, services, and task-specific skills, exhaustive tool schema injection becomes costly and imposes a closed-world assumption that limits agents to a predefined static inventory. Retrieval-augmented tool selection offers a natural alternative, but existing one-shot retrieval methods often fail to align isolated tool descriptions with the agent's true task intention, especially in long-horizon tasks where required capabilities emerge through decomposition, observations, and newly induced subgoals. We propose SING, an intention-aware active tool discovery framework that builds an intention-tool graph linking user intentions, tool capabilities, and tool collaboration patterns, and dynamically retrieves tools according to evolving task states. Using a unified corpus of 7,471 tools, we evaluate SING on three real-world tool-use benchmarks. SING improves Global Recall@5 by up to 59.8% and downstream success rate by up to 28.9% over baselines, while reducing full-corpus tool-schema exposure by 99.8%, demonstrating that intention-aware graph structure enables more accurate and context-efficient tool discovery in large-scale agentic ecosystems.

### 2. 论文提出的方法论
- **核心思想**：将用户意图作为用户查询与工具描述之间的抽象中间层。通过合成意图图，显式建模工具的功能结构、协作关系，并利用图传播与动态 ReAct 流程实现主动发现。
- **离线阶段：意图图构建**
  - **查询合成**：为每个工具，利用 LLM 根据服务信息、工具描述与模式，生成多样化的自然语言指令及对应的函数调用参数，并过滤低质量或过于相似的样本。
  - **链扩展**：将单工具查询扩展为多工具链，以捕获协作模式。利用输入‑输出类型兼容性和 LLM 判断选择下一个工具，并将查询重写为更复杂的多步任务。
  - **原子意图提取**：从合成查询和工具模式中，为每个工具提取简洁的动词‑宾语意图短语（如“geocode address to coordinates”），并跨工具、跨服务归一化，去除冗余。
  - **图构建**：构建异构图 \(G = (V, E)\)，节点包括服务 \(V_s\)、工具 \(V_t\)、意图 \(V_i\)。边包括：
    - `has_tool`：服务到工具。
    - `has_intention`：工具到意图（由于意图归一化，不同服务工具可共享节点，形成跨服务语义连接）。
    - `tool_next`（有向）、`tool_cooccur`（无向）：基于合成链建立协作边，边权重为对数频率 \(w(t_a, t_b) = \log(1 + c_{a,b})\)，供 PPR 使用。
- **在线阶段：动态 ReAct 发现**
  - 智能体在每一步选择动作类型：`DISCOVER`、`INVOKE` 或 `RESPOND`。
  - 当动作类型为 `DISCOVER`，智能体基于历史与推理生成状态条件查询 \(q^{(j)}\)，通过发现管道 \(R\) 检索新工具 \(\Delta T_j\)，更新可用工具集。
  - **发现管道**：分层检索（先服务后工具）。
    - **服务级发现**：将用户请求分解为全局请求和子任务，结合直接语义匹配和意图引导的图传播。语义得分取请求与服务的最大余弦相似度；图传播得分通过将子任务意图匹配到意图节点作为 PPR 种子，结合语义相关工具节点，传播得到服务得分。最终服务得分加权组合：
      \[
      \text{score}_{\text{server}}(s) = \lambda_{\text{sem}} \hat{d}(s) + \lambda_{\text{ppr}} \hat{g}(s)
      \]
      取 top‑\(K_s\) 个服务。
    - **工具级发现**：对候选服务中的工具，为每个子任务计算工具得分，融合描述匹配、意图匹配和图 PPR 得分：
      \[
      \text{score}_{\text{tool}}(t \mid x_i) = \lambda_{\text{desc}} \text{sim}(x_i, \text{desc}(t)) + \lambda_{\text{int}} \max_{z \in I(t)} \text{sim}(x_i, z) + \lambda_{\text{graph}} \hat{g}(t)
      \]
      每个子任务返回 top‑\(N\) 个工具，取并集。
