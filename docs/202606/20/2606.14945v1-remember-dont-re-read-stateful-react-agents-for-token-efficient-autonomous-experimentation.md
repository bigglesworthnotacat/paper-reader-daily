## 摘要
自主研究模式通过让大语言模型（LLM）迭代地修改代码以优化目标指标来实现自主实验。然而，其无状态设计在每次迭代时都从头重建实验上下文，导致每次迭代的令牌成本为 $O(n)$，总计 $O(n^{2})$。本研究使用 LangGraph 将该模式重构为有状态的 ReAct 智能体，其中类型化的持久状态通过工具调用接口在迭代间传递实验历史。评估了两个基准：超参数调优（15 次迭代，每次迭代观察量小）和代码性能优化（40 次迭代，每次迭代观察量大，包含完整源代码和基准测试结果）。在超参数调优中，有状态智能体消耗的令牌减少了 90%（2,492 对比 24,465）。在代码优化中，有状态智能体消耗的令牌减少了 52%（627K 对比 1,275K），同时在两项任务上达到了相当的优化质量。令牌的减少是结构性的：无状态智能体在每次迭代中以 $O(n)$ 成本重读完整历史，而有状态智能体在固定大小的对话窗口中运行，成本为 $O(1)$。本文详细描述了该架构，以便实践者为其自身工作流实现有状态的自主研究智能体。

## Abstract
The autoresearch pattern enables autonomous experimentation by having a large language model (LLM) iteratively modify code to optimize a target metric. Its stateless design, however, reconstructs experimental context from scratch at every iteration, incurring $O(n)$ token cost per iteration and $O(n^{2})$ total. This work reformulates the pattern as a stateful ReAct agent using LangGraph, where typed persistent state carries experimental history across iterations via a tool-calling interface. Two benchmarks are evaluated: hyperparameter tuning (15 iterations, small per-iteration observations) and code performance optimization (40 iterations, large per-iteration observations containing full source code and benchmark results). On hyperparameter tuning, the stateful agent consumes 90\% fewer tokens (2{,}492 vs.\ 24{,}465). On code optimization, the stateful agent consumes 52\% fewer tokens (627K vs.\ 1{,}275K) while achieving comparable optimization quality on both tasks. The token reduction is structural: the stateless agent re-reads the full history at $O(n)$ cost per iteration, while the stateful agent operates within a fixed-size conversation window at $O(1)$ cost. This paper describes the architecture in sufficient detail for practitioners to implement a stateful autoresearch agent for their own workflows.

## 1. 论文的核心问题与整体含义
- **核心问题**：在自主实验（autoresearch）场景中，大语言模型（LLM）需反复修改代码并观察指标来优化目标。传统的**无状态设计**在每次迭代时都会将完整的实验历史（所有过去参数、结果、甚至代码）重新发送给模型，导致每次调用的输入长度随迭代次数线性增长 $O(n)$，总 token 消耗呈平方级 $O(n^2)$。
- **整体含义**：这种结构性的浪费不仅推高了成本，还因上下文窗口限制限制了实验的可扩展性。本文试图通过引入**有状态 ReAct 智能体**，让实验历史持久化，使每轮只需传输固定大小的摘要，从而实现 $O(1)$ 的 per‑step 成本与 $O(n)$ 的总成本，本质上解决了 token 效率的瓶颈问题。

## 2. 方法论
- **架构设计**：采用 LangGraph 构建一个具有三个核心节点的状态图：
  - `REASON`：调用 LLM（Claude Haiku 4.5）按 ReAct 模式生成推理足迹和工具调用。
  - `TOOLS`：确定性执行工具调用并将观察结果返回。
  - `CHECK`：不依赖 LLM 的收敛判断（达到阈值、迭代耗尽或 LLM 主动终止）。
- **状态管理**：定义 `AgentState` 类型化字典，包含 `messages`、`experiment_history`、`current_best` 等字段，在迭代间持久化。实验状态由 `ExperimentState` 类专门维护，其 `summary()` 方法仅返回最近 5 次实验的摘要与最佳指标，而非完整历史。
- **固定窗口**：对话消息列表通过滑动窗口限定为最近 20 条，从而保证每轮输入的 token 数基本恒定。
- **工具接口**：智能体通过工具调用与环境交互，例如 `run_experiment` 运行指定超参数并返回指标，`get_experiment_history` 查询历史等。生产版本还提供 `modify_train_py`（含防护规则，禁止删除评估和记录日志的代码）、`submit_training_job` 和 `query_results` 等工具。
- **算法性质**：无状态智能体每次从头重建上下文，成本为 $O(n)$/步；有状态智能体因依赖持久化状态和固定窗口，成本为 $O(1)$/步。

## 6. 主要结论与发现
- **显著 token 节省**：在超参数调优任务中，有状态智能体的总 token 消耗仅为无状态的 **1/10**（2,492 vs. 24,465）；在代码优化任务中节省 **52%**（627K vs. 1.275 M）。
- **结构性优势**：每轮输入 token 在有状态版本中保持恒定（或很快饱和），而无状态版本持续线性增长，证明 O(1) per‑step 架构的可行性。
- **优化质量持平**：两项任务中，有状态智能体达到的最佳 macro F1（0.779）和最佳速度提升（1.91x）均与无状态版本（0.764, 1.87x）在统计上无显著差异。
- **可扩展性**：由于状态机的总成本为 $O(n)$ 而非 $O(n^2)$，进一步延长实验序列时节省比例会继续扩大，且不存在上下文溢出风险。

- **简洁而高效的改造**：用 LangGraph 的状态图和类型化字典实现了状态持久化，对原 autoreearch 模式改动最小，却带来了结构性成本降级。
- **理论分析清晰**：明确指出 token 增长的复杂度区别，并通过两个典型观察量大小验证了节省比例的变化规律。
