## 摘要
对于多步交互环境中的大语言模型智能体，一个关键挑战是如何有效利用累积的交互经验。现有工作通常将此类经验的两种用途分离：将其作为自然语言规则保留在模型外部以供后续提示，或使用轨迹和反馈来更新模型参数。前者易于解释，但可能无法与不断演化的策略保持同步；后者能更广泛地改进策略，但在稀疏奖励设置中对局部错误的纠正有限。我们提出了大语言模型智能体的经验规则与策略联合学习（JERP），该方法从相同的交互轨迹中更新长期经验规则池和策略。在决策时，JERP检索任务相关规则，并将其与交互历史一起作为智能体的条件。在每个回合结束后，它使用收集的轨迹来优化策略，并通过将当前 rollout 与参考成功轨迹进行比较来修订规则池。这种耦合使规则池与不断演化的策略保持一致，同时使稳定有效的行为逐渐吸收到模型本身中。在AlfWorld和WebShop上的实验表明，JERP在复杂交互任务中取得了持续的决策性能提升。

## Abstract
For LLM agents in multi-step interactive environments, a key challenge is to make effective use of accumulated interaction experience. Existing work has typically separated two uses of such experience: keeping it outside the model as natural-language rules for later prompting, or using trajectories and feedback to update the model parameters. The former is easy to interpret but can fall out of sync with the evolving policy; the latter improves the policy more broadly but provides only limited correction for local mistakes in sparse-reward settings. We present Joint Learning of Experiential Rules and Policies for LLM Agents (JERP), which updates a long-term experiential-rule pool and the policy from the same interaction trajectories. At decision time, JERP retrieves task-relevant rules and conditions the agent on them together with the interaction history. After each episode, it uses the collected trajectories both to optimize the policy and to revise the rule pool by comparing current rollouts with reference successful trajectories. This coupling keeps the rule pool aligned with the evolving policy while allowing stable and effective behaviors to be gradually absorbed into the model itself. Experiments on AlfWorld and WebShop show that JERP yields consistent gains in decision performance for complex interactive tasks.

## 1. 核心问题与整体含义（研究动机和背景）

本文聚焦于大语言模型（LLM）智能体在多步交互环境中的经验利用问题。现有工作对交互经验有两种典型利用方式：

- **外部化规则**：将经验转化为自然语言规则（如反思文本、操作指南），存储于模型外部，在后续决策时检索使用。
- **参数更新**：利用轨迹和反馈直接更新模型参数（如强化学习微调）。

两种方式各有局限：
- 外部规则易于解释，但会随策略演化而**脱节**：策略改变后，旧规则可能失效甚至误导。
- 参数更新能全局改进策略，但在稀疏奖励场景中，对**局部错误**的纠正不足，难以及时暴露和修正。

因此，论文提出核心问题：**能否用同一批交互数据同时支持规则更新和策略优化**，实现两种利用方式的协同。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
**JERP（Joint Learning of Experiential Rules and Policies）**：在同一个训练循环中，同时更新长期经验规则池和策略参数，使规则随策略演化而动态对齐，并将稳定有效的行为逐步吸收到模型参数中。

### 关键技术细节

#### (1) 整体框架
- 为每个任务实例维护一个长期经验规则池 `K(d)`，每条规则包含文本内容 `z_i` 和效用分数 `s_i`。
- 每回合开始，根据效用分数排序，选出前 k 条规则作为当前工作规则集 `eK(d)`。
- 决策时，智能体基于任务描述 `d`、交互历史 `h_t` 和工作规则集 `eK(d)` 生成动作 `a_t`。
- 回合结束后，轨迹组 `Td` 同时用于：
  - **策略参数更新**（基于组相对优势的 GRPO 目标）
  - **经验规则池更新**（通过对比当前轨迹与参考成功轨迹，生成编辑操作）

#### (2) 轨迹采样与决策
- 规则池按效用分数降序选择 top-k 规则，并固定在一个回合内保持不变。
- 输入提示：`x_t = (d, h_t, eK(d))`，策略输出动作 `a_t ∼ π_θ(·|x_t)`。

#### (3) 策略参数更新（Stage I）
- 对每个任务实例采样 N 条完整轨迹。
- 轨迹级奖励 `R(τ_i)` 为终止时环境奖励。
- 计算组内均值 `μ_d` 和标准差 `σ_d`，得到组相对优势 `A_i = (R(τ_i) - μ_d)/(σ_d + δ)`。
- 轨迹内所有时间步共享同一优势值。
- 优化目标为 GRPO 损失（含裁剪项和 KL 正则项）：
  ```
  L_GRPO(θ) = -1/N Σ_i 1/|τ_i| Σ_t min(ρ_i,t A_i, clip(ρ_i,t) A_i) - βD_KL(π_θ ‖ π_ref)
  ```
  其中 `ρ_i,t = π_θ(a_i,t|x_i,t)/π_θ_old(a_i,t|x_i,t)`。

#### (4) 经验规则更新（Stage II）
- 输入：当前规则池 `K(d)`、当前轨迹组 `Td`、参考成功轨迹集 `T⁺(d)`（累积的训练中成功轨迹）。
- 利用一个参数固定的 LLM 生成结构化编辑操作，包含五种原语：
  - **ADD(z)**：添加新规则
  - **EDIT(q, z)**：修改规则
  - **UPVOTE(q)**：增加效用分数
  - **DOWNVOTE(q)**：降低效用分数
  - **MERGE(Q, z)**：合并相似规则
- 然后应用操作更新规则文本和效用分数，并裁剪超过容量或低于阈值的规则。
- 如果当前任务尚未积累成功轨迹，则 `T⁺(d)=∅`，仅依赖当前规则池和轨迹组。

#### (5) 算法流程（Algorithm 1）
初始化每个任务的规则池和成功轨迹集为空 → 循环训练步骤：
- 采样任务实例，构建工作规则集
- 采样 N 条轨迹，计算奖励和优势
- 用 GRPO 目标更新策略参数
- 调用 `ReflectAndEdit` 生成编辑操作，应用更新规则池
- 更新效用分数和裁剪规则池
- 将当前成功轨迹加入 `T⁺(d)`

## 3. 实验设计

### 数据集/场景
- **AlfWorld**：基于文本的家庭操控环境，包含六种任务类型（Pick, Look, Clean, Heat, Cool, Pick2）。部分可观测，需多步交互。
- **WebShop**：在线购物交互环境，要求智能体搜索、浏览、比较并完成购买。

### 评估指标
- AlfWorld：任务成功率（%）
- WebShop：平均得分和任务成功率（%）

### 基线方法
1. **Vanilla LLM**：直接提示，无推理框架、无跨回合记忆、无参数更新。
2. **ReAct**：交替推理和动作，仅依赖当前回合上下文。
3. **Reflexion**：将自然语言反思写入记忆，用于后续决策。
4. **RLOO**：用留一法基线估计优势的 REINFORCE 风格策略更新。
5. **GRPO**：组相对优势策略优化（无独立价值模型）。

所有基线均使用相同的基础模型和 LoRA 微调（GRPO、RLOO、JERP）。ReAct 和 Reflexion 无参数更新，Vanilla 为直接提示。

## 6. 主要结论与发现

- JERP 在 AlfWorld 上整体成功率 61.5%，优于 GRPO（57.8%）和 RLOO（48.7%）；在 WebShop 上平均得分 79.0，成功率 64.1%，同样领先所有基线。
- 收益集中在需要更长操作序列和更多中间约束的任务（如 Clean、Heat、Cool、Pick2），而 GRPO 在简单任务（Pick、Look）上略优，说明规则池对复杂任务帮助更大。
- 消融实验显示：动态规则池更新比固定规则池带来持续更高的成功率，尤其是在训练中后期，证明规则池与策略同步演化的关键作用。
- 验证了“相同轨迹双重使用”的设计的有效性，规则和策略互为促进。
