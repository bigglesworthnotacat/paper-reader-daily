## 摘要
大型语言模型（LLM）多智能体系统日益依赖的不仅是个体智能体的推理能力，还有智能体之间的连接方式。本文介绍QueenBee Planner框架，该框架将智能体间通信拓扑视为一种可检索且自我改进的设计技能。工作智能体池、任务适配器和评分函数被冻结；只有外部LLM规划器学习生成时序通信有向无环图（DAG），指定谁向谁发送信息、在哪一轮发送、谁合并消息、以及谁输出最终答案。执行轨迹被提炼为基于证据的设计规则，包含三种动作：“保留”、“修改”和“避免”。为防止自我进化将偶然成功或看似合理但错误的解释固化为策略，QueenBee采用了留出接受门、方差感知信用、模体级归因、迁移信任、洞察证伪和结构去重。我们在Count-Frequency聚合任务和Silo-Bench式分布式协调任务上评估该方法。在固定工作智能体的条件下，自我进化的图生成产生的通信结构优于固定拓扑和冷生成。在CF全测试设置中，最佳生成图将均方根误差从最强固定拓扑的12.53降至7.87，同时减少了消息量、模型调用次数和令牌成本；Silo式结果在冷生成和固定拓扑基线上也显示出相同的改进方向。这些结果表明，多智能体系统能够学习可重用的架构设计知识，而不仅仅是记忆任务答案。

## Abstract
Large language model (LLM) multi-agent systems increasingly depend not only on how individual agents reason, but also on how agents are connected. This paper introduces QueenBee Planner, a framework that treats inter-agent communication topology as a retrievable and self-improving design skill. A pool of worker agents, the task adapter, and the scoring function are frozen; only an outer LLM planner learns to generate temporal communication DAGs specifying who sends information to whom, in which round, who merges messages, and who emits the final answer. Execution traces are distilled into evidence-backed design rules with three actions: \emph{Preserve}, \emph{Modify}, and \emph{Avoid}. To prevent self-evolution from turning lucky runs or plausible but false explanations into policy, QueenBee uses held-out acceptance gates, variance-aware credit, motif-level attribution, transfer trust, insight falsification, and structural deduplication. We evaluate the method on Count-Frequency aggregation and Silo-Bench-style distributed coordination tasks. With fixed workers, self-evolved graph generation produces communication structures that improve over fixed topologies and cold generation. In the CF fulltest setting, the best generated graph reduces RMSE from 12.53 for the strongest fixed topology to 7.87 while also reducing messages, model calls, and token cost; Silo-style results show the same direction of improvement over cold and fixed-topology baselines. These results suggest that multi-agent systems can learn reusable architectural design knowledge rather than merely memorizing task answers.

- **核心问题**：现有LLM多智能体系统的性能不仅依赖个体推理能力，更依赖智能体间的通信拓扑结构。然而，当前方法要么采用固定拓扑（如链式、星形、网格），要么通过搜索优化单个任务的拓扑，但无法积累可重用的架构设计知识。系统缺乏一种“成为更好的架构师”的能力——即跨任务学习如何设计通信图。

  - 单智能体层面，推理结构（CoT、ToT等）显著影响性能。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程（文字说明）
### 核心思想
- **规划器-工作者分离**：冻结智能体池（worker pool）、任务适配器（task adapter）和评分函数。只有外部LLM规划器（planner）被训练，其输出是一个时间通信有向无环图（temporal communication DAG），指定每轮谁向谁发送消息、谁合并、谁输出最终答案。
- **技能库（Skill Bank）**：存储基于证据的设计规则，每条规则包含适用条件、结构描述、动作（Preserve / Modify / Avoid）、统计摘要和证据。规则来自执行轨迹的提炼。
- **自演化循环**：规划器生成候选DAG → 执行并收集证据 → 提炼技能更新 → 通过留出接受门过滤 → 更新技能库。

### 关键技术细节
1. **时间通信DAG定义**：
   - 形式化：`G = (N, T, E, h, I, R)`，其中N为智能体数，T为通信轮数，E为有向边集`(i,j,t)`表示第t轮i向j发送消息，h为最终答案持有者，I为每轮接收方指令，R为合并规则。
   - 同步屏障语义：同一轮内的边基于上一轮结束时的状态执行。
   - 可行性约束：轮数上限`T_max`、边数上限`M`、每轮每个接收方最大入度`β`。

2. **技能结构**：
   - `s = (z, r, a, q, E)`：适用条件z、设计规则r、动作a∈{Preserve, Modify, Avoid}、统计摘要q、证据E。
   - 三种动作含义：
     - Preserve：直接保留为可执行框架。
     - Modify：保留核心信息流但修改边、汇点等。
     - Avoid：作为负面约束，避免重现失败结构。

3. **模体级信用分配**：
   - 将生成图分解为8个手绘结构特征（步骤数、消息数、最大扇入、扇入桶、是否存在汇点、汇点数、是否存在审计/反向覆盖边、归约深度）。
   - 每个模体`k`收集对应的证据行，计算带样本数量惩罚的平均损失：`˜L_k = ¯L_k + κ_m / √n_k`。
   - 候选图的结构先验为共享模体的加权平均损失。

4. **自演化更新门控**：
   - **留出生成门**：候选更新`ΔS`必须使规划器在留出任务和种子上的损失降低至少ϵ才被接受。
   - **方差感知信用**：候选规则使用风险调整损失`L_risk(s) = ¯L(s) + κ_s·σ(s)/√n_s`。
   - **洞察证伪**：LLM提出的自然语言洞察需通过留出证据的证伪检查。
   - **迁移信任**：技能即使通过门控，也只在有直接成功证据的任务槽上Preserve，有间接证据时Modify，无证据时回退冷启动。
   - **结构去重**：使用时序拓扑等价哈希合并同构结构，避免技能库膨胀和信用分裂。

5. **执行流程**：

     - 提炼技能更新ΔS。

## 6. 论文的主要结论与发现
- **自我进化能生成更优通信结构**：在CF任务中，自进化最佳图将RMSE从最强固定拓扑的12.53降至7.87，同时减少消息量74.2%、模型调用54.5%、令牌成本59.6%。
- **改进来源于架构知识而非工作者能力**：由于工作者固定，改进归因于规划器生成的通信拓扑更好。
- **技能库可迁移**：在Silo任务（不同类型信息孤岛）上，自我进化图生成（0.500 EM）优于最佳单一固定拓扑（0.333 EM）和每case oracle固定拓扑（0.417 EM），表明技能库学到了更细粒度、任务条件化的结构知识。
- **进化过程非单调但总体提升**：探索阶段可能导致性能波动，但保留最佳存档随轮次下降。
- **成功图结构多稀疏且包含归约-审计模式**：常见结构包括混合归约-审计、二元树归约、平衡树归约、双根汇聚等，而非密集通信。
- **Avoid规则同样重要**：负面记忆（如链式、网格-星形）能有效防止规划器重蹈覆辙。
