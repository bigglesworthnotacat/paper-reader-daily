## 摘要
技能文档是指导大语言模型智能体的结构化自然语言指令，对现代智能体框架至关重要，然而大语言模型却难以编写出真正有效的技能。在SkillsBench上，人类编写的技能将通过率提高了16.2个百分点，而大语言模型编写的技能则没有带来可测量的提升。我们提出了SkillAxe，一个完全无监督的框架，使大语言模型能够迭代地诊断和精炼自己的技能。SkillAxe将技能质量分解为四个可解释的维度（质量影响、触发精度、带故障归因的指令依从性以及解决方案路径覆盖度），生成结构化的改进简报，无需真实标签、测试套件或环境奖励。在SkillsBench上，SkillAxe相比未改进的LLM技能将通过率相对提升了28%，并弥合了与人类编写技能之间47%至67%的差距。我们在SpreadsheetBench上验证了该方法作为持续改进引擎的有效性，SkillAxe构建的技能库从过去的智能体轨迹中学习，仅使用22个技能就将通过率从16.0%提高到52.0%。

## Abstract
Skill documents, structured natural-language instructions that guide Large Language Model (LLM) agents, are critical to modern agent frameworks, yet LLMs struggle to write skills that actually work. On SkillsBench, human-authored skills improve pass rates by 16.2 percentage points, while LLM-authored skills provide no measurable gain. We introduce SkillAxe, a fully unsupervised framework that enables LLMs to iteratively diagnose and refine their own skills. SkillAxe decomposes skill quality into four interpretable dimensions (quality impact, trigger precision, instruction compliance with fault attribution, and solution-path coverage), producing structured improvement briefs that require no ground-truth labels, test suites, or environment rewards. On SkillsBench, SkillAxe improves pass rates by 28\% relative over unimproved LLM skills and closes 47--67\% of the gap to human-authored skills. We validate the approach as a continuous improvement engine in the wild on SpreadsheetBench, where a SkillAxe-built skill library learns from past agent trajectories and raises pass rate from 16.0\% to 52.0\% using only 22 skills.

### 2. 方法论

- **核心思想**：在同一任务上分别运行**注入技能**的智能体和**无技能**的基线智能体，对比两者的输出差异，从中提取四个可解释的诊断维度，再将诊断结果转化为结构化的“改进简报”驱动技能重写。
- **四个诊断维度**：
  - **质量影响（Quality Impact）**：通过LLM裁判比较技能注入与无技能输出的偏好方向和程度，得到在[-1,1]区间的整体增益评分，判断技能是否有用、有害或无影响。
  - **触发精度（Trigger Precision）**：从技能描述中抽取正向触发短语和负向排除短语，并在嵌入空间中计算三个几何度量——覆盖广度、负向特异性、边界清晰度，诊断技能是否被正确触发。
  - **指令依从性与故障归因（Instruction Compliance with Fault Attribution）**：先将技能拆解为可评估的规则集合（含严重度权重），再判断每条规则的执行依从度、规则质量以及故障归属（是技能规则模糊导致失败还是智能体未执行），从而决定保留正确规则还是改进规则。
  - **解决方案路径覆盖度（Solution-Path Coverage）**：列举任务的多种可行解法路径，计算技能内容与各路径的嵌入相似度，判断技能是否仅覆盖狭窄的策略子集。
- **精炼流程**：以上诊断生成改进简报，由同一个LLM阅读简报后重写技能，可多轮迭代直至收敛。

### 6. 主要结论与发现

- **技能主要提升执行可靠性而非答案质量**：SkillAxe改进的技能将使输出可评估的任务比例从46.7%提升至72.7%，而输出任务中的正确率未变（57.1%），说明技能主要帮助智能体正确运行、避免崩溃。
- **在SkillsBench上**：SkillAxe将通过率相对提升28%（41.6% vs. 32.5%），弥合与人工技能差距的47–67%。
- **在SpreadsheetBench上**：SkillAxe构建的22技能库匹配了69技能未改进库的52.0%通过率，且技能激活率高一倍（35.8% vs. 20.0%），说明技能更易被智能体识别与采纳。
- **自我精炼闭环可行**：无需真实标签或奖励函数，仅靠行为对比即可生成可操作的改进建议。

### 7. 优点

- **无监督性**：完全不依赖人工标注、测试套件或环境奖励，降低部署门槛。
- **可解释的诊断维度**：将技能质量分解为四个可独立衡量的方面，为改进提供了明确的定位。
- **故障归因机制**：区分了智能体未执行和技能本身模糊两种失败原因，防止错误地修改正确规则。
- **持续改进场景验证**：在SpreadsheetBench上展示了从零构建技能库并随任务增多而自动优化的“飞轮效应”。
- **技能压缩**：精炼后的技能库规模更小（22 vs. 69），激活率更高，有利于降低推理成本。

### 8. 不足与局限

- **单次迭代约束**：实验仅执行一轮精炼，剩余与人工技能的差距可能需要多轮优化或在线执行反馈。

- **领域泛化有限**：两个基准任务类型集中（代码、表格），更广泛领域的适用性待验证。
