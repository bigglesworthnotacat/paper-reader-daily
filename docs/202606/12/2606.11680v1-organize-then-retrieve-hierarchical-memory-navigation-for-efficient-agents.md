## 摘要
大语言模型智能体在处理长周期任务时面临困境，因为其固有的无状态性要求将所有任务相关信息编码进不断增长的输入上下文中。由此导致的推理质量下降、推理成本增加和延迟升高，使得高效的记忆工作机制成为必需。然而，现有方法要么依赖有损压缩，要么基于相似性检索，往往无法捕捉多步智能体任务所需的时间结构和因果依赖。本文提出HORMA，一种分层组织与检索记忆智能体，将经历组织成类似文件系统的层次结构，其中概括的实体与相应的原始轨迹相链接，实现高效访问而不丢失细节信息。HORMA将工作记忆分解为两个阶段：结构化记忆构建和基于导航的检索。构建模块通过区分由信息缺失导致的失败和由误导性或过载上下文导致的失败，迭代优化经历的组织方式。导航模块通过使用轻量级智能体遍历层次结构来检索任务相关上下文，该智能体经强化学习训练，能选择最少且充分的上下文，从而减少关键执行路径上的延迟。在ALFWorld、LoCoMo和LongMemEval基准上，HORMA在受限上下文预算下提升了任务性能，同时在长对话任务中最多仅需基线22.17%的令牌使用量。与现有方法相比，它始终实现了更好的效率-性能权衡，并能有效泛化到未见过任务。

## Abstract
Large language model (LLM) agents struggle with long-horizon tasks due to their inherent statelessness, requiring all task-relevant information to be encoded in growing input contexts. The resulting degraded reasoning quality, increased inference cost, and higher latency necessitate efficient working memory mechanisms. However, existing approaches either rely on lossy compression or similarity-based retrieval, which often fail to capture temporal structure and causal dependencies required for multi-step agentic tasks. In this work, we present HORMA, a Hierarchical Organize-and-Retrieve Memory Agent that organizes experience into a file-system-like hierarchical structure, where summarized entities are linked to the corresponding raw trajectories, enabling efficient access without losing detailed information. HORMA decomposes working memory into two stages: structured memory construction and navigation-based retrieval. The construction module iteratively refines how experiences are structured by distinguishing between failures caused by missing information and those caused by misleading or overloaded context. The navigation module retrieves task-relevant context by traversing the hierarchy using a lightweight agent trained with reinforcement learning to select minimal yet sufficient context, thereby reducing latency along the critical execution path. Across ALFWorld, LoCoMo, and LongMemEval, HORMA improves task performance under constrained context budgets while requiring at most 22.17% of the baseline token usage in long conversation tasks. Compared to existing methods, it consistently achieves better efficiency-performance trade-offs and generalizes effectively to unseen tasks.

### **2. 论文提出的方法论**

*   **核心思想**: **HORMA (Hierarchical Organize-and-Retrieve Memory Agent)**，其核心洞见是记忆构建（长期结构优化）和记忆检索（即时推理支持）两个过程在时间尺度和功能角色上根本不同，应被解耦并分别优化。
*   **关键技术细节**
    *   **分层文件系统式记忆体**:
        *   记忆被组织成一个类似文件系统的层次化结构（如`/entities/people`, `/events`, `/raw_logs`等）。
        *   存储的“笔记”包含概括性摘要、时间戳和指向原始对话轨迹的**来源链接**，实现信息压缩与无损追溯的统一。
    *   **模块一：层次化记忆管理器（Memory Manager，`M_m`）**
        *   **功能**: 负责将原始交互轨迹异步地组织成结构化的记忆文件。
        *   **机制（技能演化）**: 不直接通过强化学习优化记忆结构，而是视其为一种技能习得过程。
            1.  **初始化**: 用强LLM的基础提示定义通用组织原则。
            2.  **对比分析**: 通过对比使用原始历史（H）和结构化记忆（H'）的任务成败，区分两类失败模式：
                *   **外生性失败（H成功，H'失败）**: 结构化记忆丢失了关键信息。
                *   **内生性失败（H'成功，H失败）**: 结构化记忆成功过滤了噪声，避免了原始历史中的推理干扰。
            3.  **技能提炼与迭代**: 针对失败模式生成自然语言反馈，并将这些反馈整合成新的记忆管理“技能”，更新至管理器提示词中。这个过程如一种“文本梯度下降”，形成一个不断演化的技能库。
    *   **模块二：层次化检索智能体（Retrieval Agent，`M_r`）**
        *   **功能**: 在推理时，从文件系统记忆中主动选择与任务最相关的上下文`C_t`。
        *   **机制（基于强化学习的导航）**:
            *   **具身动作空间**: 该智能体使用可执行的Bash命令（`ls`, `cd`, `grep`, `cat`等）以及两个自定义动作（`select`(选择上下文), `done`(完成检索)）与记忆体交互，实现层次化导航和精准定位。
            *   **优化（GRPO）**: 使用**群体相对策略优化**算法训练一个轻量级检索器。
            *   **奖励函数（证据基础的检索奖励）**: 定义奖励为检索到的上下文`C_t`与任务相关的地面真凭据`E`之间的Jaccard重叠率：`J(C_t, E) = |C_t ∩ E| / |C_t ∪ E|`。该奖励直接评估检索质量，与下游推理任务解耦，提供了更精准的监督信号。

### **6. 主要结论与发现**

*   **性能与效率的帕累托最优**: 在ALFWorld上，HORMA在实现更高成功率的同时，消耗更少的交互步数和平均令牌数，达到了更好的效率-性能平衡。
*   **极致的上下文压缩**: 在长对话任务中，HORMA的令牌使用量极低。在LoCoMo和LongMemEval上分别仅为基线方法的**3.07%–22.17%** 和**1.24%–16.19%**。
*   **强大的泛化能力**: 仅在LoCoMo上训练的轻量级检索器（Qwen 3.5 4B），无需微调即可直接泛化到ALFWorld和LongMemEval上，并在LongMemEval上取得了最优性能，甚至超过了无上下文限制的基线。
*   **机制的有效性验证**:
    *   记忆**构建**与**检索**的解耦是有效的，且记忆管理需要强LLM的语义能力（记忆构建成为瓶颈），而检索可通过小模型加强化学习来优化。
    *   基于导航的检索相比相似性检索，能显著减少时间敏感任务中的非推理错误（如检索到过时信息）。

### **7. 优点：方法或实验设计上的亮点**

*   **雅致的解耦设计**: 将记忆构建与检索解耦是核心亮点，这不仅在功能逻辑上清晰，也为优化提供了便利（解决了信用分配问题），使得分别采用“技能演化”和“强化学习”这两种最适配的策略来优化两个模块成为可能。
*   **“先组织后检索”的范式**: 引入文件系统式的层次化记忆结构，使得记忆系统本身具有了内在的拓扑结构和来源可追溯性，检索行为从“大海捞针”似的语义匹配，演变为有逻辑的**导航**，增强了可解释性和精准度。
*   **高效的优化策略**:
    *   **非参数技能演化**: 通过对比失败轨迹来更新记忆管理提示，避免了用稀疏奖励直接端到端训练记忆构建的巨大不稳定性，且技能可以跨任务复用。
    *   **证据基础的检索奖励**: 引入与下游推理无关的中间奖励信号，为检索器优化提供了直接且精细的反馈，使小模型也能高效学习。

### **8. 不足与局限**

*   **记忆管理器成本**: 虽然检索器可以轻量化，但核心的记忆管理器仍需依靠如Claude Sonnet 4.5这类强大的前沿模型来保证高质量的层次化组织和技能演化，这可能带来较高的成本。
*   **非在线学习**: 当前的检索器训练依赖于预先收集的对话数据和地面真凭据，而非在与环境的实时在线交互中学习，这可能限制了其探索和适应极端动态环境的能力。论文将“完全在线学习”列为未来工作。
*   **技能演化的依赖性**: 技能库的演化过程依赖于对比分析（比较HORMA与原始历史基线的表现），这意味着其改进效果的评估需要一个近乎无损的原始历史基线作为参照，这本身可能会消耗大量资源。
*   **应用范围限制**: 虽然展现了良好的泛化性，但工作记忆主要针对长程任务中的上下文管理。对于需要极强创造性或在极短时间内处理巨量一次性信息的任务，该框架的优势可能不那么明显。
