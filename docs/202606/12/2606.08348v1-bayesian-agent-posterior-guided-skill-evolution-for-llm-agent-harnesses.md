## 摘要
LLM智能体越来越依赖外部推理条件：提示、工具、记忆、SOP、技能以及套件反馈。这些资产无需改变模型权重即可改进任务执行，但通常通过启发式反思或通过重用观察到的成功和失败（仿佛次数本身就是可靠的信念）来进行修改。我们提出了Bayesian-Agent，这是一个原生且跨套件的框架，它将可重用的技能和SOP视为关于冻结模型在特定提示、上下文和套件环境下是否成功的假设。Bayesian-Agent记录经过验证的轨迹证据，维护每个技能的特征条件类别后验分布，并将后验状态映射为可检查的动作，如修补、分割、压缩、退役和探索。面向模型的提示会获得可执行的护栏和故障模式补丁，而后验摘要可供审计。使用deepseek-v4-flash，增量修复将SOP-Bench从80%提升至95%，Lifelong AgentBench从90%提升至100%，RealFin-Bench从45%提升至65%。我们进一步评估了Bayesian-Agent的原生后端以及可选的GenericAgent、mini-swe-agent和Claude Code后端。结果涵盖正面、负面、饱和及案例研究设置，表明智能体技能演化最好视为后验引导的套件优化，而非未经校准的提示积累。源代码可在https://github.com/DataArcTech/Bayesian-Agent获取。

## Abstract
LLM agents increasingly rely on external inference conditions: prompts, tools, memory, SOPs, skills, and harness feedback. These assets can improve task execution without changing model weights, but they are often revised by heuristic reflection or by reusing observed successes and failures as if counts alone were reliable belief. We introduce \textbf{Bayesian-Agent}, a native and cross-harness framework that treats reusable skills and SOPs as hypotheses about whether a frozen model will succeed under a particular prompt, context, and harness environment. Bayesian-Agent records verified trajectory evidence, maintains a feature-conditioned categorical posterior over each skill, and maps posterior state into inspectable actions such as patch, split, compress, retire, and explore. Model-facing prompts receive executable guardrails and failure-mode patches, while posterior summaries remain available for audit. With \texttt{deepseek-v4-flash}, incremental repair improves SOP-Bench from 80\% to 95\%, Lifelong AgentBench from 90\% to 100\%, and RealFin-Bench from 45\% to 65\%. We further evaluate Bayesian-Agent's native backend and optional GenericAgent, mini-swe-agent, and Claude Code backends. The results include positive, negative, saturated, and case-study settings, suggesting that agent skill evolution is best viewed as posterior-guided harness optimization rather than uncalibrated prompt accumulation. The source code is available at https://github.com/DataArcTech/Bayesian-Agent.

### 2. 方法论
- **核心理念**：将每一个可重用技能 `h_k` 与一个特征条件化的成功率后验分布绑定，即 `P(y=1 | M_θ, C_t, h_k, z_t)`，其中 `z_t` 是验证轨迹的特征向量，`M_θ` 是冻结模型，`C_t` 是推理环境（提示、上下文、工具、验证器等）。技能更新被建模为对推理环境 `C_t` 的干预。
- **证据模型**：采用因子化类别贝叶斯证据模型，使用拉普拉斯平滑计算类别先验和特征似然，再通过乘积计算未归一化后验，归一化得到成功概率后验 `s_k(z)`。同时维护 Beta‑Bernoulli 共轭形式（`α/β`）用于审计与保守决策。特征离散化为上下文、失败模式、token数/步数/时延桶等，适应稀疏在线数据。
- **决策策略**：根据后验状态，执行规则化的五项技能动作——**探索**（无观测）、**修补**（同一失败模式≥2次）、**拆分**（≥3种上下文且≥4条证据）、**压缩**（≥3条证据且成功概率≥0.72）、**退役**（失败证据占优且成功概率<0.45）。该策略保守透明，避免无必要的文本漂移。
- **模型面上下文**：仅将可执行护栏和故障模式补丁注入提示，不直接暴露后验数值，防止模型对概率产生幻觉推理。
- **框架边界**：提供原生最小化 harness，并通过适配器支持 GenericAgent、mini‑swe‑agent、Claude Code 等外部后端，仅需后端输出验证轨迹并接受技能文本注入。

### 6. 主要结论与发现
- **增量修复非常有效**：在 deepseek-v4-flash 上，BA-Inc 仅对失败任务重跑，将 SOP-Bench 准确率从 80% 提升到 95%，Lifelong AgentBench 从 90% 提升到 100%，RealFin-Bench 从 45% 提升到 65%，且额外 token 开销可控。
- **全量在线进化并非始终有益**：在 Lifelong AgentBench flash 设置下，BA-Full 准确率（85%）低于基线 GA（90%），说明证据稀疏时在线更新可能引入排序效应或噪声。
- **框架跨后端泛化**：不同执行后端（原生、GenericAgent、mini‑swe‑agent、Claude Code）均可通过适配器接入贝叶斯技能层，并获得不同程度的修复提升，验证了“依赖验证轨迹而非特定运行时”的设计。
- **后验引导优于简单计数**：通过显式建模不确定性和可审计的动作（修补/拆分/压缩/退役），技能维护更稳健，避免了直接基于频率的误判。
- **技能演化是可审计的**：案例研究清晰地展示了“验证失败→后验更新→生成可执行补丁→再次成功”的可追溯链路。

### 7. 优点
- **新颖的形式化**：将外部技能可靠性问题纳入贝叶斯推理框架，统一了提示工程、上下文工程和 harness 工程，为智能体自演化提供了概率决策基础。
- **透明且可审计**：后验状态和策略动作均为显式、可记录，技能补丁的修改过程全链路可追踪，利于调试与信任。
- **实用且低成本升级**：增量修复模式可在不改变模型权重、不动原有成功运行的前提下，有针对性地挽回失败任务，且仅消耗修复所需的推理 token。
- **架构解耦**：通过适配器设计，将贝叶斯技能层与特定执行后端分离，具备良好的可移植性。
- **实验严谨性**：覆盖多基准、多模型、多后端及正负/饱和场景，分析细致（如区分修复成本和累计成本）。

### 8. 不足与局限
- **贝叶斯模型简单**：使用因子化类别模型和平滑先验，并非完整的结构学习或模型选择，可能无法捕捉特征间的依赖关系。
- **应用前提较受限**：要求具备可验证的轨迹、可重复的任务、明确的成功/失败信号以及可注入技能文本的环境，不适用于一次性任务、主观评估、非平稳环境或缺少工具/数据导致的失败。
- **后端覆盖尚未全面**：虽测试了多个后端，但数量仍有限，更广泛的即插即用通用性有待验证。
- **全量模式性能不稳定**：当在线证据稀疏时，BA-Full 可能劣于简单基线，目前默认策略保守但非最优，更智能的决策策略仍有研究空间。
- **修复不保证单调提升**：技能修改可能引入新失败，虽然框架加强了审计，但未完全消除漂移风险。
