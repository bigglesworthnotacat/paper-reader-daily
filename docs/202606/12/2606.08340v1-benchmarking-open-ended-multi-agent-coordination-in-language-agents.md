## 摘要
随着语言模型越来越多地被部署为自主智能体，它们必须在开放式的交互任务中长时间协调合作。然而，现有的评估很少同时测试这些需求，而是侧重于单智能体任务、短期交互或高度结构化的多智能体场景。我们引入了 $alem$，这是一个基于 JAX 的开放式多智能体协调基准，构建于类似 Craftax 的动态系统之上。Alem 将程序生成的协调任务、软专业化、通信以及可控的协调难度嵌入到一个包含探索、制作、交易和战斗的长视界生存世界中。我们评估了 13 个现代大语言模型在同类团队中的零样本表现，并以训练有素的多智能体强化学习（MARL）智能体作为参考点。当前的大语言模型智能体仍远未解决 Alem，平均归一化回报仅约 6%，但它们的失败并不均匀。在最困难的协调设置下，零样本的 Gemini-3.1-Pro-High 接近了经过十亿步训练的 MARL 智能体，而 GPT-5.4-High 则获得了较高的基础任务奖励，但协调奖励却低得多。这一对比表明，个体任务能力并不代表协调能力。消融实验显示，通信是协调的最大贡献因素，而记忆和推理在用于维持多步骤计划时有所帮助。总体而言，我们的结果表明，协调是前沿大语言模型智能体面临的一个明显瓶颈，与单智能体能力相分离。Alem 使这一瓶颈可测量，并为开发能够通信、分配角色和执行共享计划的智能体提供了受控测试平台。代码可在 https://github.com/alem-world/alem-env 获取。

## Abstract
As language models are increasingly deployed as autonomous agents, they must coordinate with others over long horizons in open-ended interactive tasks. Yet existing evaluations rarely test these demands together, instead emphasising single-agent tasks, short interactions, or highly structured multi-agent settings. We introduce $alem$, a JAX-based benchmark for open-ended multi-agent coordination built on Craftax-like dynamics. Alem embeds procedurally generated coordination tasks, soft specialisation, communication, and controllable coordination difficulty into a long-horizon survival world with exploration, crafting, trading, and combat. We evaluate $13$ modern LLMs zero-shot within homogeneous teams, with trained MARL agents as reference points. Current LLM agents remain far from solving alem, averaging only ~6% normalised return, but their failures are not uniform. On the hardest coordination setting, zero-shot Gemini-3.1-Pro-High approaches MARL agents trained for one billion steps, while GPT-5.4-High achieves strong base-task reward but much lower coordination reward. This contrast shows that individual task competence does not imply coordination competence. Ablations show that communication is the largest contributor to coordination, while memory and reasoning help when used to maintain multi-step plans. Overall, our results identify coordination as a distinct bottleneck for frontier LLM agents, separate from single-agent capabilities. Alem makes this bottleneck measurable and provides a controlled testbed for developing agents that communicate, allocate roles, and execute shared plans. Code is available at https://github.com/alem-world/alem-env.

### 2. 论文提出的方法论

*   **核心思想**
    *   构建一个名为 `ALEM` 的、基于 JAX 框架的开放式多智能体协调基准测试，其核心思想是**将协调作为环境的一个显式、可控且程序化生成的部分**，而非隐含或固定的设定。这使得可以单独评估协调能力，并将其与基础环境任务能力区分开来。

*   **关键技术细节**
    *   **程序化的协调任务生成**：在每个 episode 开始时，环境会为挖掘、建造、战斗和制作等不同任务独立地、程序化地生成协调需求。这包括协调类型（同步 / 交接）和所需智能体数量（2 到 N 个）。这避免了智能体记忆固定的交互模式。
    *   **协调耦合频谱**：系统性地定义了智能体行动之间的依赖性，通过其**时间分离度**（`Δt`）来表征，形成了一个从长时间依赖、短期交接（如一个智能体启动任务，另一个在时间窗口内完成）到严格同步（在同一时间步执行联行动作）的协调频谱。
    *   **软专业化与难度控制**：
        *   **软专业化**：不同于强制执行角色的“硬性阀门”，`ALEM` 允许任何智能体执行任何角色限定的动作，但角色专家必然成功（成功率 `η_spec = 1`），非专家的成功概率为 `η_non = 1 - α`。这鼓励了有效的角色分配，同时保留了适应能力。
        *   **可控的协调难度**：使用单一标量 `α ∈ [0, 1]` 来缩放整体协调难度。`α` 越高，任务执行约束越严格。论文评估了简单（`α = 0.3`）、中等（`α = 0.6`）和困难（`α = 0.9`）三个难度等级。由于世界生成是确定性的，改变 `α` 可以保持空间布局和协调机会的频率不变，从而干净地隔离执行难度。
    *   **环境接口**：
        *   **强化学习接口**：提供像素和符号观测（包括本地地图、库存和队友仪表盘），以及一套涵盖环境操纵、制作、战斗和明确通信（广播离散消息）的 60 维离散动作空间。
        *   **基于文本的语言接口**：为评估大语言模型，暴露一个结构化的文本接口。模型在每个时间步接收包含近期观察 - 行动历史、队友信息、私人草稿板（Scratchpad）和可见协调约束的文本状态，并必须返回一个包含有效动作的结构化响应。

### 3. 实验设计

    *   实验在简单（Easy）、中等（Medium）、困难（Hard）三个协调难度等级上进行，以评估不同模型在不同协调需求下的表现。

*   **基准（Benchmark）**
    *   论文本身提出了 `ALEM` 作为一个新的基准，用于衡量多智能体协调能力。

*   **对比方法**
    *   **前沿大语言模型（零样本）**：评估了 13 个现代大语言模型，包括闭源模型（如 GPT-5.4, Gemini 3.1 Pro）和开源模型（如 Gemma-4, Llama-3.1/3.3, Qwen-3.5/3.6 系列），所有模型均以零样本方式在**同类团队**（Homogeneous Teams）中运行。
    *   **训练有素的多智能体强化学习基线（MARL）**：作为参考点，训练了 IPPO、MAPPO、PQN-VDN 和 HyperMARL-IPPO 四种 MARL 算法，分别在 1 亿步和 10 亿步环境交互步数下进行评估。
    *   **消融实验**：针对表现最好的闭源和开源模型（Gemini-3.1-Pro-High 和 Gemma-4-31B-it），在困难难度下分别移除了**通信**、**草稿板记忆**和**推理**组件。
    *   **异质团队实验**：初步测试了由不同模型（如同一 Gemma 系列的不同规模模型，或 Gemini、GPT、Gemma 的最强模型）组成的**异质团队**在困难难度下的表现。

### 6. 论文的主要结论与发现

*   **协调是一个明显的瓶颈**：当前的大语言模型智能体远未解决 `ALEM`，平均总归一化回报仅约 6%，表明长期、开放式协调是一个巨大挑战。
*   **个体能力不等于协调能力**：性能分解分析揭示了隐蔽的失败模式。例如，GPT-5.4-High 在基础任务奖励（Base%）上表现出色，但其协调奖励（Coord.%）远低于更小规模的模型，如 Gemma-4-26B-A4B-it。反之，一些模型（如 Qwen3.5-122B-A10B）表现出协调奖励高于基础奖励。
*   **通信是关键因素**：消融实验表明，移除通信能力会导致协调性能出现最显著的下降，尤其是在 Gemini-3.1-Pro-High 上（协调奖励从 17.5% 降至 5.3%），证明通信主要支撑协调行为而非一般性环境进展。
*   **记忆与推理的作用是辅助性的**：
    *   **记忆（Scratchpad）**的效果取决于模型如何使用它。当被用于维持前瞻性计划时（如 Gemini 的用法），其作用显著；当主要作为当前状态的转述时（如 Gemma 的用法），则帮助不大。
    *   **推理**是唯一同时影响基础和协调性能的消融项，它的缺失会导致两种奖励均下降，表明其对于整体行为规划的支持作用。
*   **异质团队表现不佳**：初步实验表明，由不同模型组成的团队表现接近其成员同类团队的平均水平，而非继承最强个体的性能。这表明简单地引入一个更强的模型并不能自动提升团队协调水平。
*   **MARL 参考点**：在经过 10 亿步训练的 MARL 智能体中，最强基线在困难设置下也仅达到约 18% 的最大总回报，表明 `ALEM` 对当前的学习方法而言仍是一个未被充分解决的挑战。

### 7. 优点

*   **基准设计的创新性**：将协调从环境动态中解耦，并作为一个程序化生成、可控难度的独立维度，这是对现有基准的重大改进。
*   **全面的诊断能力**：通过分解奖励（基础 vs. 协调）、任务层面的协调类型分析以及系统性的消融实验，能够精准定位大语言模型智能体的具体失败模式（如通信、规划），而不仅仅是给出一个总分。
*   **评估的严谨性与可复现性**：基于 JAX 的实现保证了高性能和可复现性。统一的评估协议、详细的统计分析和多种基线（MARL、单智能体）确保了结论的可靠性。
*   **实践与研究的价值**：明确指出了“协调”是独立于“单智能体能力”的瓶颈，为未来多智能体大语言模型的研究指明了方向，并提供了直接可用的测试平台。

### 8. 不足与局限

*   **评估的智能体类型有限**：评估主要局限于基于文本的大语言模型，未包括视觉 - 语言模型（VLMs）或 Claude 系列模型，可能限制了结论的普适性。
*   **种子数限制**：由于 API 成本，对闭源的最强模型（GPT-5.4, Gemini-3.1-Pro）每个难度只运行了 10 个种子，而开源模型为 20 个种子，可能导致闭源模型性能估计的方差稍大。
*   **智能体工具的简单性**：零样本评估使用了相对基础的“推理-后-行动”（reason-then-act）范式、有限的上下文历史（最近 8 步）和草稿板记忆。更复杂的智能体工具（如代码执行、长期向量化记忆、跨回合学习）可能显著提升性能，未被探索。
*   **异质团队研究的初步性**：对异质团队的评估仅停留在初步阶段，仅测试了两种组合，未能深入探究不同模型间通信与规划协调性差的深层原因和解决方案。
