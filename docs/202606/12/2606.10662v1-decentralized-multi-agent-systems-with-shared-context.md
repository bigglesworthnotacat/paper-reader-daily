## 摘要
多智能体系统（MAS）可以通过将复杂问题分解为并行子任务，在测试时扩展大型语言模型的推理能力。然而，现有的大多数MAS依赖于集中式编排，即由一个主智能体分配工作、收集输出并合并结果。随着子任务数量的增加，这个控制器成为通信和集成的瓶颈。我们提出了去中心化语言模型（DeLM），这是一个MAS框架，通过并行智能体、共享的已验证上下文和任务队列来实现去中心化协调。智能体异步地认领子任务、读取累积进展、执行局部推理，并写回紧凑的已验证更新。共享上下文作为一个公共通信基底，使智能体能够在彼此已验证进展的基础上继续工作，而无需通过中央控制器路由每一次更新。实验表明，DeLM在软件工程测试时扩展和长上下文推理方面均有提升。在SWE-bench Verified上，DeLM在Avg.@1、Pass@2和Pass@4指标上均取得了最佳性能，与最强基线相比最高提升10.5个百分点，同时每个任务的成本降低约50%。在LongBench-v2多文档问答上，DeLM在四个前沿模型家族中取得了最高平均准确率，与最强基线相比最高提升5.7个百分点。代码可在我们的项目网站https://yuzhenmao.github.io/DeLM/获取。

## Abstract
Multi-agent systems (MAS) can scale large language model reasoning at test time by decomposing complex problems into parallel subtasks. However, most existing MAS rely on centralized orchestration, where a main agent assigns work, collects outputs, and merges results. As the number of subtasks grows, this controller becomes a communication and integration bottleneck. We propose Decentralized Language Models (DeLM), a MAS framework that decentralizes coordination through parallel agents, a shared verified context, and a task queue. Agents asynchronously claim subtasks, read accumulated progress, perform local reasoning, and write back compact verified updates. The shared context acts as a common communication substrate, enabling agents to build on one another's verified progress without routing every update through a central controller. Empirically, DeLM improves both software-engineering test-time scaling and long-context reasoning. On SWE-bench Verified, DeLM achieves the best performance across Avg.@1, Pass@2, and Pass@4, with gains of up to 10.5 percentage points over the strongest baseline, while reducing cost per task by roughly 50%. On LongBench-v2 Multi-Doc QA, DeLM achieves the highest average accuracy across four frontier model families, improving over the strongest baseline by up to 5.7 percentage points. The code is available on our project website at https://yuzhenmao.github.io/DeLM/.

    *   **扩展性差**：所有有益的中间发现、失败或部分解决方案都必须经主控点合并与再次广播，过程序列化且低效。
    *   **信息损耗**：在主控点进行合并、摘要的过程中，可能出现“稀释、省略或扭曲”关键细节，导致重要进展丢失。
*   **整体含义**：论文主张从“提示路由”转向“状态共享”，提出一种**去中心化协调范式**，旨在打破中央瓶颈，使多智能体系统能够更高效、更保真地利用并行计算资源，在测试时扩展LLM的复杂推理能力。

### 2. 论文提出的方法论
论文提出了一个名为 **DeLM（Decentralized Language Models）** 的MAS框架，其核心理念是**以“共享、已验证的问题状态”取代“中央控制器”作为协调媒介**。

*   **核心思想与架构**：
    *   DeLM框架由三大核心组件构成：**并行智能体、共享上下文（\( C \)）和任务队列（\( T \)）**。
    *   智能体不再通过主控点交互，而是以**异步**方式从 \( T \) 中认领子任务，读取 \( C \) 中的累积进展，进行局部推理，并将本地的紧凑更新写回 \( C \)。这形成了一个去中心化的、基于状态的通信基底。

*   **关键技术细节**：
    1.  **共享上下文（\( C \)）**：存储紧凑、可复用的“要点”（Gists），而非原始输出。每个要点关联详细证据，支持按需“展开”（Unfolding），使智能体能够**从粗到细**地访问全局进展，既避免了上下文窗口过载，又保留了深入挖掘细节的能力。
    2.  **准入时验证**：为了防止错误信息在共享状态中扩散，任何写入 \( C \) 的更新都必须先经过**验证**。即根据其底层证据（原始上下文或推理轨迹），由LLM校验器检查要点是否忠实。不支持的声明将被拒绝或要求重写，确保 \( C \) 成为经过整理的可信状态。
    3.  **分级压缩**：对于长文本源，采用**分层压缩**路径：原始文本 \( r_i \) → 包含具体引用的摘要 \( S_i \) → 高度紧凑的要点 \( G_i \)。\( G_i \) 存储在 \( C \) 中，\( S_i \) 和原始文本作为后备存储，仅在智能体通过“深层展开”请求时提供。这保证了导航的轻量性和证据检索的精确性。
    4.  **算法流程**：DeLM的流程是一次初始化、多次循环迭代。
        *   **初始化**：根据输入生成初始子任务并放入 \( T \)。
        *   **循环**：并行执行 \( T \) 中的任务 → 压缩并验证结果 → 将验证过的要点（\( G_i \)）并入 \( C \) → 若 \( T \) 为空且任务未完成，则基于当前 \( C \) 生成更多子任务加入 \( T \)。
        *   **结束**：\( T \) 为空且无新任务生成时，基于 \( C \) 生成最终答案。
