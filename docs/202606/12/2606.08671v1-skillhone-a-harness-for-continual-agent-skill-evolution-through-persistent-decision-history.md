## 摘要
智能体技能通过任务特定的流程、脚本和参考扩展了语言模型智能体，但它们所针对的任务和环境不断变化。现有方法在有限运行中改进技能，仅保留最终成果，丢弃了后续智能体解释先前修订、评估和拒绝的替代方案所需的决策历史。我们提出了SkillHone，一个基于持久决策历史的智能体技能持续进化平台。SkillHone将技能修订与提供实践反馈的评估侧证据相结合，记录结构化的诊断、修订、证据和结果历史。角色分离的子智能体在带有精简报告的实践探针上运行候选技能，并根据先前的决策提出修订建议，从而在无需重新发现过去推理的情况下实现跨会话的细化。我们在原始的开放式网络环境中在深度研究基准上评估SkillHone，其中智能体不提供集成搜索堆栈，必须通过可移植技能组织检索。我们将其与由商业检索服务支持的深度研究智能体进行比较。以Qwen3.6-35B-A3B作为评估时骨干，所得到的技能在GAIA上比深度研究智能体高15.8分，在WebWalkerQA-EN上高3.2分，同时也优于先前的技能进化方法。

## Abstract
Agent skills extend language-model agents with task-specific procedures, scripts, and references, but the tasks and environments they target continually change. Existing methods improve skills in bounded runs and retain only the final artifact, discarding the decision history that later agents need to interpret prior revisions, evaluations, and rejected alternatives. We introduce SkillHone, a harness for continual agent skill evolution grounded in persistent decision history. SkillHone pairs skill revisions with evaluation-side evidence that supplies practice feedback, recording structured histories of diagnoses, revisions, evidence, and outcomes. Role-separated subagents run candidate skills on practice probes with redacted reporting and propose revisions informed by prior decisions, enabling cross-session refinement without rediscovering past rationale. We evaluate SkillHone on deep-research benchmarks in a raw open-web setting, where agents are not given an integrated search stack and must organize retrieval through portable skills. We compare against a deep-research agent backed by commercial retrieval services. With Qwen3.6-35B-A3B as the evaluation-time backbone, the resulting skills outperform the deep-research agent by 15.8 points on GAIA and 3.2 points on WebWalkerQA-EN, while also exceeding prior skill-evolution methods.

### 1. 论文的核心问题与整体含义

- **核心问题**：现有的智能体技能进化方法将技能改进视为一个有限回合的优化过程，其输出仅为最终的技能产物。这一过程丢弃了关键的决策历史，包括诊断动机、被拒绝的替代方案、评估证据以及结果接受的理由。当环境变化或需要跨会话持续维护时，后来的智能体（Agent）无法理解先前的改动逻辑，导致重复错误、撤销有效更改或重新探索旧思路，严重阻碍了技能的长期演进。
- **整体含义**：论文旨在解决“智能体技能如何在不断变化的任务和环境中实现持续、可追溯的进化”这一根本问题。它强调技能的优化不应是孤立的产物生成，而应是一个包含决策上下文记忆的持续维护过程，使智能体能够继承并利用过往的完整修改记录。

### 2. 论文提出的方法论

- **核心思想**：SkillHone是一个面向智能体的技能进化“平台（Harness）”，其核心是将技能构建为一个随时间演化的“版本包”，并为其配备一个持续记录“决策历史”的机制。通过让后来的智能体同时继承当前技能和历史记录，实现无需重新推导过往决策的持续优化。
- **关键技术与架构**：
    - **角色分离的子智能体调度**：系统动态创建并分配两类权限严格区分的子智能体团队，而非固定的多智能体系统：
        1.  **优化团队**：负责读取精简后的反馈和决策历史，形成诊断并提议技能修订。它们无权访问未精简的评估素材，防止过拟合。
        2.  **评估团队**：负责在实际评估存储库上运行候选技能，分析失败原因，但只能输出经过精简的问题报告。它们无权修改技能存储库。
    - **持久决策历史**：将每一次技能进化的步骤形式化为结构化的决策记录 `ht = (诊断, 候选修订, 精简评估证据, 结果)`。这个历史记录不仅仅是代码差异，而是链接了问题、证据和最终决策的上下文。
    - **算法流程**：
        - **输入**：初始技能 $S_0$，技能存储库 $R_s$，评估存储库 $R_e$。
        - **循环进化**：在第 $t$ 次迭代，系统执行以下步骤：
          1.  **评估**：评估子代理在 $R_e$ 上运行当前技能 $S_t$，生成精简报告 $\tilde{E}_t$。
          2.  **诊断与修订**：优化子代理检索决策历史 $H_t$，结合 $\tilde{E}_t$ 提出诊断 $q_t$ 和修订提议 $r_t$。
          3.  **证据返回**：评估子代理针对 $r_t$ 返回精简评估证据 $e_t$。
          4.  **决策**：系统根据证据决定接受、修订或拒绝提议。
          5.  **更新**：若接受，技能更新为 $S_{t+1}$；否则保持不变。将本次决策步骤 $(q_t, r_t, e_t, o_t)$ 追加到历史 $H_{t+1}$ 中。
        - **输出**：最终技能 $S_T$ 和完整决策历史 $H_T$。
    - **信息屏障**：严格区分优化侧和评估侧的信息可见性，优化侧只能看到精简报告，防止直接记忆评估探针的答案
