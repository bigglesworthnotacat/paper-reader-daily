## 摘要
工具使用型大型语言模型（LLM）智能体常因在行动时刻未能筛选、压缩或凸显决定性证据而失败，尽管相关文本并未缺失。本文提出CICL，一种决策感知上下文层，将实例证据转化为上下文图，通过共享的八字段模式路由确定性、Opus辅助、Qwen、Codex/GPT-5.5和Qwen-QLoRA的判断，根据行动偏移、结果提升、必要性和负迁移风险对单元评分，并将高效用证据作为类型化记忆卡打包给有预算限制的智能体。该设计将测量得到的决策信号与评判模型解耦，使得前沿标注、局部代理和轻量级排序器可在同一可审计协议下进行比较。实验表明，CICL在暴露其局限性的同时，带来了具体公开基准的增益。在50个SWE-bench Verified文件检索实例上，对BM25前50候选直接进行Qwen3.6-plus重排序，将hit@1从0.58提升至0.78，MRR@10从0.634提升至0.790，所有2500个判断均可解析。受控诊断显示行动关键性：在预算120下，CICL在v1上达到F1 0.620，在v3上达到0.425，而移除最高效用的语义v3单元则使F1骤降至0.000。补充检查增加了Qwen-QLoRA在710个候选上的一致性、一个包含200个标签的真实代码Opus辅助信号的小规模实验，以及一个三实例的补丁冒烟验证，确认了检索到补丁的流程，但未声称官方SWE-bench成功。RepoBench-R摘要仍优于记忆卡，紧凑排序器尚未取代启发式方法。CICL为决策关键型上下文贡献了一种可复现的测量与选择层，而非端到端编码智能体修复声明。

## Abstract
Tool-using LLM agents often fail not because relevant text is absent, but because decisive evidence is not selected, compressed, or surfaced at action time. We present CICL, a decision-aware context layer that turns instance evidence into a context graph, routes deterministic, Opus-assisted, Qwen, Codex/GPT-5.5, and Qwen-QLoRA judgments through a shared eight-field schema, scores units by action shift, outcome uplift, necessity, and negative-transfer risk, and packs high-utility evidence as typed memory cards for a budgeted agent. The design separates the measured decision signal from the judge model, so frontier annotation, local surrogates, and lightweight rankers can be compared under one auditable protocol. Empirically, CICL yields a concrete open-benchmark gain while exposing its limits. On 50 SWE-bench Verified file-retrieval instances, direct Qwen3.6-plus reranking of BM25 top-50 candidates raises hit@1 from 0.58 to 0.78 and MRR@10 from 0.634 to 0.790, with all 2,500 judgments parseable. Controlled diagnostics show action-criticality: at budget 120, CICL reaches F1 0.620 on v1 and 0.425 on v3, and removing the top-utility semantic v3 unit collapses F1 to 0.000. Supplementary checks add Qwen-QLoRA agreement over 710 candidates, a small 200-label real-code Opus-assisted signal, and a three-instance patch smoke validating retrieval-to-patch plumbing without claiming official SWE-bench success. RepoBench-R summaries still beat cards, and compact rankers do not yet replace the heuristic. CICL contributes a reproducible measurement and selection layer for decision-critical context, not an end-to-end coding-agent repair claim.

- **研究动机**：现有方法（如BM25、密集检索）将上下文选择视为相关性排序问题，忽视了所选上下文对智能体下一步决策的实际影响。更长提示并不保证关键信息被保留或及时呈现。
- **整体含义**：作者提出将上下文选择重新定义为**决策感知问题**——有用的上下文是那些能够改变智能体下一步行动、提升预期结果、成为成功必要条件或引入负迁移风险的证据。CICL旨在为这种决策关键上下文提供一个可测量、可审计的选择层，而非完整编码智能体修复方案。

## 2. 方法论：核心思想与关键技术细节

CICL（Counterfactual-Inspired Context Layer）的核心理念是**将每个候选上下文单元视为决策时刻的干预变量**，通过估计其反事实效用进行筛选和压缩。

### 2.1 整体流程（如图1）

1. **构建上下文图**：对每个仓库或环境实例构建图，节点为文件、符号、任务记忆、规则、失败记录等，边捕获包含、相似、冲突等关系，并支持单跳邻居扩展。
2. **决策效用引擎**：通过多种评判模型（Opus、Qwen API、Codex/GPT-5.5、训练的Qwen-QLoRA）按**八字段模式**（见表1）生成结构化判断，计算每个候选单元的**四分量效用**：
   - `action_shift`：加入单元后，智能体最大概率行动是否改变；
   - `expected_outcome_uplift`：预期成功得分的提升；
   - `necessity`：单元是否为成功所必需；
   - `negative_transfer_risk`：单元引入负迁移的风险。
3. **效用聚合**：采用固定线性聚合（公式3）：
   \[
   s = 0.34 \Delta_\text{act} + 0.26 N + 0.28 \Delta_\text{out} - 0.22 R - 0.08 \text{cost}
   \]
   其中 cost 与 token 数量和风险相关。
4. **记忆卡打包**：选出的高效用证据被重写为**决策感知记忆卡**，包含五个必填字段：`trigger`、`evidence`、`action hint`、`failure-if-ignored`、`scope`，并额外附带诊断性因果分数用于排序。
5. **预算组装**：在推理时，根据 token 预算，优先打包高效用单元。

### 2.2 评判模型与评分多样性

CICL刻意将**评判模式**与**决策信号**解耦，允许使用不同模型产生评分，并保持可比较性：
- **Claude-Opus 4.7**：用作反事实标注辅助，生成结构化八字段判断；
- **轻量级排序器**：在 Opus 标注数据上训练 25 维线性排序器或 MLP 排序器；
- **Qwen3.5-9B QLoRA**：作为开放权重的本地替代评判器，通过监督微调习得八字段输出；
- **直接Qwen API / Codex/GPT-5.5**：用于实时重排序。

### 2.3 关键公式与算法细节

效用估计的反事实分解如公式1所述，但在实现中均通过确定性模拟器探针、LLM裁判或轻量级排序器预测来近似。压缩分为**后选择压缩**和**预算前压缩**两种模式进行报告，避免混淆压缩收益与选择变化。

## 6. 主要结论与发现

1. **决策感知效用是有效的排序信号**：在SWE-bench Verified上，Qwen3.6-plus决策感知重排序显著提升BM25的表现（Hit@1：0.58→0.78，MRR@10：0.634→0.790）。
2. **效用成分具有行动关键性**：移除得分最高的语义单元导致合成v3的F1从0.245骤降至0.000，证实高决策效用单元对成功至关重要。
3. **压缩可节省 tokens**：记忆卡压缩相比原始选择平均节省44.93 tokens，且不影响所选上下文ID。
4. **方法存在明显边界**：
   - RepoBench-R中，通用摘要方法的压缩成功率仍高于决策感知记忆卡。
   - 紧凑型排序器（线性/MLP）未能替代启发式CICL。
   - 在语义干扰物（v3）下，CICL并未持续领先VanillaRAG。
5. **评判模型可移植性**：Qwen和GPT-5.5均能解析八字段判断协议，为决策信号测量提供了审计可能。

## 7. 优点：方法或实验设计亮点

- **概念创新**：将上下文选择重新定义为决策干预问题，引入反事实启发式效用评估，超越传统的相似度排序。
- **架构解耦**：明确区分评判模型与决策信号，支持多种评判模型（前沿API、本地微调模型、轻量排序器）在同一协议下比较和替换，增强了实用性和可审计性。
- **记忆卡结构化设计**：强制带触发条件、动作提示、风险描述等字段，使压缩结果更具行动可操作性，而非仅做文本摘要。
- **详尽的诊断与消融**：通过受控合成实验、效用分量移除、预算扫描、蒸馏排序器、错误分析（有害上下文率）等，透彻揭示了方法的有效性与局限性。
- **公开透明**：负面结果如实报告，明确指出方法在哪些场景落后于基线，并开源代码、证据日志及QLoRA适配器，促进可复现性。

## 8. 不足与局限

- **因果声称受限**：文中承认所提“反事实”并非真正的因果识别，无随机干预，效用估计依赖评判模型输出，可能引入偏差。
- **评估范围未达端到端**：主要评估限于文件级检索和压缩质量，未报告官方SWE-bench补丁成功率；真实代码补丁实验仅少量冒烟测试。
- **蒸馏排序器表现不足**：轻量级排序器（线性/MLP）未能匹配启发式CICL的性能，限制了在无API场景下的实用性。
- **RepoBench-R负结果**：记忆卡在保留代码词法细节上不如通用摘要，表明此压缩格式不一定适合代码补全类任务。
- **语义干扰物下优势减弱**：在v3（语义干扰）场景中，CICL未全面超越VanillaRAG，揭示其可能对强语义混淆更为敏感。
- **实验规模较小**：真实代码上的Opus辅助标注仅200例，Qwen一致性验证局限于合成分布，推广到复杂现实场景需更大规模验证。
- **权重启发式固定**：效用聚合的系数为预置经验值，未进行自动学习或调优，可能未达最佳组合。
- **领域局限**：方法在代码代理场景验证，其推广到其他工具使用型智能体（如网络导航、数据库查询）尚需检验。
