## 摘要
LLM智能体的长期记忆不仅仅是适时检索正确的段落。当前的记忆系统将信念修正、因果耦合与跨领域抽象缩并至单一为表面回忆而调优的检索平面上，因而在需要对用户如何演变进行推理的隐式个性化上表现挣扎。我们提出DCPM，它将智能体记忆沿认知能力层级重新组织，从原始输入与原子事实向上，穿过历时信念轨迹与身份，直至领域图式、潜在意图与跨领域模式。该层级由继承双过程理论架构划分的两个过程驱动：一个同步的昼间写入器（系统1），以双向链接的取代链记录信念修正；一个异步的夜间引擎（系统2），归纳图式与意图并扫描跨领域碰撞，将其抽象为更高层的核心图式。在LongMemEval、PersonaMem和PersonaMem-v2上，启用系统2在基准测试奖励隐式跨会话推理处贡献最大（在PersonaMem-v2上提升高达+5.20），而在片段回忆处贡献最小，这与架构预测相符。

## Abstract
Long-term memory for an LLM agent is more than retrieving the right passage at the right time. Current memory systems collapse belief revision, causal coupling, and cross-domain abstraction into a single retrieval surface tuned for surface recall, and consequently struggle on implicit personalisation that requires reasoning over how a user has evolved. We propose DCPM, which reorganises agent memory along a cognitive capability hierarchy ascending from raw inputs and atomic facts, through diachronic belief trajectories and identity, to domain schemas, latent intentions and cross-domain patterns. The hierarchy is driven by two processes inheriting the architectural split of dual-process theory: a synchronous daytime writer (System1) that records belief revisions as doubly linked supersedes chains, and an asynchronous nighttime engine (System2) that induces schemas and intentions and sweeps for cross-domain collisions abstracted into higher-level core schemas. On LongMemEval, PersonaMem and PersonaMem-v2, enabling System2 contributes most where the benchmark rewards implicit cross-session inference (up to +5.20 on PersonaMem-v2) and least on span recall, matching the architectural prediction.

### 2. 论文提出的方法论：DCPM

论文提出了一个名为 **DCPM (Dual-Process Cognitive Memory)** 的认知记忆系统，其核心思想是沿认知能力层级组织记忆，并由两个继承自“双过程理论”的异步进程驱动。

*   **认知能力层级**
    DCPM将记忆划分为从低到高的五个层级，形成一个从被动存储到主动认知演化的弧线：
    1.  **被动基础层**：原始输入和原子事实的向量存储。
    2.  **结构化记忆层**：稳定的用户身份条目，以及在事实存储上建立的、记录信念修正的**历时性取代链**。
    3.  **主动认知演化层**：存在于独立图 `G` 中的三种节点。
        *   **领域内行为图式**：总结用户在单一生活领域内的行为规律。
        *   **潜在意图**：编码用户未来可能关心的潜在事件，用于主动关怀。
        *   **跨领域核心图式**：通过比对不同领域中“行为相似但语义遥远”的图式，抽象出的更高层认知模式，是系统的最高级检索目标。

*   **关键技术细节：双过程驱动**
    *   **同步昼间写入器**：负责在线、低延迟地将新对话写入记忆。它在收到显式的写入请求时触发，流程如下：
        1.  **原始持久化**：确保原始数据零丢失。
        2.  **提取**：由一个提取LLM将对话总结为原子事实和用户身份条目。
        3.  **调和**：由一个调和LLM将新条目与向量库中最近邻条目进行比较，执行`ADD`（新增）、`SUPERSEDE`（取代，表示信念修正）或`UPDATE`（更新）操作。重复项被静默丢弃。
        4.  **批量写入与建立取代链**：当执行`SUPERSEDE`操作时，系统不删除旧条目，而是建立双向指针 (`n_{new}.SUPERSEDES = n_{old}`, `n_{old}.SUPERSEDED\_BY = n_{new}`)，形成一条可遍历的**历时性轨迹链**。这是区别于“原地更新”系统的关键设计。

    *   **异步夜间引擎**：在系统空闲或夜间以较宽松的延迟预算运行，负责对写入器积累的记忆进行深度抽象，不参与在线响应。
        1.  **阶段1：预处理**：使用两阶段DBSCAN算法，对尚未被纳入图式/意图节点的“新鲜”事实进行聚类。
        2.  **阶段2：LLM智能体归纳**：为每个聚类调用一次归纳智能体，它配备了`CREATE_SCHEMA`、`CREATE_INTENTION`等工具，在记忆图`G`中创建或更新领域图式和意图节点，并建立指向支撑事实的证据边。
        3.  **阶段3：跨领域扫描器**：这是实现跨领域抽象的核心。
            *   （a）**行为抽象**：LLM为每个图式生成一段简短的行为描述，该描述被编码为行为嵌入。图式原文则生成语义嵌入。
            *   （b）**碰撞扫描**：扫描来自不同领域的图式对，当它们的**行为嵌入相似度**高于阈值，而**语义嵌入相似度**低于阈值时，将其标记为“碰撞”。
            *   （c）**核心归纳**：对每个碰撞聚类，调用LLM归纳出一个更高层的**跨领域核心图式**，并写入图`G`。

*   **读取路径**：查询时，系统通过向量相似度命中事实/身份条目，并沿其`SUPERSEDES`链遍历。同时，系统还会提取图中以该事实为证据的图式、意图和核心图式节点。整个读取路径**无需调用LLM**，延迟由向量搜索主导。
