## 摘要
现代大语言模型代理可以使用外部工具帮助用户解决复杂任务。然而，对于需要从长期历史事件或先前代理与环境的交互中学习的问题，大语言模型代理需要使用记忆机制来存储和检索经验。尽管对话代理已有复杂的记忆系统，但很少有研究实证探讨如何通过过去的用户-代理对话来提升代理的工具使用能力。我们提出 MemToolAgent，这是一个通过记忆管理改进工具使用的框架。我们的方法包含一个记忆提取模块，将过去的经验处理为结构化的记忆条目，以及一个检索模块，动态选择存储的记忆条目的子集。这使得响应更加个性化和准确，符合用户偏好和反馈，而无需对大语言模型进行微调。总之，这项工作有三个主要贡献：（1）统一的记忆条目格式，无需大语言模型微调即可改进通用和个性化工具使用；（2）基于反思的记忆提取，利用环境和用户反馈将错误执行提炼为批判性内容进行存储；（3）一个检索模块，根据记忆相似度分布选择使用多少过去经验。在 WorkBench、NESTFUL 和 PEToolBench 基准测试上，MemToolAgent 相比强基线分别实现了 29%、80% 和 17% 的相对改进。

## Abstract
Modern large language model (LLM) agents can use external tools to help users solve complex tasks. However, for problems that require learning from long-term historical events or from previous agent-environment interactions, LLM agents are required to use memory mechanisms to store and retrieve experiences. While sophisticated memory systems exist for dialogue agents, few studies have empirically examined how to improve agents' tool-using capabilities through past user-agent conversations. We propose MemToolAgent, a framework that improves tool use through memory management. Our approach contains a memory extraction module that processes past experiences into structured memory entries, and a retrieval module that dynamically selects a subset of the stored memory entries. This enables more personalized and accurate responses aligned with user preferences and feedback without requiring LLM fine-tuning. In summary, this work has three main contributions: (1) a unified memory entry format that improves both general-purpose and personalized tool use without LLM fine-tuning, (2) a reflection-based memory extraction that uses environment and user feedback to distill wrong executions into critiques to store, and (3) a retrieval module that chooses how many past experiences to use based on the memory similarity distribution. MemToolAgent achieves 29%, 80%, and 17% relative improvements compared to strong baselines on the WorkBench, NESTFUL, and PEToolBench benchmarks, respectively.

- **核心问题**：如何通过外部记忆管理，让工具型代理从环境反馈和用户反馈中学习，从而提升通用工具调用的准确性以及个性化工具选择的一致性，同时避免对大语言模型进行微调。

### 2. 方法论

#### 2.1 核心思想
- 将每次交互的经验存储为统一格式的记忆条目，并通过“反思性记忆提取”将错误执行抽象为批判，避免原始轨迹的冗余和不相关。
- 在推理时，根据用户查询与记忆条目的语义相似度分布动态决定检索条目数量（Dynamic “n” Top‑n Retrieval），再以检索到的记忆作为上下文引导代理完成任务。

#### 2.2 记忆条目格式
每个记忆条目 \( m \) 表示为：
- \( q \)：用户查询
- \( a = \{a_1, a_2, \dots, a_n\} \)：代理执行的工具调用序列
- \( f \in \{0,1\} \)：二元用户反馈（1 成功，0 失败）
- \( r \)：反思/批判，仅当 \( f = 0 \) 时由记忆提取模块生成

#### 2.3 动态 “n” Top‑n 检索
- **步骤**：
  1. 计算查询与所有记忆条目的余弦相似度。
  2. 降序排列相似度。
  3. 通过滑动窗口计算相似度的一阶导数近似（半径 \( R \) 控制平滑程度）。
  4. 使用 `scipy.signal.find_peaks` 在导数序列中寻找峰值（即相似度突然下降的位置），将峰值索引作为动态选择的条目数 \( n \)。
- **参数**：半径 \( R \)、峰值显著性 `prominence`、选取第几个峰值（如第一峰或第二峰）。
- 相比固定 k 值，动态选择可避免无关条目引入噪声，也能在信息匮乏时减少冗余，比单纯选最大相似度落差（如 Adaptive‑k）更鲁棒。

#### 2.4 记忆提取模块
- 针对失败任务，调用额外 LLM（记为 \( E \)）输入当前查询、可用工具、代理响应、环境反馈、用户反馈以及检索到的相似记忆，生成简练的批判 \( r \)。
- 批判帮助代理直接理解错误原因，而非从冗长原始轨迹中归纳。
- 对于个性化场景，记忆提取模块负责抽象出高层用户偏好（如“用户偏爱使用 API_X 下的工具”），以软指导方式给代理。

### 3. 实验设计

#### 3.1 基准与数据集
- **WorkBench**：面向工作场景（邮件、日历、CRM、分析、多域）的工具使用基准，包含 5 个数据库、27 个工具。评价指标为输出结果与预期结果的一致性准确率。测试集规模：分析 108、日历 99、CRM 72、邮件 81、多域 189 条任务。记忆填充方式：每个模板各取 1 条查询，保证测试集至少有一条直接相关的记忆。
- **NESTFUL**：评估模型生成嵌套工具调用序列的能力，基于 MathQA 和 StarCoder2-Instruct，共 1891 条测试查询。本研究取前 500 条作为测试，后续 500 条填充记忆。指标包括工具 F1、参数 F1、部分序列匹配、全序列匹配以及 Win Rate（执行后答案正确与否）。
- **PEToolBench**：个性化工具编排基准，每条测试查询需结合 80‑100 条历史指令（含正确/错误评级）推断用户偏好。过滤后使用 591 条测试。指标为工具准确率和参数准确率。

- 模型：PEToolBench 用 Claude 3.5 Sonnet，NESTFUL 和 WorkBench 用 Claude 3.7 Sonnet；嵌入模型为 Qwen3-Embedding-8B。

### 6. 主要结论与发现

- MemToolAgent 显著提升了工具使用性能：在 WorkBench 上总体准确率从 57.56% 提升至 85.06%（相对 29% 改善），NESTFUL 全序列匹配从 15.6% 升至 30.4%（相对 80%），Win Rate 从 51.2% 升至 70.2%，PEToolBench 工具准确率相对 A‑MEM/Mem0 提高 17%。
- 动态 top‑n 检索优于固定 k 值，能自适应发现最优检索数量。
- 独立于执行步骤的反思性记忆提取是关键，比直接存储原始执行轨迹更有效。基于记忆提取的批判有助于代理理解错误本质，避免重复犯错。
- 使用改写或合成查询填充记忆能缓解冷启动问题，在部分域甚至超越使用原始查询。
- 硬偏好引导（Hard Preference Guiding）作为记忆提取的低成本替代方案，在个性化场景中可进一步提高工具准确率，但可能伤害参数准确率。
- Token 使用分析表明，MemToolAgent 主要增加输入 token，输出 token 增长较缓，有利于控制成本与响应延迟。

### 7. 优点

- **统一框架**：同时处理通用工具使用和个性化工具编排，适用面广。
- **反思驱动记忆**：利用环境和用户反馈生成批判，将失败转为可学习的结构化知识，无需修改模型权重。
- **自适应检索**：基于相似度分布的峰值检测确定检索数量，避免固定 k 值的局限性，在多个任务上表现稳健。
- **低成本冷启动**：通过合成查询或改写查询预先填充记忆，展示了实际部署可能性。
- **强实证结果**：在三个不同性质基准上均取得大幅提升，且消融实验清晰证明各模块贡献。
- **输出效率**：相比扩展思考等基线，虽增加输入 token，但控制输出 token 增长，对实际服务友好。

### 8. 不足与局限

- **单次运行**：所有结果仅基于一次评估，缺乏误差棒或多轮平均，统计显著性未经验证。
- **单轮任务为主**：所用基准均为单轮问题，未探讨多轮对话中动态拼接记忆或逐轮检索的效果。
- **工具数量有限**：基准中包含的工具不超过 40 个，真实应用场景工具数量庞大时，记忆覆盖和检索鲁棒性尚未验证。
- **冷启动依赖合成数据**：合成查询生成的质量和多样性有待进一步改进，部分复杂域（如多域）表现不佳。
- **二元反馈假设**：实验假设用户仅提供成功/失败二元反馈，现实场景中需处理更丰富的反馈形式。Hard Preference Guiding 虽有效，但在某些情况下可能错误否定正确参数。
- **成本与复杂性**：引入额外 LLM 调用进行记忆提取会增加推理前或记忆写入阶段的成本，可能不适合极低延迟场景。
- **检索超参数调优**：动态 n 的参数（R、prominence、峰号）需手动按任务调整，自适应性和通用性尚待提升。
