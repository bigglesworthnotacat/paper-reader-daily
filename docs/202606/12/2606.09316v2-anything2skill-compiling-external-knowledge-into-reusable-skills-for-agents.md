## 摘要
检索增强生成（RAG）使智能体能够在推理时访问外部知识，但它主要检索碎片化的陈述性证据，导致智能体需要反复从段落、手册、示例、日志或轨迹中推断任务过程。这引发了一个根本性问题：从外部知识库中提取的技能能否安装到智能体中，使其快速逼近领域专长？在本文中，我们提出了Anything2Skill，一个基于分类法引导的框架，将异构外部知识编译为智能体可复用、可检索且可执行的技能。给定一组知识记录，\textsc{Anything2Skill}首先将每条记录分解为证据窗口，并在技能树先验下进行规划与扩展式技能提取。然后，提取出的候选技能被转换为结构化的技能合约，其中规定了调用条件、禁忌症、动作步骤、工作流步骤、约束条件、输出规范、支持证据和置信度分数。为了构建可部署的程序性记忆，Anything2Skill通过分类法感知编译、注册级协调、生命周期跟踪、版本化更新和可视化技能树投影，在持久化的SkillBank中管理提取的技能。在推理时，智能体既从原始知识库中检索任务相关段落，也从SkillBank中检索相关程序性技能，使RAG提供陈述性证据，而编译后的技能提供可复用的程序性指导。在qsv与GitHub-CLI上的实验表明，Anything2Skill与RAG结合分别达到98.85%和94.10%的成功率，显著优于仅使用RAG的智能体。这些结果表明，将潜在的程序性知识编译为显式技能，是将检索增强型智能体从知识访问扩展到能力复用的有效途径。

## Abstract
Retrieval-augmented generation (RAG) enables agents to access external knowledge at inference time, but it primarily retrieves fragmented declarative evidence, leaving agents to repeatedly infer task procedures from passages, manuals, examples, logs, or trajectories. This raises a fundamental question: can skills extracted from external knowledge bases be installed into an agent, enabling it to rapidly approximate domain expertise? In this paper, we propose Anything2Skill, a taxonomy-guided framework that compiles heterogeneous external knowledge into reusable, retrievable, and executable skills for agents. Given a corpus of knowledge records, \textsc{Anything2Skill} first decomposes each record into evidence windows and performs plan-and-expand skill extraction under a skill-tree prior. The extracted candidates are then converted into structured skill contracts that specify invocation conditions, contraindications, action moves, workflow steps, constraints, output specifications, supporting evidence, and confidence scores. To construct a deployable procedural memory, Anything2Skill manages the extracted skills in a persistent SkillBank through taxonomy-aware compilation, registry-level reconciliation, lifecycle tracking, versioned updates, and visible skill-tree projection. At inference time, agents retrieve both task-specific passages from the original knowledge base and relevant procedural skills from the SkillBank, allowing RAG to provide declarative evidence while compiled skills provide reusable procedural guidance. Experiments on qsv and GitHub-CLI show that Anything2Skill combined with RAG achieves 98.85\% and 94.10\% success rates, respectively, substantially outperforming RAG-only agents. These results suggest that compiling latent procedural knowledge into explicit skills is an effective way to extend retrieval-augmented agents from knowledge access toward capability reuse.

## 二、方法论

### 2.1 整体框架：Anything2Skill

- 输入：一组异构知识记录 \( \mathcal{K} = \{D_i\} \)（文档、手册、对话、日志、轨迹等）以及一个技能分类法 \( \mathcal{T} \)。
- 输出：持久化的技能库（SkillBank） \( \mathcal{B} \)，技能以结构化“合约”形式表示，含名称、描述、资产类型、分类节点、调用条件、禁忌、动作、工作流步骤、约束、输出规约、支持证据与置信度等字段。
- 两大组件：
  1. **基于技能树先验的技能提取**（将原始记录转化为结构化技能草案）。
  2. **技能库中的技能管理**（编译、协调、版本化、树投影），最终使技能可部署。

### 2.2 技能树先验与证据窗口

- 技能分类法 \( \mathcal{T} \) 定义稳定的内部资产类型（宏观协议、会话技能、微技能、安全规则、知识引用等），并包含节点层级、父子关系及领域桶。
- 每个输入文档被分解为**证据窗口** \( w_{ij} \)，包含文本段、标题上下文、原始区间、边界与元数据，确保每个提取技能可追溯到源证据。

### 2.3 计划-扩展式技能提取

- 为避免长文档直接提取技能的冗余和欠规范问题，采用**两阶段**：
  - **计划阶段**：LLM 规划器为每个窗口识别潜在技能候选（含目标、资产类型、分类节点、触发条件、禁忌、证据线索等轻量信息）。
  - **扩展阶段**：对每个候选独立展开成结构化技能草案；若窗口证据不足以支持该过程，则返回空草案，防止幻觉。
- 经过**程序性有效性过滤**（必须包含至少一个过程性成分且可映射到分类法）后，保留草案并生成支持证据记录。

### 2.4 技能管理：文档级编译、注册级协调、可见树投影

- **文档级编译**：利用分类感知的身份键合并同一文档内描述同一能力的窗口草案，编译器约束下生成规范技能，最大化覆盖支持性内容，减少无依据与冗余。
- **注册级协调**：新技能与 SkillBank 中已有技能进行混合检索（稠密向量、BM25 稀疏匹配、分类兼容性加权），模型决策生命周期动作（创建、强化、修订、合并、拆分、保持不变、丢弃），动作受分类兼容性严格约束，防止跨层合并。每个技能拥有生命周期状态及版本更新。
- **可见技能树投影**：将活跃技能按分类层级组织为可导航的树，通过父节点选择与同级去重整合，最终 SkillBank 包含规范技能集、证据库、层级投影、版本历史及生命周期日志。

### 2.5 技能增强检索

- 推理时，代理同时检索：
  - 来自原始知识库的任务相关证据（RAG 通道）。
  - 来自 SkillBank 的相关程序性技能（技能通道）。
- 代理基于两者联合解任务，实现“陈述性证据 + 可复用程序指导”的双重供给。

## 三、实验设计

### 3.1 数据集与场景

- **qsv 基准**：聚焦 CSV 命令行操作，需理解官方文档、选择命令与选项，完成表格数据转换。
- **GitHub-CLI 基准**：基于 GitHub CLI 文档和操作指南（110 篇源文档），评估仓库管理操作（如 issue、PR、发布、仓库检查等）。从中预编译了 179 项 GitHub-CLI 技能（含全局技能、顶层技能、二级技能、微技能）。

### 3.2 对比方法

四种配置：

- **Base Agent**：无任何额外外部上下文。
- **RAG**：仅从原始知识库检索 top-3 源条目作为上下文。
- **Anything2Skill**：仅从 SkillBank 检索编译好的技能，不提供实时文本检索。
- **Anything2Skill + RAG**：同时提供 RAG 的陈述性段落和 SkillBank 的技能指导。

所有配置使用相同的基座模型 `gpt5.4` 及相同任务协议。

### 3.3 评价指标

- **任务成功率**：以智能体能否完对自然语言任务生成有效可执行命令行方案的比例衡量。

## 六、主要结论与发现

- 仅使用编译技能（Anything2Skill）即可显著超越 Base Agent（qsv：81.60% → 91.95%；GitHub-CLI：64.70% → 82.30%），说明**程序性技能具有独立价值**。

- 对复杂 CLI 工作流（如 GitHub-CLI），程序性技能收益更大，因其减轻了任务时推断过程的负担。
- 综上，将潜在程序性知识编译为显式技能，可使检索增强型智能体从知识访问拓展到**能力复用**，显著提高成功率。
