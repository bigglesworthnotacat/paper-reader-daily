## 摘要
基于摘要的提示压缩正越来越多地被LLM智能体用于缩短长且分布式的上下文，但这改变了安全边界：过滤器检查压缩前的提示，而后端却对新生成了压缩上下文进行操作。我们识别出一种压缩边界漏洞——重链接，其中压缩器表现为一个混淆代理，将分布式的、局部良性的片段总结成一个完整的恶意指令。与提示注入不同，重链接无需在源上下文中放置显式的恶意载荷。我们证明重链接源于摘要本身：注意力机制使分离的片段联合可用，预训练使兼容的片段连接合理，后训练则偏向于产生紧凑且可被后端执行的摘要。我们将攻击者诱导的形式形式化为对抗性重链接，并提出了Relink——一个基于DSL的自动化工具，它能在压缩前将恶意载荷分割成良性片段，同时保持完整载荷缺失。在四个长上下文智能体基准测试中，Relink达到了86.9%的重链接率和后端执行率，而干净分割对照组仅为17.0%。现有防御措施无法可靠捕获对抗性重链接；我们的KBRA防御将残余后端执行率降至0.0%。

## Abstract
Summarization-based prompt compression is increasingly used by LLM agents to shorten long, distributed contexts, but it shifts the security boundary: filters inspect the pre-compression prompt while the backend acts on a newly generated compressed context. We identify relinking, a compression-boundary vulnerability where the compressor behaves as a confused deputy, summarizing distributed, locally benign fragments into a complete malicious instruction. Unlike prompt injection, relinking need not place an explicitly malicious payload in the source context. We show that relinking arises from summarization itself: attention makes separated fragments jointly available, pre-training makes compatible fragments plausible to connect, and post-training favors compact backend-actionable summaries. We formalize the attacker-induced form as adversarial relinking and present Relink, an automated DSL-based tool that splits malicious payloads into benign fragments while keeping the complete payload absent before compression. Across four long-context agent benchmarks, Relink achieves 86.9% Relink Rate and Backend Action Rate versus 17.0% for clean-split controls. Existing defenses fail to reliably capture adversarial relinking; our KBRA defense reduces residual Backend Action Rate to 0.0%.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：LLM 智能体在处理长上下文时，常采用基于摘要的 prompt 压缩（如 Claude Code 的 `/compact`）来缩短上下文。然而，这种做法引入了一个**压缩边界**：安全过滤器检查的是压缩前的源上下文，而后端 LLM 执行的是压缩后新生成的上下文。
- **核心问题**：压缩器（summarization-based compressor）可能充当 **混淆代理（confused deputy）**，将源上下文中分散的、各自良性的片段“重链接（relinking）”成一个完整的恶意指令。与提示注入不同，重链接不需要在源上下文中放置显式的恶意载荷。
- **整体含义**：该漏洞本质上是现代压缩器三个属性（注意力机制跨片段绑定、预训练的类型槽先验、后训练的帮助性偏好）共同作用的结果，因此是**内生的、结构性的**漏洞。攻击者可以利用该漏洞绕过输入侧安全防御，使后端执行未在源中出现的恶意动作。

### 2. 论文提出的方法论

- **核心思想**：将恶意指令 `τ = (p⋆, v⋆, r⋆)` 分解为动作片段（`p⋆` 及其开放槽）和值片段（`v⋆` 及其表面类型），分别插入到源上下文的不同位置，并添加非绑定的角色提示（cue），使压缩器在压缩时自动将两者绑定形成完整指令。
- **关键技术细节**：
  - **对抗性重链接形式化定义**：`Ground(H, τ)=1`（成分存在）且 `Support(H, τ)=0`（源中无绑定），但压缩后 `Express(S, τ)=1`（绑定出现），后端执行 `Act(a, τ)=1`（动作发生）。
  - **Relink 工具**：基于领域专用语言（DSL）的三阶段管道：
    1. **解构（Decompose）**：删除指令图中动作与值之间的绑定边，得到两个独立的子图 Gp（动作+开放槽）和 Gv（值+类型）。
    2. **伪装（Disguise）**：为两个子图添加相同的非绑定角色提示（如 one-hot 坐标），使压缩器能关联它们，但不恢复绑定。
    3. **分布（Distribute）**：通过掩码投影将两个子图分别放置到源上下文的不同锚点，确保它们分开且本地不形成绑定。
- **公式/算法**：注意力权重 `αij` 计算、概率间隙 `∆log P`、效用间隙 `∆util` 等数学定义用于假设验证；DSL 的操作基于语料库驱动，支持多种上下文格式（段落、JSON、列表等）。

### 3. 实验设计

- **数据集/场景**：使用四个长上下文 Agent 基准，共 619 个样本：
  - **AgentDojo**（30 个）：个人助理场景
  - **ASB**（188 个）：多域智能体安全基准
  - **InjecAgent**（200 个）：注射导向智能体环境
  - **τ₂-bench**（201 个）：客户服务智能体
- **对比方法**：
  - **干净分割控制（Clean-split）**：相同载体、动作、值，但无 Relink 的 cue 选择或载体兼容实现。
  - **其他 split 攻击**：变量重构、语义分解重构、上下文分区（用于比较源端可接受性与后端执行率）。
  - **11 种防御**：输入过滤器（Llama Prompt Guard 2, ProtectAI DeBERTa-v2, ShieldGemma）、压缩提示防御（三明治提示、定界提示、BIPIA 提醒）、源清理器（PromptLocate, AttnTrace）、后端加固（SecAlign）、运行时策略（CaMeL）以及提出的 KBRA。
- **指标**：Relink 率（RR，压缩后上下文中出现目标指令）和后端动作率（BAR，后端执行目标动作）。两者在默认设置下完全一致（RR=BAR）。

### 6. 论文的主要结论与发现

- **重链接是压缩边界的固有漏洞**：机制探针证实三个假设均成立：跨片段注意力（99.4% 正间隙）、类型槽先验（95.2% 正概率间隙）、帮助性偏好（75.3% 正效用间隙）。后训练阶段是选择性承诺的关键（instruct 模型 81.6% 正确，base 模型仅 15.7%）。
- **Relink 攻击高效**：四个基准平均 RR/BAR 达 86.9%，远高于干净分割的 17.0%；在 AgentDojo、ASB、InjecAgent 上接近饱和。
- **攻击可迁移**：在不同压缩率、压缩提示、压缩模型和后端模型下均有显著效果，但压缩提示影响最大（agent-handoff 提示最脆弱）。
- **现有防御失效**：11 种已有防御无法同时保证安全和低误报；KBRA 通过过渡审计将残余 BAR 降到 0.0%（精确匹配）或 0.2%~0.3%（语义匹配），同时后端审计变体的实用假阳性率仅 0.93%，远低于所有基线。

### 7. 优点

- **方法创新**：发现了此前未被关注的压缩边界漏洞，并给出严格的形式化定义和工具化攻击，填补了安全研究空白。
- **攻击自动化**：Relink 基于 DSL，可自动构造对抗性上下文，适应不同场景（JSON、列表、散文等），扩展性好。
- **实证深度**：不仅展示攻击效果，还通过机制探针、消融、泛化实验揭示漏洞根源，增强了可信度。
- **防御简洁有效**：KBRA 的过渡审计思路直观，不依赖复杂模型，计算成本极低（CPU 毫秒级），且理论上有界保证。
- **实验覆盖面广**：从机制到应用、从攻击到防御、从学术基准到工业系统（OpenClaw、Claude Code），比较全面。

### 8. 不足与局限

- **实验依赖特定模型**：默认使用 Gemma-4-31B-it，跨模型实验仅用 180 样本子集，未穷举所有工业压缩机。
- **攻击条件限制**：τ₂-bench 上成功率较低（63.2%），说明任务结构（如客服约束）可抑制重链接；压缩提示词影响大，某些通用提示下 RR 降至 33.9%。
- **防御局限**：
  - KBRA 需要预配置工具模式、实体别名表，这些配置本身是关键安全资产，若被篡改则防御失效。
  - 针对“单段授权绑定”攻击（A1）无法阻止，但退化为直接注入，由输入过滤器覆盖——这仍是攻击面，只是被转移而非消除。
  - 实体碰撞攻击（A2）存在理论残余风险（ϵcol），实验显示语义匹配下残余 BAR 为 0.2%~0.3%，虽小但非零。
- **未讨论社会影响**：未涉及该漏洞对隐私、版权、自动化安全审计的潜在连锁反应。
- **计算资源细节不足**：未报告攻击工具 Relink 的构建耗时（可能依赖 LLM 调用）及不同配置下的总推理成本。
