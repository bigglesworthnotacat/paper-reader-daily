## 摘要
推荐算法的迭代正在从一种依赖工程师的手工流程转向工业化的研究闭环，但这一转型仍被结构性的执行瓶颈所阻碍：从想法到上线的周期仍依赖人类工程师生成假设、修改生产代码、启动A/B实验并归因在线结果。因此，创新的规模与人力呈线性关系，而非随着证据、算力和积累的实验知识而复合增长。我们提出AgentX，一个在生产环境中部署的多智能体系统，从根本上重构了这一生产函数。AgentX作为一个自我进化的发展引擎运行：它以任何手动工作流都无法维持的规模和速度，自主地生成、实现、评估并从推荐实验中学习。该系统在一个闭环中协调四个紧密耦合的阶段。构思智能体综合来自历史实验、系统架构、数据分析和外部研究的证据，生成排序的可执行提案。开发智能体通过基于仓库的生成和多维可靠性验证，将每个提案转化为生产就绪的代码。评估智能体通过带有护栏否决的A/B判断进行安全在线部署，将成功和失败都转化为结构化的知识资产。然后，一个驾驭进化层（SGPO）将执行轨迹提炼为语义梯度更新，持续优化智能体本身——使系统不仅自动化，而且能够自我改进。

## Abstract
Recommendation algorithm iteration is moving from an artisanal, engineer-bound process toward an industrialized research loop, but this transition remains blocked by a structural execution bottleneck: the idea-to-launch cycle still depends on human engineers to generate hypotheses, modify production code, launch A/B experiments, and attribute online results. Innovation therefore scales linearly with headcount rather than compounding with evidence, compute, and accumulated experimental knowledge. We present AgentX, a production-deployed multi-agent system that fundamentally restructures this production function. AgentX operates as a self-evolving development engine: it autonomously generates, implements, evaluates, and learns from recommendation experiments at a scale and pace that no manual workflow can sustain.   The system orchestrates four tightly coupled stages in a closed loop. A Brainstorm Agent synthesizes evidence from historical experiments, system architecture, data analysis, and external research into ranked, executable proposals. A Developing Agent translates each proposal into production-ready code through repository-grounded generation and multi-dimensional reliability verification. An Evaluation Agent conducts safe online rollout with guardrail-vetoed A/B judgment, converting both successes and failures into structured knowledge assets. A Harness Evolution layer (SGPO) then distills execution trajectories into semantic-gradient updates that continuously sharpen the agents themselves -- making the system not merely automated, but self-improving.

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 将推荐系统迭代分解为四个紧密耦合的阶段，由专门智能体执行，并通过轨迹驱动自进化机制（SGPO）持续优化智能体自身，形成完整闭环。

### 关键技术细节
- **构思智能体（Brainstorm Agent）**：将模糊的用户意图转化为有限且排序的可执行实验提案。核心挑战是意图的模糊性（ambiguity problem）。通过“有界探索”（bounded exploration）和“证据加权生成”（evidence-weighted generation）从四个知识源（实验知识库、系统知识库、数据分析、模型研究）中检索证据，并利用加权评分（公式含目标对齐度、业务有效性、实现可行性、交接完整性、风险等）筛选候选提案。
- **开发智能体（Developing Agent）**：分为两个并行轨道：
  - **在线策略轨道**：将提案转化为生产就绪的代码。通过“仓库接地生成”（repository-grounded generation）和“验证导向实现循环”（verification-oriented implementation loop）减少幻觉，利用八个维度的质量评分（如属性幻觉权重22%、正确性循环权重18%等）衡量代码可靠性。
  - **离线模型轨道**：实现模型架构提案并进行可信的训练实验。采用单轮流水线：策略声明 → 代码实现与验证 → 专家投票 → 执行 → 最终评审。强制“可证伪归因”（falsifiable attribution），要求对每个因果链接进行独立验证，未经验证的指标提升不被记录。
- **评估智能体（Evaluation Agent）**：负责安全部署、A/B判断和负面结果资产化。采用“带护栏否决的A/B判断”（guardrail-vetoed A/B judgment），护栏设计遵循业务域隔离、复合经济交换指标、阈值作为注意力信号而非绝对阻断三个原则。负面结果被收录为可复用的约束知识。
- **驾驭进化层（Harness Evolution）**：提出“语义梯度提示优化”（SGPO），分两阶段：
  - **SGPO-I**：利用执行轨迹生成语义梯度（自然语言诊断），更新子智能体的提示、验证规则和输出契约，仅通过成对重放（paired replay）确认改进后才准入生产。
  - **SGPO-II**：使用历史合并请求作为编码示例，通过编码补丁重放评估（五个维度：语义正确性40%、需求覆盖25%、文件覆盖20%、默认安全10%、风格一致性5%），进一步优化开发智能体。

### 算法流程
- 用户意图 → Brainstorm Agent（有界探索+证据加权）→ 提案 → Developing Agent（在线/离线实现与验证）→ 代码/实验 → Evaluation Agent（安全部署+A/B判断）→ 结构化结果 → 轨迹 → SGPO（语义梯度更新智能体）→ 下一轮。
