## 摘要
无线网络正从以连接为中心的基础设施演变为智能化和个性化的服务平台。现有的无线智能仍然集中在网络侧优化，改善吞吐量、延迟和覆盖等目标。然而，除了网络性能外，无线智能还取决于用户通过应用上下文、移动规律、服务成本、隐私偏好和长期使用行为所感知的体验。本文提出WISPA，一种基于大语言模型（LLM）代理的无线智能自我演化个人代理框架，用于自动化终端侧资源管理。为克服终端上的资源限制，WISPA将延迟敏感的在线资源执行与离线LLM代理反思解耦。这样，轻量级在线执行器使用可解释的偏好参数做出确定性资源决策；而离线LLM代理分析终端侧轨迹、细化用户画像并更新在线偏好参数以用于后续决策。最后，我们展示了WISPA在校园通勤路线上对终端侧资源分配的实际适用性和优势。数值结果表明，WISPA能够学习用户特定的连接风格，并随着偏好变化调整接入决策。

## Abstract
Wireless networks are evolving from connectivity-oriented infrastructures into intelligent and personalized service platforms. Existing wireless intelligence remains centered on network-side optimization, improving objectives such as throughput, latency, and coverage. Nevertheless, besides network performance, wireless intelligence also depends on user-perceived experience via application context, mobility routine, service cost, privacy preference, and long-term usage behavior. This article proposes WISPA, a Wireless Intelligent Self-evolving Personal Agent framework for automated terminal-side resource management based on large language model (LLM)-based agent. To overcome the resource constraints on terminals, WISPA decouples the latency-sensitive online resource execution from offline LLM agent reflection. In this way, a lightweight online executor makes deterministic resource decisions using interpretable preference parameters; While an offline LLM agent analyzes terminal-side traces, refines user profiles, and updates online preference parameters for subsequent decisions. At last, we demonstrate the practical applicability and benefits of WISPA for terminal-side resource allocations on a campus commute route. Numerical results show that WISPA learns user-specific connection styles and adapts access decisions as preferences change.

## 2. 方法论：核心思想、关键技术细节、公式/算法流程
- **核心思想**：将终端资源管理分为**在线执行**（轻量、确定、可解释）与**离线反思**（LLM驱动、偏好校准）两个解耦的阶段。LLM不直接控制实时无线接口，而是通过分析历史记录来更新偏好参数。
- **关键技术细节**：
  - **在线阶段**（实时执行）：
    - 上下文适配器：收集应用需求、无线电测量、电池状态、移动性信息、可用动作。
    - 选项评估器：为每个候选选项预测QoE（基于服务类型的满意度）和终端侧负担（如费用、能耗）。
    - 决策执行器：使用当前偏好参数对选项打分（体验分数 = f(QoE, 友好度分数, 偏好参数)），选择最高分且通过服务层级检查的选项。
    - 记录缓冲区：存储上下文、候选分数、所选动作、实际服务结果。
  - **离线阶段**（空闲窗口，如夜间）：
    - **观察模块**：将服务记录转换为对当前策略的评估（如体验分数变化、低体验事件原因）。
    - **反思模块**：结合当前评估和历史记忆，提取稳定的偏好线索、异常模式、重复上下文-动作模式。记忆采用紧凑画像形式，保护隐私。
    - **策略模块**：将理解转化为有界、可验证的偏好参数更新，通过验证后加载到在线执行器。
  - 公式与算法流程（文字说明）：
    - 体验分数 = 服务敏感QoE + 偏好参数 × 友好度分数。
    - 服务层级检查：当延时、吞吐量、丢包超过严重阈值时，体验分数置零（防止低费用选项掩盖严重服务失败）。
    - 偏好参数更新被限制在[0,1]范围内，且每步最大变化量受限；若更新无效则保持之前参数。

## 6. 主要结论与发现
- WISPA平均体验分数比Online Only提升高达13.6%（成本导向用户）和11.25%（质量导向用户）。
- 在动态偏好漂移实验中，WISPA每次切换偏好后能在几个周期内恢复分数，表现出自适应性，且不记忆固定策略，保留历史结构的同时保持塑性。
- 个性化不是全局偏好于WiFi或SIM2，而是上下文感知调整：在可容忍的低费用场景选择友好选项，在服务需求高时切换至高保障接入。
- 离线LLM反思阶段提供的有界、可验证更新保证了循环稳定性。

## 7. 优点
- **架构创新**：将实时控制与长周期反思解耦，既保证终端响应快速、轻量、可解释，又能利用LLM的推理能力进行个性化和自适应，而不必在实时路径上部署大模型。
- **隐私保护**：离线阶段只处理摘要记录和筛选事件，原始轨迹留在终端；记忆采用紧凑画像，避免长期暴露敏感数据。
- **安全性设计**：偏好参数更新有界、有校验环节，无效更新时回退到先前参数，防止LLM推理偏差导致系统行为异常。
- **实验验证清晰**：通过详细的路由部署、服务建模、对比方法，直观展示了WISPA在不同用户偏好下的表现和动态适应性。

## 8. 不足与局限
- **实验覆盖不足**：仅单个校园场景、单一通勤路线、有限服务类型（三种）和接入选项（三种）。缺乏跨地域、跨环境、多种移动模式的验证。

- **未对比学习基线**：仅对比了固定规则和无反思的在线选择，未与强化学习、贝叶斯优化等自适应方法比较，难以评估WISPA的相对性能。
