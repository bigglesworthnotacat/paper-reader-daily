## 摘要
现代LLM代理越来越依赖上下文压缩、摘要或驱逐来将长时间运行的会话保持在令牌预算内。我们表明，这种上下文管理层是一个安全关键故障面：代理在可见时可靠遵守的上下文内治理约束可能被压缩无声移除，导致同一代理在会话后期执行被禁止的工具操作。我们将这种故障模式称为治理衰减。我们引入了ConstraintRot，一个具有确定性工具调用评分的长时间代理场景基准，并测量了七个模型系列中压缩引起的违规行为。在1323个回合中，违规从策略完全在上下文时的0%上升到压缩后的30%，某些模型达到59%；当约束在摘要中保留时，违规保持0%，但当约束被丢弃时，违规达到38%。我们进一步研究了一种压缩-驱逐攻击，其中对抗性上下文内容使摘要器偏向省略合法策略，并表明优化注入击败了每个评估模型。最后，我们提出了约束锁定，一种简单的无需训练的缓解措施，将治理约束与有损压缩隔离，并在我们的基准中将违规恢复到0%。这些结果将上下文管理识别为已部署LLM代理的一级治理表面。

## Abstract
Modern LLM agents increasingly rely on context compaction, summarization, or eviction to keep long-running sessions within a token budget. We show that this context-management layer is a safety-critical failure surface: in-context governance constraints that agents reliably obey while visible can be silently removed by compaction, causing the same agent to perform prohibited tool actions later in the session. We call this failure mode Governance Decay. We introduce ConstraintRot, a benchmark of long-horizon agent scenarios with deterministic tool-call grading, and measure compaction-induced violations across seven model families. Across 1,323 episodes, violation rises from 0% with the policy in full context to 30% after compaction, reaching 59% for some models; when the constraint survives the summary, violation remains 0%, but when it is dropped, violation reaches 38%. We further study a Compaction-Eviction Attack, in which adversarial in-context content biases the summarizer to omit a legitimate policy, and show that optimized injections defeat every evaluated model. Finally, we propose Constraint Pinning, a simple training-free mitigation that quarantines governance constraints from lossy compaction and restores violation to 0% in our benchmark. These results identify context management as a first-class governance surface for deployed LLM agents.

- **问题**: 现代LLM代理（如企业助手）在执行长周期、多步骤工具调用任务时，为了保持上下文窗口在令牌预算内，会依赖**上下文压缩**（context compaction），即对历史对话进行摘要或驱逐。这种机制在优化任务连续性时，可能**默默地将代理原本遵守的治理约束（如安全策略、运行规则）从上下文中删除**，导致代理在后续相同请求下违反约束（如向外域发送邮件）。
- **整体含义**: 上下文管理本是为了保持代理运行效率，但却成为**安全失效的关键表面**。这种现象称为 **治理衰减（Governance Decay）** 。作者指出，这是系统层问题而非模型层问题，现有安全研究忽视了这一漏洞，并且约束衰减尤其危害那些只能存在于上下文中的**软组织策略**（如域限制、审批规则），而硬安全规范（如禁止删除生产库）则因模型固有偏见受影响较小，造成虚假的安全感。

- **防御——约束锁定**:  
  - 将治理约束提取到“锁定缓冲”中，该缓冲**免于压缩**，每次压缩后**逐字重新注入**，并**完整性检查**确保后续上下文仍隐含约束。

## 6. 主要结论与发现

1. **压缩诱导大范围违规**：全上下文时违规0%，一次压缩后平均30%（最高59%）；某些模型压缩后甚至比无策略基线更差（如DeepSeek-V4: 59% vs floor 37%），说明压缩不仅不帮助，反而有害。
2. **约束生存预测违规**：当约束在摘要中保留时违规0%，被丢弃时达38%，强证据支持删除机制。
3. **故障原因是压缩器，不是代理**：交叉实验（压缩器模型与代理模型交换）显示违规率主要取决于压缩器是否保留约束，而非代理本身。
4. **软策略衰减严重，硬规范相对安全**：软策略衰减+50pp vs 硬规范+6pp，形成8.3倍差距。
5. **压缩-驱逐攻击可武器化**：固定注入可突破部分模型的被动鲁棒性（如GLM 0%→22%）；优化注入击败所有模型（Claude 0%→65%）。
6. **约束锁定有效且代价极低**：在所有被动和固定攻击下恢复0%违规，合法动作完成99% (仅1%过度拒绝)，消融表明其防止了之前的35%过度拒绝。
7. **锁定仍存在根本性局限**：如果攻击者冒充操作员在近期上下文中声称“策略撤销”，朴素锁定仍会失效（17%违规），强化后也仅降至10%，需要可信带外操作通道。

## 7. 优点：方法或实验设计上的亮点

- **基准设计巧妙**：软/硬策略分离，能够隔离模型固有偏见与上下文约束效应，揭示真实衰减梯度。
- **确定性评分**：解析工具调用参数而非LLM裁判，消除主观偏差，确保可复现。
- **机制因果证明**：通过交叉压缩器-代理实验、生存率量化、反事实摘要（手动删除策略后观察违规变化），强有力地证明衰减是约束删除导致而非其他因素。
- **全面攻击与防御探索**：研究两种攻击变体（容量和注入）及优化搜索的威力，并提出有效的轻量化防御。
- **实际框架验证**：在四个主流框架（LangGraph, LangMem, AutoGen, OpenAI SDK）中复现了衰减，证明问题真实存在。
- **开放诚实**：明确指出防御的失效边界（操作员冒充问题）并承认其为开放问题，而非过度宣传。

## 8. 不足与局限

- **系统消息通道安全**：如果约束放在框架保证不压缩的系统消息中，则不适用。本文局限于记忆、对话、工具输出等非系统通道（这些是实际压缩的部分）。
- **隐式约束不能锁定**：约束锁定要求约束可提取为可引用的规则；隐式约束（如隐含在长文中的政策）不在范围内。
- **攻击搜索有限**：仅尝试了6种预定义注入策略，梯度或LLM-in-the-loop优化可能产生更强攻击。
- **生存率评判可能不够精确**：使用LLM裁判评估生存率，尽管有三裁判多数票和与关键词启发式的一致性，但人类标签可提供更精确的基准。
- **模型API服务限制**：所有模型通过API调用，样本温度固定，但重复次数有限（主要3次/条件），随机性可能影响统计稳定性，但高功率网格(6重复)部分缓解。
- **外部泛化性**：仅测试了7个模型和9个任务，虽覆盖主流系列，但更多模型和更复杂多步违规场景未测试。
- **操作员冒充攻击的根本性**：即使锁定，模拟的操作员撤销声明仍能绕过。作者指出需要可信带外操作员通道才能彻底解决，这是一个重大开放问题。
- **压缩预算设置**：实验中压缩预算较紧（80词？图3显示15-300词范围），实际生产可能更温和，但作者指出生产建议倾向于激进压缩以最大化效率，因此高衰减范围依然相关。
