## 摘要
现代大型语言模型（LLM）服务引擎越来越依赖检索增强生成（RAG）和非前缀键值（KV）缓存融合来加速长上下文、多租户推理。虽然现有的KV缓存侧信道攻击需要严格的线性前缀对齐——这使它们无法应对包含唯一且用户特定的私有前缀的现实RAG查询——但我们发现了一类关键的结构性漏洞，这些漏洞源于分块感知的内存调度。我们证明了用于对齐和融合不连续内存块的确定性微架构机制无意中泄露了连续的“阶梯波”时序特征。利用这一物理观察，我们引入了SpliceLeak，这是首个针对非前缀KV缓存融合的端到端侧信道攻击。SpliceLeak执行系统性的两阶段隐私泄露：首先，它在结构上指纹识别隐藏私有提示的确切长度；随后，它操纵边界碰撞以逐个标记提取确切语义内容。在生产级框架（集成了LMCache的vLLM）上的广泛评估表明，在有限熵场景中，SpliceLeak实现了高达100%的提取成功率。受确定性+104毫秒硬件延迟空洞驱动，该攻击每个标记仅需63次请求，即可穿透现实的连续批处理噪声。为了解决内存去重与安全之间的固有冲突，我们提出了SpliceDefense，一个双部分缓解框架，包括量化块填充（QCP）和恒定时间边界融合（CTBF）。我们的评估证实，SpliceDefense能有效平铺侧信道信号（Delta TTFT ~ 0），且吞吐量开销可忽略不计，从而保留了全局缓存共享的关键优势。

## Abstract
Modern Large Language Model (LLM) serving engines increasingly rely on Retrieval-Augmented Generation (RAG) and non-prefix Key-Value (KV) cache fusion to accelerate long-context, multi-tenant inference. While existing KV cache side-channel attacks require strict linear prefix alignment--rendering them ineffective against real-world RAG queries that contain unique, user-specific private prefixes--we uncover a critical class of structural vulnerabilities inherent to chunk-aware memory scheduling. We demonstrate that the deterministic micro-architectural mechanisms used to align and fuse disjoint memory chunks inadvertently leak a continuous "Step-Wave" timing signature. Exploiting this physical observation, we introduce SpliceLeak, the first end-to-end side-channel attack targeting non-prefix KV cache fusion. SpliceLeak executes a systematic two-phase privacy breach: it first structurally fingerprints the exact length of hidden private prompts, and subsequently manipulates boundary collisions to extract exact semantic content token-by-token. Extensive evaluations on production-grade frameworks (vLLM integrated with LMCache) demonstrate that SpliceLeak achieves up to a 100% extraction success rate in bounded-entropy scenarios. Driven by a deterministic +104 ms hardware latency void, the attack requires as few as 63 requests per token, piercing through realistic continuous batching noise. To resolve the inherent conflict between memory deduplication and security, we propose SpliceDefense, a bipartite mitigation framework consisting of Quantized Chunk Padding (QCP) and Constant-Time Boundary Fusion (CTBF). Our evaluations confirm that SpliceDefense effectively flattens the side-channel signal (Delta TTFT ~ 0) with negligible throughput overhead, preserving the critical benefits of global cache sharing.

- **研究含义**：本文首次揭示了**非前缀KV缓存融合**机制中的结构漏洞。该机制将检索到的文档块划分为固定大小的块，并在融合时进行选择性重计算：对齐的完整块直接从共享内存加载，未对齐的尾部令牌则需要全注意力重计算。这种确定性调度产生了可测量的**“阶梯波”（Step-Wave）时序特征**，使攻击者能够绕过前缀隔离，恢复隐藏的提示长度并逐令牌提取语义内容。

- **威胁模型**：攻击者是与该RAG服务同租户的普通用户，仅能通过标准API提交文本探针并观察TTFT（首令牌时间），无特殊系统权限。攻击者知道目标LLM的公共分词器和RAG框架的非前缀融合机制，并能识别或猜测受害者可能检索的公共知识块（C_target）。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 两阶段攻击框架：SpliceLeak

#### 阶段一：结构指纹识别（Phase I: Structural Fingerprinting）
- **核心思想**：利用非前缀KV缓存融合中固定大小块（如B=256个标记）导致的**阶梯波效应**。当攻击者改变其探针提示前缀长度时，TTFT呈现分段线性模式：
  - **宏观层**：TTFT与完整块数量（⌊L/B⌋）严格线性相关（每块开销b ≈ 38.6 ms），零误差暴露宏长度。
  - **微观层**：每个块组内，TTFT与未对齐尾部长度（L mod B）线性相关（斜率a ≈ 0.13 ms/tok，组内相关系数r ≈ 0.91），高精度暴露精确长度（±30个标记）。

- **机制**：通过测量不同前缀长度下TTFT的阶梯变化，攻击者能反推出受害者私有前缀的确切长度N_vic，从而为后续语义提取提供精确锚点。

#### 阶段二：语义提取（Phase II: Semantic Extraction via SpliceLeak）
- **核心思想**：利用精确的N_vic，攻击者构造探针使私有前缀恰好对齐固定块边界（如256标记块）。通过**边界碰撞**机制：若攻击者猜测的边界令牌与受害者真实令牌匹配，则语义连续性得以保持，引擎触发零碰撞状态，选择性重计算预算降低，TTFT出现可测量的显著下降（如104 ms）；若猜错，则强制完整块重计算，TTFT较高。

- **关键技术细节**：
  - **边界锚定提取机制**：攻击者首先猜测紧邻共享文档块的最后一个令牌t_N_vic。构造探针X_adv = P_pad || t_guess || C_target，其中P_pad长度为N_vic - 1，t_guess ∈ 词汇表V。通过观察TTFT是否显著下降来判断t_guess是否匹配。
  - **Agent驱动的提取框架**：采用**LLM驱动的提取Agent（A_LLM）**，如Gemini-3-Pro API（或本地模型Llama-3-8B）。Agent利用已知的公共块C_target和已提取的令牌，预测高概率候选子集V_sub（大小可调，如20个）。然后对每个候选发送探针，根据TTFT下降幅度确定正确令牌。提取过程为**从右向左迭代**：每次提取边界令牌后，将其追加到已提取序列，并向左移动一位，继续提取前一个令牌（类似逆序遍历）。

- **算法流程（文字描述）**：
  1. 从阶段一获取受害者私有前缀长度N_vic。
  2. 初始化已提取序列S_extracted为空。
  3. 对于i = N_vic 到 1（从最接近C_target的令牌开始向左）：
     a. 生成填充序列P_pad，长度为i - 1（确保探针中前缀总长度等于N_vic）。
     b. Agent A_LLM利用C_target和已有S_extracted生成候选子集V_sub（如20个候选）。
     c. 对每个候选t_guess，构造探针X_adv = P_pad || t_guess || S_extracted || C_target，测量TTFT。
     d. 选择TTFT最低且低于阈值δ_th的t_guess作为正确令牌，追加到S_extracted开头。
     e. 若找不到匹配，则停止或回退到束搜索。
  4. 返回S_extracted作为提取的私有前缀。

### 防御机制：SpliceDefense
- **量化块填充（QCP）**：在提示处理阶段，将私有前缀长度向上取整到块大小B的整数倍（附加空令牌），使所有前缀对齐块边界，从而消除阶梯波效应（掩盖长度指纹）。
- **恒定时间边界融合（CTBF）**：强制边界融合时执行固定的最坏情况重计算预算（无论令牌是否匹配），使正确和错误猜测的TTFT无差异，消除语义提取的时序信号。

## 6. 论文的主要结论与发现

1. **非前缀KV缓存融合存在结构性侧信道漏洞**：固定大小块调度产生可测量的“阶梯波”时序特征，可作为高分辨率长度指纹，单令牌精度可达±30标记。
2. **SpliceLeak攻击在有限熵场景下极其有效**：在场景A（已知模板）下提取成功率100%，每输入仅需~21次请求；在场景B/C（连续提取）下每个令牌仅需63次请求，成功率100%，中位Delta TTFT约+104 ms，信号稳定且不衰减。
3. **注意力架构不影响漏洞存在**：MHA（LongChat-7B）和GQA（Qwen2.5-7B）均被成功攻击，且GQA模型下信号实际上更强（Delta=358ms vs 446ms），表明KV缓存压缩技术不提供安全防护。
4. **SpliceDefense可有效消除侧信道**：CTBF将Delta TTFT从+138ms降至约-9ms（接近零），QCP引入小于5ms的额外延迟（<1%预填充时间），吞吐量开销可忽略。
5. **短到中等长度上下文更易受攻击**：对比16K长上下文（Delta约464ms），~2000标记上下文的Delta高达1057ms，因为融合开销占比降低，使时序信号更突出。

  - 仅在7B规模模型上测试，未验证更大模型（如LLaMA-70B）或异构架构（如MoE）。
  - 背景噪声模型简单（单一泊松过程），实际生产环境可能包含突发性高并发、优先级调度、缓存淘汰等复杂因素，结果可能有所不同。
  - 未测试跨云网络环境（WAN jitter）下的攻击鲁棒性，论文附录提到极端WAN波动可能降低SNR。
- **防御效果评估不足**：
  - CTBF虽消除了时序信号，但增加了固定计算开销，论文未详细分析其在不同硬件和批大小下的性能上界。
  - QCP引入的空令牌填充可能增加显存占用，在极高内存压力下可能导致更长延迟或batch size下降。
- **未探讨可迁移性**：
  - 是否能在其他非前缀融合框架（如CacheBlend的不同实现版本）或基于Radix-Tree的系统（SGLang）中复现？论文仅简短提及，未进行实验验证。
