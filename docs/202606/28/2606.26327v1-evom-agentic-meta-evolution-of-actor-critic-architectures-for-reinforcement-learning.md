## 摘要
在演员-评论家强化学习中，网络架构通常需要手动设计。自动化这一设计具有挑战性，因为每个候选架构在评估前必须经过训练，且设计空间是开放的。为了解决这些挑战，我们引入了EVOM，一个用于发现高性能演员-评论家架构的智能体元进化框架。我们将架构搜索构建为双层优化：内环通过低保真度的近端策略优化（PPO）训练权重，而外环通过迭代优化架构程序驱动元进化。关键的是，这个外环由一个基于LLM的设计智能体驱动，该智能体仅作为架构设计者，与策略执行和环境控制完全解耦。实验表明，EVOM优于手动设计的基线、LLM引导的随机搜索以及最先进的LLM引导程序化策略搜索方法MLES，在Ant-v4和HalfCheetah-v4上展现出更优的性能。消融研究验证了元进化循环和LLM设计智能体对于最终性能都是不可或缺的。

## Abstract
In actor-critic reinforcement learning, network architectures are typically manually designed. Automating this design is challenging because each candidate must be trained before evaluation, and the design space is open-ended. To address these challenges, we introduce EVOM, an agentic meta-evolution framework for discovering high-performance actor-critic architectures. We frame architecture search as a bi-level optimization: an inner loop trains weights via the low-fidelity proximal policy optimization (PPO), while an outer loop drives meta-evolution by iteratively refining architecture programs. Crucially, this outer loop is powered by an LLM-based design agent that operates purely as an architecture designer, completely decoupled from policy execution and environment control. Experiments reveal that EVOM outperforms the manually designed baseline, an LLM-guided random search, and the state-of-the-art LLM-guided programmatic policy search method MLES, delivering superior performance on Ant-v4 and HalfCheetah-v4. Ablation studies validate that both the meta-evolution loop and the LLM Design Agent are indispensable for final performance.

## 1. 核心问题与整体含义（研究动机和背景）
- **研究动机**：在强化学习的Actor-Critic方法（如PPO）中，网络架构（策略网络和价值网络）通常由人工手动设计，这既耗时又可能错失更优设计。自动化架构搜索面临两大挑战：
  1. **评估成本高**：每个候选架构必须完整训练后才能评估其性能，导致全预算比较计算量巨大。
  2. **设计空间开放**：策略头、价值头、共享骨干、归一化层、激活函数等可组合方式极多，不存在固定的拓扑搜索空间。
- **整体含义**：针对上述挑战，本文提出**EVOM**，一个将LLM（大语言模型）作为设计代理的**双层元进化框架**，自动发现高性能的Actor-Critic架构。该框架将架构搜索形式化为双层优化：内环用低保真PPO训练权重，外环通过LLM驱动进化迭代优化架构程序。

## 2. 方法论
### 核心思想
- 将架构搜索建模为**双层优化**：
  - **内环（Inner Loop）**：给定一个架构程序 \(p\)，使用PPO在**低保真预算**（低时间步数）下训练其权重 \((\theta, \phi)\)，并计算平均奖励作为适应度信号。
  - **外环（Outer Loop）**：维护一个**精英种群**，每代通过LLM设计代理执行变异（Mutation）和交叉（Crossover）生成后代，评估后选择适应度最高的候选作为下一代精英。
- LLM设计代理**仅作为架构设计者**，不接触梯度、参数值或环境状态，与策略执行和环境交互**完全解耦**。

### 关键技术细节
1. **架构表示**：每个候选 \(p\) 存储为可执行的Python程序，包含 `PolicyNet(obs_length, act_length)` 和 `ValueNet(obs_length)` 类定义及结构化JSON描述（层类型、宽度、激活函数、归一化、残差结构等）。程序通过编译、形状检查、有限输出等验证，否则赋予惩罚适应度 -1000。
2. **LLM引导的进化循环**：
   - **初始化**：LLM根据环境描述、输入输出维度等生成初始种群 \(P^{(0)}\)。
   - **繁殖**：每代从当前精英种群中，LLM通过变异（修改单一父代）和交叉（融合两个父代）各生成 \(N/2\) 个后代，构成后代种群 \(P'\)。
   - **评估**：每个后代用PPO在低保真预算下训练，计算 \(K\) 个确定性评估episode的平均奖励。
   - **选择**：合并当前精英与后代，按适应度排序，保留前\(N\)个作为下一代精英。
3. **PPO评估与最终重训**：
   - **低保真搜索**：使用 100k 时间步训练，\(K=3\) 个评估episode，快速得到适应度代理。
   - **全预算重训**：进化结束后，选择最佳架构用 5M 时间步从头训练，报告最终性能。
4. **提示模板**：使用三种模板（初始化、变异、交叉），包含环境描述、固定输入输出信息、架构设计JSON和代码框架。变异和交叉提示中包含当前父代架构信息，要求LLM生成有意义的新设计。

### 算法流程（文字说明）
1. LLM生成初始种群 \(P^{(0)}\)（N个架构程序）。
2. 对每个初始架构，用PPO（100k步）训练并计算评估奖励。
3. 循环 \(t=0\) 到 \(G-1\)（G=20）：
   - 使用LLM对当前精英 \(P^{(t)}\) 执行变异（N/2）和交叉（N/2）生成后代 \(P'\)。
   - 对每个后代进行PPO低保真评估（100k步+3个episode）。
   - 将 \(P^{(t)}\) 与 \(P'\) 合并，按评估奖励排序。
   - 选择前N个作为下一代精英 \(P^{(t+1)}\)。
4. 最终评估：选择所有精英中最好的架构，用PPO全预算（5M步）重新训练并评估，输出最终奖励。
