## 摘要
LLM代理越来越多地被用于在开放的图形用户界面环境中自动化用户任务。它们不可避免地会遭遇包含用户敏感信息的屏幕，此时由用户接管任务执行是非常可取甚至必要的。最先进的LLM驱动的代理通常针对完成任务进行微调，而不考虑其行为的安全影响。这使得它们在实际部署中很困难，并对可靠性产生不利影响。因此，识别和分类用户敏感状态并定义用户敏感查询至关重要。该数据集将帮助工程师在关键场景中识别并请求移交用户。这篇短文开发了一个探索代理，它从一个演示任务开始系统地探索查询空间，以识别那些如果执行将导致GUI环境中用户敏感状态的查询。

## Abstract
LLM agents are increasingly being used to automate tasks for users within an open GUI environment. They inevitably encounter screens containing user-sensitive information, for which takeover of task execution by the user is highly desirable or even necessary. State-of-the-art LLM-driven agents are usually fine-tuned to complete tasks regardless of the safety implications of their actions. This makes their real-world deployment difficult and adversely affects the reliability. Therefore, it is crucial to identify and categorize user-sensitive states and define user-sensitive queries. This dataset would be to engineers to recognize and request handover to the user in critical scenarios. This short paper develops an explorer agent that systematically explores the query space starting from one demonstrated task to identify queries that, if executed, would lead to user-sensitive states in a GUI environment.

## 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：当前LLM驱动的GUI代理在开放环境中执行任务时会不可避免地遇到包含用户敏感信息的屏幕（如个人数据、账户管理、交易确认等），而这些代理通常被微调为“无论如何完成任务”，缺乏对安全风险的感知；一旦误操作（如发送邮件、删除文件、支付）可能导致不可逆后果，从而阻碍实际部署并损害可靠性。

## 论文提出的方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：构建一个“探索代理”（Explorer LM），以纯探索为目标，通过蒙特卡洛树搜索（MCTS）迭代式地生成并执行查询，利用奖励机制（新颖性+用户敏感性）来导向未探索且可能敏感的状态。搜索空间由“基础查询”和从该查询分叉出的“敏感节点”组成。
- **关键技术细节**：
  - **双模型架构**：①原生LM（如M3A）负责在模拟器中执行具体动作；②探索LM（本文贡献）负责生成新的查询（任务）及其步骤。
  - **查询选择与扩展**：基于UCT公式简化，使用余弦相似度过滤重复查询（设定阈值τ=0.85），保留与已有查询和池中查询均不相似的查询作为新候选。选择方法`Asel`计算保留索引`{i: V(ui, s0)>0}`，其中`V`基于相似度差异；累计池`P`逐步扩大。
  - **饱和检测算法`Asat`**：在每个训练轮次中批量生成查询（M=10），直到新增查询数量下降（通过多项式拟合检测拐点），确保覆盖收敛。
  - **执行与回滚**：连接Android模拟器（基于SPABench），原生LM执行动作，记录屏幕截图和动作日志。
  - **训练**（替代MCTS的Backpropagation）：
    - **监督微调（SFT）**：基于回滚经验提升步骤预测准确性。
    - **组相对策略优化（GRPO）**：基于三种新颖性得分（查询新颖性、步骤新颖性、类别新颖性）构成复合奖励函数，鼓励探索历史未覆盖的查询和敏感类别。
    - 总奖励 = query novelty score × (step novelty score + category novelty score)。

  3. 输出最终模型。
