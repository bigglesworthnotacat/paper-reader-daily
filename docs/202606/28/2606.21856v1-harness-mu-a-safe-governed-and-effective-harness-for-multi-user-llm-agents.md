## 摘要
随着大语言模型（LLM）智能体在协作工作流中的日益部署，需要强大的多用户、多主体交互机制，以执行访问权限、解决权威冲突并防止未经授权的数据泄露。然而，当代LLM的单用户训练范式与多主体治理所需的硬约束之间存在根本性不匹配，使得基于概率和提示的安全措施在多轮对抗性交互中显得脆弱。我们的关键见解是：治理约束——谁被授权、什么被限制、谁的指令优先——是确定性的运行时变量，应由执行钩子强制执行，而非交由LLM负责。我们提出了Harness-MU，这是首个与模型无关、零微调的多用户LLM智能体基础设施框架。通过将语言生成与安全编排解耦，Harness-MU保证了不可打破的权限边界，同时最大化合规需求满足。在Muses-Bench基准测试中，针对四个前沿的开源权重和专有模型，Harness-MU在所有访问控制攻击中实现了隐私保护目标，效用得分比标准基线高出0.28–0.39，指令遵循准确率提升高达48.9个百分点。Harness-MU推进了“管理工程”的理念，确立了系统级基础设施对于解决LLM多主体治理挑战的必要性。代码和数据见https://github.com/YuanJrShiuan/Harness-MulUser。

## Abstract
The increasing deployment of large language model (LLM) agents in collaborative workflows demands robust multi-user, multi-principal interaction mechanisms capable of enforcing access permissions, resolving authoritative conflicts, and preventing unauthorized data disclosure. However, a fundamental mismatch exists between the single-user training paradigm of contemporary LLMs and the hard constraints required for multi-principal governance, rendering probabilistic, prompt-based safeguards vulnerable under multi-turn adversarial interactions.Our key insight is that governance constraints -- who is authorized, what is restricted, and whose instructions take precedence -- are deterministic runtime variables that should be enforced by execution hooks rather than entrusted to the LLM. We present \textbf{Harness-MU}, the first model-agnostic, zero-tuning infrastructure framework for multi-user LLM agents. By decoupling language generation from safety orchestration, Harness-MU guarantees unbreakable permission boundaries while maximizing compliant demand satisfaction. Across four frontier open-weight and proprietary models on the \textit{Muses-Bench} benchmark, Harness-MU achieves the goal of privacy preservation across all access-control attacks, outperforming the standard baseline by 0.28--0.39 in utility score and improving instruction-following accuracy by up to 48.9 percentage points. Harness-MU advances the philosophy of \textit{Harness Engineering}, establishing that systematic infrastructure is essential for solving LLM multi-principal governance challenges. The code and data are available at https://github.com/YuanJrShiuan/Harness-MulUser.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **问题**：LLM 智能体在组织协作中需同时服务多个用户，每个用户拥有不同角色、权限级别和隐私边界。然而，当前 LLM 基于**单主体-单智能体（SPA）范式**训练，无法自然处理多用户场景，导致三类系统级故障：未经授权的信息泄露、指令冲突无法按权威等级裁决、低权限指令覆盖高权限指令。
- **背景**：现有安全方法（如提示工程、RLHF）仅提供概率性防御，在多轮对抗交互中易被攻破。同时，“管理工程（Harness Engineering）”范式已在单智能体场景中证明，系统基础设施而非模型能力是可靠性的瓶颈，但尚无支持多用户的管理框架。
- **整体意义**：本文提出 Harness-MU，首个专为**多主体-多用户（MPA）场景**设计的全栈运行时管理框架，通过解耦语言生成与安全编排，以确定性执行钩子强制执行隐私和权限硬约束，实现模型无关、零微调的安全治理。

## 2. 论文提出的方法论：核心思想、关键技术细节
### 核心思想
- **治理约束是确定性运行时变量**，应由执行钩子而非 LLM 自身负责。将语言生成与安全编排解耦，在 LLM 外部通过四个独立模块完成准入控制、策略分解、隔离生成和合规检查。

### 关键技术细节
- **形式化 MPA 优化目标**（公式 3）：
  - 在**零泄露隐私约束**（I_leak = 0 for unauthorized users）和**最高权威硬目标强制满足**（I_sat = 1 for G⋆）条件下，最大化权威加权总效用。
  - 现代 LLM 基于概率分布，无法保证该约束边界，需外部基础设施强制。

- **Harness-MU 四模块运行时管道**：
  1. **Gatekeeper（准入控制层）**：在模型调用前，根据声明式组织策略为每个用户构建不可变的权限记录 `pk = ⟨uk, αk, Ak, Rk⟩`，明确允许的敏感令牌集合 `Ak`。不调用 LLM，完全确定性。
  2. **Mediator（策略编排层）**：将全局多主体目标分解为每个用户的独立执行计划 `πk`，包含权威权重、过滤后的硬目标和行为姿态（授予/拒绝/延迟），不直接与 LLM 交互，而是将策略注入下游 Worker。
  3. **Worker 集群（隔离生成层）**：为每个用户启动独立的并行执行环境，各自构建隔离的上下文 `Γk`，只包含该用户的历史、共享上下文和 Mediator 计划。**关键**：Worker 之间历史不混合，阻止用户间指令注入或权限提升攻击。
  4. **ComplianceChecker（可行域投影层）**：后处理阶段，对每个 Worker 生成的原始响应执行确定性检查（子串匹配、IHEval 格式/词汇约束）。违规则 fail-closed，用固定拒绝载荷覆盖，并生成结构化遥测触发修正循环。

- **六组件管理框架实例化**（E, T, C, S, L, V），适配多用户治理。

## 6. 论文的主要结论与发现
1. **AC：绝对隐私保护**：在所有 12 个攻击配置下，Harness-MU 实现隐私分数 1.000（零方差），原始基线最低仅 0.926（Deepseek）。总体分数提升 +0.240（Qwen）至 +0.427（Deepseek）。

3. **ComplianceChecker 的关键作用**：AC 中闭合剩余泄漏（如 Deepseek 原始隐私 0.932 → 最终 1.000），且零效用损失；IF 中显示**逆缩放**：底层模型越弱，合规修正的收益越大（GPT 触发率 66.8%，提升 +7.94pp）。
4. **延迟-令牌权衡**：Harness-MU 增加总令牌消耗（约 5 倍），但通过 Worker 并行化使关键路径延迟反而降低 11%（Deepseek 上 148s→131s），因为单用户上下文更短、推理更快。
5. **管理工程范式再次被验证**：系统基础设施是解决 LLM 多主体治理挑战的关键，而非单纯提升模型能力。
