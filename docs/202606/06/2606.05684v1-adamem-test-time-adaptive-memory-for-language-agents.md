## 摘要
语言智能体的一个核心挑战是如何利用过去经验来适应动态的测试时条件。近期工作展示了智能体记忆机制的潜力，但大多数系统将检索限制在回合开始时。因此，随着长周期任务的展开，智能体不得不依赖逐渐失配的静态指导。为了解决这种僵化，我们提出了自适应记忆智能体（AdaMEM），一个用于智能体测试时适应的新型框架。AdaMEM 无需在线更新模型参数，而是通过混合记忆架构调整智能体行为：它维护一个由离线收集的原始经验组成的长期轨迹记忆，同时动态生成短期策略记忆以实时指导决策。这一机制实现了在不同推理时计算水平下词元效率与适应性之间的权衡。实验表明，AdaMEM 显著优于静态记忆基线，在 ALFWorld 和 WebShop 上分别取得了高达 13% 和 11% 的相对增益，并且在 HotpotQA 的智能体搜索任务上持续领先。为进一步增强这种适应性，我们开发了 STEP-MFT，一种逐步记忆微调技术，通过训练策略从检索到的经验中合成高质量策略，从而带来额外的性能提升。我们的工作为智能体记忆建立了一个新的扩展维度，支持在现实环境中部署后进行持续推理和自我进化。代码见 https://github.com/yunx-z/AdaMEM。

## Abstract
A central challenge for language agents is utilizing past experience to adapt to dynamic test-time conditions. While recent work demonstrates the promise of agentic memory mechanisms, most systems restrict retrieval to episode initiation. Consequently, agents are forced to rely on static guidance that becomes increasingly misaligned as long-horizon tasks unfold. To address this rigidity, we propose the Adaptive Memory Agent (AdaMEM), a novel framework for agent test-time adaptation. Without updating model parameters online, AdaMEM adapts agent behavior via a hybrid memory architecture: it maintains a long-term trajectory memory of raw experiences collected offline while generating dynamic short-term strategy memory on-the-fly to guide decision-making. This mechanism enables the trade-off between token efficiency and adaptability across varying inference-time compute levels. Empirically, AdaMEM significantly outperforms static memory baselines, achieving relative gains of up to 13% on ALFWorld and 11% on WebShop, with consistent leading performance extending to agentic search on HotpotQA. To further enhance this adaptation, we develop STEP-MFT, a Step-wise Memory Fine-Tuning technique that trains the policy to synthesize high-quality strategies from retrieved experiences, yielding additional performance gains. Our work establishes a new scaling dimension for agentic memory, supporting continuous reasoning and self-evolution post-deployment in real-world environments. Our code is available at https://github.com/yunx-z/AdaMEM.

## 1. 研究动机与核心问题
- 语言智能体在长周期任务中，需要利用过去经验适应动态的测试时环境条件。
- 现有智能体记忆机制（例如 Synapse、ReasoningBank）通常在回合开始时一次性检索静态轨迹或策略，随着任务状态演变，这种固定指导逐渐失效，造成“僵化瓶颈”。
- 核心问题：**如何让智能体记忆摆脱静态检索的限制，在推理过程中持续自适应；以及如何高效训练模型合成能主动优化测试时适应的策略。**

## 2. 方法论

### 2.1 混合记忆架构：AdaMEM
- **长期轨迹记忆 (Long-Term Trajectory Memory, \(M\))**：离线存储成功轨迹片段，包含状态、动作及后续成功路径，按状态嵌入索引。
- **短期策略记忆 (Short-Term Strategy Memory, \(z_t\))**：在线合成简洁的自然语言策略，显式条件化于当前状态，用于指导下一步动作。
- **两种推理模式**（无需更新模型参数）：
  - **AdaMEM-HIGH**：每一步可选择是否触发检索与策略合成，生成瞬时策略 \(z_t\)，使用后即丢弃，最大化适应性。
  - **AdaMEM-LOW**：维护一个持久策略 \(z_{curr}\)，仅在智能体判定需要时刷新，以降低词元消耗。

### 2.2 策略微调：STEP-MFT
- **动机**：零样本策略可能过于泛化，缺乏针对具体失败的诊断能力。单纯基于最终结果的监督信号含噪声，无法区分有效策略与冗余文本。
- **方法**：
  1. **数据收集**：用 AdaMEM-HIGH 收集决策元组 \((s_t, E_{ret}, z_t, a_t, a'_t, r)\)，其中 \(a'_t\) 为无记忆的基线动作（仅采样，不执行）。
  2. **过程级过滤（双条件）**：
     - 最终轨迹成功（\(r=1\)）；
     - **动作改变**：\(a_t \neq a'_t\)，确保策略确实改变了决策。
  3. **形式依据**：在确定性贪心解码与 “成功轨迹对应的动作足够有效” 假设下，动作不一致是获得严格正策略优势的必要条件（优势定义为有策略与无策略的期望成功率之差）。
  4. **训练**：对过滤后的策略样本做标准监督微调（SFT），得到 **AdaMEM-MFT**。

### 2.3 关键流程
1. 当前状态输入。
2. 从长期记忆检索相似经验。
3. 合成短期策略。
4. 基于策略生成动作。

其中长期记忆可用于跨模型复用（off-policy），而短期策略始终由当前策略模型生成，保持 on-policy。

## 6. 主要结论与发现
- AdaMEM 在所有任务上均超越静态记忆基线，且在未见环境和跨模型设定下表现稳定。
- 动态检索纠正了 WebShop 上的负迁移现象，静态记忆在该任务上甚至不如无记忆。
- 混合架构支持 off-policy 记忆共享，不同模型可共用长期记忆库。
- AdaMEM 的检索表现随 \(k\) 增加单调上升，而 Synapse 因文本膨胀反而性能下降。
- STEP-MFT（动作变化过滤）显著优于仅靠结果过滤的微调，后者甚至可能损害性能。
- 推理计算增加时，AdaMEM 性能持续提升，形成更优的帕累托前沿。

## 7. 优点
- **灵活性**：两种推理模式实现词元效率与适应性的可控权衡。
- **无需参数更新**：纯测试时自适应，易于部署。
- **记忆抽象**：将原始轨迹压缩为策略，避免上下文窗口膨胀与无关信息干扰。
- **训练信号精细化**：STEP-MFT 利用动作变化作为“过程级奖励”代理，无需额外价值模型或 rollout。
- **跨模型泛化**：长期记忆与短期策略分离，支持离线记忆的跨模型复用。
- **开源**：代码公开，可复现。

## 8. 不足与局限
- **策略惯性**：AdaMEM-LOW 依赖 LLM 自我评估是否刷新策略，可能误判而坚持失效策略，导致任务失败。
- **仅使用成功轨迹**：长期记忆不包含失败经验，可能遗漏宝贵的纠正信息；模型对失败模式的吸收尚待研究。
- **微调算法**：STEP-MFT 使用拒绝采样微调，未探索在线强化学习，后者的稳定性和效果提升潜力仍开放。
- **实验规模有限**：未在更大规模的环境或更多样化的任务上验证，且仅使用了 4B/7B 模型，更大模型与更多 token 预算的扩展行为未讨论。
- **过程奖励假设**：动作变化作为必要的正优势条件，需建立在“成功轨迹动作足够有效”的近似之上，可能在高度随机或稀疏奖励环境中适用性下降。
