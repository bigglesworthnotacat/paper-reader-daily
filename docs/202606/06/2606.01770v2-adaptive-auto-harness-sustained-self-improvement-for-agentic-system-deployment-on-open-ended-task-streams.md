## 摘要
Auto-harness系统（如A-Evolve、GEPA和Meta-Harness）通过从执行反馈中优化提示、技能、工具、记忆和支持基础设施来改进LLM智能体，但它们通常在固定的离线基准上进行评估。真实的部署则呈现开放式的任务流：历史不断增长且没有固定终点，异构任务需要不同的装备，问题分布随时间变化。这些挑战使得单一且频繁密集更新的装备变得脆弱，导致性能在准确率早期达到峰值后下降。这促使了具有任务级适应的持续装备构建。我们提出了Adaptive Auto-Harness，一个针对此类流的框架和系统。该框架将与理想装备之间的差距分解为演化损失和适应损失。系统通过一个有状态的多智能体演化器、一个具有求解时路由的装备树以及在历史缺乏所需信号的情况下的人类引导钩子来解决这些损失。在预测市场、安全竞赛和事件预测流上，Adaptive Auto-Harness优于五个现有的自动装备基线，消融实验将增益归因于更好的构建、路由或有针对性的人类引导。代码可在https://github.com/A-EVO-Lab/a-evolve/tree/release/adaptive-auto-harness获取。

## Abstract
Auto-harness systems such as A-Evolve, GEPA, and Meta-Harness improve LLM agents by optimizing prompts, skills, tools, memories, and supporting infrastructure from execution feedback, but they are typically evaluated on fixed offline benchmarks. Real deployments instead present open-ended task streams: histories grow without a fixed endpoint, heterogeneous tasks require different harnesses, and problem distributions shift over time. These challenges make a single repeatedly and densely updated harness brittle, causing performance degradation as accuracy peaks early and then declines. This motivates sustained harness construction with task-wise adaptation. We introduce Adaptive Auto-Harness, a framework and system for such streams. The framework decomposes the gap to an oracle harness into evolution loss and adaptation loss. The system addresses these losses with a stateful multi-agent evolver, a harness tree with solve-time routing, and human-steering hooks for cases where history lacks the needed signal. Across prediction-market, security-competition, and event-forecasting streams, Adaptive Auto-Harness outperforms five existing auto-harness baselines and ablations attribute gains to better construction, routing, or targeted human steering. Code is available in \href{https://github.com/A-EVO-Lab/a-evolve/tree/release/adaptive-auto-harness}{Link}.

# Adaptive Auto-Harness: 面向开放式任务流的智能体系统持续自我改进

## 1. 研究动机与核心问题

- **背景**：现有Auto-Harness系统（如A-Evolve、GEPA、Meta-Harness）通过执行反馈自动优化LLM智能体的提示、技能、工具等“装备”，在固定离线基准上取得显著提升。
- **核心问题**：真实部署场景为**开放式任务流**，具有三个关键挑战：
  - **无界流（Unbounded Stream）**：任务持续到来，历史不断膨胀，无固定训练/测试划分，进化器面临上下文窗口瓶颈。
  - **任务异质性（Heterogeneity）**：流中混合多种任务类型（如政治、体育、金融），单一固定装备无法普适优化。
  - **分布非平稳性（Non-stationarity）**：任务分布随时间漂移，先前拟合的装备迅速过时。
- **根本缺陷**：重复、密集更新单一装备会导致性能早期达到峰值后迅速下降（即过度拟合早期流证据），无法实现持续部署中的自我改进。
- **总体目标**：设计一种能在开放式任务流中实现**持续装备构建**与**求解时任务适应性**的框架，解决上述部署维度挑战。

## 2. 方法论

### 2.1 分析框架：遗憾分解

将装备构建形式化为遗憾最小化问题，将相对于神谕装备（oracle harness）的期望遗憾分解为两项可操作的损失：

- **命题 1（遗憾分解）**：
  \[
  \mathbb{E}_{x_t}[\text{Regret}(\phi, x_t)] = L_{\text{evo}}(\Phi) + L_{\text{adapt}}(\phi)
  \]
  - **进化损失 \(L_{\text{evo}}\)**：进化器类 \(\Phi\) 本身的能力上限，即从历史中构建最优装备的结构性差距（如单智能体提示编辑器无法生成多文件基础设施）。
  - **适应损失 \(L_{\text{adapt}}\)**：已部署进化器 \(\phi\) 在未看到任务 \(x_t\) 前就提交单一装备的损失，即因缺乏逐任务定制而产生的核心差距。

### 2.2 系统设计：三大核心机制

#### (1) 持续自动装备：有状态多智能体进化器
- 针对进化损失，将进化过程分解为**四个阶段**的专用智能体，每个阶段享有完整上下文预算：
  - **分析（Analyst）**：识别失败模式，生成任务板，标注优先级及目标分支（`main`或`branch/<name>`）。
  - **研究（Researchers）**：并行探索多样假设，测试实现方法，写入研究日志。
  - **构建（Builder）**：基于验证后的研究实现代码，更新提示/技能/工具。
  - **验证（Verifier）**：测试构建产物，通过/失败后回馈构建者重试（最多3次）。
- 支持**时间戳揭示反馈**：根据任务解决时间异步提供评估信号，避免未来信息泄露。
- 维持**跨周期持久状态**：工作空间包含任务板、研究日志、架构文档和验证测试，供进化器跨周期累积经验。

#### (2) 求解时适应：装备树路由
- 针对适应损失，由进化器构建结构化装备库，求解时仅需轻量选择操作。
- **装备树**：使用Git仓库组织，按任务领域（regime）创建专用分支（如`branch/sports`、`branch/crypto-classical`），每个分支携带自己的提示、技能、工具注册表，实现领域隔离。
- **智能体路由**：求解时，路由智能体读取每个分支的README（含“何时路由到此”指南），根据任务 \(x_t\) 的文本描述匹配最合适分支；若无强匹配，回退到`main`分支。

#### (3) 人在回路（HITL）通道
- 处理历史中完全缺失的信号（如API密钥、独家数据源），通过两个钩子触发：
  - **任务板引导**：人类审阅/编辑分析阶段生成的任务板，添加新数据源方向或优先级。
  - **研究阶段交互协助**：当研究智能体遇到认证墙等屏障时，实时索要凭据或人类指导。
- 输入限定为源引导/凭据，不泄露答案或直接选择分支。

## 6. 主要结论与发现

- **统一框架有效**：Adaptive Auto-Harness在所有三个基准上均取得领先，Full System在PolyBench达到80.9%准确率和330%回报率，CTF-Dojo 50.2% Pass@1，FutureX 47.3% Pass@1，同时显著优于先前的自动装备系统。
- **损失来源可分离**：基准特性差异导致不同的瓶颈（PolyBench瓶颈为置信校正，FutureX为信源获取，CTF-Dojo为载荷处理），但均能通过强化进化器能力和任务级适应来解决。
- **持续性优于单次进化**：有状态多智能体进化（相比单智能体线性链）持续提升：PolyBench CWR从20.3→44.3，CTF-Dojo从38%→43%，FutureX从38%→44%；去除反馈或记忆会广泛退化。
- **装备专业化带来真实增益**：装备树暴露了显著的适应空间，智能体路由可部分捕获该增益（CTF-Dojo提升17.5pp，PolyBench CWR提升2.7pp），但远未穷尽可用潜力。
- **人类引导具针对性**：当缺失外部信号时，HITL在目标切片（金融与技术类问题）带来+20%的提升，而在已有信号的任务中增益为零，表明HITL应局限于特定缺口而非广泛干预。

## 7. 优点

- **理论与系统整合**：首次将开放式部署的遗憾分解为进化损失和适应损失，为后续系统设计提供清晰的理论指导。

- **轻量适应机制**：装备树配合智能体路由，将重适应推向构建时，求解时仅做分支选择，高效兼顾领域隔离。
- **实用的人机协作**：结构化HITL钩子仅针对缺失信号开口，避免人类干预削弱自动化，符合实际部署需求。

## 8. 不足与局限

- **基准覆盖有限**：仅在预测市场、CTF、事件预测三个流中评估，未在更广泛时空扩展的真实部署环境中测试。
- **损失项非直接可算**：进化损失和适应损失为分析性定义，实验通过消融和代理诊断间接论证，缺乏对神谕装备的直接估计。
- **路由未充分利用分支潜力**：部分基准上（如FutureX）路由增益为负，自适应算子与分支构造的协同仍可改进。
- **人类引导依赖预定义缺口**：HITL有效性取决于是否提前识别所需外部信号，不适用于完全不可预见的缺口。
