## 摘要
多智能体推理系统采用“生成-然后-传递”范式，迫使端到端延迟与流水线深度呈线性增长。我们提出 StreamMA，一种多智能体推理系统，在每个推理步骤生成后立即将其流式传输给下游智能体，实现相邻智能体的流水线化，从而减少延迟。令人惊讶的是，这种流水线化还提升了有效性：由于多步推理质量不均，早期步骤比后期步骤更可靠，使用这些可靠的早期步骤而非完整链可以防止易出错的后期步骤误导下游智能体。我们通过首个流式、串行和单智能体协议的闭合形式联合分析，形式化了这两个优势，推导出有效性排序、加速上限和成本比率。在涵盖数学、科学和代码的八个推理基准测试、两个前沿大型语言模型（Claude Opus 4.6 和 GPT-5.4）以及三种拓扑结构（链式、树状、图状）上，StreamMA 均优于两个基线（平均提升 7.3 个百分点，在 HMMT 2026 上最大提升 22.4 个百分点；使用 Claude Opus 4.6-high）。除了这些贡献，我们还发现了一条“步骤级缩放定律”：增加每个智能体的步骤数可以持续提升有效性和效率，这是一个新的缩放维度，与智能体数量缩放正交并可组合。

## Abstract
Multi-agent reasoning systems adopt a "generate-then-transfer" paradigm that forces end-to-end latency to scale linearly with pipeline depth. We introduce StreamMA, a multi-agent reasoning system that streams each reasoning step to downstream agents as soon as it is generated, pipelining adjacent agents and thus reducing latency. Surprisingly, this pipelining also improves effectiveness: because multi-step reasoning quality is non-uniform and early steps are more reliable than later ones, working with these reliable early steps instead of the full chain prevents error-prone late steps from misleading downstream agents. We formalize both advantages with the first closed-form joint analysis of stream, serial, and single protocols, deriving the effectiveness ordering, speedup upper bound, and cost ratio. Across eight reasoning benchmarks spanning mathematics, science, and code, two frontier LLMs (Claude Opus 4.6 and GPT-5.4), and three topologies (Chain, Tree, Graph), StreamMA outperforms both baselines (avg. +7.3 pp, max +22.4 pp on HMMT 2026; Claude Opus 4.6-high). Beyond these contributions, we discover a "step-level scaling law": increasing per-agent steps consistently improves both effectiveness and efficiency, a new scaling dimension orthogonal to and composable with agent-count scaling.

### 1. 研究动机与核心问题
- **背景**：主流多智能体推理系统采用“生成‑然后‑传递”的串行协议，上游智能体必须完整生成回复后下游才能开始，导致端到端延迟随流水线深度线性增长。
- **问题**：这种“等待至完整”的设计忽略了一个关键事实——多步推理中不同步骤的质量并不均匀，早期步骤通常更可靠，后期步骤容易出错。强迫下游智能体读取完整回复会使错误累积，反而损害决策质量。
- **意图**：论文尝试将“推理步骤”而非“完整回复”作为智能体间的传输单元，通过流式通信实现流水线并行，在降低延迟的同时利用可靠早期步骤提升有效性。

### 2. 方法论：流式通信协议与理论分析
- **核心思想**：每个智能体基于 LLM 进行流式推理，每生成一个推理步骤立即将其推送给所有下游智能体，下游智能体无需等待上游完成即可开始处理，从而形成智能体间的流水线。
- **算法流程**（基于 DAG 拓扑）：
  - 所有源节点接收原始问题 Q。
  - 每个智能体并行执行：从队列获取消息（上游步骤）并追加到上下文；使用流式调用生成步骤，每生成一步就推送给所有后继节点，同时将步骤附加到自身上下文以便后续 KV‑缓存复用。
  - 无全局同步栅栏，多前驱节点即时处理到达的步骤。
- **有效性形式化**（定理1）：
  - 设上游步骤 j 的正确概率为 \(p_j\)，有益/有害增益为 \(\delta, \varepsilon\)，临界阈值 \(p^* = \varepsilon/(\delta+\varepsilon)\)。
  - 定义三种加权均值：均匀均值 \(\bar{p}\)、头部加权 \(p_{\text{head}}\)、尾部加权 \(p_{\text{tail}}\)。
  - 根据这三者与 \(p^*\) 的大小关系，将 Stream、Serial、Single 三种模式的有效性排序划分为六种情况：
    - **Stream 优势**：\(p_{\text{head}} > p^*\) 且 \(p_{\text{tail}} < p^*\)（典型错误累积模式）。
    - **Serial 优势**：\(\bar{p} > p^*\) 且 \(p_{\text{tail}} > p^*\)（全程可靠或后期修正）。
    - **Single 优势**：\(p_{\text{head}} < p^*\) 且 \(\bar{p} < p^*\)（全程有害或前期严重错误）。
  - 这使得协议选择具备闭式判决条件。
- **效率上界**（定理2）：加速比上界为  
  \[
  \text{Speedup} = \frac{A[(S + r_{\text{po}})r_{\text{vdp}} + S]}{(S + A - 1)(1 + \alpha r_{\text{vdp}} + \beta r_{\text{vdc}})}
  \]
  当预填充和缓存读取远快于解码时，简化为 \(AS/(S + A - 1)\)。
- **成本比**（定理3）：  
  \[
  \frac{\text{Cost}_{\text{stream}}}{\text{Cost}_{\text{serial}}} = \rho \cdot \frac{r_{\text{cpd}}(\alpha + r_{\text{ccp}}\beta) + 1}{r_{\text{cpd}}(1 + r_{\text{po}}/S) + 1}
  \]
  在解码价格主导且 KV‑缓存命中率高的情况下，成本可低于串行协议。

### 6. 主要结论与发现
- **有效性**：StreamMA 在所有 8 个基准和 3 种拓扑上均优于 Serial，Claude 模型平均提升 +7.3 pp，最大 +22.4 pp（HMMT 2026，链式）。GPT‑5.4 同样呈正增益但不显著区域受天花板效应限制。
- **加速与成本**：流水线使延迟接近理论加速上界 \(AS/(S+A-1)\)；在 KV‑缓存命中率较高时，Stream 的 API 成本可低于 Serial。
- **步骤级缩放定律**：固定智能体数量 A，增加每智能体步骤 S 可同时提升准确率和加速比，与智能体数量缩放互补。在 A=64 时，S 从 auto 提升至 64 带来额外 +5.3 pp 并实现 26.9× 的墙钟加速。
- **协议选择理论**：定理1给出了闭式的有效性排序，揭示了“何时 Stream 优于 Serial”取决于头部可靠性，而非直觉上的“信息越多越好”。
