## 摘要
语言模型对现实世界问题的规划通常涉及世界约束和用户约束，这些约束可能不会预先完全指定，而是通过交互逐步揭示。然而，现有基准测试对此类逐步揭示的双重约束下的自适应规划仍探索不足。为填补这一空白，我们提出 AdaPlanBench，一个动态交互式基准，用于评估大型语言模型（LLM）智能体能否在逐步揭示的世界和用户约束下进行自适应规划与重新规划。AdaPlanBench 基于 307 项家庭任务构建，并配备可扩展的约束构建流程，为每项任务增添双重约束。运行时，智能体通过多轮协议与环境交互，隐藏约束仅在智能体提出违反它们的计划时才会被揭示，从而要求其在累积反馈下迭代修订计划。这使得规划极具挑战，因为智能体必须从反馈中推断并跟踪约束，同时进行有效重规划。在十个主流 LLM 上的实验表明，双重约束下的自适应规划依然困难，最佳模型准确率仅为 67.75%。我们进一步观察到，随着约束的累积性能会下降，其中用户约束构成尤为显著的挑战，失败往往源于较弱的物理基础与效率降低。这些结果将 AdaPlanBench 确立为双重约束交互式规划的测试平台，并凸显了 LLM 智能体在动态揭示的约束下实现可靠适应的挑战。

## Abstract
Planning for real-world problems by language models often involves both world and user constraints, which may not be fully specified upfront and are progressively disclosed through interaction. However, existing benchmarks still underexplore adaptive planning under such progressively revealed dual constraints. To address this gap, we introduce AdaPlanBench, a dynamic interactive benchmark for evaluating whether Large Language Model (LLM) agents can adaptively plan and re-plan under progressively revealed world and user constraints. AdaPlanBench is built on 307 household tasks, with a scalable constraint construction pipeline that augments each task with dual constraints. At runtime, agents interact with the environment in a multi-turn protocol where hidden constraints are revealed only when the agent proposes a plan that violates them, requiring iterative plan revision under accumulating feedback. This makes planning challenging, as agents must infer and track constraints from feedback while re-planning effectively. Experiments on ten leading LLMs show that adaptive planning under dual constraints remains challenging, with the best model reaching only 67.75% accuracy. We further observe that performance degrades as more constraints accumulate, with user constraints posing a particularly large challenge and failures often stemming from weaker physical grounding and reduced effectiveness. These results establish AdaPlanBench as a testbed for dual-constrained interactive planning and highlight the challenge of reliable adaptation to dynamically revealed constraints in LLM agents.

### **2. 论文提出的方法论**

论文提出了一个名为 **AdaPlanBench** 的动态交互式基准，其设计分为两部分：静态数据集构建和动态运行时协议。

*   **核心思想**：通过模拟一个“用户”和一个“世界环境”，让智能体在一个具有大量隐藏约束的开放式任务中进行多轮交互。智能体只有在其提出的计划违反了某项隐藏约束时，才会收到相应的约束反馈，并被要求重新制定计划。
*   **数据集构建（约束生成管道）**：
    *   **基础任务**：选取并清洗了 **MacGyver** 数据集中 **307** 个符合要求的家庭任务（如：干燥弄湿的排球）。
    *   **约束生成**：采用一个包含多种 LLM 组件的多智能体框架，进行最多 3 轮的迭代式约束采样。
        1.  **计划采样**：使用多个规划器（GPT-4.1、DeepSeek-V3.2 等）在空约束或累积约束下，为同一个任务生成不同的候选计划。
        2.  **约束提取**：从生成的计划中提取所用的工具。**世界约束**直接将这些工具标记为“不可用”（如，家中没有毛巾）；**用户约束**则从工具的使用属性中推断用户偏好，并将其作为限制（如，不喜欢用电风扇，怕它太慢）。
        3.  **约束合并与验证**：将不同规划器、不同轮次产生的约束进行去重、合并且最终由检查器验证，确保约束集内部没有矛盾且任务依然有解。
    *   **难度分级**：根据约束的数量，为每个任务生成了三个难度等级的环境配置：`E_low`、`E_mid` 和 `E_high`。
*   **运行时交互协议**：
    *   **多轮交互**：智能体在每一轮提出一个计划，由 LLM 评判器分别检查其是否违反隐藏的世界约束和用户约束。
    *   **约束逐级揭示**：违反的约束会通过一个“用户模拟器”转述为自然语言反馈，告知智能体。揭示策略遵循**硬约束优先**原则：优先揭示世界约束，在所有世界约束都被满足后，再揭示用户约束。
    *   **终止条件**：当计划满足所有约束并通过质量评估时，任务成功；若连续两轮未违反任何新约束（陷入停滞状态）或达到最大轮次，则任务失败。

### **3. 实验设计**

*   **基准与场景**：实验就是基于论文提出的 **AdaPlanBench** 基准，场景为 **307 项家庭任务**。主要实验设置是在难度级别为 **`E_mid`** 的环境下进行的。
*   **评估对象（LLM 模型）**：
    *   **闭源模型**：GPT-5 系列（GPT-5, GPT-5-Mini, GPT-5-Nano），DeepSeek-v4-Flash, Gemini 系列（Gemini-3.1-Pro, Gemini-3-Flash）。
    *   **开源模型**：Qwen3 系列（8B, 14B, 32B），Llama-3.3-70B-Instruct。
    *   总共**评估了10个主流LLM**。
*   **评估指标**：
    *   **准确率**：最终计划同时满足所有约束并通过质量评估的百分比。
    *   **有效计划率**：最终计划满足所有约束（不论质量评估）的百分比。
    *   **平均轮次**：完成任务的平均交互轮数。
    *   **平均重复违反次数（世界/用户）**：平均每个任务中，智能体违反已揭示约束的次数。
    *   **平均触发约束数（世界/用户）**：每个任务中，智能体触发的不同约束数量与交互轮数的比值。
    *   **规划质量评分**：从工具使用可行性、物理合理性、有效性、安全性等8个维度对计划进行1-5分的评分。

### **6. 论文的主要结论与发现**

1.  **双重动态约束是重大挑战**：所有模型表现均不理想，最强模型 **GPT-5 的准确率也仅为 67.75%**，多数开源模型低于 30%。
2.  **用户约束是主要难点**：通过消融实验发现，用户约束导致的性能下降远大于世界约束。单一用户偏好可能排除大量可行的动作空间。
3.  **高有效计划率≠高成功率**：一些模型（如 Gemini-3.1-Pro）能保持较高的计划有效性（避免重复违规），但最终准确率却不高，说明它们难以提出正确有效的最终方案。
4.  **成功的关键在于主动探索**：性能更好的模型（如 GPT-5）倾向于触发更多种类的约束，表明它们通过多样化尝试来探索约束边界，从而更快找到可行方案。
5.  **简单修复收效甚微**：显式地提供约束历史或基于评分的反馈，只能略微提升计划的合规性，但对最终任务成功率帮助不大，甚至会因“近期偏差”而破坏计划稳定性。
6.  **失败模式**：在累积约束下，模型的失败主要表现为**任务有效性差**和**物理合理性弱**。

论文在第七章坦诚地指出了以下局限：

*   **领域覆盖有限**：目前仅聚焦于**家庭任务**。虽然这是研究双重约束的天然场景，但旅行、办公等其他领域的特性未被覆盖。
*   **依赖 LLM 评估的潜在偏差**：尽管有人工验证，但使用 LLM 作为评判器来检查约束和评分，仍可能存在模型特有的偏好或系统性偏差。
