## 摘要
移动代理越来越多地被期望根据屏幕截图和语言目标来操作日常应用，其中可靠的控制需要对屏幕可供性、多步导航和未来状态变化进行推理。然而，许多代理将这种计算外化为冗长的文本思维链，这减慢了交互速度，增加了监督成本，并使部署复杂化。我们提出了MIRAGE，一个从可见的文本推理轨迹中学习连续潜在推理表示的框架。MIRAGE将显式推理转化为紧凑的隐藏状态，使代理能够在内部进行推理，而无需解码冗长的推理过程。它还包含一个生成式世界模型目标：潜在推理向量与未来的屏幕截图对齐，鼓励代理在行动前预测即将出现的界面状态。这将隐藏计算转变为既是一个压缩的思维表示，也是一个前瞻性的环境动态模型。在推理时，MIRAGE在连续潜在空间中进行推理，减少了令牌生成，同时提高了执行效率。在AndroidWorld上，MIRAGE在4B消融实验中与显式思维链监督微调的性能相当，且解码令牌预算降低了3-5倍，并将可比的指令调优基线提高了10.2个百分点；在AndroidControl上，它在生成令牌减少超过75%的同时改善了动作定位。

## Abstract
Mobile agents are increasingly expected to operate everyday applications from screenshots and language goals, where reliable control requires reasoning over screen affordances, multi-step navigation, and future state changes. However, many agents externalize this computation as long textual chains of thought, which slows interaction, increases supervision cost, and complicates deployment. We introduce MIRAGE, a framework that learns continuous latent reasoning representations from visible textual reasoning traces. MIRAGE transfers explicit reasoning into compact hidden states, enabling the agent to reason internally without decoding long rationales. It also incorporates a generative world-model objective: latent reasoning vectors are aligned with future screenshots, encouraging the agent to anticipate upcoming interface states before acting. This turns hidden computation into both a compressed thought representation and a forward-looking model of environment dynamics. At inference time, MIRAGE reasons in continuous latent space, reducing token generation while improving execution efficiency. On AndroidWorld, MIRAGE matches explicit chain-of-thought supervised fine-tuning in the 4B ablation with a 3-5x lower decoded-token budget and improves a comparable instruction-tuned baseline by 10.2 points; on AndroidControl, it improves action grounding while generating over 75% fewer tokens.

## 2. 方法论：核心思想、关键技术细节与算法流程
MIRAGE 的核心思想是**将显式推理“内化”为连续潜在空间中的隐式思维链**，并利用**世界模型对齐**来提升潜在状态的预测能力。

### 2.1 从显式思维到隐式思维的建模
- 传统显式 CoT 分解为 `p(τ, a|o) = ∏ p(τ_i | τ_<i, o) · ∏ p(a_j | τ, a_<j, o)`，其中 τ 是文本推理序列。
- MIRAGE 引入 N 个连续潜在变量 `z_t = (z_{t,1}, ..., z_{t,N})`，将移动控制建模为 `p(a|o) = ∫ p(a|z, o) q(z|o) dz`。这些潜在变量不通过词汇表解码，而是隐含在解码器隐藏状态中。

### 2.2 两阶段训练流程
- **第一阶段：显式 CoT 预热**  
  使用结构化思维格式训练，包含 `<THOUGHT> [observation] [rationale] [predict]` 和 `<ACTION_DESC>`、`<ACTION>`。模型通过标准下一 token 交叉熵学习动作格式和推理模式。
- **第二阶段：潜在 CoT 与世界模型训练**  
  将整个 `<THOUGHT>` 块替换为 N 个潜在 token，插入到序列中。这一阶段包含两个关键创新：

#### 2.2.1 APLR（Approximate Parallel Latent Refinement，近似并行潜在细化）
- 传统潜在 CoT（如 Coconut）要求逐个串行更新所有潜在槽，前向传播次数多。
- APLR 利用因果注意力的严格下三角特性，采用 **雅可比风格并行迭代**：在每一轮中，同时利用上一轮所有旧值计算所有潜在槽的新值，经过 K 轮（默认 K=3）后：
  - 前 K 个潜在状态 **精确等于** 串行解；尾部槽存在结构化残差，其误差可通过严格下三角 Jacobian 描述为 `δ^{(K)} ≈ A^K δ^{(0)}`。
- 训练时仅对最终轮次的输出计算动作交叉熵损失 `L_ce`。

#### 2.2.2 Q-Former 世界模型下一帧对齐
- 针对尾部潜在状态因 APLR 近似带来的环境预测误差，引入世界模型头：从最终潜在隐藏状态 `C_t` 通过轻量级 Q-Former 预测下一帧视觉特征 `ŷ`。
- 目标帧特征来自 frozen 视觉塔提取的下一张截图 patch 特征 `V_{t+1}`（使用 stop-gradient）。
- Q-Former 使用可学习的行/列位置查询，与 `C_t` 进行交叉注意力，投影输出作为预测特征，计算 cosine 距离损失 `L_wm`。
- 联合损失：`L = λ L_ce + (1-λ) L_wm`，λ 默认 0.8。世界模型头仅在训练时使用，推理时丢弃。

### 2.3 推理时
- 仅插入潜在 token，经 APLR 细化后直接解码动作 token，不产出任何文本推理。这使得输出 token 大幅减少，延迟显著降低。
