## 摘要
大语言模型进化为工具驱动型代理，带来了与真实世界执行相关的、而非简单文本生成的新一类安全挑战。现有的对齐方法通常依赖粗略的拒绝信号或静态监督，难以在多种代理风险下平衡安全与有用的工具执行。我们提出RUBAS，一个基于量规的代理安全强化学习框架。RUBAS将代理行为分解为四个维度：工具使用安全、参数安全、响应安全和有用性。这些结构化的量规为完整的代理轨迹提供了细粒度且可解释的奖励，使得强化学习能够优化安全的工具使用，同时保持任务完成度。在多个代理安全基准和模型上的大量实验表明，RUBAS相较于标准对齐基线提升了安全性，减少了工具锚定的幻觉，并保持了有竞争力的效用。我们的结果说明，多维度量规奖励为在安全关键的工具使用场景中对齐大语言模型代理提供了有效的训练信号。

## Abstract
The evolution of LLMs into tool-enabled agents creates a new class of safety challenges associated with real-world execution rather than simple text generation. Existing alignment methods often rely on coarse refusal signals or static supervision, making it difficult to balance safety with useful tool execution across diverse agentic risks. We introduce RUBAS, a rubric-based reinforcement learning framework for agent safety. RUBAS decomposes agent behavior into four dimensions: tool-use safety, argument safety, response safety, and helpfulness. These structured rubrics provide fine-grained and interpretable rewards over complete agent trajectories, enabling reinforcement learning to optimize safe tool use while preserving task completion. Extensive experiments across multiple agent safety benchmarks and models show that RUBAS improves safety over standard alignment baselines, reduces tool-grounded hallucinations, and maintains competitive utility. Our results suggest that multi-dimensional rubric rewards provide an effective training signal for aligning LLM agents in safety-critical tool-use settings.

### 2. 论文提出的方法论

*   **核心思想**：利用“评估比生成更容易”的不对称性，将复杂的代理安全需求分解为可验证的量规清单，并将其转化为密集的标量奖励，以驱动强化学习优化。
*   **关键技术细节：四维量规系统 (Rubric System)**
    *   **维度设计**：代理行为被分解为四个可评估的维度 `R = {r_tool, r_arg, r_res, r_help}`：
        *   **工具使用安全 (Tool-use Safety)**：是否调用了未授权工具、执行了不安全的动作序列。
        *   **参数安全 (Argument Safety)**：传递给工具的输入是否安全、非恶意。
        *   **响应安全 (Response Safety)**：对用户的最终响应是否安全，尤其是在工具返回错误或不安全信息时。
        *   **有用性 (Helpfulness)**：是否在合法、安全的前提下完成了用户的目标。
    *   **量规设计**：每个维度下包含多个二值化、可验证的准则，例如"是否调用了必需的工具"、"参数是否匹配禁止的正则表达式"等。这些准则会自动生成，并针对每个任务实例进行实例化。
    *   **奖励框架**：
        *   每个维度的得分是其下属准则得分的加权平均：`r_i(τ|x) = norm( Σ(α_ij * c_ij(τ|x)) / Σα_ij )`。最终奖励 `R_rubric` 是各维度得分的加权和。
        *   **可验证性**：除“拒绝判断”准则由外部裁判模型评估外，其余所有准则均设计为通过程序化检查的确定性函数，确保了奖励信号的可靠性和可解释性。
        *   加入了**完整性奖励**（对达到停止条件的轨迹）和**推理奖励**（惩罚省略推理过程的轨迹），以辅助稳定训练。
*   **训练方法**：采用**组相对策略优化 (GRPO)** 算法。对于每个提示，策略模型会采样多条工具调用轨迹，使用实例特定的量规计算标量奖励，然后利用组内相对优势对模型进行在线策略更新。

    *   **评估基准**：

*   **安全与效用的平衡**：RUBAS在所有非退化方法中实现了最低的平均风险评分，并且没有牺牲工具调用能力，在安全性和有用性之间取得了最佳平衡。

### 7. 优点

*   **方法论创新**：首次将基于量规的细粒度、可解释强化学习奖励框架应用于工具型LLM代理安全对齐这一复杂领域。
*   **全面的安全视角**：通过四个维度的量规设计，覆盖了用户指令、工具执行和外部环境反馈等多元风险源头，超越了仅关注最终响应的传统方法。

### 8. 不足与局限

*   **数据依赖**：训练数据主要依赖强大的LLM合成。虽然是可扩展的，但可能与真实部署场景存在分布偏差，泛化到真实世界中模糊多变的用户意图和领域特定策略的能力有待进一步验证。

*   **安全方案的局限性**：作者明确指出，RUBAS并非完整的安全解决方案，不能保证抵御所有未知攻击，实际部署中需结合运行时监控、人工确认等多重安全措施。
