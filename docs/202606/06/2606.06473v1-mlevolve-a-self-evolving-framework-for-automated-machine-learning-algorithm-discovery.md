## 摘要
大型语言模型（LLM）智能体越来越多地被应用于长期任务，如科学发现和机器学习工程（MLE），其中持续的自我进化成为关键能力。然而，现有的MLE智能体存在分支间信息隔离、无记忆搜索和缺乏分层控制的问题，这些共同阻碍了长期优化。我们提出了MLEvolve，一个基于LLM的自我进化多智能体框架，用于端到端的机器学习算法发现。通过将树搜索扩展为渐进式MCGS，MLEvolve通过基于图的参考边实现跨分支信息流，并通过熵启发的渐进调度将搜索从广泛探索逐步转向聚焦利用。为了使智能体能够通过积累的经验进化，我们引入了回顾性记忆，它结合了冷启动领域知识库和动态全局记忆，用于任务特定的经验检索和重用。为了实现稳定的长期迭代，我们进一步通过自适应编码模式将策略规划与代码生成解耦。在MLE-Bench上的评估显示，MLEvolve在12小时预算（标准运行时间的一半）下，在平均奖牌率和有效提交率等多个维度上达到了最先进的性能。此外，MLEvolve在数学算法优化任务上也优于包括AlphaEvolve在内的专门算法发现方法，展示了强大的跨领域泛化能力。我们的代码可在https://github.com/InternScience/MLEvolve获取。

## Abstract
Large language model (LLM) agents are increasingly applied to long-horizon tasks such as scientific discovery and machine learning engineering (MLE), where sustained self-evolution becomes a key capability. However, existing MLE agents suffer from inter-branch information isolation, memoryless search, and lack of hierarchical control, which together hinder long-horizon optimization. We present MLEvolve, an LLM-based self-evolving multi-agent framework for end-to-end machine learning algorithm discovery. By extending tree search to Progressive MCGS, MLEvolve enables cross-branch information flow through graph-based reference edges and gradually shifts the search from broad exploration to focused exploitation with an entropy-inspired progressive schedule. To allow the agent to evolve with accumulated experience, we introduce Retrospective Memory, which combines a cold-start domain knowledge base with a dynamic global memory for task-specific experience retrieval and reuse. For stable long-horizon iteration, we further decouple strategic planning from code generation with adaptive coding modes. Evaluation on MLE-Bench shows that MLEvolve achieves state-of-the-art performance across multiple dimensions including average medal rate and valid submission rate under a 12-hour budget (half the standard runtime). Moreover, MLEvolve also outperforms specialized algorithm discovery methods including AlphaEvolve on mathematical algorithm optimization tasks, demonstrating strong cross-domain generalization. Our code is available at https://github.com/InternScience/MLEvolve.

### 2. 论文提出的方法论

- **整体框架**：MLEvolve 是一个基于 LLM 的自我进化多智能体系统，其核心由三大组件构成，以解决上述瓶颈：

- **核心组件1：渐进式蒙特卡洛图搜索（Progressive MCGS）**
    - **核心思想**：将标准树搜索 MCTS 扩展为图搜索，并引入基于熵的自适应探索策略。
    - **图结构搜索**：在传统的主干边（`Primary edges`）基础上引入**参考边（`Reference edges`）**。参考边连接不同分支或非相邻层级的节点，实现跨分支信息流动和组合式传输，但不参与反向传播的信用分配。
    - **四种图扩展操作**：
        1.  **基础扩展**：仅从父节点生成。
        2.  **分支内进化**：引用同分支历史节点，反思成败经验，避免重复错误。
        3.  **跨分支参考**：当分支停滞时，引用全局高评分节点，汲取外部成功经验。
        4.  **多分支聚合**：全局停滞时，融合多个分支的顶级轨迹，开辟全新搜索方向。
    - **渐进式探索调度**：设计一个基于熵的概率软开关，随时间从主要以 UCT 准则探索（高熵），平滑过渡到以精英引导（Elite-Guided）利用（低熵）为主，使搜索后期能集中资源在最有潜力的分支上。

- **核心组件2：回顾性记忆（Retrospective Memory）**
    - **核心思想**：为智能体配备可自动累积和检索的经验系统，实现从“无记忆搜索”到“经验驱动决策”的转变。
    - **构成与机制**：
        - **冷启动领域知识库**：一个静态的、按任务类型（图像、NLP、表格等）组织的轻量级模型先验库，用于初始化解决方案，减少冷启动错误。
        - **动态全局记忆**：在搜索过程中自动累积极的每次有效执行的结构化记录（计划、结果、分析）。采用结合关键词匹配和 FAISS 语义搜索的**混合检索**，并通过**阶段感知过滤**，在规划和调试阶段分别检索相关经验。

- **核心组件3：分层规划与自适应代码生成**
    - **核心思想**：将“为何修改/修改什么”（战略）与“如何修改”（实现）解耦，并根据搜索状态灵活选择代码生成粒度。
    - **规划器-编码器解耦**：规划器决定修改模块和原因，编码器专注于代码实现。
    - **自适应编码模式**：
        1.  **基础模式**：全新重写，用于初始起草。
        2.  **分步模式**：按规划器的模块级指令逐步生成，应对复杂管线。
        3.  **差异模式**：在现有代码上进行精准的局部补丁编辑，实现稳定、可控的迭代优化。
