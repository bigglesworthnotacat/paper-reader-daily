## 摘要
AI智能体依赖一套由技能、工具和工作流组成的装备来解决复杂问题。持续改进这一装备对于适应新任务至关重要。然而，现有的优化方法通常需要真实的验证集，但在实际部署环境中，此类标注数据难以获取。为了解决这一问题，我们提出了回顾式装备优化（RHO），这是一种自监督方法，仅利用过去的轨迹来优化智能体装备。具体而言，RHO从过往轨迹中筛选出具有多样性的核心挑战任务子集，并并行地重新求解这些任务。智能体通过自我验证与自我一致性来分析这些回放结果，随后生成候选装备更新，并根据自身的成对自我偏好选出最有效的更新。我们在软件工程、技术工作和知识工作这三个不同领域对RHO进行了评估。值得注意的是，在无需任何外部评分的情况下，单轮优化便将SWE-Bench Pro上的通过率从59%提升至78%。此外，我们的分析表明，RHO能有效针对先前的失败模式。因此，优化后的装备改变了智能体的行为模式，并在长时间会话中保持了更高的准确性。

## Abstract
AI agents rely on a harness of skills, tools, and workflows to solve complex problems. Continually improving this harness is essential for adapting to new tasks. However, existing optimization methods typically require ground-truth validation sets, yet such labeled data is difficult to acquire in practical deployment settings. To address this problem, we introduce Retrospective Harness Optimization (RHO), a self-supervised method that optimizes the agent harness using only past trajectories. Specifically, RHO selects a diverse coreset of challenging tasks from past trajectories and re-solves them in parallel. The agent analyzes these rollouts using self-validation and self-consistency, then generates candidate harness updates and selects the most effective one by its own pairwise self-preference. We evaluate RHO across three diverse domains, spanning software engineering, technical work, and knowledge work. Notably, a single optimization round improves the pass rate on SWE-Bench Pro from 59% to 78% without any external grading. Furthermore, our analysis demonstrates that RHO effectively targets prior failure modes. As a result, the optimized harness alters the agent's behavior patterns and sustains higher accuracy during long-horizon sessions.

## 2. 方法论：回顾式装备优化（RHO）
RHO 是一种完全自监督的方法，以单次回顾的方式优化装备，不依赖任何外部评分或真实标签。

### 2.1 核心思想
将装备优化重构为**对过往轨迹的回顾分析过程**：智能体重新求解过去的部分任务，对比不同执行轨道的差异，用“自我偏好”作为内部信号来指导装备更新。

### 2.2 关键技术流程与算法
算法分为三个阶段（参见图2和算法1）：

- **① Coreset Selection（核心集选择）**
  - 目标：从历史轨迹池 $D$ 中选出一个规模为 $k$ 的、兼具**挑战性**与**多样性**的子集 $D_{\text{core}}$。
  - 实现方式：
    1. 利用语言模型判断每条轨迹 $\tau_i$ 的难度分数 $r_i$ 和抽象描述指纹，并计算指纹的嵌入向量。
    2. 基于行列式点过程（DPP）构造核矩阵 $K = \operatorname{diag}(e^{\mathbf r}) \mathbf S \operatorname{diag}(e^{\mathbf r})$，其中 $\mathbf S$ 为余弦相似度矩阵。
    3. 通过参数 $\theta$ 权衡难度与多样性，DPP 采样选出 $k$ 条轨迹作为核心集。

- **② Group Rollout（分组回放与诊断）**
  - 目标：在核心集任务上，通过并行回放产生改进信号。
  - 实现方式：
    1. 对每个核心任务 $t$，用当前装备 $h_0$ 并行执行 $G$ 次，得到轨迹组 $\{\tau_{t,g}\}$。
    2. 执行两类自偏好分析，生成结构化 JSON 格式的改进指令 $I_t$：
       - **自我验证**（self-validation）：检查单条轨迹内部是否正确完成任务，标记错误工具调用、错误假设、过早停止等。
       - **自我一致性**（self-consistency）：跨轨迹检查行为分歧（如不同计划、工具序列差异），识别高不确定性的区域，生成提高一致性的建议。
    3. 合并所有核心任务的指令形成总指令集 $I$。

- **③ Best-of-N Harness Proposal（优选装备生成）**
  - 目标：基于诊断指令生成多个候选装备，并用自我偏好选出最优者。
  - 实现方式：
    1. 并行调用优化算子 $N$ 次，输入 $h_0$ 和 $I$，生成 $N$ 个候选装备 $\{h_j\}$。
    2. 对每个 $h_j$，在核心集任务上重新执行一次，得到新轨迹 $\tau_{t}^{(j)}$。
    3. 用成对排序函数 `rank`，将每个新轨迹与原始装备的基线轨迹 $\tau_{t}^{(0)}$ 进行自偏好比较，计算每个候选装备的平均相对优势分数 $S_j = \frac{1}{|D_{\text{core}}|} \sum_{t} \text{rank}(t, \tau_{t}^{(j)}, \tau_{t}^{(0)})$。
    4. 选择 $S_j > 0$ 且得分最高的装备 $h_{j^*}$ 作为最终更新装备；如果所有候选得分均 $\le 0$，则保留原装备。

### 2.3 关键公式
- 核心集 DPP 核：$K = \operatorname{diag}(e^{\mathbf r}) \mathbf S \operatorname{diag}(e^{\mathbf r})$，其中难度缩放因子 $\tilde r_i$ 经过 $\theta$ 调控。
- 候选优势分：$S_j = \frac{1}{k} \sum_{t \in D_{\text{core}}} \text{rank}(t, \tau_{t}^{(j)}, \tau_{t}^{(0)})$。
- 优化目标形式化：$h^\star = \arg\max_{h'} \mathbb{E}_{t,\tau \sim \text{solve}(h',t)}[U(t,\tau)]$，但用自偏好替代隐式的 $U$。

## 6. 主要结论与发现
- RHO 在不使用任何外部标注的情况下，**单轮优化显著提升装备**。SWE-Bench Pro 通过率从 59% 提高至 78%（绝对提升 19%），Terminal-Bench 2 和 GAIA-2 亦稳定提升。
- 其优势来源于**对全装备（工具、技能、指令）的编辑能力**，而基线方法多仅更新记忆或文本技能。
- 优化后的装备新增了**针对历史失败模式的工具和技能**（如非标准工具链检测、缓存清理脚本、多边形验证工具等）。
- RHO 主要提升**长时程任务的成功率**，并重塑了智能体的**行为模式**（如在 SWE-Bench 中更频繁进行验证，在 Terminal-Bench 中减少无效编辑，增加执行）。
- 消融实验证实，**兼顾难度与多样性的核心集选择** 以及 **self-validation + self-consistency 双重诊断信号** 是性能增益的关键。

## 7. 优点
- **完全自监督、无需验证集**：直接使用历史轨迹，解决了部署中标注缺失的根本局限。
- **操作全装备**：不受限于记忆或提示词，可生成新工具和可执行脚本，修改幅度与灵活性远超以往“经验自改进”方法。
- **单次回顾，计算可接受**：相比需要多轮验证迭代的方法，RHO 在同等甚至更少预算下获得更优或可比的性能。
- **跨域通用性强**：在代码修复、命令行操作、动态环境知识任务上均有效，验证了方法的普适性。
- **可解释的更新**：附录完整列出了优化出的技能和工具，展示了具体、可审计的改进内容。
- **全面的消融与行为分析**：从任务选择信号、诊断信号到行为分布变化，深入揭示了优化机理。

## 8. 不足与局限
- **环境依赖**：group rollout 需要任务环境可重置并允许多次尝试，不适用于一次性的、不可逆的物理环境任务。
- **装备表面限制**：方法假设智能体的能力很大程度上承载于可编辑的“装备”（提示、脚本、工具文件），若装备表面抽象不足，优化效果可能受限。
- **自偏好可靠性风险**：优化完全基于模型的自我偏好判断，可能放大模型自身的偏好偏见、错误推理或不符合安全规范的行为，需部署中增加人工审核与安全回退机制。
- **历史轨迹质量敏感**：核心集选择与诊断信息均来自历史轨迹，若原轨迹集缺乏典型失败或过于简单，改进效果将打折扣。
- **计算开销**：相比于简单的记忆积累方法，RHO 需要多次并行回放和诊断，计算成本较高（虽与某些基线相当，但仍高于轻量级方法）。
- **尚未在非 LLM 智能体或更广泛领域验证**：当前实验限定在使用相同语言模型背骨的 Codex 代理上。
