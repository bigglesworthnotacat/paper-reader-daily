## 摘要
大型语言模型在通用任务上表现出色，但在训练标签成本高昂的专业化、高风险领域中，性能不如较小的监督模型。我们通过 EvoPool 来解决这一问题，这是一个受达尔文进化论启发的进化式多智能体框架。三个专业化智能体迭代提出可执行的标注器代码，一个小型验证集提供适应度信号，一个确定性门控只保留那些通过跨代可行性、多样性和边际贡献检查的标注器。池投票通过 EvoAgg 映射为软训练标签，EvoAgg 是一种文本感知聚合器，结合了语义特征与标注器投票特征。所构建的池以近乎零的单例成本运行，在10万样本上比LLM标注快4500到31000倍。在涵盖生物医学关系抽取、法律条款分类、复杂推理和密集多标签生物医学分类的8个LLM弱势专业化复杂任务中的7个上，EvoPool平均以+0.141的宏F1值超越了最强的LLM标注基线，在ChemProt上达到峰值+0.301，在PubMed上达到+0.265。代码可在以下地址获取：https://github.com/tianyi0216/EvoPool。

## Abstract
Large language models excel at general tasks but underperform smaller supervised models in specialized, high-stakes domains where training labels are costly. We address this regime with EvoPool, an evolutionary multi-agent framework inspired by Darwinian evolution. Three specialized agents iteratively propose executable annotator code, a small validation set provides a fitness signal, and a deterministic gate keeps only annotators that pass viability, diversity, and marginal-contribution checks across generations. Pool votes are mapped to soft training labels by EvoAgg, a text-aware aggregator combining semantic features with annotator-vote features. The authored pool runs at near-zero per-example cost and is 4500 to 31000x faster than LLM annotation on 100K examples. Across 7 of 8 LLM-weak specialized and complex tasks spanning biomedical relation extraction, legal-clause classification, complex reasoning, and dense multi-label biomedical classification, EvoPool beats the strongest LLM annotation baseline by an average +0.141 macro-F1, peaking at +0.301 on ChemProt and +0.265 on PubMed. Code is available at: https://github.com/tianyi0216/EvoPool

*   **核心问题**：在专业化、高风险领域（如生物医学、法律），训练一个高性能的监督模型需要大量准确标注的数据，但获取这些标签（如请临床医生或律师标注）的成本极其高昂。
*   **现有方案的不足**：
    *   **直接LLM标注**：虽然可以利用大语言模型的通用知识，但为每个训练样本进行推理，导致成本随语料库规模线性增长，且在专业化领域准确性欠佳。
    *   **一次性LLM标注器合成**（如Alchemist, DataSculpt）：利用LLM一次性生成一套标注函数（标注器池），部署成本低，但生成的标注器会继承LLM的先验偏见，忽略特定领域的薄弱类（如长尾类），导致覆盖率不足。
*   **研究动机**：探索一种新范式，将LLM的推理能力摊销到一个可进化的、可复用的标注器池中，使其能根据领域反馈自我优化，从而在极低的部署成本下，达到甚至超越昂贵的直接LLM标注或一次性合成的性能。

### **2. 方法论与技术细节**

EvoPool的核心思想是将程序化标注建模为在可执行标注器种群上的进化搜索过程。其技术流程如下：

*   **进化式多智能体循环**
    *   **框架**：借鉴达尔文进化论，通过**变异**、**选择**和**继承**三个原语来迭代优化一个标注器池。
    *   **三种角色代理**：
        *   **生成器**：在循环开始时（k=0），负责广度优先探索，覆盖所有类别，生成多样化的候选标注器。
        *   **改进器**：根据每轮迭代的验证集诊断报告，针对当前池中宏F1最弱的类别，进行有针对性的变异，提出新的标注器。
        *   **精炼器**：识别池中高精度但低召回率的标注器，通过扩展关键词库或放宽条件等方式，在保留区分信号的同时扩大其覆盖率。
    *   **继承单位**：跨代继承的唯一通道是**标注器池本身**，而不是代理的状态或经验（达尔文主义而非拉马克主义），这使得后续迭代的提示词不会因历史信息而膨胀，成本更低且效果略优。
*   **池级别的自然选择（确定性门控）**
    *   每轮迭代结束时，通过三个过滤器对新生成的候选标注器进行筛选：
        1.  **生态位冗余**：移除与池中已有成员相似度过高（Jaccard阈值0.95）的标注器。
        2.  **最低竞争力**：移除在验证集上精确度低于阈值（0.25）或触发次数过少的标注器。
        3.  **边际贡献**：通过后向消融，贪婪地移除那些删除后反而能提升池集合验证宏F1的标注器，确保每个新成员都有正向贡献。
    *   **查询选择**：使用**BatchBALD**方法，从训练集中挑选出当前标注器池最不确定且彼此信息冗余度低的一小批样本，作为改进器和精炼器下一轮推理和改进的素材。
*   **EvoAgg：文本感知聚合器**
    *   **问题**：传统的多数投票法无法处理标注器集体沉默（弃权）或少数派意见被淹没的情况，尤其在长尾类别上。
    *   **解决方案**：EvoAgg将文本的低维向量嵌入与标注器的票型向量拼接起来，组成输入特征`X = [ϕ(x); V]`，然后用逻辑回归模型在验证集上学习如何生成软标签。
    *   **作用**：文本嵌入作为标注器沉默或错误时的后备信号，并能根据每个样本的内容动态仲裁文本和投票的权重，确保全面的覆盖率并提升性能。
