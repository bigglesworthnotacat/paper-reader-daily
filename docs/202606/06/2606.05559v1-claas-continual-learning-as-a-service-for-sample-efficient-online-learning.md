## 摘要
部署的大语言模型智能体必须适应动态环境中的分布偏移。理想情况下，适应可以基于累积的智能体经验进行，并在迁移到未来任务时保留先前的能力。然而，智能体动作和环境转换在每个场景中只能采样一次，因为真实世界环境无法轻易重置。为此，我们研究了一种经验性的在线持续学习设置，其中智能体从一系列场景中学习。我们提出了持续学习即服务（CLaaS），这是一个使智能体在部署期间能够不断改进的系统，隐藏在聊天API背后。为了提高样本效率，CLaaS将推演轨迹存储在经验回放缓冲区中，以便在异步训练期间重用梯度。我们在一个对抗性任务上评估了CLaaS，证明了参数更新相较于上下文学习能带来更优的前向迁移和更低的遗忘，回放是样本效率的关键选择。

## Abstract
Deployed large language model agents must adapt to distribution shift in dynamic environments. Ideally, adaptation can be performed from accumulated agent experiences and retain prior capabilities while transferring to future tasks. However, agent actions and environmental transitions can only be sampled once per scenario, as real-world environments cannot be trivially reset. To this end, we investigate an experiential and online continual learning setting in which agents learn from a stream of scenarios. We propose continual learning as-a-service (CLaaS), a system which enables agents to improve during deployment, abstracted behind a chat API. To increase sample efficiency, CLaaS stores rollouts in an experience replay buffer for gradient reuse during asynchronous training. We evaluate CLaaS on an adversarial task, demonstrating that parametric updates lead to superior forward transfer and less forgetting than in-context learning, with replay being a critical choice for sample efficiency.

### 2. 论文提出的方法论

- **核心思想**
  - 构建一个闭环系统：从部署环境中收集单次推演轨迹（rollouts），存储到经验回放缓冲区中，然后异步地用这些轨迹对策略模型进行参数更新（LoRA微调），最后将更新后的权重热重载到推理服务器。这一过程循环进行，实现持续学习。

- **关键技术细节**
  - **系统架构**：由推理服务器（vLLM）收集推演数据，训练引擎（veRL）执行异步梯度更新，并通过热重载LoRA权重实现实时改进。
  - **问题建模**：学习过程被定义为一个在线持续学习问题，模型从N个场景`S_1:N`的顺序流中学习。在每个场景`i`的第`t`步，智能体基于当前策略`π_θk`生成响应，并获得一个由验证器给出的奖励`r_i`。场景只能执行一次。
  - **经验回放缓冲区**：核心机制是存储每条轨迹`(s_i, τ_i, r_i)`，并在设置`B_min`个样本后开始异步训练。缓冲区有最大容量`B_max`，采用先进先出（FIFO）策略驱逐旧数据。更重要的是引入“最大回放年龄”`A_max`，允许轨迹在最多`A_max`次梯度更新中被重用，显著提升了样本效率。
  - **目标函数与多算法支持**：论文评估了三种兼容单轨迹更新的在线策略算法：
    1. **PPO**：使用Actor-Critic架构，通过价值网络估计优势函数。
    2. **REINFORCE++**：使用归一化回报与KL惩罚。
    3. **SDPO（自蒸馏策略优化）**：这是论文表现最佳的方法。它维护一个EMA教师模型，在任务失败时根据反馈生成“修正后”的正确响应，然后最小化学生模型和教师模型输出logits之间的Jensen-Shannon散度。这使得模型能从二元的验证信号中获得平滑的梯度。
  - 更新都使用了基于重要性采样的裁剪目标，以修正因回放缓冲区中轨迹的策略滞后带来的偏差。

### 3. 实验设计

- **数据集与场景**
  - **基准**：使用**IH-Challenge**基准测试中的“复合”类别。该基准包含对抗性提示注入场景，要求防御者（LLM）在系统指令下遵守多种复杂规则。
  - **非稳态环境构造**：不只是一个静态测试，实验特意构造了一个**自适应攻击者**（另一个LLM），它会根据之前的攻击是否成功来调整策略（最多3轮攻击），创造出持续变化、非稳定的分布偏移。这使得静态微调或简单的ICL难以应对。
  - **数据划分**：随机采样100个场景，划分为5个连续的数据分割（splits），每个包含20个场景。模型在每个分割上进行异步学习，之后保存检查点并评估其在所有已见和未见分割上的表现。

- **对比方法**
  1. **基线模型**：未做任何适应的模型。
  2. **ICL（上下文学习）**：将历史场景的验证反馈累积到系统提示词中作为上下文，不进行参数更新。
  3. **PPO + CLaaS**
  4. **REINFORCE++ + CLaaS**
  5. **SDPO + CLaaS**（最佳方案）

- **评估指标**
  - **前向迁移（Forward Transfer）**：在训练后的检查点上，评估其对“未来”未见数据分割的性能。
  - **遗忘（Forgetting）**：衡量模型在持续学习新分割后，在“先前”已学过的数据分割上性能下降的程度。
  - **最终性能（Final）**：最后一个检查点在所有数据分割上的平均成功率。

### 6. 论文的主要结论与发现

- **参数更新显著优于上下文学习**：CLaaS框架下的参数化方法（尤其是SDPO）在对抗性任务上远胜于ICL。SDPO的最终成功率是ICL的**3倍多**（75.2% vs 24.1%），遗忘率只有ICL的**一半**（4.2% vs 8.9%）。
- **SDPO是最优算法**：SDPO表现出最佳的前向迁移、最少的遗忘和最高的最终性能，其效果比第二好的PPO高出1.5倍。SDPO还能容忍更老的离线数据（`A_max=50`），而REINFORCE++和PPO在`A_max`过大时会崩溃。
- **经验回放至关重要**：消融实验证明，适当的回放年龄（`A_max`）能显著提升样本效率和泛化能力。年龄太小则数据利用不充分，年龄太大则因策略过于滞后导致训练不稳定甚至崩溃。
