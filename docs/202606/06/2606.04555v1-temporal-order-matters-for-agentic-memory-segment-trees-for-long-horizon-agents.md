## 摘要
长周期对话智能体需要通过与不断演变的事件、任务和目标交互来服务用户。这类历史记录天生具有时间性，然而许多现有记忆系统主要按主题相似性组织信息，可能忽略事件发生的顺序。我们提出线段树记忆（SegTreeMem），一种将对话历史表示为基于话语的时间序列线段树的记忆架构。SegTreeMem 通过在线最右前沿更新规则增量插入新话语，在保留时间顺序的同时形成层次化记忆片段。在检索时，SegTreeMem 将相关性分数沿树传播，结合局部语义匹配与层次化时间上下文。在三个长周期记忆基准测试和两个大语言模型骨干上，SegTreeMem 均比平面检索、图结构记忆和树结构记忆基线提升了答案质量。额外的时间顺序排列分析表明，性能增益依赖于记忆构建过程中保持时间顺序，这支持了时间顺序是智能体记忆关键结构的观点。

## Abstract
Long-horizon conversational agents need to interact with users through evolving events, tasks, and goals. Such histories are naturally temporal, yet many existing memory systems organize information primarily by topical similarity and may ignore the order in which events occur. We introduce Segment Tree Memory, or SegTreeMem, a memory architecture that represents conversation history as a temporally ordered Segment Tree over utterances. SegTreeMem incrementally inserts new utterances through an online rightmost-frontier update rule, preserving chronological order while forming hierarchical memory segments. For retrieval, SegTreeMem propagates relevance scores through the tree to combine local semantic matching with hierarchical temporal context. Across three long-horizon memory benchmarks and two LLM backbones, SegTreeMem improves answer quality over flat retrieval, graph-structured memory, and tree-structured memory baselines. Additional temporal-order permutation analysis shows that the performance gain depends on preserving temporal order during memory construction, supporting the claim that temporal order is a key structure for agentic memory.

## 1. 论文的核心问题与整体含义

长周期对话智能体（Long‑horizon conversational agents）在多轮交互中需要持续记住并利用先前话语中携带的事件、任务与偏好。对话历史天然具有时间性：相邻话语通常围绕同一话题，随后再转移到新的上下文。然而，现有记忆系统大多通过主题聚类、语义相似性来组织记忆，往往忽视话语发生的先后次序，导致检索出的证据在语义上可能相关但在时序上错位，从而影响回答准确性。

本文的核心观点是：**时间顺序对智能体记忆至关重要**。为此，作者提出一种名为 **Segment Tree Memory（SegTreeMem）** 的记忆架构，将对话历史组织成严格按时间排序的线段树（Segment Tree），并在其上进行层次化的相关性传播检索。通过三个长周期记忆基准、两种大语言模型骨干的比较实验，以及时间顺序打乱实验，论文验证了保持时间顺序能够显著提升记忆构建与检索的性能。

## 2. 论文提出的方法论

### 2.1 核心思想
- 将对话历史 `X = (x₁,...,x_T)` 表示为**有根线段树**，每个节点对应一个连续时间段（区间）的话语片段。
- 叶子节点对应单句话语；内部节点对应若干孩子区间的不交并集，并递归保持时间连续。
- 每个节点携有 LLM 生成的摘要 `A(v)`，概括该区间的内容。

### 2.2 在线线段树构建（Online Segment Tree Construction）
- 维护一个**最右前沿** `F_t`：所有右端点为 `t` 的节点序列（从叶子到根）。
- 当新话语 `x_{t+1}` 到来时，在 `F_t` 的非叶节点中（最多 `O(log T)` 个）选择一个兼容性最高的节点 `f_{ℓ*}`。
- **兼容性模型**可选择：
  - 余弦相似度（取注释向量与话语最相似的节点）；
  - 逐点 LLM（逐个判别是否合并）；
  - 批量 LLM（一次性比较所有候选）。
- 若不存在兼容节点（NONE 情况），则新建根，将原树作为左子树，并为新叶子创建右侧内部链，保证时间有序。
- 整个更新只影响前沿及其祖先节点，`O(log T)` 节点，满足实时性要求。

### 2.3 基于树结构的检索（Structure‑aware Retrieval）
- 对每个节点 `v` 计算其与查询 `q` 的局部相关性 `r_q(v)`（使用文本嵌入的余弦相似度，归一化得初始分布 `s_q^{(0)}`）。
- 引入**传播矩阵 `W_P`**，根据传播策略 P（自顶向下或自底向上）定义节点间的结构转移概率。
- 通过多跳传播：
  `s_q^{(k)} = W_P · s_q^{(k-1)}`  
  得到 k 跳后的分布。
- 最终得分由各阶分布加权求和：
  `\tilde{s}_q = \frac{\sum_{k=0}^H \alpha^k s_q^{(k)}}{\sum_{k=0}^H \alpha^k}`  
  其中 `α` 控制远跳的贡献（衰减因子），`H` 为传播视野。
- 取最终得分最高的 `K` 个节点作为检索证据。

设计的关键在于**通过树结构将相关性从局部匹配点扩展到其时间邻近上下文（上下层或兄弟节点）**，从而检索出更完整的语义与时序一致的证据。

## 6. 论文的主要结论与发现
- **SegTreeMem 在所有基准、两种 LLM 骨干下均显著优于现有基线**（最多提升近 20%）。
- **保持时间顺序构建的线段树优于非时间相似性聚类树**，即使使用相同的检索传播设置。
- **时间顺序打乱实验**：非时间树受影响小，而 SegTreeMem 性能大幅下降，直接佐证了时间顺序结构对性能的关键贡献。
- **批量 LLM 构建效果最好**，但余弦相似度构建在成本‑质量上更具性价比（无 LLM 调用）。
- **传播策略的效益与数据集相关**：细节导向问题（LoCoMo）适合自顶向下，项目状态类问题（RealMem）适合自底向上；适度传播可提升结果，过度传播会引入噪声。
- 结构感知检索通过改变检索证据的**时间粒度**来影响答案质量，而非简单地增加上下文量。

## 7. 优点
- **原理创新**：将经典线段树数据结构引入智能体记忆，天然保证内容的时间连续性与层次性。
- **在线增量构建**：右前沿更新机制严格局部、高效，满足实时交互需求。
- **结构感知检索**：基于矩阵的分数传播同时利用局部语义与时序层次，且传播方向和强度可调，灵活性高。
- **实验扎实**：多基准、多基线、多骨干、消融设计和排列实验，结论可信。
- **解释性强**：通过节点层次分布分析和具体 QA 案例，清晰阐述了传播如何改变时间粒度及为什么数据集效应不同。

## 8. 不足与局限
- **最坏情况性能**：极端频繁主题切换会导致树高度退化和节点数二次增长，结构优势减弱（但实际对话极少出现）。
- **长程话题重复**：同一话题在时间上分散时，纯时间树无法直接链接远处相关节点，有限跳传播可能连接不上。
- **摘要丢失细节**：内部节点摘要可能压缩掉精确事实（如日期、名字），影响需要精确召回的问题。

- **应用范围有限**：仅关注文本对话，尚未拓展到多模态观察、工具调用等长周期智能体场景。
- **资源与算力信息缺失**：未提供硬件配置、总计算费用或执行时间总量，不便于精确复现成本。
