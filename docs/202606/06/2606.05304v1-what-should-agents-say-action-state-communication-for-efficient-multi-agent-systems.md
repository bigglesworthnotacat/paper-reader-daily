## 摘要
基于大型语言模型的多智能体系统通常围绕角色、管道和轮次调度进行组织，而智能体之间传递的内容往往是不加约束的自然语言。然而，这种自由形式的通信会迅速增加词元用量，消耗共享上下文窗口，最终影响系统性能和推理成本。我们分析了两种多智能体系统拓扑结构下的五种常见智能体间通信策略，发现没有一种固定策略是普遍最优的。相反，有效的智能体间消息会持续保留下游智能体所需的以动作为中心的信息。基于此，我们提出了PACT（协议化动作-状态通信与传输），它将智能体间通信视为一个公共状态更新问题，并在每个原始智能体输出进入共享历史之前，将其投射为紧凑的动作-状态记录。在不同的多智能体系统拓扑结构中，PACT始终改善了性能-成本权衡，以显著更少的词元实现了相当或更强的任务性能。这些收益还延伸到了生产级编码工具：PACT 在 OpenHands 上以每解决一项减少 10% 词元的代价提升了解决率，并且在 SWE-agent 上保持解决率不变的同时将输入词元减半。我们的代码已在 https://github.com/iNLP-Lab/PACT 公开提供。

## Abstract
Multi-agent systems (MAS) built on large language models are typically organized around roles, pipelines, and turn schedules, while the content that agents pass to one another is often left as unconstrained natural language. However, this free-form communication can rapidly inflate token usage, consume the shared context window, and ultimately affect both system performance and inference cost. We analyze five common inter-agent communication strategies across two MAS topologies, finding that no fixed strategy is universally optimal. Instead, effective inter-agent messages consistently preserve action-centered information needed by downstream agents. Building on this, we propose the PACT (Protocolized Action-state Communication and Transmission), which treats inter-agent communication as a public state-update problem and projects each raw agent output into a compact action-state record before it enters shared history. Across different MAS topologies, PACT consistently improves the performance-cost trade-off, achieving comparable or stronger task performance with substantially fewer tokens. The gains extend to production coding harnesses: PACT lifts OpenHands' resolve rate at -10% tokens-per-resolved, and is resolve-neutral on SWE-agent while halving input tokens. Our code is publicly available at https://github.com/iNLP-Lab/PACT.

### 2. 论文提出的方法论

*   **核心思想**：将智能体间的通信重新定义为**公共状态更新问题**。一个智能体发送给另一个智能体的消息，不应是其私有推理过程的摘要，而应是下游智能体继续执行任务所需的**最小化、以动作（Action）为中心的信息**。
*   **关键技术：PACT (Protocolized Action-state Communication and Transmission)**
    *   **本质**：一种**免训练、与框架无关**的通信协议，它界定了智能体“私有计算”与“公开通信”的边界。
    *   **工作原理**：
        1.  **信息过滤与投射**：在智能体的原始输出（包括思考过程、最终答案等）被追加到共享历史 *之前*，一个名为 `ΠPACT` 的发送端投射函数会对其进行处理。
        2.  **动作-状态消息格式**：`ΠPACT` 将原始输出压缩为一个结构化的公开消息记录，只包含三个接收方（下游智能体）所需的核心字段：
            *   **ACTION**： 明确陈述发送方已经完成的动作，或要求下一个智能体即将执行的动作。
            *   **STATE**： 记录支撑该动作的任务相关状态、观察到的证据或工具执行结果，使消息成为一个有根据的公开状态，而非无支撑的结论。
            *   **RESULT**： 传递动作的直接输出产物，如检索到的事实、修订后的计划等。
        3.  **公式表达**：共享历史的更新方式从 `H_{t+1} = H_t ⊕ y_t` (y_t为原始输出) 变为 `H_{t+1} = H_t ⊕ ΠPACT(y_t, o_t, r_t)`，其中 `o_t` 和 `r_t` 分别是当前智能体的观察和角色。
        *   **关键属性**：该协议仅约束进入公共频道的信息，不限制智能体自身的内部推理和规划过程（即私有计算）。

### 6. 论文的主要结论与发现

*   **无通用最优策略**：在 MAS 中，没有固定的通用最优通信策略。**有效的消息内容是“动作中心”的**。
*   **“全量”和“简洁”均非正解**：传递完整输出昂贵且冗余；通用的简洁生成或简短摘要不稳定，可能丢失任务关键信息。
*   **PACT 显著改善性能-成本前沿**：
    *   在受控 MAS 设定中，PACT 平均减少 **38.7%** 的词元消耗，同时保持或提升了任务性能。
    *   在编码智能体上，PACT 使 **OpenHands** 的解决率提升了 **3.6 个百分点**，且词元消耗降低 **10.3%**；在 **SWE-agent** 上，输入词元减少 **50.4%**，解决率几乎不变。
*   **更强的模型更适配紧凑通信**：模型规模越大，PACT 的优势越明显，能更好地利用紧凑、有根据的证据和制品。

### 7. 优点

*   **问题视角新颖**：将研究焦点从 MAS 的组织结构转向了**消息内容**这一底层且关键的设计要素。
*   **方法论巧妙**：PACT 将其定义为一个“公共状态更新”问题，通过三个结构化字段（Action, State, Result）优雅地解决了信息冗余与关键信息丢失之间的矛盾。
*   **实用性强**：PACT 是**免训练的 (training-free)**，并且被设计为 **“束带无关” (harness-agnostic)** 的协议，易于作为轻量级插件（如代理钩子）集成到现有的各种 MAS 框架中，无需改变内部架构或重新训练模型。
*   **实验扎实**：覆盖了从受控实验到生产环境的多个层次，包含多维度（任务、成本、模型规模）的全面比较，验证了方法的有效性和泛化能力。

### 8. 不足与局限

*   **场景覆盖局限**：论文主要聚焦于**历史密集型**的 MAS 架构。对于其他形式的协作，如开放式的头脑风暴、工具交互极其频繁但无需长久对话记忆的规划场景，或动态路由的智能体网络，其有效性尚未被验证。
*   **协议通用性代价**：严格的三字段格式可能导致一些微妙的、非结构化的交互信息（如隐式的偏好、语气）丢失，这在某些依赖此类微调信号的协作场景（如辩论）中可能有害。
*   **对模型能力的依赖**：PACT 的效果部分依赖于模型从原始输出中精确提取“动作”、“状态”和“结果”的能力。对于较小或指令遵循能力较弱的模型，该压缩过程本身可能引入新的信息损失或误解。
*   **测试覆盖**：尽管测试了多个模型规模，但均来自 Qwen 系列，未验证在 GPT、Claude 等其他模型家族上的表现。对真实世界编码任务的评估也仅限于两个平台和一个基准。
