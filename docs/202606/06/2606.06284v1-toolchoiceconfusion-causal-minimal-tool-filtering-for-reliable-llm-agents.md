## 摘要
大型语言模型智能体日益依赖外部工具，但更大的工具菜单可能会因增加错误工具调用、过早行动和令牌成本而降低可靠性和效率。现有的工具选择方法通常优化语义相关性，暴露名称或描述与用户请求匹配的工具。我们认为相关性不足：一个工具可能与任务有关，但在当前步骤中仍可能是不必要或过早的。我们提出因果最小工具过滤（CMTF），这是一种无需训练的方法，通过因果充分性来选择工具。CMTF使用轻量级的前置条件-效果契约，仅暴露从当前状态推进至用户目标所需的最小下一步工具边界。在多步骤工具使用任务中，我们将CMTF与全工具暴露、关键词检索、状态感知过滤和因果路径消融进行比较，衡量任务成功率、错误工具调用、过早行动、工具暴露和令牌成本。在包含102个任务、100个工具、四个LLM后端和2448次任务-方法-模型运行的主要基准测试中，CMTF在总体成功率上与最强的因果基线相当，同时将可见工具从每步100个减少到一个，并相对于全工具暴露减少了约90%的令牌使用。

## Abstract
Large language model agents increasingly rely on external tools, but larger tool menus can reduce reliability and efficiency by increasing wrong-tool calls, premature actions, and token cost. Existing tool-selection methods often optimize semantic relevance, exposing tools whose names or descriptions match the user request. We argue that relevance is insufficient: a tool may be related to the task while still being unnecessary or premature at the current step.   We propose Causal Minimal Tool Filtering (CMTF), a training-free method that selects tools by causal sufficiency. CMTF uses lightweight precondition-effect contracts to expose only the minimal next-step tool frontier needed to advance from the current state toward the user goal. Across multi-step tool-use tasks, we compare CMTF with all-tools exposure, keyword retrieval, state-aware filtering, and causal-path ablations, measuring task success, wrong-tool calls, premature actions, tool exposure, and token cost. In the main benchmark with 102 tasks, 100 tools, four LLM backends, and 2448 task-method-model runs, CMTF matches the strongest causal baseline in aggregate success while reducing visible tools from 100 to one per step and reducing token usage by about 90% relative to all-tools exposure.

## 1. 论文的核心问题与整体含义

- **核心问题**：大型语言模型（LLM）智能体在执行多步任务时，需要从大型工具库中选择正确的工具。然而，工具菜单过大会导致 **ToolChoiceConfusion**——智能体因看到大量「看似相关但当前并不可用、不必要或过早」的工具，而产生错误工具调用、过早行动、轨迹变长和令牌成本飙升。
- **现有方法的不足**：主流工具过滤方法（如关键词检索、嵌入匹配）基于**语义相关性**，只考虑工具名称/描述与用户查询的相似性，而忽略了**当前任务状态下的因果必要性**。一个工具可能与任务相关，但在当前步骤既不可执行，也可能干扰正确决策。
- **整体含义**：论文主张工具过滤应从“语义匹配”转向“因果充分性”——仅暴露在当前状态下能推进任务向目标迈进、且因果上绝对必要的**最小工具集**，从而从根本上消除选择混淆，提升可靠性和效率。

## 2. 方法论

### 核心思想
- 提出 **Causal Minimal Tool Filtering（CMTF）**，一种免训练、轻量级的工具过滤方法。
- 核心原则：不基于语义相关性，而是基于**前提-效果契约（precondition-effect contracts）**，动态暴露当前步骤上因果充分的**最小工具边界**。

### 关键技术细节
- **工具表示**：每个工具 `t` 被建模为轻量级契约 `(d, R, E, c, ρ)`，其中 `d` 为自然语言描述，`R` 为所需状态变量集合，`E` 为产生状态变量集合，`c` 为代价，`ρ` 为风险等级。
- **状态与目标**：任务状态 `s_t` 是一组已知状态变量；目标 `g` 是任务完成所需的状态变量集合。
- **因果充分性定义**：
  - 工具 **可执行** 当且仅当 `R ⊆ s_t`。
  - 工具 **因果充分** 当且仅当它既可执行，又位于一条从当前状态到目标状态的**最小依赖路径**上。
- **最小路径**：一条由工具序列构成的因果路径，其累积效果最终满足目标，且路径长度或代价最小。
- **因果边界暴露**：CMTF 仅在每步暴露当前最小路径的**首个可执行工具**（若有多条最小路径并列，可暴露多个工具）。

### 算法流程
1. 基于当前状态 `s_t`、目标 `g` 和所有工具的前提-效果，构建**前提-效果依赖图**。
2. 利用 **广度优先搜索（BFS）** 寻找从 `s_t` 到达 `g` 的最短因果路径 `π⋆`。
3. 仅将路径 `π⋆` 上的**第一个工具**加入可见工具集 `V_t`，返回给 LLM 智能体。
4. 执行工具后更新状态，并重复上述过程，直到目标满足。

### 与其他方法的区别
- **全工具暴露**：展示所有工具，选择负担最大。
- **关键词/top‑k 检索**：基于语义相似度保留若干工具，但可能遗漏因果上必需的、或引入过早的干扰工具。
- **状态感知过滤**：仅移除不满足前提条件的工具，但保留所有可执行工具（即使其对目标无贡献）。
- **完全因果路径暴露**：暴露一条路径上的所有工具（包括未来步骤的工具），CMTF 则仅暴露**当前因果边界**。

## 3. 实验设计

### 数据集与场景
- **受控合成基准**，专门设计以剥离工具选择行为，避免外部 API 的不确定性影响。
- 3 个工作流领域：**日历（calendar）、邮件（email）、文件/文档（files/documents）**。
- 共 **102 个任务**（每领域 34 个），每个任务包含：自然语言查询、初始状态、目标状态、黄金工具链、模拟的工具输出和成功判定标准。
- **工具库大小**：**100 个合成工具**，包含核心工具和多种干扰源：
  - 相关但错误的工具（如用 `create event` 代替 `update event`）
  - 过早工具（如未知 ID 时暴露 `update event`）
  - 近似重复工具（如 `search emails` 与 `search email ids`）
  - 高风险工具（如 `send email`, `delete file`）
  - 跨域干扰工具（如金融、支付、数据库等工作流工具）
- 所有工具输出均为**确定性模拟**，避免外部变化对结论的干扰。

### 对比方法
- **All tools**：展示全部 100 个工具。
- **Keyword top‑5 / top‑10**：基于任务文本与工具元数据的令牌重叠度，保留前 5 或 10 个。
- **State-aware**：保留所有前提条件已满足的工具。
- **Full causal path**：暴露一条最短完整因果路径上的所有工具（包括未来步骤工具）。
- **CMTF**：仅暴露当前因果边界工具。

### 评估指标
- 任务成功率（能否在步长限制内满足目标）
- 错误工具调用次数（与黄金下一步工具比较）
- 过早行动次数（提前执行写、删、发等高风险操作）
- 平均每步可见工具数
- 轨迹长度和令牌总消耗

## 4. 资源与算力

- **未明确提及训练算力**。CMTF 是一种**免训练**方法，仅依赖轻量级的前提‑效果图搜索，无需 GPU 进行模型训练。
- 实验中使用四个 LLM 后端（Amazon Nova 2 Lite、Nova 2 Pro Preview、Claude 3.5 Haiku、Claude Sonnet 4）通过 API 进行推理，但**未说明推理所使用的 GPU 型号、数量或总计算时间**。因此，本研究主要关注方法论有效性，而非计算资源消耗。

## 5. 实验数量与充分性

- **实验规模**：102 任务 × 6 种过滤方法 × 4 个 LLM 后端 = **2,448 次独立任务‑方法‑模型运行**。
- **充分性**：
  - 覆盖多领域、多种干扰类型，实验设计控制严格（确定性模拟、统一提示、定额步长限制），保证了对比的**客观性与公平性**。
  - 仅用合成基准，缺乏与真实 API 集、开放场景的对比，但专门针对工具过滤问题提供了**清晰的因果归因**，在方法论验证层面是充分的。
  - 同时提供了不同模型家族的表现，验证了方法在强、弱模型上的普适性。

## 6. 主要结论与发现

- **CMTF 实现了近乎完美的成功率（0.99）**，与“全因果路径”基线持平，但可见工具从 100 骤降至**每步 1 个**。
- 与全工具暴露相比，CMTF 将平均令牌消耗**降低约 90%**（从 24,569 降至 2,405），同时将错误工具调用从 1.25 次/任务降至 0.01 次，完全消除了过早行动。
- **语义相关性和可执行性不足以取代因果必要性**：关键词 Top‑k 和状态感知过滤虽然减少了可见工具数量，但成功率显著更低（0.61–0.72），错误调用更多。
- **因果边界比完整因果路径更优**：全因果路径暴露了未来工具，导致非零错误率和过早行动；CMTF 仅暴露当前必需工具，进一步提升了决策可靠性。
- **模型级别一致性**：CMTF 对较弱模型（如 Claude 3.5 Haiku）提升巨大（成功率从 0.48 升至 0.94），对强模型（如 Claude Sonnet 4）则在维持成功的同时清洗残余错误和令牌成本。

## 7. 优点

- **概念创新**：首次将工具过滤从“语义相关性”转向“因果充分性”，定义了 **ToolChoiceConfusion** 这一新故障模式，提供了清晰的理论框架。
- **方法轻量且普适**：免训练、仅需前提‑效果契约，易于集成到现有 LLM 代理系统中，无需额外的模型微调或大量计算资源。
- **实验设计严谨**：使用受控合成基准消除外部干扰，多维评估指标（成功、错误类型、工具暴露、Tokens）全面衡量了方法的可靠性与效率。
- **效果显著**：在维持成功率的同时，大幅压缩了可见工具数量和令牌成本，兼具高可靠性、高效率和低开销。
- **与强基线的系统对比**：不仅与简单检索对比，还与可执行性过滤、全因果路径等因果相关方法进行了消融，明确展示了“因果边界”的价值。

## 8. 不足与局限

- **合成基准局限性**：所有工具输出均为确定性模拟，未考虑真实工具系统中的延迟、失败、验证、权限等问题，所得结论不一定直接迁移至生产环境。
- **工具契约依赖**：CMTF 效果强依赖于前提‑效果契约的准确性和完整性；若契约错误或遗漏，会直接导致过滤失误。
- **任务类型受限**：实验任务主要为搜索‑读‑写等结构化工作流，对于开放域、创造性探索或多目标交替的任务，当前完全基于逻辑状态的因果路径可能不适用。
- **状态与目标预设**：要求任务目标能被表示为清晰的状态变量集合，且当前状态能被准确追踪；这在复杂、动态或主观任务中难以满足。
- **评估指标偏严格**：使用黄金工具链计算错误率，可能惩罚真实场景中同样有效但路径不同的的决策。
- **模型和平台覆盖有限**：尽管测试了四个现代 LLM，但尚未在更广泛的开源模型、真实工具栈和用户研究中验证；也未评估端到端的延迟或货币成本。
