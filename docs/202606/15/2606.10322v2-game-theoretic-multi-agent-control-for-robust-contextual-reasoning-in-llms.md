## 摘要
在多次交互中，大型语言模型（LLM）维护着不断变化的上下文，而不是生成孤立的响应，这使得它们容易受到提示注入和上下文污染攻击，在这些攻击中，局部看似合理的对抗性片段会逐渐扭曲推理轨迹。现有的防御主要过滤单个输出，往往忽略了跨轮次的上下文演化，导致长期推理暴露在风险之中。尽管模型上下文协议（MCP）标准化了上下文交换和工具调用，但它只作为一个被动的路由层，并不强制保证上下文的稳定性。为了解决这些局限性，我们引入了博弈论安全模型上下文协议（GT-MCP），这是一种控制器驱动的多智能体方法，将上下文管理视为一个闭环动力学过程。GT-MCP协调三个异构的LLM智能体，并通过一个信任函数选择输出，该函数联合评估相对于已验证上下文图的因果一致性、智能体间的语义一致性以及随时间变化的分布漂移。当检测到不稳定时，基于回滚的自愈机制会恢复已验证的上下文，并阻止无依据的片段传播。在自适应对抗威胁模型下对500个交互轮次进行的实证评估表明，上下文漂移在99.6%的轮次中保持有界，仅0.4%的轮次需要恢复。每轮效用保持高度集中，中位数 = -0.19，P05 = -0.72，P95 = 0.30；低于-1的严重退化仅发生在0.4%的情况中，并且在控制器级别没有注入尝试成功。所选输出保持稳定，胜率高于98%，计算开销保持可预测，每令牌延迟 = 1.63e-3秒。

## Abstract
Large Language Models (LLMs) in multi-turn interactions maintain evolving context rather than generating isolated responses, making them vulnerable to prompt-injection and context-poisoning attacks in which locally plausible adversarial fragments gradually distort reasoning trajectories. Existing defenses mainly filter individual outputs and often ignore context evolution across turns, leaving long-horizon reasoning exposed. Although the Model Context Protocol (MCP) standardizes context exchange and tool invocation, it functions as a passive routing layer and does not enforce contextual stability. To address these limitations, we introduce the Game-Theoretic Secure Model Context Protocol (GT-MCP), a controller-driven multi-agent method that treats context management as a closed-loop dynamical process. GT-MCP coordinates three heterogeneous LLM agents and selects outputs through a trust function that jointly evaluates causal consistency against a validated context graph, semantic agreement among agents, and distributional drift over time. When instability is detected, a rollback-based self-healing mechanism restores the validated context and prevents unsupported fragments from propagating. Empirical evaluation over 500 interaction turns under an adaptive adversarial threat model shows that contextual drift remains bounded in 99.6% of turns, with recovery required in only 0.4%. Per-turn utility remains tightly concentrated, with median = -0.19, P05 = -0.72, and P95 = 0.30; severe degradation below -1 occurs in only 0.4% of cases, and no injection attempt succeeds at the controller level. Selected outputs maintain stable win rates above 98%, and computational overhead remains predictable, with latency per token = 1.63e-3 s.

### **方法论：GT-MCP 框架**

论文提出了一个名为**博弈论安全模型上下文协议（GT-MCP）** 的上下文控制层，其核心思想和关键技术细节如下：

*   **核心思想**：
    *   **闭环动态控制**：将多智能体、多轮次的LLM推理视为一个受扰动的**闭环动态过程**。系统的“状态”是由“已验证上下文”（`c_t`）构成的。
    *   **博弈论建模**：将防御方（控制器）和攻击方（试图注入恶意内容的对手）之间的交互建模为一个**重复的Stackelberg安全博弈**。控制器通过选择一个稳健的选择-恢复策略，使攻击者的预期收益最小化。
    *   **轨迹级控制**：核心目标不再是筛选单个不安全的响应，而是**稳定整个上下文随时间的演化轨迹**，防止无根据的对抗性片段在多次交互中积累和传播。

*   **关键技术细节与流程**：
    1.  **上下文状态与验证**：控制器维护一个**已验证上下文**（`c_t`），它只包含经过验证、有依据的推理元素。任何外部输入（如用户查询、检索结果等）均被视为**不可信流入**（`Δc_t`），与已验证上下文通过带来源标记的结构化拼接（`⊕`）形成“观测上下文”（`~c_t`），但不会直接污染已验证状态。
    2.  **异构多智能体生成**：三个异构的LLM智能体（如GPT-5.3, Llama-3.1-70B, DeepSeek-R1）同时接收“观测上下文”并各自生成一个候选响应。
    3.  **信任评估与选择**：控制器通过一个**信任函数**（`T_i(t)`）为每个候选响应打分，并选择得分最高者作为本轮输出（`ŷ_t`）。信任函数由三个互补信号联合计算：
        *   **因果一致性**：通过**因果上下文图**（`G_t`）的支撑度来衡量，确保响应的声明有图结构支持。
        *   **跨智能体一致性**：衡量候选响应与其他智能体响应的语义一致性，防止单个被感染的智能体误导结果。
        *   **候选特有的上下文漂移**：计算如果接受该响应，将导致的已验证上下文分布性偏移程度，作为惩罚项。
        *   **信任公式**：`T_i(t) = α * CCI_i(t) + β * AGR_i(t) - γ * CDS_i(t)`。
    4.  **自愈恢复机制**：当所选响应的上下文漂移（`CDS`）超过阈值（`δ_c`）时，触发**基于回滚的自愈**。该机制会隔离低支撑度的图节点和相关片段，评估不同深度的历史上下文状态，选择既能消除风险又不损失过多信息的检查点进行恢复，并仅重新整合那些有坚实依据的片段。

*   **实验评估严谨**：采用了**自适应对手模型**和**多种攻击场景**，并通过**详尽的消融研究**和**配对比较**，令人信服地证明了每个组件对最终鲁棒性的贡献。

### **不足与局限**

*   **算力信息缺失**：论文未能明确说明支撑其复杂闭环系统的**硬件成本**，这使得部署可行性的评估不够完整。
*   **固定模型集合的偏差风险**：实验仅在三种特定LLM上进行。**模型异构性的广度**（如包含更小或更大、多模态模型）可能会影响控制器的性能，结论的普适性有待进一步验证。
*   **对抗模型的限制**：威胁模型假设“最多一个智能体被攻陷”，这可能简化了现实中更复杂的**协同攻击**或**一致性模仿攻击**的风险。
*   **评估范围**：评估是在500轮的有限范围内进行的，无法捕捉在**超长程部署**中可能出现的极其缓慢的漂移累积现象。
*   **“正确性”代理的局限**：因果一致性和跨智能体一致性是“推理正确性”的操作性代理指标，当已验证的上下文本身不完整或包含噪声时，该方法并不能保证认识论意义上的正确性。
