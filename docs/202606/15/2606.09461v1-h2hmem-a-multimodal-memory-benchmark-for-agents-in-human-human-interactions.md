## 摘要
大型语言模型智能体越来越多地部署在人类-人类交互环境中，如会议助理和临床文档系统，它们必须观察对话并保留信息以备下游查询。与传统的助手设置不同，这些环境本质上是多模态的，涉及复杂的篇章现象（如回指和指示），并包含来自多个参与者的异步或冲突信息。然而，现有的记忆基准主要关注单用户、纯文本交互，未能捕捉这些挑战。为填补这一空白，我们引入了H2HMem，一个用于评估复杂人类-人类交互中记忆能力的多模态记忆基准。H2HMem包括双人和多人对话，具有多模态信息流，并从三个维度评估智能体：记忆回忆、推理和应用。使用先进智能体进行的实验揭示了在跨模态、参与者和会话中构建、保留和利用记忆方面的重大局限，突显了下一代LLM智能体的巨大改进空间。

## Abstract
Large language model agents are increasingly deployed in human-human interaction settings, such as meeting assistants and clinical documentation systems, where they must observe conversations and retain information for downstream queries. Unlike traditional human-assistant settings, these environments are inherently multimodal, involve complex discourse phenomena such as anaphora and deixis, and contain asynchronous or conflicting information from multiple participants. However, existing memory benchmarks largely focus on single-user, text-only interactions, failing to capture these challenges. To address this gap, we introduce H2HMem, a Human-to-Human Multimodal Memory Benchmark for evaluating memory capabilities in complex human-human interactions. H2HMem includes both dyadic and multi-party conversations with multimodal information streams, and evaluates agents along three dimensions: memory recall, reasoning, and application. Experiments with advanced agents reveal substantial limitations in constructing, retaining, and utilizing memories across modalities, participants, and sessions, highlighting substantial room for improvement in next-generation LLM agents.

*   **研究背景与动机**：大型语言模型（LLM）智能体正从传统的人机对话（作为参与者）扩展到人-人交互的观察者角色，如会议记录、临床文档生成等。此新范式要求智能体被动观察多参与者间的复杂对话并记忆信息，以回答下游查询。
*   **核心问题与差距**：现有记忆基准存在明显不足，主要聚焦于**单用户、纯文本**的人机交互，未能覆盖真实人-人交互的三个核心挑战：
    *   **多模态性**：对话中图文信息交织。
    *   **复杂篇章现象**：如回指、指示等，要求智能体基于上下文记忆解析。
    *   **多参与者异步/冲突信息**：多人对话中信息分散、异步更新，甚至存在观点冲突。
*   **研究目标**：构建一个名为 **H2HMem** 的综合性基准，以系统评估LLM智能体在接近真实的人-人交互场景下的多模态记忆能力，涵盖双人和多人对话场景。

### 2. 论文提出的方法论

*   **核心思想**：通过一个**人机协同（Human-in-the-Loop）** 的生成流水线，在保护隐私的前提下，自动生成大规模、多会话、多模态的双人与多人对话数据集，并设计覆盖多个记忆维度的评估任务。
*   **技术路线：数据集构建流水线（共5阶段）**
    1.  **参与者画像生成**：基于预定义的结构化模式，由LLM（DeepSeek-V3）生成包含性格、背景等属性的参与者档案。
    2.  **场景构建**：LLM根据参与者和选定话题，生成一个包含多个时间顺序会话（Session）的连贯大纲，并为每个会话生成图像检索关键词。
    3.  **图像收集与人工优化**：基于关键词，通过在线搜索、文本生成图像和手动编辑等方式收集图像，并由人工标注员筛选和调整，确保图像与对话大纲内容对齐。
    4.  **图像描述与对话生成**：先用多模态大模型（GPT-4o）为图像生成详细描述，再将参与者档案、会话大纲和图像描述提供给LLM（DeepSeek-V3），生成最终包含图像引用的多轮对话。
    5.  **问题-答案对构建**：基于生成对话，由LLM初稿生成，再由人工标注员验证、修正，确保问题的清晰性、答案的正确性和难度适当性，并分配至特定任务类别。
*   **评估维度与任务设计（3大维度，9类任务）**
    *   **记忆回忆**：评估直接检索信息的能力。
        *   `单模态精确回忆 (UPR)`: 从单一模态（文本或图像）中检索事实。
        *   `跨模态关联检索 (CRR)`: 对齐文本与对应图像内容。
        *   `知识解析 (KR)`: 在跨会话更新或冲突信息中找到当前正确信息。
    *   **记忆推理**：评估跨时间、参与者和模态的推理能力。
        *   `多模态因果推理 (MCR)`: 推断文本和图像内容间的因果关系。
        *   `指代与演变追踪 (RET)`: 解析回指/指示，追踪实体在多会话中的演变。
        *   `时序推理 (TR)`: 对跨会话事件进行时间排序。
    *   **记忆应用**：评估在动态场景中应用和更新记忆的能力。
        *   `测试时学习 (TTL)`: 在新场景中利用已存储的记忆进行推理。
        *   `冲突检测 (CD)`: 判断新陈述是否与记忆中信息矛盾。
        *   `答案拒绝 (AR)`: 当记忆缺失或信息模糊时拒绝回答。

### 3. 实验设计

*   **数据集/场景**：使用自建的 **H2HMem** 数据集，包含25个对话（20个双人对话，5个多人对话），总计309个会话，7078轮对话，1300张图像，以及2236个评估用问答对。
*   **基准方法**：对比了6种记忆系统，分为文本类（用GPT-4o将图像转为描述后存储）和多模态类（直接存储图像）：
    *   **文本类方法**：
        *   `Full Memory (Text)`: 将所有历史文本作为上下文输入。
        *   `NaiveRAG`: 基于稠密检索，从历史会话块中检索Top-5相关块增强生成。
        *   `A-Mem`: 智能体记忆方法，构建结构化记忆片段并建立链接。
    *   **多模态方法**：
        *   `Full Memory (Multimodal)`: 将混合文本和原始图像作为上下文输入。
        *   `MuRAG`: 使用多模态稠密检索器检索Top-5相关多模态段落。
        *   `NGM`: 神经图记忆，在潜空间维护压缩的对话历史表示。
*   **评测方式**：
    *   **主要指标**：`LLM-as-Judge`，使用GPT-4o-mini作为零样本评估器，对模型回复与标准答案的语义一致性进行打分（0-1分），并经验证与人类判断高度一致（Cohen’s κ = 0.84）。
    *   **辅助指标**：词汇重叠指标（精准度P、召回率R、F1值、BLEU-1）。
*   **基座模型**：实验使用了GPT-4.1-Nano、Qwen2.5-VL-3B-Instruct和Qwen2.5-VL-7B-Instruct等多种多模态大语言模型作为骨干网络。

### 6. 论文的主要结论与发现

*   **整体性能低下**：即使是当前最先进的记忆系统，在所有任务上的最佳加权平均得分也仅为 **0.5757**，表明在人-人多模态交互的记忆方面存在巨大挑战。
*   **四大核心瓶颈**：
    1.  **跨模态对齐困难**：所有方法在需要将文本与图像内容关联的跨模态检索任务（CRR）上，性能均显著低于单模态回忆（UPR）。
    2.  **噪声过滤能力弱**：尽管检索增强了召回率，但精准度普遍偏低，说明系统成功检索到相关信息后，难以从多个参与者的噪声信息中有效过滤出关键事实。
    3.  **深层推理能力不足**：在需要因果推理（MCR）、指代消解（RET）等复杂推理任务上得分最低，模型难以整合分布式证据，尤其难以遵循人类含蓄的指代习惯。
    4.  **冲突检测脆弱**：冲突检测（CD）任务表现极差，词汇精准度和召回率接近零，表明当前智能体在发现和解决多人交互中的信息矛盾方面能力极其有限。
*   **交互结构影响**：双人和多人场景呈现互补的性能模式。长时间跨度的双人对话对一致性任务（如知识解析KR）更具挑战，而人员密集的多人对话则使跨模态检索等任务受益于集中证据，但冲突检测难度剧增。

*   **问题定义新颖且现实**：准确捕捉了LLM智能体在人-人观察者场景下的新挑战，填补了从单用户文本记忆向多用户多模态记忆评估的空白。
*   **基准设计系统全面**：
    *   构建了保护隐私的人机协同数据生成流水线，同时涵盖了双人和多人交互场景。

### 8. 不足与局限

*   **数据集局限性**：
    *   **语言与领域**：数据为合成生成的英文对话，可能无法完全代表真实世界的语言多样性、文化差异和更复杂的现实场景。

*   **方法覆盖的广度**：虽然对比了多种记忆方法，但主要评估了它们在该基准下的原生能力，未进行精细的任务适配或提示工程优化，因此结论可能偏向于揭示“零样本”或基础设置下的问题。
*   **评估偏差风险**：
    *   主要评估指标`LLM-as-Judge`虽与人类判断高度一致，但仍可能存在系统性偏差（如对流畅但信息错误的回复打分不准确）。
    *   辅助的词汇评估指标（BLEU等）与人类判断相关性较弱，尤其在开放式推理任务上作用有限。
*   **应用限制**：论文明确指出该基准存在被滥用于开发监控系统的风险，因此限制了其应用场景，仅建议用于会议助理、临床文档等良性应用的研究。
