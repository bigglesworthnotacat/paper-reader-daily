## 摘要
近期智能体集群的成功将基于大语言模型（LLM）的智能体范式从单智能体工作流转向多智能体系统，突显了智能体编排在任务分解与协作中的重要性。然而，现有的编排框架仅限于狭窄的模态集合，难以泛化到异构模态共存并交互的更复杂场景。这一局限性在全模态场景中尤为突出，此类场景下的任务需要对文本、图像、音频和视频等多样化输入的统一理解与协调。本工作提出Orchestra-o1，一个旨在支持跨多模态高效智能体协作的全模态智能体编排框架。Orchestra-o1引入统一编排机制，实现模态感知的任务分解、在线子智能体特化及并行子任务执行。这种可扩展设计使智能体系统能够有效处理涉及异构信息源的复杂现实任务，在OmniGAIA基准上比次优方法准确率高出10.3%。此外，我们还提出决策对齐的组相对策略优化（DA-GRPO），一种用于训练Orchestra-o1-8B的高效自主强化学习方法，该方法也在所有现有开源全模态智能体中取得了最先进的性能。

## Abstract
The recent success of agent swarms has shifted the paradigm of large language model (LLM)-based agents from single-agent workflows to multi-agent systems, highlighting the importance of agent orchestration for task decomposition and collaboration. However, existing orchestration frameworks are limited to a narrow set of modalities and struggle to generalize to more complex settings where heterogeneous modalities coexist and interact. This limitation becomes particularly pronounced in omnimodal scenarios, where tasks require the unified understanding and coordination of diverse inputs such as text, image, audio, and video. In this work, we propose Orchestra-o1, an omnimodal agent orchestration framework designed to support efficient agent collaboration across multiple modalities. Orchestra-o1 introduces a unified orchestration mechanism that enables modality-aware task decomposition, online sub-agent specialization, and parallel sub-task execution. This scalable design allows agent systems to effectively tackle complex real-world tasks involving heterogeneous information sources, surpassing the second-best approach by 10.3% accuracy on the OmniGAIA benchmark. Furthermore, we introduce decision-aligned group relative policy optimization (DA-GRPO), an efficient agentic reinforcement learning approach for training Orchestra-o1-8B, which also achieves state-of-the-art performance against all existing open-source omnimodal agents.

当前基于大语言模型（LLM）的智能体正从单代理工作流向多代理系统演进，**智能体编排**成为关键。然而，现有编排框架存在明显的模态局限：
- 多数框架仅支持文本或简单的视觉语言任务，难以泛化到文本、图像、音频、视频等**全模态**共存且相互作用的真实场景。
- 现有的全模态代理可分为两类：**原生全模态代理**（如 Gemini-3-Pro）将感知、推理与行动耦合于单一模型，在长程推理、工具使用上能力有限；**编排式代理**（如 AOrchestra）虽解耦了高层决策与底层处理，但其工具集不完整、工作流多为线性，且部分闭源框架难以扩展。
- 因此，论文的核心问题是：**如何设计一个可扩展的全模态代理编排框架，以高效协调跨异构信息源的复杂任务，并训练出高性能的开源编排模型**。

### 2. 方法论
论文提出 **Orchestra-o1** 框架，其核心思想是**将高层编排与低层专用执行完全解耦**，通过主代理动态分解任务、调度并行子代理来实现高效协作。

- **分层编排架构**
  - **主代理（编排者）**：负责模态感知的任务分解，构建依赖图，识别可并行执行的子目标；为每个子目标分配最适合的代理后端（根据技能向量与成本/延迟匹配）和工具集合；在证据充足时生成最终答案。
  - **子代理（执行者）**：独立的 ReAct 风格代理，携带主代理分配的特定指令、上下文、后端模型和工具子集，执行单一的感知或行动子任务并返回结构化结果。
  - **统一工具生态**：将感知工具（图像分析、音频分析、视频分析）与行动工具（网络搜索、页面访问、代码执行）纳入同一需求匹配接口，实现“按需分配”。
- **并行与依赖感知调度**
  - 主代理在每个编排轮次中，构造潜在依赖图 \(G_t\)，确定所有前置任务已完成的**就绪子目标集合** \(R_t\)。
  - 从 \(R_t\) 中根据预算和并行上限选择一批子任务 \(P_t\)，然后**异步并行执行**。所有子任务在条件独立假设下可完全并行化，理论上有明确的轮次级延迟优势。
- **记忆压缩与迭代精炼**
  - 维持一个结构化的子任务历史记忆 \(H_t\)，并每轮在 token 预算约束下压缩上下文 \(C_t\)，以便主代理做出更全面的决策。
- **训练方法：DA-GRPO**
  - 为获得开源的主代理模型，提出**决策对齐的组相对策略优化（DA-GRPO）**。
  - **数据构建**：基于 FineVideo、LongVideoBench、COCO 等种子数据，通过模态锚点提取、多策略改写（枢轴交换、时间偏移、数值重组等）和多重质量验证，生成约 1200 个高质量合成任务及其专家编排轨迹。
  - **奖励设计**：对每个编排决策，从格式正确性、动作有效性、工具合理性和决策质量四个维度进行李克特评分（Claude-Haiku-4.5 作为轻量级评判模型），加权求和形成即时奖励。
  - **优化目标**：在组内归一化优势的基础上，使用 PPO 风格的裁剪损失和 KL 正则项更新策略，使模型输出的决策在语法有效的同时，战略上与高质量行为对齐。训练仅在重建的状态-决策对上离线进行，无需重复执行昂贵的子代理系统。

### 6. 主要结论与发现
- **全模态编排的显著优势**：Orchestra-o1-GPT-5 在 OmniGAIA 上达到 **72.8%** 准确率，超出次优的 Gemini-3-Pro 达 **10.3%**，远超同类型的 AOrchestra-GPT-5 达 **32.8%**。
- **高效并行与成本控制**：得益于依赖感知的并行调度和精确的工具/模型匹配，Orchestra-o1 在取得最高精度的同时，总调用成本远低于线性编排的 AOrchestra。
- **开源编排模型的有效训练**：基于 8B 主干的 Orchestra-o1-8B 达到 **30.0%**，大幅度超越此前最佳的开源全模态代理（20.8%），证明小模型经过 DA-GRPO 训练可具备强大的编排决策能力。
- **DA-GRPO 关键作用**：实验表明，直接对编排决策层面的多维质量进行奖励优化，优于仅靠最终答案奖励的 GRPO 或模仿学习的 SFT。

### 7. 亮点
- **系统性的框架设计**：将全模态代理系统形式化为一个集任务分解、并行调度、工具分配、记忆压缩于一体的闭环决策问题，提供了清晰的理论分析（如并行加速比、信息增益）。
- **高效的离线训练范式**：DA-GRPO 通过重建状态和专家决策进行离线打分，避免了训练时运行复杂多代理系统的极高成本，同时提供了比仅关注最终答案更密集、更有针对性的监督信号。
- **实验全面且说服力强**：在极具挑战的全模态基准上对闭源顶尖模型形成了“吊打”之势，同时证明了轻量级模型通过编排也能达到极强水准，兼具学术价值与应用潜力。

### 8. 不足与局限
- **系统复杂度**：相对于单一模型，编排框架引入了更多工程组件（子代理管理、历史维护、异步执行等），增加了实现和调试的成本。
- **训练未端到端**：DA-GRPO 仅优化了主代理的决策策略，子代理后端模型在训练期间是固定的，整个编排系统尚未实现端到端的联合学习与优化。
- **对强基线的依赖**：最优性能仍依赖 GPT-5 作为主代理，开源版本的性能与闭源版本仍有较大差距，编排策略的泛化能力可能受主代理基模型能力的影响。
