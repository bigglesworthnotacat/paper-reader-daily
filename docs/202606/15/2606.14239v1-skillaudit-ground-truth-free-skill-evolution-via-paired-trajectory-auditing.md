## 摘要
智能体技能是结构化的程序包，用于引导冻结的大型语言模型智能体执行专门的工作流程。技能在部署后很少能保持充分适用：边缘案例、应用程序接口变更和部署限制只有通过使用才会显现，这使得技能演化成为实际必需。现有方法依赖特权反馈，如留出的验证分数、隐藏测试结果或环境奖励——当实践者只有任务描述和工作空间数据时，这些信号往往不可得。我们提出了SkillAudit，一个在没有真值反馈的情况下演化智能体技能的框架。其核心思想是成对轨迹审计：在每次迭代中，同一任务分别在包含和不包含候选技能的情况下执行，从而隔离出技能如何改变智能体行为，而无需外部标签。为了将行为差异转化为编辑指导，SkillAudit使用过程对齐对比评估（PACE），这是一组评估器集群，将轨迹偏差映射到与技能文档中特定段落相关联的诊断信号。一个结构验证器根据任务规范一次性编译并固定，用于检查任务约束并回滚有害更新。SkillAudit通过两条管道进行编辑：提炼（Refine）从广泛有用的技能中去除噪声或不相关的指导，而修复（Repair）替换与任务冲突的段落。在涵盖8个专业领域的89个容器化任务中，SkillAudit的平均任务奖励达到73.9%，优于无技能智能体（40.9%）和静态专家技能（56.7%）。这些增益是在演化过程中未访问隐藏测试、参考解决方案或外部评分函数的情况下获得的。

## Abstract
Agent skills are structured procedural packages that guide frozen LLM agents in specialized workflows. Skills rarely remain sufficient after deployment: edge cases, API changes, and deployment constraints become visible only through use, making skill evolution a practical necessity. Existing methods depend on privileged feedback such as held-out validation scores, hidden test outcomes, or environment rewards -- signals often unavailable when a practitioner has only a task description and workspace data. We introduce SkillAudit, a framework for evolving agent skills without ground-truth feedback. The key idea is paired trajectory auditing: at each iteration, the same task is executed with and without the candidate skill, isolating how the skill changes agent behavior without external labels. To turn behavioral differences into edit guidance, SkillAudit uses Process-Aligned Contrastive Evaluation (PACE), a cluster of evaluators that maps trajectory divergences to diagnostic signals linked to specific passages in the skill document. A structural verifier, compiled once from the task specification and then fixed, checks task constraints and rolls back harmful updates. SkillAudit routes edits through two pipelines: Refine removes noisy or irrelevant guidance from broadly useful skills, while Repair replaces passages that conflict with the task. Across 89 containerized tasks spanning 8 professional domains, SkillAudit achieves 73.9% average task reward, outperforming an agent without skills (40.9%) and the static expert skill (56.7%). These gains are obtained without accessing hidden tests, reference solutions, or external scoring functions during evolution.

## 1. 研究动机与核心问题
技能演化在现实部署中面临一个根本矛盾：
- **必要性**：代理技能（Agent Skills）作为引导冻结大模型执行专业工作流的指令包，在部署后会因新边缘案例、工具变化、API 变更或部署特有约束而迅速过时，必须进行演化以保持有效性。
- **障碍**：所有现有技能演化方法（如 SkillOpt、SkillForge、SkillClaw）皆依赖外部真值信号——留出验证分数、隐藏测试结果、环境奖励或专家参考解答——这些信号在真实部署环境下往往不可得。

因此，论文提出了一个**高度实用的问题**：在仅拥有任务描述（T）和工作空间数据（W）的现实部署场景中，如何在没有外部真值反馈的情况下自主改进智能体技能？这一设问从根本上改变了技能演化的研究范式，将其从依赖“神谕”信号引导转为从任务执行本身的对比中自我诊断、自我改进，具有极强的工程应用价值与现实意义。整体来看，该工作试图填补“暗箱部署下技能自适应”这一关键空白。

## 2. 方法论
SkillAudit 的方法论围绕 **“成对轨迹审计”** 这一核心思想展开，共包含三个紧密耦合的层次：

**核心机制——成对轨迹审计 (Paired Trajectory Auditing)**
- 在每次迭代中，对同一任务执行两次：一次注入候选技能 S (产生轨迹 τw)，一次不注入任何技能 (产生轨迹 τwo)。
- 两者行为差异直接隔离出技能对智能体行为的影响，提供无需外部标签的自含式证据。

**过程对齐对比评估 (PACE)**
- 一个由12个评估器模板构成的集群，跨四个维度展开评估：过程遵循度、工件证据、一致性、有效性增量。
- 关键创新在于 **段落级锚定诊断**：PACE 不只是一个通过/失败的判定器，它会将 τw 与 τwo 之间的行为差异映射到技能文档的特定段落，输出锚定于该段落的“手术目标”与“保护提示”，直接驱动编辑的具体内容。
- 采用不对称的决策规则：任何评估器报告“技能有害”即零容忍回滚；否则，工件证据评估器优先裁决。

**结构锚点验证器 (Anchor Verifier)**
- 在演化开始时，根据 T 和 W 一次性编译一个确定性检查脚本并锁定，包含所有可客观验证的约束（文件存在性、格式合规性、基于工作空间可重计算的值等）。
- 作用：作为硬边界，防止 PACE 软信号导致的评估漂移，任何锚点验证失败即强制回滚。

**双管道编辑策略：Refine 与 Repair**
- 初始阶段，任务解释器对 S0 与 T 的兼容性进行预评估，自动路由任务至两条管道之一：
  - **Refine (提炼)**：适用于核心流程正确但有噪声冗余的技能，采用“减法优先”策略，消除干扰，保护有效核心。
  - **Repair (修复)**：适用于核心指导与任务冲突的技能，允许替换、诊断驱动定位并切除有害段落，并可从未注入技能的轨迹 τwo 中提取经过验证的片段进行替换。

整个演化循环最多迭代5次，当满足三项收敛标准（近两次迭代无“技能有害”判决、锚点验证通过、无剩余可操作手术目标）时终止。

## 3. 实验设计
**数据集与场景**
- **基准**：SkillsBench（最新版+ v1.1 扩展），使用其中无环境错误的89个可运行任务。
- **任务域**：合并后可得到8个专业领域：软件工程、办公与白领、自然科学、工业与物理系统、金融与经济、数学与运筹、网络安全、媒体与内容生产。每个任务在Harbor容器内独立运行，终端验证器在演化结束后执行并给出 [0, 1] 的奖励。

**对比基线**
- **无技能基础基线**：智能体不注入任何技能直接执行任务。
- **含技能静态基线**：使用 SkillsBench 自带的人工专家精心编写的初始技能，代表无迭代改进能力下技能的最高水平。
- **核心限制**：演化过程严格运行在“桩容器”中，无权访问 pytest 验证器和任何测试内容，奖励函数仅在演化完全结束后才执行计算。

**模型**
- 所有代理执行、PACE评估、技能编辑器均使用 Claude Opus 4.8。

## 6. 主要结论与发现
- **总体效果**：SkillAudit 平均任务奖励达 73.9%，显著优于无技能基线(40.9%)和人工静态技能(56.7%)，提高幅度分别达 +33.0 和 +17.2 个百分点。
- **保护与恢复能力**：对已工作技能的保护可靠性高达92%；对失效技能的恢复成功率约43%。对初始技能有害于任务的情况（with-skill < no-skill）有强制恢复能力。
- **核心边界——可观察性假说**：成功的关键不是任务领域，而是技能知识是否具备“可观察性”。若技能关键信息能留下结构性检查痕迹（如编译、输出文件、计算结果），则演化有效；若涉及“领域规程”（无结构性痕迹的知识），演化则盲视。这个发现是本文最深刻、最有价值的见解，直接指出了无真值反馈优化的适用边界。
- **技能结构收敛**：演化操作高度符号化、模式化（去捆绑、去硬编码、内联约束、添加导航索引等），整体趋势是使技能从“教程”收敛向“可验证的执行契约”，形成信息密度极高的紧凑中间状态。

## 7. 优点
- **问题定义的突破性**：首次在学术层面严格的阐述并解决了“零真实反馈下的技能演化”这一设置，填补现有方法的实用空白。
- **机制设计巧妙**：成对审计将难以量化的“技能影响”通过自身对比转化为可操作信号；段落级锚定诊断将模糊的“好/坏”判定精细化为“修改何处、如何修改”的具象编辑指令。
- **系统完整且稳健**：软评估(PACE)与硬约束(锚点验证器)的配合、双管道编辑机制以及考虑退化保护的不对称裁决，构成了一个健壮的自主演化框架。
- **深入的机理分析**：不仅证明优势，还通过知识/任务类型等维度揭示方法的“能力边界”（即可观察性边界），这种自我限制的分析极大增加了研究的学术深度与诚实性。
- **具有工程启发的质化发现**：从编辑操作的词汇表到技能导航层的发明，为未来人工设计更强韧的智能体技能提供了直接可操作的范本。

## 8. 不足与局限
- **缺少关键的消融实验**：未能解耦 PACE、锚点验证器、双管道策略、甚至成对审计本身的独立贡献，全系统的好坏无法归因到任一组件，削弱了方法上的严谨结论。
- **计算成本未量化**：成对执行和12个LLM评估器在每次迭代中造成的 Token 和时间开销未报告，实用前景不够透明。
- **搜索、规划类任务表现差**：方法对需要“语义判断、多步推理、缺少结构性痕迹”的任务提升有限（如搜索25%、规划57.5%），这构成了一项应用硬限制。
- **依赖无技能基线的有效性**：成对审计要求 τwo 具有足够连贯性以作为参照，这在极复杂任务中可能不成立，方法在 τwo 完全崩溃时会退化为单轨迹评估，此时能力有限。
- **模型特异性的存疑**：演化完全套用 Claude Opus 4.8 这一个模型，虽声称技能是模型无关的，但是否真能跨模型转移未经检验。
- **进化深度有限**：最多5次迭代且修复成功率不足一半，对于深层知识缺失的问题只能“意识其缺失”而无法凭空创造，仍需人类或强监督信号进行知识注入。
