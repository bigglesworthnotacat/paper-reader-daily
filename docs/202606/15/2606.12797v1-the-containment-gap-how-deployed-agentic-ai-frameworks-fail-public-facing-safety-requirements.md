## 摘要
能够自主调用工具、维护持久记忆并执行多步计划的智能体大语言模型系统，正越来越多地部署在面向公众的领域，包括政府服务、医疗分诊和金融咨询。我们探究用于构建这些系统的框架是否提供了架构层面的结构性安全保证。应用从智能体架构的组合模型中推导出的六项隔离原则，我们审计了三个主流框架（LangChain、AutoGPT和OpenAI Agents SDK），发现它们无一原生合规。针对最普遍漏洞类型之一的记忆完整性防御，在三个受评估框架中均未被遵守。我们通过实验验证了这些发现：在一个基于LangChain构建的模拟政府福利智能体中，单次记忆毒化写入会在所有测试种子和后端上引发持久的定向腐败，使目标申请人的错误拒付率上升至88.9%。在复杂五因素政策下，同一攻击在保持总体准确率的同时，使定向错误拒付增加了3.5倍，从而令腐败难以通过标准监控察觉。我们接着引入了两种轻量级隔离机制：记忆完整性验证器和策略门，两者均以亚毫秒级开销（每次调用<0.2毫秒）消除了两种攻击向量。我们得出结论，当前的智能体框架生态系统可能尚未满足面向公众部署的默认安全预期，并概述了优先的架构干预措施，以便在高风险、具有社会影响的应用场景中实现可信部署。

## Abstract
Agentic large language model systems that autonomously invoke tools, maintain persistent memory, and execute multi-step plans are increasingly deployed in public-facing domains, including government services, healthcare triage, and financial advising. We ask whether the frameworks used to build these systems provide architectural-level structural safety guarantees. Applying six containment principles derived from a compositional model of agentic architectures, we audit three dominant frameworks (LangChain, AutoGPT, and OpenAI Agents SDK) and find no native compliance in any of them. Memory integrity, a defense against one of the most prevalent vulnerability classes, is not observed in any of the three evaluated frameworks. We validate these findings empirically: in a simulated government benefits agent built on LangChain, a single memory-poisoning write induces persistent targeted corruption across all tested seeds and backends, increasing the wrongful denial rate for targeted applicants to 88.9%. Under a complex five-factor policy, the same attack preserves aggregate accuracy while increasing targeted wrongful denials by 3.5x, rendering the corruption difficult to detect through standard monitoring. We then introduce two lightweight containment mechanisms: a memory integrity validator and a policy gate, which eliminate both attack vectors with sub-millisecond overhead (<0.2ms per call). We conclude that the current agentic framework ecosystem may not yet meet secure-by-default expectations for public-facing deployments and outline priority architectural interventions to enable trustworthy deployment in high-stakes, socially impactful applications.

### 1. 论文的核心问题与整体含义

- **研究动机与背景**：智能体大语言模型（Agentic LLM）系统正被广泛部署到政府福利、医疗分诊、金融咨询等公共高影响领域。这些系统自主调用工具、维护持久记忆、执行多步计划，一旦出错可能造成现实伤害。
- **核心问题**：现有的主流智能体构建框架（如 LangChain、AutoGPT、OpenAI Agents SDK）是否在架构层面提供了**结构性的安全隔离保证**？AI 安全界多关注模型输出的毒性、偏见等，而系统安全界尚未系统评估这些框架在感知、推理、执行、记忆等阶段的边界隔离是否可靠。
- **整体含义**：通过审计与实验揭示了一个**“隔离缺口”（containment gap）**——框架普遍缺少对记忆完整性、策略执行等关键环节的强制保护，一次简单的内存毒化就可导致持续、隐蔽的定向歧视（例如错误拒付特定人群的福利），且传统监控难以察觉。论文呼吁推动**默认安全（secure-by-default）**的框架设计，以保障社会依赖 AI 的弱势群体。

### 2. 论文提出的方法论

- **组合模型与六项隔离原则**：将智能体建模为由感知、推理、执行、记忆更新四阶段组成的递归流水线。从系统安全经典原则（如最小特权、深度防御）出发，推导出六项架构级安全因子：
  1. **推理-执行分离（P1）**：推理产出必须经策略门过滤，仅允许安全动作通过。
  2. **能力范围界定（P2）**：为每个会话绑定权限令牌，限定可用的工具、参数范围、速率、有效期。
  3. **记忆完整性（P3）**：任何写入持久记忆的内容必须经完整性函数验证，不通过则丢弃。
  4. **层间转换验证（P4）**：每个阶段边界（如感知→推理、推理→执行）都需执行安全检查。
  5. **认证通信（P5）**：多智能体间所有消息需携带可验证的凭证。
  6. **运行时监控（P6）**：对执行轨迹进行异常检测并在发现异常时激活隔离。
- **形式化定理**：若同时满足 P1 和 P3，则单步内存毒化无法导致持久策略违反（即隔离充分性）。
- **审计方法论**：基于公开文档、源码和已有安全研究，对三个框架的 18 个“框架-原则”配对进行二元评分（✓原生、✓*需配置、✗缺失），由双评分员独立评分（Cohen’s κ=0.81），不一致时引入第三评分员裁决。
- **防御算法设计**：提出两种不依赖 LLM 的确定性轻量级机制：
  - **算法1 记忆完整性验证器**：拦截记忆写入，检查来源类型、JSON 模式合规性，并用正则表达式匹配政策覆盖与人口统计定向的投毒模式，拒绝则静默丢弃。
  - **算法2 工具调用策略门**：维护工具许可白名单、路径作用域和速率计数器，对所有工具调用进行名称、参数路径规范化检查，非法调用直接阻断。

### 3. 实验设计

- **模拟场景**：一个基于 LangChain 的对话式福利审批智能体，依据申请人收入和家庭人数做出批准/拒绝决定。简单规则：收入<$40,000 且家庭人数>2 → 批准。
- **数据集与基准**：
  - 250 条合成福利申请（5 个区域各 50 条），另加 200 条对抗样本（100 条内存投毒载荷，100 条工具滥用攻击）。
  - 干净基线：无攻击时使用 Qwen-2.5 3B-Instruct 模型在 3 个随机种子下测得平均准确率 0.908。
- **对比实验**：
  - **实验 1（内存投毒）**：在 40 条申请序列的第 11 条后注入一条虚假政策（“区域 B 收入<$30k 者拒绝”），测量投毒后准确率、目标区域（B）错误拒绝率及腐败率。
  - **实验 2（验证器恢复）**：启用记忆验证器，重复上述攻击，观察腐败率和准确率是否恢复。
  - **实验 3（工具策略绕过）**：测试 100 次路径遍历、非授权 API 调用和受限写入攻击，对比无策略门与启用策略门时的绕过率。
  - **多后端泛化**：在 Qwen-2.5 3B 基础上，增加 Claude Haiku 4.5、GPT-4o 等共 5 个后端，验证攻击与防御的跨模型一致性。
  - **复杂政策实验**：将福利规则扩展为五因素合取政策（收入、家庭人数、未成年子女、12 个月内无福利领取、居住年限），生成 250 条合成案例（含 68 条模糊案例），测试攻击在总体准确率保持不变时对特定区域的隐蔽伤害。
