## 摘要
现代基于大语言模型的自主代理越来越依赖丰富的用户界面状态观测来在复杂数字环境中实现可靠的动作基础。然而，许多部署将完整的UI状态传输到远程推理服务器，即使大多数元素与当前任务无关，这可能会泄露敏感但不必要的上下文，如验证码、私人通知和后台应用状态。我们提出MINIM，一个可信的本地代理，在观测数据离开设备之前在客户端执行隐私感知的最小化。基于情境完整性（CI），MINIM通过预测每个UI元素的固有敏感度分数（s）和任务条件必要性分数（n）来学习双分数表示。这些分数驱动一个三元披露策略：保留必要元素，在需要时抽象敏感属性，并删除与任务无关的内容。我们优化一个CI感知目标，该目标对高风险内容上的必要性错误施加更大的惩罚，从而能够在保留任务关键信息的同时进行积极的修剪。基于WebArena的真实UI观测实验表明，MINIM大幅减少了与任务无关的敏感信息泄露，同时保留了任务关键的语义上下文和代理可靠动作所需的交互可及性。

## Abstract
Modern LLM-powered autonomous agents increasingly rely on rich user interface (UI) state observations to achieve reliable action grounding in complex digital environments. However, many deployments transmit the full UI state to remote inference servers even when most elements are irrelevant to the current task, which can leak sensitive but unnecessary context such as authentication codes, private notifications, and background application states. We propose MINIM, a trusted local broker that performs privacy-aware minimization on the client side before any observation leaves the device. Grounded in Contextual Integrity (CI), MINIM learns a dual-score representation for each UI element by predicting an inherent sensitivity score (s) and a task-conditioned necessity score (n). These scores drive a ternary disclosure policy that keeps essential elements, abstracts sensitive attributes when needed, and removes task-irrelevant content. We optimize a CI-aware objective that penalizes necessity errors more strongly on high-risk content, enabling aggressive pruning while preserving task-critical information. Experiments on real-world UI observations derived from WebArena show that MINIM substantially reduces task-irrelevant sensitive leakage while preserving task-critical semantic context and the interactive affordances required for reliable agent actions.

## 2. 方法论
- **核心思想**：将 UI 元素的披露决策解耦为两个预测器（敏感度分数 \( s \) 和任务条件必要性分数 \( n \)）和一个固定的三元策略（保留 K、抽象 A、移除 R），从而实现可解释且可调节的隐私-效用平衡。
- **关键技术细节**：
  - **双分数表示**：对可访问性树的每个节点 \( e_i \) 同时预测其**固有敏感度** \( \hat{s}_i \)（与任务无关）和**任务条件必要性** \( \hat{n}_{i,T} \)。
  - **模型架构**：使用图注意力网络 v2（GATv2）在可访问性树上进行图推理。节点特征由 384 维 MiniLM 文本嵌入、UI 属性（角色、状态等）和结构特征拼接成 512 维向量；网络包含 3 个 GATv2 层（隐藏维度 256，4 头注意力），后接两个独立 MLP 头分别输出 11 类分数（0‑10）。
  - **CI 感知损失函数**：训练目标包含敏感度回归损失 \( L_{\text{sens}} \) 和带情境权重的必要性损失 \( L_{\text{nec}} \)。\( L_{\text{nec}} \) 对高敏感度（\( s_i \) 大）但低必要性（\( n_{i,T} \) 小）的节点预测错误施以更大惩罚，通过加权因子 \( 1 + \lambda \cdot \frac{s_i}{10} \cdot (1 - \frac{n_{i,T}}{10}) \) 实现，从而强制模型尽量减少不必要的高风险信息传输。
  - **部署时披露策略**（算法 1）：
    - 若 \( \hat{n}_{i,T} < \tau_{\text{nec}} \) → 移除（R）；
    - 若 \( \hat{n}_{i,T} \geq \tau_{\text{nec}} \) 且 \( \hat{s}_i \geq \tau_{\text{sens}} \) → 抽象（A，保留元素结构但用占位符替换敏感值）；
    - 否则 → 保留（K）。
  - **抽象机制**：保留节点结构（如按钮、输入框）但隐藏原始文本，代以 `[REDACTED]` 或类型占位符，并假定存在残余泄露（占原文本 5%）。
- **公式与流程**：整体视为约束上下文瓶颈的松弛优化，\( \min_{Z_t} I(X_t^{\text{out}}; Z_t | T) \) 满足 \( \text{TaskSuccess} \ge \tau \)，训练时通过 CI 加权损失近似求解。

## 6. 主要结论与发现
- **隐私‑效用高效平衡**：MINIM 在测试集上实现 TCIP‑I 0.9931、TCNP 0.9491，同时将任务无关敏感泄露 TISL 降低至 0.101（仅为全观测的 10.1%），明显优于只考虑敏感度或只考虑必要性的单维度策略。
- **优于通用 LLM 评分器**：与各类开源 LLM 相比，MINIM 的 TISL 最低（0.101 vs. 0.194~0.312），且仅保留约 12% 的节点，而 LLM 普遍保留 26%~35% 的节点却泄露更多敏感内容，说明通用指令微调无法自动实施任务条件最小化。
- **抽象动作不可或缺**：将抽象（A）改为完全移除会导致交互式节点保留率大幅下降（0.9650），而改为完全保留则泄露不显著增加，表明适度的语义抽象能够在几乎不增加隐私成本的情况下恢复任务关键交互元素。
- **任务语义自适应**：对于事务性、管理类任务，MINIM 自动保留更多节点和交互元素，支持安全关键流程；对信息浏览类任务则更积极压缩，体现 CI 原则的自适应能力。
- **CI 合规性**：在反事实任务（改为仅浏览模式）下，必要性分数降低，模型对敏感但无关的元素仍稳定执行移除，基本符合“需要知道”的传输规范。

## 7. 优点
- **理论‑实践结合紧密**：以情境完整性为框架，将隐私规范转化为可训练的双分数机构与明确的三元决策策略，可解释性强。
- **优雅的解耦设计**：预测（评分模型）与执行（阈值策略）分离，允许事后调整隐私‑效用均衡而无需重新训练。
- **轻量本地部署**：仅依赖 GATv2 等小型模型，无需依赖远程大型语言模型进行隐私决策，减小隐私风险并降低延迟。
- **保留交互可及性**：引入抽象操作（A）而非简单二元删除，在隐藏敏感内容的同时保留按钮、输入框等结构，确保代理仍能正常执行任务。
- **实验系统全面**：涵盖多个域、多种任务语义，与多种基线对比，并进行了详尽的消融和鲁棒性分析，验证了方法的有效性。
