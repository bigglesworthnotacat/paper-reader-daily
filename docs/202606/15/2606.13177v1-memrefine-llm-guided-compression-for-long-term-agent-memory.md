## 摘要
大型语言模型（LLM）智能体越来越多地需要在长期交互中运行，其中来自过去对话的信息必须被保留和召回，以支持未来的任务。然而，随着交互的累积，记忆存储无限制地增长，并充满冗余条目，这不仅增加了存储成本，还挤占了最有用证据，从而降低了检索效果。此外，这在具有严格内存预算的资源受限平台上尤其受限，促使我们提出了存储预算化的记忆管理，即在一个固定预算内保持已构建的记忆存储，同时保留对未来交互有用的信息。为此，我们提出了MemRefine，一个LLM引导的框架，由于表面相似性不能很好地反映事实价值，该框架仅使用相似性来提出候选对，并将删除、合并和保留的决定委托给基于事实内容的LLM裁判，迭代进行直到满足预算为止。在多个记忆框架和长期对话基准测试中，MemRefine始终满足目标预算，同时保持下游性能，并在紧张预算下优于基于规则的基线方法。

## Abstract
Large language model (LLM) agents are increasingly expected to operate over long-term interactions, where information from past dialogues must be preserved and recalled to support future tasks. However, as interactions accumulate, the memory store grows without bound and fills with redundant entries that inflate storage cost and degrade retrieval by crowding out the most useful evidence. Furthermore, this is especially limiting on resource-constrained platforms with hard memory budgets, motivating us to formulate storage-budgeted memory management, the task of keeping an already constructed memory store within a fixed budget while preserving information useful for future interactions. To this end, we then propose MemRefine, an LLM-guided framework that, since surface similarity poorly reflects factual value, uses similarity only to propose candidate pairs and defers delete, merge, and preserve decisions to an LLM judge based on factual content, iterating until the budget is met. Across multiple memory frameworks and long-term conversation benchmarks, MemRefine consistently meets target budgets while preserving downstream performance and outperforming rule-based baselines under tight budgets.

- **整体含义**：论文首次将“存储预算化的记忆管理”形式化为一个独立任务，并提出解耦的压缩框架，使记忆系统能在不改变构建与检索管线的前提下，持续清理冗余、保持紧凑。

## 2. 方法论
- **核心思想**：利用嵌入相似度快速筛选出可能重叠的记忆对，但将最终的**删除、合并或保留**决策权交给基于事实内容的 LLM 裁判，而不是依赖表面句法相似度或图结构中心性。
- **问题形式化**：将压缩视为一个查询无关的 max‑min 优化问题：
  ```
  max_{M′∈F(M0)} min_{q∈Q} U(q,M′)   s.t. size(M′) ≤ ρ·size(M0)
  ```
  通过要求压缩后的记忆在面对最差查询时仍能保持效用，避免误删非冗余的事实。
- **算法流程（MemRefine）**
  1. **候选对选择**：从尚未被标记为“保留”的记忆对中，选择余弦相似度最高的 pair `(u,v)`。
  2. **LLM 裁判**：将该 pair 送入 LLM，根据事实内容判断关系为冗余（delete）、互补（merge）或相异（preserve）。若为 merge，同时产出一条融合指令。
  3. **存储更新**：
     - `DELETE`：移除被指定的一个节点，若有图结构则剪除悬挂边。
     - `MERGE`：由另一个 merge‑LLM 依据指令合并双方事实，生成为单一新记忆，嵌入重新计算，并继承两节点的链接并集。
     - `PRESERVE`：保留两节点，并将该对加入跳过集，不再重复判断。
  4. **终止**：当存储大小 ≤ 目标预算 或 再无未判断的有效候选对时停止。
- **关键设计**：压缩模块插入在记忆构建之后、检索之前，不修改宿主记忆管线，对图结构和非图存储均适用。

## 3. 实验设计
- **数据集与基准**
  - **LoCoMo**（主要基准）：10 个会话样本，含 1,986 道问题，覆盖单跳、多跳、时间推理、开放式、反事实等 5 类。
  - **Scaled LoCoMo‑style**：构造了 3× 和 10× 更长时间线的对话（5 个样本 / 规模），用于压力测试。
  - **LongMemEval S**：60 道问题，6 种类型，用于验证方法在另一评测上的泛化性。
- **记忆框架**
  - **A‑MEM 风格图记忆**：节点含内容、嵌入、链接，仅保留事实图表示（去除辅助字段）。
  - **Mem0**：通过摄入‑更新流水线处理后的条状记忆。
- **对比方法**
  - `Base Memory`（无压缩）、`RULE_SIM`（固定相似度阈值决策）、`RULE_PR`（结合 PageRank 保护的规则基线，仅在图记忆上对比）。
- **压缩预算**：70%、60%、50%、40%、30% 存储比例。
- **评价指标**：LoCoMo 上使用 token‑level F1 和完全匹配（EM）；LongMemEval S 上使用自动评分准确率。

## 6. 主要结论与发现
- **存储‑效用可控权衡**：MemRefine 在 70%‑50% 的中等压缩率下，下游 F1 几乎与基准持平，即使在 30% 的激进压缩下，性能也呈现**渐进衰减**而非骤降。
- **LLM 裁判的优势**：在宽松预算下规则基线尚可，当预算收紧、需要区分语义关联但事实独立或互补的记忆时，MemRefine 的 **LLM 裁判显著优于固定相似度或图中心性策略**，差距随压缩加剧而拉大。
- **信息冗余度受记忆框架影响**：A‑MEM 图记忆因直接存储原始对话节点，冗余较多，压缩空间更大；Mem0 在摄入时已过滤和合并，可移除的冗余更少，被压缩后性能更快开始下降。
- **类别级影响差异**：压缩后单跳、时间推理等依赖局部证据的题型往往因去噪而提升；开放式、反事实题型因证据分散，对压缩更敏感。
- **对更长对话的鲁棒性**：在 3× 和 10× 缩放数据集上，存储‑效用曲线仍然平滑，表明方法可扩展到长期运行的智能体。
- **跨基准泛化**：在 LongMemEval S 上也验证了 A‑MEM 图记忆在适度压缩后甚至可获得更好准确率的现象。
