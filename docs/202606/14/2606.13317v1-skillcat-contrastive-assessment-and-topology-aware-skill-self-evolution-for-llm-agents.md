## 摘要
面向LLM智能体的技能自我进化方法旨在将执行轨迹转化为可复用的技能文档，但当前流程通常每个任务仅从一条轨迹学习，在检查之前合并候选技能补丁，并在推理前加载完整的技能语料库。我们提出了SkillCAT，一个免训练框架，将该过程划分为三个阶段。对比因果提取（CCE）对每个任务采样多条轨迹，并比较同任务的成功/失败对，以识别解释结果差异的证据。评估增强的进化（AAE）在源任务克隆上回放每个候选补丁，并仅保留能改进或维持任务结果的补丁，之后进行分层技能补丁合并。拓扑感知任务执行（TTE）将进化的技能编译为可路由的子技能拓扑，使得推理时仅加载与任务相关的能力节点。我们在常见的智能体基准上评估了SkillCAT，包括SpreadsheetBench、WikiTableQuestions和DocVQA，并进一步测试了跨模型和分布外泛化。在这些设置下，SkillCAT将平均得分相较于基线最高提升了40.40%，证明了无需模型训练即可实现可靠的技能进化。

## Abstract
Skill self-evolution methods for LLM agents aim to turn execution trajectories into reusable skill documents, but current pipelines typically learn from one trajectory per task, merge candidate skill patches before checking them, and load the full skill corpus before inference. We propose SkillCAT, a training-free framework that separates this process into three stages. Contrastive Causal Extraction (CCE) samples multiple trajectories for each task and compares same-task success/failure pairs to identify evidence that explains outcome differences. Assessment-Augmented Evolution (AAE) replays each candidate patch on source-task clones and keeps only patches that improve or preserve task outcomes before hierarchical skill patch merging. Topology-Aware Task Execution (TTE) compiles the evolved skills into a routable sub-skill topology, so inference loads only the capability nodes relevant to the task. We evaluate SkillCAT on common agent benchmarks, including SpreadsheetBench, WikiTableQuestions, and DocVQA, and further test cross-model and out-of-distribution generalization. Across these settings, SkillCAT raises the average score over baselines by up to 40.40%, demonstrating reliable skill evolution without model training.

### **1. 核心问题与研究动机**
当前LLM智能体的技能自我进化方法（如Trace2Skill）通常遵循离线批量流程，将执行轨迹提炼为技能文档。然而，随着任务和技能库规模增长，存在三个关键问题：

- **单轨迹偏差**：每个任务仅基于一条轨迹提取技能，成功轨迹可能包含偶然策略，失败轨迹难以定位根因，导致提取的证据不可靠。
- **无验证合并**：候选技能补丁在未经验证的情况下直接合并，低质量或有害补丁会“污染”技能语料，损害下游性能。
- **上下文过载**：合并后的技能持续膨胀，推理时加载全量技能会导致提示过长、干扰任务相关规划，降低效率与效果。

论文旨在提出一种训练无关的框架SkillCAT，将技能进化过程解耦为证据提取、补丁验证与整合、测试时技能部署三个可观测阶段，从根本上解决上述问题。

### **2. 方法论**
SkillCAT将技能自进化划分为三个顺序阶段：

#### **2.1 对比因果提取（CCE）**
核心思想是利用同任务内成功与失败轨迹的对比，定位导致结果差异的关键行为。
- **轨迹采样与配对**：对每个进化任务使用多个随机种子（默认5个）采样，获得成功集和失败集。从成功和失败轨迹中各随机抽取一条，构建**同任务成功/失败对比对**。
- **因果分水岭识别**：找出成功轨迹与失败轨迹中行动序列首次出现分歧的时间步`w_i`，作为因果分水岭。
- **记录提取**：围绕`w_i`提取局部证据、推断失败原因和可编辑的经验教训，形成候选记录`r_i`，而非总结整条轨迹。
- **回退机制**：若某任务所有轨迹均成功或均失败，可选择性使用单轨迹提取器（可关闭以保持严格对比）。

#### **2.2 评估增强的进化（AAE）**
将每个候选补丁视为需验证的假设，通过**源任务回放**评分，再按分数进行分层合并。
- **回放评估与评分**：对每个候选补丁`p_j`，在关联的源任务克隆上分别记录基线结果`y_j`和应用补丁后的结果`ŷ_j`，根据结果转换赋予分数量表：
  - 3.0：失败→成功（最佳）
  - 2.0：成功→成功（保持）
  - 1.0：失败→失败（微弱）
  - 0.0：成功→失败（应拒绝）
- **阈值筛选**：仅保留评分≥θ（默认θ=2.0）的补丁，确保只接纳能修复失败或维持成功的补丁。
- **分层合并**：按分数层级从低到高合并补丁（`S^(ℓ) = μ(S^(ℓ-1), P_θ^(ℓ))`），初始技能为`S_0`，最终进化技能`S* = S^(L)`。这种层次化合并让高分补丁在后期享有更高优先级。

#### **2.3 拓扑感知任务执行（TTE）**
将进化技能编译为**可路由的子技能拓扑**，推理时仅加载任务相关部分以控制上下文长度。
- **拓扑构建**：解析`S*`的Markdown层级，一级标题内容作为始终加载的核心技能`S^◦`，二级标题作为可路由节点集合`V`；每个节点存储正文`B_v`和元数据（标题、摘要、依赖等）。
- **路由选择**：对于测试任务`c`，由LLM路由器根据元数据摘要`Σ`选择最多`k`个主要节点（`Ṽ_c = ρ(c, Σ; k)`），再通过确定性扩展加入依赖节点与基础节点，得到最终节点集`V_c`。
- **技能组装**：拼接核心技能与所选节点正文，形成任务专用的路由技能`S_c = S^◦ ⊕ ⊕_{v∈V_c} B_v`。该过程不修改技能原文，仅控制加载范围。

整体流程可参见 Algorithm 1。

### **3. 实验设计**

### **7. 优点**
- **问题针对性明确**：准确识别并系统性地解决了技能进化中的单轨迹偏差、无验证合并与上下文过载三大痛点。
- **免训练且模块化**：无需微调LLM，各阶段解耦，可独立分析或替换。
- **验证机制严谨**：基于源任务回放的评分与阈值筛选，有效过滤有害补丁，避免性能恶化。
- **拓扑路由创新**：通过编译可路由技能拓扑，实现按需加载，大幅精简推理上下文而不牺牲精度。
- **实验全面**：涵盖多个任务类型、多模型对、跨家族泛化、多模态迁移以及详细的消融与分析，结论可靠。
- **开放性与复用性**：生成的技能可跨模型复用，具有实际部署意义。

### **8. 不足与局限**
- **任务领域较集中**：实验主要围绕电子表格、表格问答和文档VQA，未展示在代码、工具调用或多步交互等更广泛智能体任务上的效果。
- **超参数依赖**：CCE的种子数（5）、AAE的阈值（2.0）和TTE的top_k（7）均需根据具体场景调整，可能缺乏通用性指导。
- **进化成本**：CCE需为每个进化任务采样多条轨迹，虽可并行但时间开销随种子数增加（如9种子达4.7小时），在更大规模任务集上可能受限。
- **路由依赖LLM**：TTE的路由器本身是一个LLM调用，可能引入额外推理延迟，且路由质量影响最终效果。
- **未探索噪音累积**：分层合并虽优先高分补丁，但长期多轮进化后低分补丁的累积噪音影响未深入分析。
- **评估指标单一**：主要依赖任务通过率或准确率，缺少对生成技能可读性或维护性的评估。
