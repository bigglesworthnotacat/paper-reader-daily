## 摘要
尽管大型语言模型（LLMs）推动了开放域对话系统的发展，但由于长上下文推理的固有局限和处理大量原始文本的低效性，维持长期一致性仍是一项挑战。现有方法通常依赖非结构化记忆存储，易导致信息丢失，或使用计算成本高昂且延迟高的大型语言模型。为解决这些限制，我们提出了G-Long，一个图增强框架，利用微调的小语言模型（sLM）进行结构化三元组提取和关联检索，显著降低运营成本。此外，我们引入了新颖的注意力感知重要性评分机制，借助T5摘要器固有的交叉注意力信号来识别显著记忆。在多个基准上的广泛实验证明，G-Long在响应生成和记忆检索方面均达到了最先进的性能，在MSC上的响应质量提升高达9.8%，在LME上的检索召回率提升40.8%，同时显著减少了计算开销。

## Abstract
While Large Language Models (LLMs) have advanced open-domain dialogue systems, maintaining long-term consistency remains a challenge due to inherent limitations in long-context reasoning and the inefficiency of processing extensive raw text. Existing approaches typically rely on either unstructured memory storage, which is prone to information loss, or computationally expensive LLMs that incur high latency. To address these limitations, we propose G-Long, a graph-enhanced framework that utilizes a fine-tuned small Language Model (sLM) for structured triplet extraction and associative retrieval, significantly reducing operational costs. Furthermore, we introduce the novel attention-aware importance scoring mechanism that leverages the intrinsic cross-attention signals of a T5 summarizer to identify salient memories. Extensive experiments across diverse benchmarks demonstrate that G-Long achieves state-of-the-art performance in both response generation and memory retrieval, yielding performance gains of up to 9.8% in response quality on MSC and 40.8% in retrieval recall on LME, while significantly minimizing computational overhead.

## 2. 论文提出的方法论
- **整体框架 G-Long**：以图结构记忆银行替代非结构化文本存储，通过微调的小语言模型（sLM）完成结构化三元组提取和关联检索，减少对昂贵 LLM-API 的依赖。
- **四个核心组件**：
  - **(1) 高效记忆构建**：
    - **三元组提取模块**：利用微调的 sLM（如 Qwen3-8B 在 PersonaExt 数据集上指令微调）将原始对话解析为 `(subject, relation, object)` 三元组，输出结构化 JSON。
    - **注意力感知重要性评分模块**：使用在 SAMSum 上微调的 T5-small 摘要器的交叉注意力图，计算每个三元组的显著性得分。公式如下：
      - 平均最后解码层的注意力权重：\(A_{i,j} = \frac{1}{H} \sum_{h=1}^H \text{Attn}_h(x_i, y_j)\)
      - 三元组 \(t_k\) 的原始得分：\(S(t_k) = \sum_{x_i \in \text{Tokens}(t_k)} \sum_{j=1}^M A_{i,j}\)
      - 归一化权重：\(w_k = \frac{S(t_k)}{\max_{k'} S(t_{k'})} \in (0,1]\)
  - **(2) 图结构记忆银行**：
    - 节点（实体）使用稠密向量索引（MiniLM + ChromaDB），支持语义近似匹配。
    - 边为属性丰富的有向容器：\(e_{s \to o} = \{r, w, \tau_{\text{create}}, \tau_{\text{access}}\}\)，包含关系文本、重要性得分、时间戳。
  - **(3) 关联记忆检索**：
    - **锚点节点识别**：从当前用户话语 \(u_t\) 中找到最相似的 Top-N 实体节点。
    - **多跳子图扩展**：从锚点出发，进行 L 跳（文中最优为 L=1）扩展，生成候选三元组集 \(T_{\text{cand}}\)。
    - **两阶段混合重排序**：
      - 先按语义相似度取 Top-N 三元组 \(T_{\text{sem}}\)。
      - 再结合重要性 \(w\) 和时间衰减因子 \(\lambda\) 计算最终得分：\(S_{\text{final}}(t) = \beta_1 \cdot S_{\text{sem}}(t) + \beta_2 \cdot w + \beta_3 \cdot e^{-\lambda \Delta \tau}\)，选出 Top-K 最终三元组。
  - **(4) 响应生成**：将检索到的三元组序列化拼接为 \(M_{\text{Long}}\)，与系统指令、当前对话历史一同输入骨干 LLM 生成回复。

## 3. 实验设计
- **数据集**：
  - **Multi-Session Chat (MSC)**：多会话对话，5 个时间分离的会话，评估人物信息保留。
  - **Conversation Chronicles (CC)**：多会话，含时间依赖和关系演化。
  - **LoCoMo**：极长对话（平均 27 个会话），用于可扩展性测试。
  - **LongMemEval (LME)**：大规模长对话记忆 QA 任务，专门评估检索准确度与回答正确性。
- **评估指标**：
  - 生成任务：BLEU-n、ROUGE-L、BERTScore；G-Eval（Coherence, Engagingness, Fluency）；人类偏好评估。
  - 检索任务：Recall@k, Accuracy@k, MRR。
- **对比基线**：
  - 非结构化文本框架：No History, Long Context, MemoryBank, LD-Agent。
  - 结构化/图框架：FraCom, HippoRAG。
  - 均统一使用 gpt-4o-mini 作为生成骨干（HippoRAG 仅用于检索评估），确保公平性。

## 7. 优点（亮点）
- **结构化表征**：通过三元组图显式建模实体关系，缓解文本摘要的模糊与信息丢失。
- **高效低成本**：sLM 本地化记忆提取和紧凑三元组序列化，减少对昂贵 LLM API 的依赖，token 效率极高。
- **注意力感知重要性评分**：利用 T5 交叉注意力天然信号，无需额外标注，实现轻量级记忆显著性评估。
- **两阶段混合重排序**：融合语义、重要性和时间衰减，兼顾相关性与时效性。
- **鲁棒性验证**：跨多种 LLM 和 sLM，证明框架的模型无关性和模块化程度。
- **可解释性与控制**：显式图结构支持记忆的精确删除，符合“被遗忘权”需求。

## 8. 不足与局限
- **语义粒度折中**：三元组会丢失原始文本中的情感色彩、风格等细粒度信息。
- **管道依赖风险**：整体性能受三元组提取模块精度影响，极端指代消解或歧义可能引入噪声；目前缺乏图内事实一致性自校正机制。
- **检索边界受限**：
  - 对包含未解析回指（如“它”）的查询，容易出现参照断裂，检索不准。
  - 对局部关键词过度敏感时可能产生语义漂移（如将颜色词误关联到发色）。
- **实验样本量**：部分子实验（如 LoCoMo N=50，基线测试 N=100）样本较少，结论的统计稳健性有待更大规模验证。
- **安全与偏见**：依赖预训练模型，仍可能产生幻觉或偏见，部署需人工监督。
