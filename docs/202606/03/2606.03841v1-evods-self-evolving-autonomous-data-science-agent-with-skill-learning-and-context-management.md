## 摘要
大型语言模型智能体的近期进展推动了自动化数据科学的显著进步，但现有方法仍受限于静态动作集和缺乏原则性的长程上下文管理，阻碍了其跨任务积累可重用经验以及在多阶段、迭代式数据科学流程中可靠运行的能力。为解决这些挑战，我们提出了 EvoDS，一个通过智能体强化学习来学习扩展技能并自适应管理长期上下文的自我进化自主数据科学智能体。具体而言，EvoDS 引入了两项关键策略：（1）自主技能获取机制，使智能体能够合成、验证并重用可执行技能；（2）自适应上下文压缩策略，将上下文管理视为一个学习控制问题，而非被动截断。这些策略在一个两阶段多智能体训练方案中协同运作，使 EvoDS 能够随时间自主提升。理论上，我们证明了 EvoDS 的分层设计减少了工具选择错误，且其优化目标符合信息瓶颈原理，确保了高效的上下文利用。实验表明，EvoDS 在四个不同基准测试中平均优于最先进的开源数据科学智能体 28.9%，同时消除了词元溢出故障。我们的代码和数据已开源，地址为 https://github.com/usail-hkust/EvoDS。

## Abstract
Recent progress in Large Language Model (LLM) agents has enabled promising advances in automated data science. However, existing approaches remain fundamentally limited by their static action sets and lack of principled long-horizon context management, hindering their ability to accumulate reusable experience across tasks and operate reliably in multi-stage, iterative data science pipelines. To address these challenges, we introduce EvoDS, a self-evolving autonomous data science agent that learns to expand its skills and adaptively managing long-term context through agentic reinforcement learning. Specifically, EvoDS introduces two key strategies: (1) Autonomous Skill Acquisition (ASA) mechanism, which enables agents to synthesize, validate, and reuse executable skills; and (2) Adaptive Context Compression (ACC) strategy, which treats context management as a learned control problem rather than passive truncation. These strategies are orchestrated within a two-stage multi-agent training scheme, enabling EvoDS to autonomously improve over time. Theoretically, we prove that EvoDS's hierarchical design reduces tool-selection error, and its optimization objective aligns with an information bottleneck principle, ensuring efficient context use. Empirically, EvoDS outperforms state-of-the-art open-source data science agents by an average of 28.9% across four diverse benchmarks while eliminating out-of-token failures. Our code and data are available at https://github.com/usail-hkust/EvoDS.

### 1. 研究动机与核心问题

基于大语言模型（LLM）的自主数据科学智能体虽取得进展，但现有方案普遍存在两类根本性瓶颈：

- **静态动作空间**：智能体依赖预设的固定工具集，无法将成功的解题经验抽象为可复用的技能，导致跨任务学习能力缺失，每次面临新任务都需要重新“试错”。
- **长程上下文管理失当**：数据科学任务天然需要多阶段迭代推理，产生大量中间结果与执行日志。现有方法多采用被动截断或简单总结，容易丢失关键信息，引发“迷失在中间”现象和词元溢出（out-of-token）失败，严重制约长程任务的可靠性。

这些局限使当前智能体难以像人类数据科学家那样，在反复探索中持续积累经验并有效管控记忆。EvoDS 旨在构建一个能够**自我进化**的智能体系统，同时攻克技能学习与上下文管理两大难题。

### 2. 方法论

EvoDS 的设计围绕三个核心部分展开：分层多智能体架构、自主技能获取（ASA）、自适应上下文压缩（ACC），并通过两阶段智能体强化学习联合优化。

#### 2.1 分层多智能体架构

- **Manager Agent**：顶层控制器，负责全局推理、任务分解与子智能体调度，并装备少量基础工具（如 Bash、Python、SQL、上下文总结工具）。
- **专项子智能体**：包括 `Cleaner`、`Featurizer`、`Modeler`、`Visualizer`、`Debugger`，各自维护专属的**局部技能空间**（工具集互不重叠），接受 Manager 委托执行具体子任务。
- 每个智能体保留两种记忆：**全局记忆**（整体任务目标）与**局部记忆**（子任务目标与中间上下文）。子智能体执行完任务后，**将原始执行结果压缩为面向全局目标的摘要**返回给 Manager，以减少上下文长度。

这种分解将庞大的决策空间拆分为多个小规模的子问题，理论上可降低工具选择的错误风险（定理 5.1）。

#### 2.2 自主技能获取（ASA）

当子智能体发现现有技能无法完成子任务时，可自主合成新技能。过程包含四个阶段：

1. **合成（Synthesis）**：利用 LLM 生成新技能 `a_new = ⟨n, d, c⟩`（名称、文本描述、可执行代码）。
2. **验证（Verification）**：在环境中实际执行并检查其可运行性与输出有效性。
3. **缓存（Caching）**：通过验证的技能暂存入合成技能仓库 `ΔA_i`，避免低质量技能污染动作空间。
4. **扩展（Expansion）**：采用**使用频次感知**策略，仅当某技能的合成次数 `c(a_new) ≥ τ`（τ=3）时，才将其永久加入子智能体的动作空间。

该机制使智能体能够在连续任务中识别并填补能力缺口，逐步积累可复用的执行工具。

#### 2.3 自适应上下文压缩（ACC）

不同于传统的被动截断，EvoDS 将上下文管理内化为智能体的主动决策：

- **子智能体层面**：将执行结果 `o_t` 基于全局目标 `G` 压缩为摘要 `õ_t = φ(o_t | G)`，成功时保留关键成果，失败时保留错误原因。
- **Manager 层面**：装备专门的 `context_summarize` 工具，Manager 可根据当前推理状态自主决定何时调用该工具压缩历史上下文，更新记忆 `C ← g(C | G)`，实现**自适应**压缩而非被动触发。

#### 2.4 两阶段多角色强化学习

所有智能体共享同一个 Qwen3-8B 基座模型，通过角色化系统提示和不同动作空间实现行为差异。

- **SFT 阶段**：使用更强教师模型（DeepSeek-V3.1）在 8K 训练实例上收集轨迹（Manager 主轨迹与子智能体子轨迹），进行监督微调，提供稳定的初始化。
- **RL 阶段**：采用 GRPO 进行在线联合优化。
  - **奖励设计**：子轨迹采用二元规则奖励（成功 0.1，失败 -0.1）；Manager 主轨迹奖励 `R_main = R_outcome + 0.2 R_sub - 0.1 P_context - 0.1 P_turn`，其中包含最终解质量、子任务完成率、上下文长度惩罚和交互轮次惩罚。
  - 因上下文压缩将主轨迹分割为多段，采用**奖励广播**将最终奖励赋予所有段。
  - 总损失函数同时对主轨迹和子轨迹的剪切代理目标求期望，并加入 KL 散度正则项。

理论分析表明，Manager 的优化目标等价于求解信息瓶颈问题，即在压缩中滤除任务无关信息、保留关键信号（定理 5.2）。
