## 摘要
大语言模型（LLM）研究的一个核心目标是构建能够在动态环境中通过持续交互进行规划、行动和适应的智能体系统。尽管近期基于LLM的智能体展现出令人印象深刻的上下文推理能力，但它们在长期决策中仍显脆弱，常常遭遇目标漂移，即目标和计划在长期交互过程中发生偏移。我们提出了Multi$^2$，一个分层多智能体决策框架，将智能体行为显式分解为互补的角色。高层智能体（系统1）利用监督微调（SFT）专注于上下文感知的子目标生成，而低层智能体（系统2）则在交互环境中通过离线到在线强化学习（RL）执行原子动作。这种分离实现了稳定的长期控制，缓解了目标漂移，并支持高效适应。在多种交互环境中，Multi$^2$始终优于强大的智能体基线，在多轮交互中表现出更强的鲁棒性和协调性。除了性能提升，我们还引入并发布了三个分层基准数据集，填补了基于LLM的智能体在分层决策训练和评估方面长期存在的空白。

## Abstract
A central goal of large language model (LLM) research is to build agentic systems that can plan, act, and adapt through sustained interaction with dynamic environments. While recent LLM-based agents exhibit impressive contextual reasoning, their long-horizon decision-making remains fragile, often suffering from objective drift, where goals and plans drift over extended interactions. We introduce Multi$^2$, a hierarchical multi-agent decision-making framework that explicitly decomposes agent behavior into complementary roles. A high-level agent (System 1) focuses on context-aware sub-goal generation using supervised fine-tuning (SFT), while a low-level agent (System 2) executes atomic actions through offline-to-online reinforcement learning (RL) in interactive environments. This separation enables stable long-horizon control, mitigates objective drift, and allows efficient adaptation. Across diverse interactive environments, Multi$^2$ consistently outperforms strong agentic baselines, demonstrating improved robustness and coordination in multi-turn interaction. Beyond performance, we introduce and release three hierarchical benchmark datasets, filling a long-standing gap in training and evaluating hierarchical decision-making for LLM-based agents.

## 1. 论文核心问题与研究背景
- **核心问题**：基于大语言模型（LLM）的智能体在动态交互环境中进行**长视界（long-horizon）决策**时，容易出现**目标漂移**——随着交互步数增加，行为逐渐偏离最初意图，且推理过程**令牌（token）效率低下**。
- **整体含义**：现有方法或仅依赖提示（prompting）进行层次分解，或未能对执行层进行有效训练，导致长期交互中误差累积、约束违反和死循环。需要一套**稳定、高效、角色明确**的分层决策框架来解决上述瓶颈。

## 2. 方法论：Multi^2 框架
### 2.1 核心思想与整体架构
- 将智能体分成两个角色明确的系统：
  - **系统1（高层规划者）**：负责根据任务描述和当前观测生成**上下文感知的子目标**。
  - **系统2（低层执行者）**：在给定子目标条件下，执行**原子动作**并与环境交互。
- 两个系统使用同一个预训练骨干网络，但**各自独立的 LoRA 适配器**，实现参数级解耦。

### 2.2 关键技术细节与训练流程
- **分层数据集构造**：将专家轨迹（来自 GPT-4-Turbo 并经规则化处理）拆分为：
  - \( D_{\text{sys1}} \)：任务-子目标序列对，用于规划。
  - \( D_{\text{sys2}} \)：子目标条件下的状态转移序列 \((o_t, a_t, r_t, o_{t+1})\)，用于执行。
- **系统1训练（SFT）**：使用监督学习模仿专家子目标，优化交叉熵损失 \( \mathcal{L}_{\phi} = -\mathbb{E}[\log \pi_{\phi}(g_h \mid o_t; K_n)] \)，确保任务级语义规划的一致性。
- **系统2训练（离线到在线 RL）**：
  - **离线阶段**：结合**策略锚定（policy-anchored）的优势加权模仿**与 Q 函数、值函数学习，稳定初始化执行策略。损失函数包含 TD 误差、期望回归（expectile regression）以及带优势加权的对数似然项。
  - **在线阶段**：从离线策略出发，持续与环境交互，使用**KL 散度正则化**约束在线策略不远离离线策略，并通过优势加权强化学习更新，实现稳定自提升。损失包含加权对数项 \( w_t \log \pi_{\theta}^{\text{on}} \) 和 KL 惩罚项。
- **推理模式**：系统1仅在子目标完成时被重新调用，输出下一个子目标，从而避免不必要的上下文累积，提升令牌效率。

## 6. 主要结论与发现
