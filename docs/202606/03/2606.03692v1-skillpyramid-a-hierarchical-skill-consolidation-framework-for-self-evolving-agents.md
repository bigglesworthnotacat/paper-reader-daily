## 摘要
近期的人工智能智能体能够灵活调用技能来解决复杂任务，但其长期改进从根本上受限于缺乏系统性的技能构建、积累与迁移。特别是，由于缺少统一的技能整合框架，智能体往往在不同任务中重复构建相似能力，无法有效地将经验转化为可复用的资产，并且难以将任务特定的技能泛化到新场景中。为了解决这一局限，我们提出了SkillPyramid，一个技能整合框架，通过复用已有技能经验来实现更广泛的任务泛化。SkillPyramid基于分层技能拓扑，并进一步引入了一种自进化机制，使智能体能够在任务执行过程中组合、验证并吸收新技能。在ALFWorld、WebShop和ScienceWorld上使用四种骨干模型的实验表明，SkillPyramid将平均奖励大幅提升了38.0%，并将执行步骤减少了27.7%。总体而言，我们的方法将技能集合从静态资源池转变为一个动态进化系统。

## Abstract
Recent AI agents can flexibly invoke skills to solve complex tasks, but their long-term improvement is fundamentally constrained by a lack of systematic skill construction, accumulation, and transfer. In particular, without a unified framework for skill consolidation, agents tend to redundantly construct similar capabilities across different tasks, are unable to effectively transform experience into reusable assets, and struggle to generalize task-specific skills to novel scenarios. To address this limitation, we propose SkillPyramid, a skill consolidation framework that reuses existing skill experience for broader task generalization. Operating on a hierarchical skill topology, SkillPyramid further introduces a self-evolution mechanism that enables agents to compose, validate, and incorporate new skills during task execution. Experiments on ALFWorld, WebShop, and ScienceWorld across four backbone models show that SkillPyramid substantially increases the average reward by 38.0% and reduces execution steps by 27.7%. Overall, our method transforms a skill collection from a static resource pool into a dynamic evolution system.

## 2. 论文提出的方法论

### 2.1 核心思想

- 把技能组织成一个**多层次金字塔结构**（SkillPyramid），其中：
  - **底层**：细粒度、可复用的原子操作（atomic skills）；
  - **上层**：抽象的问题解决模式（abstract skills）。
- 通过明确的**重用关系与依赖建模**，使已有技能中的经验能够被提取并用于生成新技能，而非从零开始开发。

### 2.2 关键技术细节与流程

- **技能表示**：技能 `s = (n, d, c)`，包含名称、描述、技能内容（适用条件、执行流程、输入/输出），并增加依赖字段 `δ` 记录对其他技能的引用。

- **金字塔构建**（§2.2）：
  - **关系分析器**（Relation Analyzer）：
    - **粗分组**：基于技能名称和描述，将潜在相关的技能聚类为粗粒组 `G0`。
    - **细粒度组分析**：对每个组内技能进行多轮精细分析，输出关系构建任务 `D = {(g_k, τ_k, e_k)}`，其中 `τ_k` 为关系类型（向下原子提取或向上抽象归纳），`e_k` 为自然语言理由与构建指导。
  - **关系构建器**（Relation Builder）：
    - **向下原子提取** (`Downward Atomic Extraction`)：
      - 针对共享低层可复用操作的技能组 `g_k`，提取**原子技能** `a_k = B_atom(g_k)`；
      - 将源技能 `s_i ∈ g_k` 重写为 `s'_i`，并通过格式化的重用引用 `ρ(a_k)` 调用 `a_k`；
      - 若源技能被完全覆盖，则直接将其作为可复用模块并更新引用；高相似原子技能合并为规范技能；
      - 原子技能放置在**比源技能低一层**的位置：`ℓ(a_k) = min_{s∈g_k} ℓ(s) − 1`。
    - **向上抽象归纳** (`Upward Abstract Induction`)：
      - 针对共享高层问题解决模式的技能组 `g_k`，归纳**抽象技能** `u_k = B_abs(g_k)`；
      - `u_k` 描述如何组织下层技能来求解一类相关任务，并引用组内技能；
      - 抽象技能放置在**比源技能高一层**的位置：`ℓ(u_k) = max_{s∈g_k} ℓ(s) + 1`。
  - 分析器–构建器管道**迭代运行**，直至无新技能产生或达到最大轮数 `T`。

- **技能创造**（§2.3）：
  - 给定新任务 `x`，**技能创建器**从金字塔中检索相关技能集 `R(x)`；
  - 分两阶段生成新技能 `ŝ = C_create(x, R(x), P)`：
    - **框架构建**：优先使用高级抽象技能勾勒任务结构与求解逻辑；
    - **细节实例化**：优先使用低级原子技能填充可执行操作、输入、检查、输出及重用引用。

- **自进化机制**（§2.4）：
  - 当新技能 `ŝ` 被验证后，将其**增量式吸收**进金字塔：
    - 关系分析器仅以 `ŝ` 为查询，检索与之潜在相关的已有技能；
    - 关系构建器按相同流程构建 `ŝ` 与已有技能之间的新重用关系 `∆P_t`；
    - 更新金字塔结构：`P_{t+1} = Update(P_t, ŝ, ∆P_t)`，避免每次全量重建。

## 3. 实验设计

### 3.1 数据集 / 场景

- **ALFWorld**：文本化具身家务任务，包含 seen（140 个任务）和 unseen（134 个任务）两个分割。
- **ScienceWorld**：小学科学实验文本环境，包含 seen（194 个任务变异）和 unseen（211 个任务变异）两个分割。
- **WebShop**：模拟电商网站购物任务，使用 200 个随机采样的任务。
- **GAIA‑Lite**：用于验证金字塔对网络挖掘技能的整合作用，取自 GAIA 的 75 个轻量任务子集（Level 1 和 Level 2）。

### 3.2 对比方法

- **ReAct**：推理与动作交替，不使用外部技能或经验。
- **Reflexion**：在 ReAct 基础上增加基于失败轨迹的语言自我反思。
- **ExpeL**：提取跨任务经验教训并以自然语言知识形式复用，保持扁平记忆。
- **ReAct+Skills**：提供相同的初始技能库，但以扁平方式检索使用，无金字塔组织与进化。

### 3.3 骨干模型

- DeepSeek-V3.2、GPT‑4.1、Gemini 2.5 Pro、Qwen3‑235B，采样温度均设为 0。

## 6. 论文的主要结论与发现

- **性能与泛化**：在所有骨干模型和基准上，SkillPyramid 均取得最佳平均奖励（平均 73.7 v.s. 最佳基线 65.8），尤其在未见任务上优势显著（84.8 v.s. 73.9）。
- **效率**：平均交互步数从 ReAct+Skills 的 17.7 步降至 14.6 步，层次化重用能引导更短、更精准的动作序列。
- **组件贡献**：
  - 原子技能主要提升富含可复用低层操作的任务；
  - 抽象技能在需要多步过程组合的任务上更为关键；
  - 自进化使系统持续适应任务流，冻结金字塔会导致性能退化；
  - 抛弃金字塔、从零生成技能会产生严重噪声，效果甚至低于扁平技能基线。
- **网络挖掘技能整合**：对网络获取的 2,271 个技能进行金字塔整合后，准确率与步数均优于扁平使用，说明金字塔能从嘈杂的大规模技能中提取结构价值。
