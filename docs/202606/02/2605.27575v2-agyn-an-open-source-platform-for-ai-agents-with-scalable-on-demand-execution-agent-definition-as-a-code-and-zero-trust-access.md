## 摘要
随着组织将AI智能体投入生产部署，这些智能体执行非确定性工作流、维护有状态会话，并且通常以特权访问内部服务，工程挑战从构建单个智能体转向以适当的隔离、治理和安全大规模地运营它们。在本文中，我们介绍了Agyn，一个围绕智能体工作负载量身定制的三个关键原则设计的开源平台：基于Kubernetes的信号驱动、有状态的无服务器运行时；用于智能体及工具定义的Terraform提供者；以及基于零信任和最小权限原则的安全模型。Agyn是智能体无关、模型无关和云平台无关的。

## Abstract
As organizations move toward production deployments of AI agents, which execute non-deterministic workflows, maintain stateful sessions, and often operate with privileged access to internal services, the engineering challenge shifts from building individual agents to operating them at scale with proper isolation, governance, and security. In this paper we present Agyn, an open-source platform designed around three key principles tailored for agent workloads: a signal-driven, stateful serverless runtime on Kubernetes; a Terraform provider for agent and harness definition; and a security model grounded in zero-trust and least-privilege principles. Agyn is agent-agnostic, model-agnostic, and cloud-agnostic.

## 1. 研究动机与核心问题
- **核心问题**：企业将基于大语言模型（LLM）的AI智能体投入生产时，面临从单体开发到大规模安全运营的转型挑战。智能体工作负载具有非确定性、有状态、需特权访问内部系统等特性，现有基础设施难以同时满足以下三项关键需求：
  - **高效按需执行**：避免为每个智能体预分配计算资源，需要动态生成实例并在空闲时立即回收。
  - **智能体定义即代码**：智能体的提示、工具、密钥、资源限制等配置应像基础设施一样通过版本控制、代码审查和声明式部署进行治理。
  - **零信任访问**：访问权限必须基于身份、默认拒绝，且凭据绝不能暴露给LLM模型，防止提示注入或模型行为异常导致的泄露。
- **整体含义**：Agyn 作为开源平台，旨在为 AI 智能体提供一套集成上述需求的运行环境，实现智能体无关、模型无关、云平台无关的大规模安全运营。

## 2. 方法论：核心思想与关键技术细节
Agyn 围绕三个设计原则构建系统：

### 2.1 信号驱动的有状态无服务器运行时
- **触发机制**：当对话线程收到新消息时，通知服务发布事件，智能体编排器接收事件并进行协调。
- **按需生成与状态保持**：
  - 编排器解析智能体定义、获取密钥、申请 OpenZiti 身份，并指示 k8s-runner 创建 Kubernetes Pod。
  - Pod 挂载专用持久卷，恢复先前会话的工作区状态，使智能体从中断点继续执行。
- **空闲回收**：活跃时每10秒发送心跳；若空闲超时（默认5分钟），编排器停止Pod并删除网络身份，但保留持久卷和对话历史，下次消息到达时重新附着同一状态。

### 2.2 智能体与工具链的代码化定义
- **Terraform 提供者**：通过声明式代码定义整个智能体及其“全套装备”（容器镜像、工作区配置、系统提示、模型、密钥、MCP 服务器、持久卷等）。
- **模块化与复用**：支持将常用工具链模式打包为可共享的 Terraform 模块，消除重复配置。
- **版本控制**：`terraform apply` 后配置通过网关进入智能体服务，新消息触发新 Pod 时使用更新定义，无需重启。

### 2.3 基于零信任与最小权限的安全模型
- **每容器隔离**：
  - 智能体主容器运行智能体进程，MCP 服务器以 sidecar 容器方式运行，各自拥有独立文件系统和进程树，仅共享回环网络接口。
  - 密钥仅注入需要它们的容器（通常是 MCP sidecar），永不进入驱动 LLM 的智能体容器。
- **零信任覆盖网络**：
  - 采用 OpenZiti 实现，每个智能体 Pod 生成时获得唯一 x509 身份，通过 mTLS 在握手阶段确立调用方身份。
  - 网关提取身份并沿调用链传播，基于属性的访问控制（ABAC）策略限制每个智能体只能访问其显式授权的服务。
- **基于关系的授权（ReBAC）**：
  - 使用 OpenFGA（灵感源于 Google Zanzibar）执行图遍历式细粒度权限控制。
  - 定义智能体角色（所有者、维护者、参与者）控制配置与交互权限，对话线程访问按参与者限定。
