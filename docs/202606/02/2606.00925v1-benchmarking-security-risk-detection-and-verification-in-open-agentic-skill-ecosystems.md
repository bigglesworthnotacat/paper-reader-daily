## 摘要
开放智能体平台允许社区贡献者发布可重用技能，供智能体在运行时调用。这种可扩展性也带来了供应链风险：恶意贡献者可以将有害行为隐藏在表面上无害的技能中。然而，现有防御措施难以评估，因为缺乏同时衡量恶意技能检测与运行时验证的基准。我们提出 SkillVetBench，一个面向开放智能体技能生态的两阶段安全审查基准。第一阶段对每个技能的自然语言规范进行语义审查，以检测隐藏的恶意意图。第二阶段在插桩沙箱中执行被标记的技能，观察运行时行为并收集可审计证据。我们基于公开的 OpenClaw 生态系统中已确认的恶意技能构建基准，包括来自近期 ClawHavoc 供应链攻击的样本。与仅用静态方法不同，SkillVetBench 通过执行轨迹验证检测到的威胁。实验表明：(1) 仅语义和基于签名的基线方法不够充分，会遗漏高达 89% 的恶意技能，这些技能的威胁源于自然语言指令、多组件逻辑或跨组件交互；(2) 运行时攻击集中在少数高权限原语上，尤其是 exec、write_file、install_skill 和 spawn；(3) SkillVetBench 提供的案例研究表明，沙箱执行能够以具体的运行时证据直接支持恶意判定。

## Abstract
Open agent platforms allow community contributors to publish reusable skills that agents can invoke at runtime. This extensibility also creates a supply-chain risk: malicious contributors can hide harmful behavior inside skills that appear benign under superficial inspection. However, existing defenses are hard to evaluate because there is no benchmark that measures both malicious-skill detection and runtime verification. We present SkillVetBench, a two-stage security vetting benchmark for open agentic skill ecosystems. The first stage performs semantic vetting over each skill's natural-language specification to detect hidden malicious intent. The second stage executes flagged skills in an instrumented sandbox to observe runtime behavior and collect auditable evidence. We build a benchmark from confirmed malicious skills in the live OpenClaw ecosystem, including samples from the recent ClawHavoc supplychain campaign. Unlike static-only methods, SkillVetBench verifies detected threats with execution traces. Our experiments show that: (1) semantic-only and signature-based baselines are insufficient, missing up to 89\% of malicious skills whose threats arise from natural-language instructions, multicomponent logic, or cross-component interactions; (2) runtime attacks are concentrated in a small set of high-permission primitives, especially exec, write\_file, install\_skill, and spawn; and (3) SkillVetBench provides case studies in which sandbox execution directly supports malicious verdicts with concrete runtime evidence.

### 2. 论文提出的方法论：核心思想、关键技术细节

**核心思想**：可靠的安全审查必须同时分析技能的“语义意图”和“运行时行为”，并以后者作为最终恶意判定的证据。因此，SkillVetBench采用一个两阶段审查框架：

**阶段一：基于LLM-as-a-Judge的语义分析**

- **输入**：技能的完整构件，包括自然语言说明书（SKILL.md）、可执行脚本、配置文件等。
- **机制**：利用大语言模型（LLM）作为“评判员”，从多个安全维度对技能进行评分，而非简单的二分类。这旨在捕捉隐藏在自然语言指令、跨组件逻辑中的攻击意图。
- **评估维度与指标**：
  - **指令保真度风险 (IFR)**：衡量技能被提示注入或指令覆盖操纵的可能性。
  - **数据引力 (DG)**：评估技能可访问数据的敏感度（公开、机密、受限）。
  - **操作不可逆性 (AI)**：判断技能执行后果是否可撤销（只读、可逆、不可逆）。
  - **爆发半径 (BR)**：估算一次成功攻击可能影响的用户或系统范围。
  - **链式放大效应 (CA)**：评估该技能与其他技能组合时，危险性是否会倍增。
- **综合评分**：由以上维度得分加权计算出一个统一的**技能代理风险评分（SARS）**。SARS得分将技能分为“良性（Benign）”、“可疑（Suspicious）”和“恶意（Malicious）”三类。公式为：SARS = (2*IFR + 1.5*DG + 1.5*AI + 2*BR + 2*CA) / 2.7。

**阶段二：基于Docker沙箱的程序化分析**

- **输入与触发条件**：仅对第一阶段被标记为“可疑”或“恶意”，且同时被独立扫描器（如VirusTotal）告警的技能，才进入沙箱执行。
- **执行环境**：在一个隔离的Docker沙箱中部署标准的OpenClaw代理环境。
- **行为监测**：对技能发出良性和对抗性两种提示进行探测，监控并记录所有工具调用、网络活动、文件系统操作、子进程调用和凭证访问，生成详细的网关日志。
- **最终判定**：“可疑”技能只有在执行日志中产生具体、可归因的恶意行为轨迹时，才会被升级为“恶意（Malicious）”。

### 3. 实验设计：使用了哪些数据集/场景，它的benchmark是什么，对比了哪些方法

- **数据集**：
  - 从公开的ClawHub市场中收集了 **100个技能**，涵盖了`Auto-Updaters`、`ClawHub Typosquats`等10个功能类别。
  - 从中识别出 **78个已被确认为恶意** 的技能（包括ClawHavoc攻击样本）和 **22个良性** 控制样本。

- **Benchmark（评估基准）**：
  技能自身既是评估对象，也构成了基准。评估指标包括：**检出率（Catch Rate）**、**正确告警率（Correct Alarm）**、**检出质量（Detection quality）**、**漏报率（Miss Rate）**。其中，漏报率被视为最关键的安全指标。
  运行时的攻击行为根据一个包含七大类别（命令注入、提示注入、不安全文件操作等）的漏洞分类法进行映射。

### 6. 论文的主要结论与发现

- **现有方法存在巨大盲点**：签名和仅语义方法会遗漏高达**89%** 的恶意技能，其威胁主要隐藏在自然语言指令（如提示注入）和多组件的组合逻辑中，而非独立存在的恶意代码。
- **运行时威胁高度集中**：绝大多数的恶意行为集中于`exec`、`write_file`、`install_skill`、`spawn`等少数几个高权限原语，而只读工具几乎不产生已确认的攻击事件。
- **两阶段审查的必要性**：SkillVetBench是唯一在所有78个恶意技能上实现**零漏报**的系统，且能通过沙箱执行将语义上的“可疑”升级为有据可查的“恶意”判定。静态评分工具（如CVSS）无法捕捉到由链式放大或指令层威胁带来的“组合性风险”。
- **证据驱动的判定是关键**：沙箱生成的运行时证据（如孤儿会话、意外的软件安装、内存投毒行为）可以直接支撑安全判定，弥补了纯静态分析的推测性缺陷。
