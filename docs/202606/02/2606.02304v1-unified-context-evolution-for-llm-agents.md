## 摘要
基于LLM的智能体通过结合推理与环境反馈可以解决多步交互任务，但每个回合都从相同的固定语境开始，沿途中发现的任何有用策略在任务结束时便丢失了。现有方法要么将学习限制在当前任务中，要么将所有经验汇集到一个非类型化的存储中，而不区分知识类型、通过使用追踪质量，或平衡库中仍欠缺的内容。我们提出统一语境演化（UCE），这是一个无梯度框架，将智能体经验外化为一个由类型化的可演化语境单元（ECU）组成的不断进化的库。UCE将经验分解为四种互补类型（记忆、策略、工作流和技能），每种类型在特定条件下从轨迹生成，在决策时检索，通过重复使用结果评分，并在不再有价值时被剪枝。调度模块将每个周期的生成预算分配给库中最薄弱的类型。在两个交互式基准测试中，UCE将ALFWorld成功率从75.4%提升至96.3%，将WebShop任务得分从45.1%提升至61.3%，并且积累的库可在不重新训练的情况下迁移到替代的actor主干上。

## Abstract
LLM-based agents can solve multi-step interactive tasks by combining reasoning with environment feedback, yet each episode starts from the same fixed context and any useful strategy discovered along the way is lost once the task ends. Existing approaches either limit learning to the current task or pool all experience into a single untyped store, without distinguishing knowledge types, tracking quality through use, or balancing what the library still lacks. We introduce Unified Context Evolution (UCE), a gradient-free framework that externalizes agent experience into an evolving library of typed Evolvable Context Units (ECUs). UCE decomposes experience into four complementary types (Memory, Strategy, Workflow, and Skill), each generated from trajectories under type-specific conditions, retrieved at decision time, scored through repeated usage outcomes, and pruned when no longer valuable. A scheduling module allocates each cycle's generation budget toward the types where the library is weakest. Across two interactive benchmarks, UCE raises ALFWorld success from 75.4% to 96.3% and WebShop task score from 45.1% to 61.3%, and the accumulated library transfers to alternative actor backbones without retraining.

## 1. 论文的核心问题与整体含义

*   **核心问题**：当前基于LLM的智能体在执行多步交互任务时，每个回合都从零开始（固定系统提示），无法跨任务保留和复用已探索出的有效经验（如环境机制、决策规则）。现有方法要么将学习限制在当前任务（如自我反思），要么将经验不加区分地混合存储（如集中经验池），缺乏对知识类型（机制、策略、流程、技能）的精细区分、持续质量追踪，以及库内知识结构的平衡。
*   **整体含义**：论文提出 **统一语境演化（UCE）** 框架，其核心思想是 **将语境作为学习的载体**。它通过一个无需梯度更新的外部库，将智能体的历史经验结构化地积累、评估和管理，从而实现跨任务、可自我进化和迁移的智能体。

## 2. 论文提出的方法论

UCE的核心是维护一个由**可演化语境单元（ECU）** 组成的结构化库，并通过一个周期性的**演进循环**来管理其完整生命周期。

*   **核心组件：四种类型的 ECU**
    *   将智能体经验结构化地分解为四种功能互补的类型，每种类型有独立的生成条件、检索范围和淘汰规则。
    *   **记忆 (Memory)**：记录环境如何运作的机制（如“开放水槽上的水盆可以直接清洗可携带物品，无需先开关水龙头”）。
    *   **策略 (Strategy)**：在特定条件下的决策规则，表述为“IF-THEN”形式（如“如果放置失败，则先打开容器再重试”）。
    *   **工作流 (Workflow)**：完成某一类任务的标准执行序列（如“找到目标→转化它→放置到指定容器”）。
    *   **技能 (Skill)**：可跨任务类型复用的多步子操作（如“搜索容器、打开关闭的容器、发现目标后取走”）。

*   **关键机制：ECU 的生命周期与演进循环**
    *   **检索**：在执行新任务时，使用一个结合了语义相似度、任务类型匹配度、质量（适应度分数）的评分函数检索最相关的ECU，并按其类型分别注入Agent的系统提示中。
    *   **适应度评分 (Fitness Score)**：通过拉普拉斯平滑的公式 `f = (s+1)/(u+2)`（`s`为成功/得分，`u`为使用次数）来持续追踪每个ECU的经验价值。新ECU初始分数为0.5。
    *   **五个阶段的演进循环**：每次循环（Cycle）将库从`L_c`更新至`L_{c+1}`：
        1.  **评估 (Evaluate)**：在固定的评估集上测试不带fitbess更新的性能。
        2.  **收集 (Collect)**：在不相交的收集任务集上运行Agent，收集轨迹并更新ECU的适应度分数。
        3.  **调度 (KYS)**：**知识产出调度**模块根据库的当前状态（覆盖率、类型质量缺口、历史生成回报）动态决定本周期为每种ECU类型分配的生成预算。
        4.  **生成 (Generate)**：利用类型特定的生成器LLM，从收集的轨迹中提炼候选ECU，并通过“重排测试 (Reshuffle Test)”过滤掉实例特化的观察。
        5.  **清理 (Cleanup)**：淘汰低价值的ECU，并在同类型/任务组内去冗余，只保留最优的ECU。

## 3. 实验设计

*   **数据集与场景**：使用两个文本交互基准测试：
    *   **ALFWorld**: 一个家居任务模拟环境，包含6种任务类型（如拾取、清洁、加热）的134个任务，评价指标为**成功率 (%)**。
    *   **WebShop**: 一个在线购物环境，包含100个基于Amazon产品的任务，评价指标为**任务得分 (%)** 和**购买成功率 (%)**。
*   **对比方法**：

    *   **跨范式基线**: **ExpeL**（提取并注入混合经验池）和 **Reflexion**（单任务内多次反思重试）。

## 6. 论文的主要结论与发现

*   **性能显著提升**：UCE将ALFWorld成功率从75.4%提升至**96.3%**，WebShop任务得分从45.1%提升至**61.3%**，购买成功率从30.0%提升至**42.0%**。

## 7. 优点

*   **创新性的知识外化框架**：“将经验作为可演进的、类型化的语境库”这一设计，为解决LLM智能体的持续学习问题提供了新颖且有效的思路。
*   **细粒度的知识生命周期管理**：从生成、评估到淘汰的完整生命周期设计，特别是基于使用结果的适应度评分和知识产出调度，使得知识的积累是自驱动、自净化的。
*   **无梯度、易集成**：方法完全基于提示工程，无需修改模型权重，对封闭源LLM友好，且ECU库可迁移至不同Agent架构，松耦合、实用性高。
*   **实验扎实、分析深入**：实验覆盖了主实验、多维度消融、迁移学习、统计检验和深入的案例研究，论证全面、说服力强。

## 8. 不足与局限

*   **环境与模型单一**：仅在两个离散动作空间、文本输入的基准上，使用一组特定的闭源模型对进行了验证。在连续控制、多模态、更长周期规划任务上的有效性未知，且模型特异性（生成器强大蒸馏能力）未被分离。
*   **任务规模有限**：任务池相对较小（134/100个任务），驱动调度信号的样本量有限，在更大规模、更长演进周期下是否会性能饱和或退化是未解问题。
*   **ECU类型的人工设计**：四种知识类型是目前人工定义的，缺乏跨领域时自动发现新知识类型的能力。
*   **固定的生成器预算和退出机制**：生成预算和ECU淘汰等参数可能需要针对新环境进行人工调整。
