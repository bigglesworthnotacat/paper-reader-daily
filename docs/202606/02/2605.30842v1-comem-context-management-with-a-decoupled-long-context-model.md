## 摘要
上下文管理使智能体模型能够通过迭代总结先前的交互历史来解决长周期任务。然而，这一过程通常会因额外的总结词元而产生大量解码开销，从而严重影响部署时的端到端响应延迟。本文提出CoMem，一种将内存管理与主智能体工作流解耦的新框架，使这些过程能够并行执行。我们提出一种k步异步流水线，将内存模型的总结与智能体的推理重叠进行，有效掩蔽上下文处理的延迟。为确保这种异步设置下的鲁棒性，我们引入奖励驱动的训练策略，使内存模型能够捕获对智能体决策而言充分的统计量。理论分析证实，与耦合架构相比，CoMem提供了更优的效率-效果权衡。我们在SWE-Bench-Verified上的大量实验结果表明，CoMem相较普通长上下文方案实现了1.4倍的延迟改进，同时保留了大部分性能。此外，我们证明这些延迟增益随系统吞吐量增加而良好扩展，为智能体推理与内存压缩的独立优化提供了一条模块化路径。

## Abstract
Context management enables agentic models to solve long-horizon tasks through iterative summarization of previous interaction histories. However, this process typically incurs substantial decoding overhead for the extra summarization tokens, which significantly affect the end-to-end response latency at deployment. In this paper, we introduce CoMem, a novel framework that decouples memory management from the primary agent workflow, enabling these processes to execute in parallel. We propose a $k$-step-off asynchronous pipeline that overlaps the memory model's summarization with the agent's inference, effectively masking the latency of context processing. To ensure robustness under this asynchronous setting, we introduce a reward-driven training strategy that aligns the memory model to capture sufficient statistics for the agent's decision-making. Theoretical analysis confirms that CoMem offers a superior efficiency-effectiveness trade-off compared to coupled architectures. Our extensive experimental results on SWE-Bench-Verified show that CoMem provides 1.4x latency improvements upon vanilla long-context solutions while preserving most of the performance. Furthermore, we demonstrate that these latency gains scale favorably with increased system throughput, offering a modular path forward for the independent optimization of agent reasoning and memory compression.

## 1. 研究动机与核心问题
- 现有大语言模型（LLM）智能体在解决长周期任务（如软件工程、网页浏览）时，需要通过迭代总结交互历史来管理上下文，以减少不断增长的输入长度。
- 然而，每次生成这类“摘要”令牌会带来大量额外的解码开销，表现为**高带宽内存（HBM）中 KV 缓存的线性增长**，使解码阶段成为内存带宽瓶颈，严重拖慢部署中的端到端响应延迟。
- 已有的上下文削减（滑动窗口、RAG）会丢失长程依赖；系统级优化（如 PagedAttention、稀疏注意力、KV 缓存量化）虽能提升吞吐，但未解决“每一步解码都要重新编码/关注庞大且相对静态的历史”这一根本冗余。
- 本文核心含义：**将记忆管理（历史压缩）与智能体推理完全解耦**，用一个轻量专用模型在后台异步完成压缩，主智能体仅需输入压缩后的摘要和少量最近步骤，从而大幅降低解码延迟，同时保持决策能力。

## 2. 方法论
### 2.1 CoMem 框架整体设计
- **记忆模型** $f$（轻量长上下文模型，如 Qwen3-4B）将长交互历史 $\tau_{<t}$ 压缩为固定长度的摘要状态 $s_t$。
- **智能体模型** $\pi$（大参数推理模型，如 GLM-4.7）仅依赖 $s_t$ 和一个最近步骤的原始历史缓冲 $C_t$ 产生动作：
  $$a_t \sim \pi(a \mid s_t, C_t;\theta_\pi)$$
  由于 $|s_t| \ll |\tau_t|$，$\pi$ 的输入长度大幅缩短，KV 缓存读取减少，解码延迟下降。

### 2.2 k 步异步管道（核心创新）
- 记忆模型**滞后 k 步**运行：每 k 步启动一次异步压缩，生成整个历史（截至 k 步前）的摘要；在该周期内，智能体继续使用**前一周期生成的摘要**与最近的 k 步原始交互来推理。
- 具体执行（算法1）：
  - 每周期的第一步执行一次**完整预填充**（uncached prefill），因为摘要更新导致 KV 缓存失效。
  - 该周期剩余的 $k-1$ 步则**复用 KV 缓存**，仅对新增的少量观察进行增量预填充，开销极小。
- 这种设计将记忆延迟“掩蔽”在智能体推理的过程中，使两者并行执行，不会增加端到端等待时间。

### 2.3 摘要长度的理论约束
- 设定每词元 KV 缓存大小 $S$、HBM 带宽 $W$、预填充吞吐 $P$、平均生成长度 $Y$，推导出摘要长度相对于原始上下文长度必须满足：
  $$\frac{L_{\text{sum}}}{L_{\text{full}}} < \frac{1}{1 + \frac{W}{Y \cdot S \cdot P}}$$
- 以 Qwen3-32B 在 A100 上为例，该界限约为 0.23，即 32K 词元的上下文可压缩至 7.36K 以内，这通过摘要完全可以实现。

### 2.4 奖励驱动的记忆模型训练
- **目标**：记忆模型的摘要应能恢复全上下文下的智能体行为（功能等价），而非追求表面文本相似。
- **训练管线**：
  1. 冻结智能体模型 $\pi$，用全上下文管道生成**参考轨迹**，记录每一步的“真值动作” $a^*_t$。
  2. 对记忆模型，输入 $\tau_{<t-k}$ 产生候选摘要 $s$；用冻结 $\pi$ 基于 $(s, C_t)$ 产生动作 $\hat{a}_t$。
  3. 定义奖励 $R(s) = \text{sim}(\hat{a}_t, a^*_t)$（余弦相似度）。
  4. 使用 **GRPO**（分组相对策略优化）最大化该奖励，并在训练中施加长度裁剪，使记忆模型学会保留对决策关键的信息。

- 整个过程**无需在线环境交互**，仅用离线数据，大幅降低训练成本。

## 6. 主要结论与发现
- **效率大幅提升**：CoMem 在多种智能体上实现 **1.45× 至 2.08× 的端到端延迟加速**（无 CPU 卸载），GLM-4.7 最高可达 2.08×；在高并发（64）下单步加速更可达 4.95×。
- **性能保持**：经 GRPO 训练后的 CoMem 在解决率上接近甚至超越全上下文基线（如 DeepSWE 上 41.0% vs 40.4%），说明有效过滤噪声。
- **吞吐扩展性**：加速比随批大小增加而提升（256 批 2.52×），证明该架构能缓解内存带宽瓶颈。
- **训练策略关键**：动作一致性奖励显著优于纯 SFT 和基座摘要，且训练无需在线环境。
- **k 值设置容量大**：k=2,4,8 性能稳定，推荐取值较为灵活。
- **模块化部署**：单个 4B 记忆模型可服务约 300 个并发智能体，代价可忽略。

## 7. 优点
- **创新解耦设计**：独立且轻量的记忆模型 + 异步流水线，明确分离上下文压缩与推理，架构清晰。
- **理论支撑**：量化摘要长度界限，为系统配置提供指导。
- **离线训练**：记忆模型训练完全离线，仅需全上下文轨迹，避免在线采样的高成本。
- **高并行性与掩蔽延迟**：k 步异步使记忆处理完全被智能体推理隐藏，不引入额外延迟。
- **即插即用与正交性**：可与稀疏注意力、KV 缓存优化等系统优化共存，不影响主流推理引擎。
- **泛化能力证据**：对 BrowseComp（网页代理）和不同脚手架均有验证，表明方法非局限 SWE 领域。

## 8. 不足与局限
