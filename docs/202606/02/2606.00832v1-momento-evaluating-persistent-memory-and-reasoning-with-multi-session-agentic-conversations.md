## 摘要
近期智能代理领域的进展已使代理能够通过工具使用、推理和多步骤规划来完成复杂任务。然而，现有的基准测试仅评估单一会话中的代理，忽视了代理为满足个性化用户目标而必须整合的过往操作、已声明偏好和先前决策。我们提出了Momento，这是一个用于在多会话服务环境中完成持久代理任务的基准，要求代理在解决时间依赖关系和跨会话不断演变的用户目标的同时，采取具有后果的工具中介行动。实验结果揭示，当前代理的主要失败源于对用户状态的错误估计，它们将先前会话历史视为当前上下文的可靠代理，而非需要重新验证的过时信息，这凸显了当前代理能力与现实长期人机交互之间的巨大差距。

## Abstract
Recent advances in agentic AI have enabled agents to complete complex tasks through tool use, reasoning, and multi-step planning. Yet existing benchmarks evaluate agents within a single session, ignoring past actions, stated preferences, and prior decisions that agents must integrate to fulfill personalized user goals. We introduce Momento, a benchmark for persistent agentic task completion in multi-session service environments, requiring agents to take consequential, tool-mediated actions while resolving temporal dependencies and evolving user goals across sessions. Experimental results reveal that current agents fail primarily through misestimation of user state, treating prior session history as a reliable proxy for current context rather than stale information requiring re-validation, highlighting a substantial gap between current agent capabilities and realistic long-horizon human-agent interaction.

### 2. 方法论

- **核心框架**：Momento 是一个角色扮演型仿真框架，包含 **用户智能体 (User Agent)** 和 **助理智能体 (Assistant Agent)**，二者通过多轮、多会话的对话进行交互，共享一个 **代理记忆模块 (Agentic Memory)**。
- **用户智能体 (πu)** ：接收用户策略 (Φu) 和对话历史 (d<i)，生成多模态回应 ui，并将每条回应写入共享记忆 M，以保持状态连续性。其行为受预定义的人物画像、长期目标、偏好和跨会话条件约束。
- **助理智能体 (πa)** ：依次完成三个主要步骤：
  1. 从记忆模块检索最相关的历史会话摘要 ˆs，采用一种混合检索函数 Θ：先识别时间约束（如某日期范围内的会话），再基于语义相似度排序并选择 top-k 个会话。
  2. 根据检索到的上下文、用户输入和历史，生成工具调用指令 ci，并在环境中执行得到结果 oi。
  3. 基于工具执行结果生成最终回复 ai。
- **代理记忆模块 (Agentic Memory)**：包含两个互补机制：
  - **长期记忆**：通过 LLM 生成 SQL 查询从会话数据库中检索关联历史，结合语义匹配、时间过滤和结构条件，实现对跨会话依赖的显式推理。
  - **短期记忆**（上下文压缩）：在有限的 128k token 上下文中，保留最近 80% 的对话细节原文，对较老的前 20% 进行摘要压缩，确保关键信息留存同时控制开销。
- **基准构建**：由人工标注员设定 161 个任务实例，每个实例包含高层次用户目标、任务目标和一组可执行工具。部分任务涉及多模态理解（如食物图像）。任务跨越多会话，通过隐式记忆依赖、偏好演化、矛盾更新和时间分布信息等设计来增强现实性。历史会话通过人工角色扮演生成。

### 3. 实验设计

- **数据集与场景**：Momento 基准包含 **161 个多会话任务实例**，围绕餐厅服务场景设计，提供 26 种工具（预订、检索、点餐、会员管理、SQL 查询等）。任务涵盖个人助理、规划、调度、推荐和继续任务等，强调偏好变化、中断、未完成任务的重拾、以及隐式引用。
- **对比方法与模型**：使用 **用户智能体仿真器** 固定为 Qwen 3.6-35B-A3B，以控制用户侧行为。评估的助理智能体模型包括：

  - Qwen 3.6-27B

- **评估指标**：
  - **DB_State**：最终数据库状态是否符合目标。
  - **Output**：回应是否满足用户请求。
  - **Tool Recall**：是否按正确顺序调用了正确的工具（对数据变更类检查参数，对查询类检查执行结果）。仅当所有三项指标满足时，任务计为成功。

### 6. 主要结论与发现

- **持久记忆与跨会话推理存在显著缺口**：当前智能体在多会话任务中频繁失败，主要因为**过高估计用户状态的可靠性**，即把陈旧的历史对话视为当前真实状态，未主动调用验证或检索工具来更新假设。
- **失效模式集中于工具召回错误**：研究显示，绝大部分遗漏的工具调用属于**验证和检索类**（如 `get_membership_benefits`），而非最终执行类。这表明智能体在形成当前状态的心智模型时过度依赖记忆，缺乏必要的质疑和确认。
- **稠密模型优于混合专家 (MoE) 模型**，且模型间的一致性较差，Pass^3 普遍较低，揭示了多步任务执行过程的不稳定性。
- **Momento 基准为未来研究提供了基础**，其设计可推动开发能够处理时间依赖、动态目标并能有效利用记忆的持久化智能体。

### 7. 优点

- **首创性**：是第一个明确评估多会话持久任务完成能力的基准，填补了智能体记忆与时间推理评测的空白。
- **真实感设计**：通过角色扮演生成历史会话，并将偏好演化、中断、多重依赖等真实交互特征融入基准任务，贴近现实应用。
- **细粒度错误分析**：不仅给出整体分数，还深入分析工具调用的失败类型和遗漏模式，精准定位了“过度信赖历史状态”这一根本问题。
- **共享记忆与混合检索**：提出的 Agentic Memory 混合了 SQL 时间查询和语义相似度排序，能够更有效地处理跨会话的时间依赖，比纯向量检索更具可解释性和针对性。

### 8. 不足与局限

- **用户仿真依赖 LLM**：用户智能体使用 LLM 模拟，其行为可能不完全真实，存在模拟偏差 (sim-to-real gap)，文中也承认了此局限性。

- **算力与训练细节缺失**：论文不涉及训练，但未说明各模型推理时的硬件环境，也未提及推理延迟等效率指标，部署实用性讨论不足。
