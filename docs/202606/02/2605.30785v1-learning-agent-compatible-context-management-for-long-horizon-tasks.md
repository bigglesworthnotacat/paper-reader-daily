## 摘要
大语言模型智能体在现实应用（如网络搜索与深度调研）中日益面临长时域任务，累积的上下文可能导致长上下文退化与推理失败。先前的工作通过智能体端上下文控制或固定策略（如摘要）进行上下文管理来缓解此问题，但这需要训练智能体自身以适配——这对于闭源智能体不切实际，且忽略了不同智能体可能需要不同策略。我们提出自适应上下文管理（AdaCoM），通过训练外部大语言模型以灵活的修改动作和端到端强化学习来管理冻结智能体的上下文。在多种智能体上进行网络搜索与深度调研基准测试，AdaCoM 通过保留任务约束与进度同时剪除过时内容，显著提升了性能。学习到的策略揭示了一种保真度-可靠性权衡：具有较高原始 ReAct 性能的智能体受益于更高保真度的上下文保留，而性能较低的智能体则需要更激进的压缩以保持在可靠的推理范围内。迁移实验表明，AdaCoM 在能力相似（以原始 ReAct 性能衡量）的智能体之间泛化最为有效，这为智能体系统提供了一条可复用上下文管理器的实用路径。

## Abstract
LLM agents increasingly face long-horizon tasks such as web search and deep research in real-world applications, where accumulated context can cause long-context degradation and reasoning failures. Prior work mitigates this through context management with agent-side context control or fixed strategies such as summarization, which require training the agent itself for adaptation - making it impractical for closed-source agents and ignoring that different agents may require different strategies. We introduce Adaptive Context Management (AdaCoM), which trains an external LLM to manage the context of a frozen agent through flexible modification actions and end-to-end reinforcement learning. Across diverse agents on web search and deep research benchmarks, AdaCoM substantially improves performance by preserving task constraints and progress while pruning stale content. The learned strategies reveal a Fidelity-Reliability Trade-off: agents with higher vanilla ReAct performance benefit from higher-fidelity context preservation, whereas lower-performing agents require more aggressive compression to stay within a reliable reasoning regime. Transfer experiments show that AdaCoM generalizes most effectively across agents with similar capability (measured by vanilla ReAct performance), suggesting a practical path toward reusable context managers for agent systems.

## 2. 方法论

论文提出了**自适应上下文管理框架 AdaCoM**，其核心思想是**架构解耦**与**操作级灵活性**。

- **核心思想**：训练一个外部小型LLM作为“上下文管理器”，对冻结的智能体（保持其参数和提示词不变）进行上下文管理。管理器无需预先定义固定操作（如摘要），而是可以自由地对上下文消息序列进行**修改、删除、合并或保持原样**，从而自主发现与智能体兼容的管理策略。
- **工作流程**：
    1. 智能体基于当前管理后的上下文生成动作，环境返回观测。
    2. 将新动作和观测追加到上下文，形成管理前上下文。
    3. 管理器根据管理前上下文与提示，输出结构化的修改动作列表（JSON格式，包含目标消息ID、角色、新内容、编辑理由）。
    4. 应用修改后得到管理后上下文，供智能体下一步使用。
- **训练方法**：采用**端到端强化学习**训练管理器，保持智能体冻结。训练分为两步：
    - **监督微调（SFT）预热**：用大模型生成的高质量编辑轨迹训练管理器掌握输出格式。
    - **强化学习优化**：使用**GRPO**算法，以任务成功率作为结果奖励，并引入**过程奖励**（如超出长度惩罚、冗余动作惩罚、格式惩罚，以及任务特有的文档命中正向奖励）来改善长期交互中的信用分配。过程奖励与结果奖励通过**两级优势归一化**后结合，以稳定训练。

## 6. 主要结论与发现

- **AdaCoM有效且通用**：在不修改目标智能体的前提下，AdaCoM在4个智能体上平均相对提升39.0%（网络搜索），在深度研究任务上也取得15.2%的平均提升，显著优于固定操作的SumCoM和需智能体自身管理的SumAgent/MemAct。
- **保真度-可靠性权衡**：智能体原始ReAct性能越高，管理器倾向于保留更多原始上下文（高保真度），仅在接近有效上下文长度时进行批量压缩；性能越低，管理器越需要频繁激进压缩（高可靠性），以保持推理健壮。这表明理想的上下文管理不是单一固定操作，而需根据智能体能力在保真度和可靠性间寻求平衡。
- **有效迁移基于能力相似性**：训练好的管理器可在未经专门训练的智能体上迁移，且当源智能体与目标智能体的原生能力（ReAct性能）相近时迁移效果最佳，这为实践中复用管理器提供了明确指导。

## 7. 优点

- **新颖的架构解耦思想**：将上下文管理与智能体完全解耦，不侵入智能体内部，使方法适用于任意闭源或开源智能体。
- **灵活的修改动作空间**：不预设固定操作，让管理器通过RL自主学习每个智能体独有的管理风格，策略可解释、适应性强。
- **扎实的实验分析**：不仅证明了性能提升，还深入揭示了能力与压缩策略之间的内在联系（保真度-可靠性权衡），并系统地验证了跨智能体迁移规律，具有很好的理论价值和工程指导意义。

## 8. 不足与局限

- **任务类型覆盖面有限**：实验仅集中在知识密集型搜索和报告生成任务，未验证在代码智能体、具身智能体或长文本写作等场景下的有效性。
- **管理器容量可能不足**：管理器为4B模型，对依赖极精细证据的高能力智能体可能存在压缩失真，限制了与最新超强闭源智能体的搭配效果。文中指出更强的智能体需搭配更强的管理器。
- **推理开销增加**：每步都需管理器额外推理，增加延迟和token成本。虽然论文指出可按需触发来减轻，但当前系统未体现该优化。
- **KV缓存效率受损**：运行时上下文修改会导致KV缓存频繁失效，论文未针对此进行优化，这是多数上下文管理方法的共有问题。
- **MCP-Bench-Wiki测试智能体较少**：深度研究实验仅包含两个智能体，结论外推性可能受限。
