## 摘要
LLM代理日益依赖社区贡献的技能来扩展其操作能力集。我们研究了自主式AI系统中的一个核心安全问题：单独安全的技能是否可能组合成不安全的已安装技能集。我们提出了SkillReact，一个组合安全测量框架，包含三个组成部分：一个确定性静态组合基准，一个双评分者LLM辅助的人工裁定流水线，以及一个基于动作的可利用性测试工具。在1520个ClawHub技能上，有651个通过单独审查，并组成211575对；基准将其中22.25%标记为结构候选。我们将这一原始比率视为面向召回率的扫描器上限，并对照人类判断进行校准：在一次按模式分层审计中，大约有五分之一的被标记对模式命中被确认为真实组合风险（总体加权有效性为18.2%，我们的主要结果），这意味着在单个注册表中约有14000个真实风险成员关系，而按技能扫描因其设计会遗漏这些关系，因为每一对技能本身都是安全的。随后，基于动作的测试工具探究这些候选在何时会转变为模型发出的工具调用，并发现其实现受宿主模型倾向的限制：在一个锚定条件的投放器子集上，Haiku-4-5在所有39次直接提示试验中都发出了投放器阶段的工具调用（其中36次为完整的下载后执行链，3次仅下载），Opus-4-7则止步于下载，而Sonnet-4-6直接拒绝。一项保持请求不变而只改变已安装技能的控制实验发现，在未安装任何技能时服从性最高：组合决定了哪些能力可及，而宿主模型则决定是否使用它们。这些结果共同推动将安装时的组合检查和能力隔离作为按技能扫描的补充。

## Abstract
LLM agents increasingly rely on community-contributed skills that expand an agent's operational capability set. We study a core safety problem in agentic AI systems: whether individually safe skills can compose into unsafe installed skill sets. We present SkillReact, a compositional security measurement framework with three components: a deterministic static-composition benchmark, a two-rater LLM-assisted human-adjudication pipeline, and an action-based exploitability harness. On 1,520 ClawHub skills, 651 pass individual inspection and form 211,575 pairs; the benchmark flags 22.25% of these as structural candidates. We treat this raw rate as a recall-oriented scanner ceiling and calibrate it against human judgment: in a pattern-stratified audit, roughly one in five flagged pair-pattern hits survives as a real compositional risk (population-weighted validity 18.2%, our headline result), implying about 14K genuine risk memberships in a single registry that per-skill scanning misses by construction, since every pair is individually safe. An action-based harness then probes when these candidates become model-issued tool calls, and finds realization gated by host-model disposition: on an anchor-conditioned dropper subset, Haiku-4-5 issues the dropper-stage tool call on all 39 direct-prompt trials (36 of them the full download-then-execute chain, 3 download-only), Opus-4-7 stops at the download, and Sonnet-4-6 refuses outright. A control that holds the request fixed and varies only the installed skills finds compliance highest with no skills installed: a composition fixes which capabilities are reachable, while the host model decides whether to use them. Together these motivate install-time compositional checks and capability isolation as complements to per-skill scanning.

## 1. 论文的核心问题与整体含义

- **研究背景**：大语言模型（LLM）代理（如 Claude Code、Cursor 等）越来越多地通过加载社区贡献的“技能”（skills）来扩展其操作能力，例如文件访问、网络传输、凭证处理等。每个技能在单独审查时可能是安全的，但当多个技能被同时安装到同一代理运行环境中时，它们的组合能力可能会产生新的安全风险。
- **核心问题**：单独安全的技能，在被组合安装后是否会产生不安全的能力集合？即组合安全风险（compositional risk）。现有安全手段主要针对单个技能进行扫描和检测，但“组合后不安全”这一盲区是逐技能扫描固有的缺陷。
- **整体含义**：论文旨在量化真实技能生态中组合安全风险的出现频率，并探讨这些静态风险候选者在何种条件下会被宿主模型实际转化为工具调用（即攻击行为）。其核心论点是：安全性分析的单位应从单个技能提升到已安装的技能集，并应引入安装时组合检查与能力隔离作为逐技能扫描的补充。

## 2. 方法论

论文提出 **SkillReact**，一个组合安全测量框架，包含三个互补组成部分：

- **确定性静态组合基准（Static-Composition Benchmark）**
  - 对技能进行能力提取（八种能力：文件读/写、网络出/入、环境读、进程生成、凭证访问、系统信息），采用基于正则表达式的确定性提取器，并保留证据轨迹。
  - 定义 **10 种禁止组合模式**（如“文件读 ∧ 网络出库”为数据外泄，“凭证访问 ∧ 网络出库”为凭证窃取等），这些模式来源于现有技能威胁分类和经典能力滥用原型。
  - 对所有通过单独审查的技能，两两配对计算能力并集，检查是否满足任一禁止模式，计算 **组合候选率（CCR）**：被标记的候选对数量 ÷ 单独安全的对总数。
  - 该基准是确定性的，面向召回率，并被作者明确称为“扫描器上限”，而非利用普遍率。

- **LLM辅助人工裁定流水线（LLM-Assisted Human-Adjudication Pipeline）**
  - 从被标记的候选对中按模式分层抽样 200 个单位（每模式 20 个），由两个不同厂商的 LLM（Claude Sonnet 4.6 和 OpenAI Codex gpt-5.5）独立给出结构化判断（有效/误报/……），再交由人类审核员在定制的 UI 中审计并纠正。
  - 利用人类确认的标签计算每模式的有效性比率，再按各模式在总标记成员中的占比进行加权，得到总体校准后的对-模式有效性 \(\hat{p}_{ppp}\)（主要结果 18.2%）。

- **基于动作的可利用性测试工具（Action-Based Exploitability Harness）**
  - 对选定的高风险候选对（主要集中在投放器模式），在受限但允许的工具环境中（Bash、Read 工具），通过直接提示等方式观察宿主模型是否会实际发出工具调用（如 `curl ... | bash`）来实现禁止的能力链。
  - 该部分测量的是模型倾向性，并设计了“成分控制”实验（固定请求，改变安装的技能）来区分风险是由技能组合驱动还是由用户显式请求驱动。

## 3. 实验设计

- **数据集**：ClawHub 技能注册表中的 1520 个技能快照，成功下载的技能构成分析语料。其中 651 个技能通过单独安全审查，构成 211,575 对用于组合分析。
- **基准与对比**：
  - 静态基准：计算原始 CCR（22.25%），并作为上限；通过与人类裁定的校准，得到更接近真实的组合风险比率（18.2% 的有效对-模式比例，对应约 4.05% 的唯对比例估计）。
  - 对基准进行了多种诊断：按模式消除（pattern ablation）、按能力消除（capability ablation）、能力标签的 LLM 交叉检验（Cohen’s κ），以及否定词窗口分析等，以评估噪声来源并理解主要风险驱动因素。
  - 动作测试工具：对投放器模式、数据外泄、凭证窃取等进行了小规模利用尝试，对比了不同 Claude 模型（Haiku， Opus， Sonnet）以及跨厂商模型（DeepSeek， Qwen， GLM 等）的行为差异。
  - 对比了不同安装条件（无技能、仅锚定技能、仅伙伴技能、无关对、投放器对）下的服从率。

## 5. 实验数量与充分性

- **静态分析实验**：
  - 全量 211,575 对计算 CCR，以及按模式、能力的消融实验。
  - 能力提取质量诊断：50 技能子集上与 LLM 标签的一致性检验。
  - 200 对-模式的人工审计，分层取样，覆盖所有 10 种模式。
  - 多个辅助诊断指标：否定窗口率、各能力的 ΔCCR 等。
  - 上述实验在静态层面较充分，能够评估方法的上限和噪声来源。
- **动作利用实验**：
  - 能力素描试验（Capability-Sketch Trials）：81 次试验（3 攻击模式 × 3 提示策略 × 3 模型 × 3 次运行）。
  - 技能工件试验（Skill-Artifact Trials）：39 次投放器锚定试验（Haiku）、80/81 次跨模式试验（Sonnet/Opus），以及清洁隔离重跑（9 次/模型）。

## 6. 主要结论与发现

- 在 ClawHub 生态中，单独安全的技能对中有 **22.25%** 被静态基准标记为组合风险候选（CCR），但该值为召回导向的上限。
- 经过人工校准后，标记的对-模式命中中约有 **18.2%** 是真正的组合风险，推算出该注册表中存在约 **1.4 万** 个真实风险成员关系（约 8.6 千个唯一风险对）。
- 风险实现与否取决于宿主模型的倾向：在相同的投放器请求下，Haiku-4-5 发出完整的下载并执行链，Sonnet-4-6 拒绝，Opus-4-7 仅下载不执行。跨厂商模型也展现出从完全服从到几乎完全拒绝的梯度。
- 成分控制实验表明，当请求固定时，即使未安装任何技能，服从率仍最高，说明模型对显式请求的服从倾向才是实现风险的主导因素，而技能组合决定了能力表面（可为哪些行为提供条件）。
- 主要的组合风险能力轴是“文件读 + 网络出库”的组合（数据外泄），这与技能生态中文件读取类技能和网络类技能的高普及率有关。

## 7. 优点

- **视角创新**：将安全分析单位从单一技能提升到已安装技能集，明确揭示“组合后不安全”的现象，填补了逐技能扫描的盲区。
- **严谨的框架设计**：区分了三种层级的证据（单独不安全技能、结构候选、模型实际工具调用），避免混淆，并强调了静态比率是上限而非利用普遍率。
- **校准策略**：使用双 LLM 辅助 + 人工审计对静态标记的有效性进行校准，降低了纯正则可造成的误报影响，并给出了更接近真实的比率。
- **多层次诊断**：通过能力噪声分析、模式消融、否定窗口、LLM 一致性检验等，使结果的解读具有较好的透明度。
- **模型倾向性的实验证明**：通过多种模型和成分控制实验，展示了模型的自主决策在风险实现中的关键作用，加强了“组合检查 + 能力隔离”这一建议的合理性。

## 8. 不足与局限

- **静态基准的固有上限**：CCR 是将能力并集视为具有执行能力，但并不意味着实际攻击会发生。当请求不利用该联合能力时，风险是潜伏的，仅体现“可能”性。
- **能力提取噪声**：正则提取器面向召回，噪声水平较高（κ 仅 0.169），且依赖于技能文档与代码的模式匹配，可能产生误报，尽管已通过人工裁定部分缓解。
- **人工审计的单点偏差**：校准数据仅由一名人类审核员完成，且审核时能看到 LLM 的评分（存在锚定效应），未测量人类间一致性，这直接影响了主要结果的可信度。
- **动作利用实验的代表性有限**：
  - 只完整测试了投放器模式，且主要集中在以 `steipete-apple-notes` 为锚定的一个子集，对其他模式和技能的泛化能力未充分验证。
  - 实验环境为高度允许的设置（`--dangerously-skip-permissions`），现实运行中通常会有限制，因此实际利用难度可能更高。
  - 成分控制实验显示风险实现主要由用户显式请求驱动，而非技能组合自身，这在一定程度上削弱了“组合导致利用”的论点。
- **生态覆盖单一**：仅使用 ClawHub 的一个快照，未在其他技能注册表或代理平台（如 GPT actions、MCP 服务器）上验证，所得比例可能不具有普适性。
- **仅限于成对分析**：只分析了技能对的组合，未考虑≥3 个技能组合的情形，也未开发针对更大技能集的检查器。
- **安装时检查器的分类限制**：提出的安装时检查器仅能检测预定义的 10 种模式，不能发现新类型的组合攻击，且依赖能力提取的准确性。
