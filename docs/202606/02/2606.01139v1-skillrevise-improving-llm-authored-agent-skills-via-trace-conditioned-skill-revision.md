## 摘要
智能体技能是使LLM智能体能够执行工作流、验证约束并从故障中恢复的程序性构件。现有的自我进化方法利用累积的轨迹来优化技能。然而，它们在冷启动环境中表现挣扎，此时只有初始的不完善技能可用。因此，技能构建默认依赖于专家编写或一次性LLM生成。专家编写的技能成本高昂，且可能不与LLM智能体实际执行任务的方式相吻合，而一次性生成的技能可能在语法上形式完备但在行为上薄弱。为了弥合这一差距，我们提出了SkillRevise，一个基于执行的框架，旨在迭代优化这些初始技能。SkillRevise从执行证据中诊断技能缺陷，从通用记忆中检索相关的修复原则，并应用基于执行的编辑。通过重新执行候选技能并衡量经验效用，它系统地保留最优技能版本。在三个基准测试和五个LLM上的评估中，SkillRevise显著优于一次性基线，将基础智能体在SkillsBench上的成功率从36.05%提高到61.63%。此外，修订后的技能表现出强大的跨模型可迁移性，捕获了超越模型特定产物的通用程序性知识。

## Abstract
Agent skills are procedural artifacts that enable LLM agents to execute workflows, verify constraints, and recover from failures. Existing self-evolving methods refine skills using accumulated trajectories. However, they struggle in cold-start settings, where only an initial, imperfect skill is available. Consequently, skill construction defaults to expert authoring or one-shot LLM generation. Expert-authored skills are costly and may not align with how LLM agents actually execute tasks, while one-shot generated skills can be syntactically well formed yet behaviorally weak. To bridge this gap, we propose SkillRevise, an execution-grounded framework designed to iteratively refine these initial skills. SkillRevise diagnoses skill defects from execution evidence, retrieves relevant repair principles from a general memory, and applies execution-anchored edits. By re-executing candidates and measuring empirical utility, it systematically retains the optimal skill version. Evaluated across three benchmarks and five LLMs, SkillRevise substantially outperforms one-shot baselines, improving the base agent's success rate on SkillsBench from 36.05% to 61.63%. Furthermore, the revised skills exhibit strong cross-model transferability, capturing generalized procedural knowledge over model-specific artifacts.

### 2. 方法论核心思想与关键技术细节
SkillRevise 是一个基于执行的、带约束的迭代修正回路。其关键组成包括：

- **诊断模块（Diagnosis）**：任务特定组件，将原始执行证据转化为结构化的修复约束：
  - \(D_i = (V_i, A_i, K_i)\)，其中 \(V_i\) 为验证规范（输出路径、模式、断言等），\(A_i\) 为失败归因（检查失败、原因、缺陷标签），\(K_i\) 为**保留约束**（已通过的检查及其触因，避免修订破坏已正确行为）。
- **原则记忆（Principle Memory）**：存储**通用修复原则**（而非具体解决方案），每条原则包含触发条件、缺陷类型、修复规则、执行动作模板、验证方式及迁移禁忌。初始记忆包含 7 条原则。
- **修订操作（Revision Operator）**：将当前技能 \(S_i\)、诊断 \(D_i\) 和绑定的原则集 \(P_i\) 映射为候选修订技能 \(\hat{S}_{i+1}\) 及修订轨迹 \(z_i\)。每次编辑必须带有**执行锚**（指定如何修改执行器行为，例如“重新加载文件、解析 JSON、检查必需键”）。
- **有界修订回合（Bounded Revision Episode）**：
  1. 执行当前技能，获得轨迹、验证反馈和成本。
  2. 从执行证据中构造诊断，若缺陷属于技能层面则触发修订；否则跳过。
  3. 基于任务元数据和诊断，利用加权倒数排名融合（sparse + dense）从原则记忆检索前 \(m\) 条候选原则，并绑定满足证据且未违禁迁移原则的原则集 \(P_i\)。
  4. 调用修订算子生成候选技能，**重新执行**该候选并记录结果。
  5. **效用门控选择**：在所有观察到的候选技能集 \(H_{\leq B}\) 中，选取效用 \(U(S,T)\) 最高者作为最终输出（\(S^*_{\leq B} = \arg\max_{S \in H_{\leq B}} U(S,T)\)）。效用函数综合考虑成功率、效率、迁移收益及干扰成本，且效率奖励仅在任务成功时生效，避免偏好过早退出的低成本候选。
- **可选记忆吸收**：事后可从执行证据中抽象出新的通用原则扩充记忆库，但基准测试中未使用在线吸收以避免污染测试任务。

### 7. 优点
- **无需人工干预**：完全自动化，从执行证据中诊断、检索原则并生成修订。
- **结构化修订**：将缺陷诊断、通用原则绑定和执行锚结合，使修订可解释、可追踪，避免自由改写产生的“幻觉”或偏离。
- **效用驱动选择**：不盲目信任最后生成的版本，而是通过经验性重执行和效用比较保留最优技能，减少回归风险。
- **通用且鲁棒**：在多个不同性质的基准和多个强大执行器上均取得一致提升，显示出框架的通用性。
- **跨模型抽象**：修订产出的技能不局限于某一模型，能捕捉通用过程知识，有利于知识复用和迁移。

### 8. 不足与局限
