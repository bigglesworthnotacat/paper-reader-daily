## 摘要
微调大型语言模型通常会破坏其安全对齐，而有害微调攻击进一步加剧了这一问题，此类攻击利用对抗性数据移除安全防护并诱导不安全行为。我们提出 SPARD，一个集成了安全投影交替优化与相关-多样性感知数据选择的防御框架。SPARD 采用 SPAG，在效用更新和基于一组安全数据的显式安全投影之间交替优化，以强制执行安全约束。为了精选安全数据，我们引入了一种相关-多样性行列式点过程，以选择紧凑的安全数据，平衡任务相关性和安全覆盖。在 GSM8K 和 OpenBookQA 上针对四种有害微调攻击的实验表明，SPARD 持续实现最低的平均攻击成功率，显著优于最先进的防御方法，同时保持高任务准确度。代码可在 https://github.com/shuhao02/SPARD 获取。

## Abstract
Fine-tuning large language models often undermines their safety alignment, a problem further amplified by harmful fine-tuning attacks in which adversarial data removes safeguards and induces unsafe behaviors. We propose SPARD, a defense framework that integrates Safety-Projected Alternating optimization with Relevance-Diversity aware data selection. SPARD employs SPAG, which optimizes alternatively between utility updates and explicit safety projections with a set of safe data to enforce safety constraints. To curate safe data, we introduce a Relevance-Diversity Determinantal Point Process to select compact safe data, balancing task relevance and safety coverage. Experiments on GSM8K and OpenBookQA under four harmful fine-tuning attacks demonstrate that SPARD consistently achieves the lowest average attack success rates, substantially outperforming state-of-the-art defense methods, while maintaining high task accuracy. Code is available at https://github.com/shuhao02/SPARD.

## 2. 方法论
### 2.1 安全投影交替优化（SPAG）
- **问题形式化**：将安全微调建模为约束优化问题  
  `min L(D_ft, θ)  s.t.  L(D_safe, θ) ≤ τ`  
  其中 τ 为预设安全损失阈值。
- **交替投影步骤**：
  1. **效用更新**：利用微调数据执行一步梯度下降，得到临时参数 `θ⁺`。
  2. **安全投影**：在 `θ⁺` 处对安全损失做一阶泰勒展开，构建线性化安全半空间约束。若 `L(D_safe, θ⁺) ≤ τ`，则直接接受该更新；否则计算显式投影修正：
     `θ_new = θ⁺ - ((L(D_safe, θ⁺) - τ) / ||g_safe||²) · g_safe`
     并引入信任域半径 `η_safe` 限制单步修正幅度，避免过度调整。
- **特点**：提供闭式投影解，无需手动调节惩罚系数，通过几何投影直接强制安全约束，既保证可行性又保持可解释性。

### 2.2 相关性‑多样性数据选择
- **动机**：实验表明，与微调任务高度相似的安全样本能提供更强的安全信号；但过度相似会导致冗余，使防御覆盖狭窄，攻击成功率反而回升。因此需同时兼顾**相关性**与**多样性**。
- **方法**：提出**相关性‑多样性行列式点过程（Relevance‑Diversity DPP）**。
  - 定义样本 `x_i` 的质量分数 `q_i = max_{x_z∈D_ft} sim(x_i, x_z)`，表征其与微调任务的相关性。
  - 构建核矩阵 `L̂_ij = (q_i·q_j)^β · K(x_i, x_j)`，其中 `K` 捕捉安全样本间的内在相似性，指数 `β` 控制相关性权重。
  - 子集 `C` 的选择概率与 `det(L̂_C)` 成正比，该行列式可分解为 `Π q_i^{2β} · det(L_C)`：前一项鼓励高相关性样本，后一项强制多样性。
- **高效贪婪选择**：因精确求解最大后验子集开销大，采用增量贪心策略。利用 Cholesky 分解递推维护每个候选样本的增益 `L̂_ii - ||w_i||²`，每一步选择增益最大者加入子集，总复杂度为 `O(Nk²)`（N 为候选池大小，k 为目标子集大小），极为高效。

### 2.3 整体框架（SPARD）
- 在微调外循环中，利用 **Relevance‑Diversity DPP** 从事先构建的通用安全池 `GeneralSafe` 中选出大小约为微调数据 3% 的安全子集 `D_safe`。
- 每一训练步执行 **SPAG**：交替进行效用更新与安全投影，保证模型始终不脱离安全区域。

## 8. 不足与局限
- **一阶近似风险**：安全投影基于一阶泰勒展开，在高度非凸的损失曲面下线性化可能不精确，论文虽用信任域和逐步校正缓解，但未给出全局收敛保证。

- **攻击覆盖范围**：实验仅包含四种特定类型的攻击构造，尚不能断言对所有潜在攻击模式均有效。
