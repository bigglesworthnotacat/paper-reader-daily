## 摘要
大型语言模型（LLM）智能体越来越依赖可复用的外部技能来解决长程交互任务。现有的免训练技能适应流程通常基于完整轨迹或会话级反馈来更新技能，这使得故障归因粗粒度，并常常产生不稳定或过于宽泛的修订。我们提出SkillAdaptor，一个具有显式故障归因的免训练步骤级技能适应框架，可接入OpenClaw类智能体框架。给定失败轨迹，SkillAdaptor识别首个可操作的故障步骤，将责任关联到候选技能，并在冻结主干网络的前提下，基于显式接受检查应用目标化更新。我们在WebShop、PinchBench和Claw-Eval上使用Kimi-K2.5、GLM-5和GPT-5.2进行了评估。SkillAdaptor在三个套件上均优于无技能和技能适应基线，最大单项指标提升为：PinchBench平均分% +1.5分，Claw-Eval平均分+1.8分，WebShop成功率+1.7分。这些结果表明，步骤级归因有助于实现更稳定、可审计的免训练技能维护。代码将发布于https://github.com/zjunlp/SkillAdaptor。

## Abstract
Large language model (LLM) agents increasingly rely on reusable external skills to solve long-horizon interactive tasks. Existing training-free skill adaptation pipelines usually update skills from full trajectories or session-level feedback, which makes failure attribution coarse and often produces unstable or overly broad revisions. We propose SkillAdaptor, a training-free step-level skill adaptation framework with explicit failure attribution, and it can plug into OpenClaw-class agent harnesses. Given a failed trajectory, SkillAdaptor identifies a first actionable fault step, links responsibility to candidate skills, and applies targeted updates under explicit acceptance checks while keeping the backbone frozen. We evaluate on WebShop, PinchBench, and Claw-Eval with Kimi-K2.5, GLM-5, and GPT-5.2. SkillAdaptor improves over no-skill and skill-adaptation baselines on all three suites, with the largest single-metric improvements of +1.5 points on PinchBench Avg Score%, +1.8 on Claw-Eval Avg Score, and +1.7 on WebShop success rate. These results indicate that step-level attribution supports more stable and auditable training-free skill maintenance\footnote{The code will be released at https://github.com/zjunlp/SkillAdaptor.}.

### 1. 研究动机与核心问题
- **背景**：LLM 智能体在长程交互任务（网页操作、编程、工具调用）中愈发依赖可复用的外部“技能”（如触发条件、动作模式、验证流程）。现有免训练的适应方法多从完整轨迹或会话级反馈中更新技能库，以整个轨迹为反思和修订的基本单元。
- **问题**：长程轨迹中，最终失败往往仅由少数几个前期错误导致，但轨迹级更新容易将失败信号扩散至无关步骤和技能，导致修订过于宽泛或误导（信用分配问题），尤其当单步早期错误使后续大量动作无效时，技能更新变得不稳定且难以审计。
- **论文目标**：提出一种免训练的、步骤级故障归因的技能适应框架 SkillAdaptor，将适应粒度从轨迹级细化到步骤级，以提升可维护技能库的稳定性和有效性。

### 2. 方法论
#### 核心思想
- 将技能适应过程分解为 **归因（Attribution） → 修改（Modification） → 资格验证（Qualification）** 三个阶段，并仅更新与被定位故障步骤直接相关的技能，冻结主干 LLM。

#### 关键技术细节
- **技能初始化与注入**：
  - 初始无外部技能，先用裸模型执行任务集，归档成功轨迹，从中蒸馏初始技能库 \(K_0\)。
  - 后续执行时，利用嵌入模型 (Qwen3-Embedding-8B) 计算任务描述与技能描述的余弦相似度，取 top-k 候选，再通过主干 LLM 重排序，将精选子集 \(S_q\) 注入上下文。
- **技能适应流程**（迭代至多10轮，或连续3轮技能库不变时提前停止）：
  1. **归因（Attribution）**  
     - **Localizer**：输入失败轨迹，预测最早可操作的故障步骤 \(t^*\) 以及故障行为描述和改进建议 \(\pi\)。  
     - **Linker**：基于 \(t^*\) 估计候选技能的责任权重 \(\{(s_j, w_j)\}\)，并预测适应动作 \(\hat{a} \in \{ \text{REVISE, GENERATE} \}\)（若因已有技能误导则修订，若技能缺失则生成新技能）。
  2. **修改（Modification）**  
     - 若 \(\hat{a} = \text{REVISE}\)：针对权重最高的技能进行最小化定向修订（保留有效部分，添加前提条件、消歧、负面示例、安全检查等）。  
     - 若 \(\hat{a} = \text{GENERATE}\)：从故障上下文合成新技能，并通过语义相似度（阈值 \(\theta_{dup}=0.95\)）过滤冗余项后加入技能库。
  3. **资格验证（Qualification）**  
     - 分别在原技能库 \(K\) 和候选更新库 \(K^+\) 下重新执行任务，比较期望反馈指标差 \(\Delta\)；仅当 \(\Delta \ge 0\) 时接受修订，否则丢弃修改并保留原库，以此抑制有害更新。

#### 公式与流程要点（文字表述）
- 任务执行：\( Run(q, K) \) 使用从 \(K\) 中检索的技能子集。
- 目标：最大化期望成功率 \( \max_K \mathbb{E}_{q \sim Q}[ success(Run(q, K)) ] \)。
- 更新判定：\( \Delta = \mathbb{E}_{q \sim Q}[M(q;K^+)] - \mathbb{E}_{q \sim Q}[M(q;K)] \)，只接受 \(\Delta \ge 0\) 的修订。

### 6. 主要结论与发现
- SkillAdaptor 在所有三个基准和三个主干模型上一致超越无技能及多种技能适应基线，证实步骤级故障归因的有效性。
- 最大单指标提升：PinchBench Avg Score% +1.5，Claw-Eval Avg Score +1.8，WebShop 成功率 +1.7，尤其在长程工具使用环境中增益显著。
- 消融实验表明，移除步骤级归因组件或验收门控会导致性能下降、方差增大，验证了各模块的必要性；仅靠成功轨迹蒸馏的初始技能库几乎无增益，说明有效的适应取决于精准的失败归因和资格控制，而非技能数量堆积。
- 适应更新集中在最初几轮，大部分主要故障模式在前两轮被修正，后期性能提升趋于平缓，此时性能受限于主干模型本身能力。
- 技能注入虽增加单步上下文 token 数，但降低了平均交互步数，表明决策负担部分从多步探索转移到单步上下文推理。

### 7. 优点
- **步骤级归因**：首次在免训练技能适应中引入显式步骤级故障定位和责任链接，大幅缓解粗粒度轨迹级更新的信用分配问题。
- **模块化与通用性**：可插入 OpenClaw 类智能体框架，无需修改主干模型，训练零开销。
- **稳定化机制**：通过资格验证只接受非负增益更新，有效抑制有害修订，控制技能库质量。
- **实验严密**：在多个主流基准和多种模型上验证，消融充分，结果一致性强，并分析了资源开销与适应轮次特性。
