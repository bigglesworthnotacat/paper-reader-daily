## 摘要
智能体系统越来越多地使用文本技能来编码可复用的任务流程，但在每一步将这些技能注入提示中会带来大量上下文开销，并以明文形式暴露技能内容。我们提出LatentSkill框架，通过预训练的超网络将文本技能转化为即插即用的LoRA适配器。LatentSkill将技能知识存储在权重空间而非上下文空间中，消除了每步的技能令牌，同时保留了模块化加载、缩放和组合能力。在ALFWorld和Search-QA任务中，LatentSkill优于对应的上下文技能基线，同时显著减少了预填充令牌数量：在ALFWorld的已知和未知划分上分别提升了21.4和13.4个成功点，预填充令牌减少了64.1%；在Search-QA上精确匹配提升了3.0个点，技能令牌开销降低了72.2%。进一步分析表明，生成的技能LoRA形成了结构化的语义几何，可通过LoRA缩放系数精确控制，并且在技能组件对齐时可通过参数空间算术进行组合。这些发现表明，权重空间技能为扩展大语言模型智能体提供了一种高效、模块化且较少暴露的基底。

## Abstract
Agent systems increasingly use textual skills to encode reusable task procedures, but injecting these skills into the prompt at every step incurs substantial context overhead and exposes skill content as plaintext. We present LatentSkill, a framework that converts textual skills into plug-and-play LoRA adapters through a pretrained hypernetwork. LatentSkill stores skill knowledge in weight space rather than context space, removing per-step skill tokens while preserving modular loading, scaling, and composition. On ALFWorld and Search-QA, LatentSkill outperforms the corresponding in-context skill baseline while using substantially fewer prefill tokens: it improves ALFWorld success by 21.4 and 13.4 points on the seen and unseen splits with 64.1% fewer prefill tokens, and improves Search-QA exact match by 3.0 points with 72.2% lower skill-token overhead. Further analysis shows that generated skill LoRAs form a structured semantic geometry, can be precisely controlled via the LoRA scaling coefficient, and can be composed through parameter-space arithmetic when skill components are aligned. These findings suggest that weight-space skills provide an efficient, modular, and less exposed substrate for extending LLM agents.

# LatentSkill：将大语言模型智能体的上下文文本技能转化为权重内的潜在技能

## 1. 研究动机与核心问题
- **核心痛点**：现代LLM智能体广泛采用“文本技能（textual skills）”——用自然语言描述可复用的任务流程、工具模式与异常恢复策略，并在每一次决策时将相关技能注入提示（prompt）。这种方式虽然模块化，但带来两方面严重问题：
  - **上下文开销巨大**：技能文本在每一步重复注入，消耗大量预填充（prefill）令牌，增加推理成本。
  - **明文暴露风险**：技能内容直接出现在提示中，可能泄露专有流程，且易受恶意指令注入攻击。
- **现有方案的困境**：直接进行agent微调或课程学习虽能避免推理时注入技能文本，但会将技能不可逆地融合进模型参数，失去模块化更新、移除和组合的灵活性。
- **研究目标**：构建一个框架，既能消除推理时技能文本的重复开销，又能保持技能的模块化、可更新性与可组合性，同时削弱明文暴露风险。

## 2. 方法论
- **核心思想**：将文本技能从“上下文空间”迁移至“权重空间”——通过预训练的超网络（hypernetwork）把技能描述文本直接映射为LoRA适配器权重，推理时挂载适配器而不输入技能文本。
- **关键技术细节**：
  - **技能编译器 \(G_\phi\)**：一个基于Transformer的超网络，输入技能文本 \(s\)，输出LoRA增量 \(\Delta_s = G_\phi(s)\)。对骨干LLM的每一个目标模块 \(m\)，生成低秩更新 \(\Delta W_s^{(m)} = B_s^{(m)} A_s^{(m)}\)，挂载时通过缩放系数 \(\alpha\) 控制注入强度：\(W' = W + \frac{\alpha}{r} B_s A_s\)。
  - **训练流程**：
    1. **文档级预训练**：从GitHub爬取约17万篇技能文档（约3亿tokens），训练编译器通过重建或补全技能文档，使超网络学会将流程文本编码为有用适配器。
    2. **轨迹监督微调（SFT）**：使用教师智能体生成的完整任务轨迹（来自ALFWorld和Search-QA），编译器为每个技能生成一个适配器，该适配器在整个轨迹共享，目标为最大化教师输出的对数似然。骨干LLM始终冻结。
  - **推理技能控制与组合**：技能适配器可预计算并缓存；挂载单个技能时可通过 \(\alpha\) 调节影响强度；组合多技能时直接在权重空间相加：\(\Delta_K = \sum_{k \in K} \alpha_k C[k]\)；对于共享子成分的技能，支持组件级分解后再组合，避免过度放大公共行为。
- **公式要点**：条件概率 \(p_{\theta,\phi}(y_t \mid h_t, s) = p_{\theta \oplus \alpha \Delta_s}(y_t \mid h_t)\)，技能文本不再出现在 \(h_t\) 中。

## 3. 实验设计
- **数据集与场景**：
  - **ALFWorld**：文本交互式家居任务，含6类（Pick、Look、Clean、Heat、Cool、Pick2），分Seen/Unseen分割。
  - **Search-QA**：7个搜索增强问答数据集，包括NQ、TriviaQA、PopQA（单跳）以及HotpotQA、2WikiMultihopQA、MuSiQue、Bamboogle（多跳）。
- **基线方法**：Vanilla（无技能）、Few-shot、Reflexion、AdaPlanner、RAG、CoT、R1-Instruct等，核心对比为**In-Context Skill**（将同一技能文本直接插入提示）。
- **评估指标**：ALFWorld用成功率（%）与平均步数；Search-QA用精确匹配（EM）；同时统计预填充/解码令牌数。
- **骨干模型与超网络实现**：Qwen3-8B作为冻结骨干，技能编译器为Transformer超网络，LoRA秩等配置固定。

## 6. 主要结论与发现
- **性能与效率双提升**：LatentSkill在ALFWorld和Search-QA上均显著超越In-Context Skill，同时预填充令牌分别减少64.1%和72.2%；交互步数也下降。
- **结构化语义几何**：生成的技能LoRA在权重空间按领域形成清晰的聚类，且该特性可泛化至未见过的OOD技能（代码、金融、写作）。
- **可控性强**：通过改变LoRA缩放系数 \(\alpha\) 可连续调制技能影响，性能呈倒U曲线，任务难度越高往往需要越大的最优\(\alpha\)。
- **组件级组合可行**：技能按语义组件分解再分别编译后，在参数空间组合能避免直接合并或文本拼接带来的干涉，实现功能互补。
- **安全鲁棒性**：LatentSkill对技能文本的格式、顺序、噪声扰动更为稳健，且能有效抵御提示中的劫持和提取攻击，因为技能不再以明文出现在上下文中。

## 7. 亮点
- **新颖的范式转换**：将“技能即提示”转变为“技能即权重”，是技能存储与调用方式的根本性创新。
- **保留模块化与灵活性**：生成的LoRA可像普通适配器一样加载、卸载、替换或缩放，打破了参数融合方案中技能不可分割的缺陷。
- **训练仅更新超网络**：骨干LLM始终冻结，超网络的训练成本相对较低，且一次生成可复用。
- **多维度性质验证**：不仅关注性能，还系统揭示了权重空间的**结构化**、**可控性**和**可组合性**，为后续研究提供了理论依据。
- **实际部署友好**：推理时零技能令牌，显著降低延迟和成本，同时天然具备一定的防御对抗攻击能力。

## 8. 不足与局限
- **任务与模型范围有限**：仅在两个智能体基准（ALFWorld、Search-QA）和Qwen3-8B上评估，未涉及网页浏览、代码生成、多智能体等更广泛场景，也未探索不同模型家族或规模的影响。
- **LoRA配置固定**：所有实验使用统一的LoRA配置（秩、注入模块等），虽经消融找到高效配置，但未充分讨论配置变化对技能编码性质的影响。
- **组合依赖语义对齐**：组件合并要求技能被分解为语义对应的组件，实际中自动分解与对齐仍是一个挑战。
- **超网络容量**：技能复杂度受超网络参数量限制，对于极长或极其复杂的技能文本，映射能力可能不足。
- **安全性的边界**：虽然明文暴露风险降低，但生成的适配器仍可能被逆向攻击或通过其他侧信道泄露部分信息，文中未深入探讨。
