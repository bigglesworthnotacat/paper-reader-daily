## 摘要
大型语言模型（LLM）作为自进化代理在CUDA内核生成中展现出强大的实证优势，其驱动力来自于跨代际的反馈条件规划。然而，规划决策如何归因并整合异构反馈信号仍不透明。标准端到端消融实验无法解决这一问题，因为迭代规划会放大早期扰动，并将反馈效应与轨迹依赖性漂移混为一谈。我们引入了\texttt{CUDAnalyst}，一个统一的分析层，通过轨迹冻结和选择性反馈注入，实现可控的、代际级别的规划决策对反馈组件的归因。\texttt{CUDAnalyst}支持稳定的代际级别评估，以及反馈效应与交互的基于原则的联盟式归因。我们的结果表明，仅当反馈对齐时，显式规划才是有益的；有效的规划源于结构化的多反馈交互；来自更强推理模型的高级规划可部分迁移到较弱模型。这些趋势在参考骨干网络、代表性工作负载和参考归纳机制中保持一致，表明所识别的反馈到规划结构在所研究的受控轴内是稳健的。

## Abstract
Large language models (LLMs) have shown strong empirical gains as self-evolving agents for CUDA kernel generation, driven by feedback-conditioned planning across generations. However, how planning decisions attribute and combine heterogeneous feedback signals remains opaque. Standard end-to-end ablations fail to resolve this question, as iterative planning amplifies early perturbations and conflates feedback effects with trajectory-dependent drift.   We introduce \texttt{CUDAnalyst}, a unified analysis layer for controlled, generation-level attribution of planning decisions to feedback components via trajectory freezing and selective feedback injection. \texttt{CUDAnalyst} enables stable generation-level evaluation and principled coalitional-style attribution of feedback effects and interactions. Our results show that explicit planning is beneficial only when feedback is aligned, that effective planning emerges from structured multi-feedback interactions, and that high-level plans from stronger reasoning models can partially transfer to weaker ones. These trends hold across reference backbones, representative workloads, and reference induction regimes, indicating that the identified feedback-to-plan structure is robust within the controlled axes studied.

#### 2. 论文提出的方法论：核心思想与关键技术细节
论文提出了一个名为 **CUDanalyst** 的分析层，其核心思想是：**反馈归因必须在固定的“代”（generation）进行**。

- **核心思想 (Trajectory Freezing)**:
    1.  **冻结轨迹**：首先，让一个“参考骨干”模型在标准配置下跑完完整的进化过程，并记录下每一代产生所有程序样本的快照（frozen program state）。
    2.  **局部干预**：在这些冻结的代中进行控制变量实验。保持其他一切（如代码生成器、解码配置、评估流程）不变，只**选择性地注入或删除特定的反馈信号**。
    3.  **归因分析**：通过比较干预前后该代程序样本的执行结果差异，就能将结果变化归因于反馈信号的改变，从而避免轨迹漂移的干扰。
- **关键技术细节**:
    - **模块化设计**：CUDanalyst 将反馈处理与规划解耦，包括一个分析管道（图2），该管道包含调试器（Debugger）、分析器（Analyzer）和性能剖析器（Profiler），并可选择性启用。其输出被送入一个独立的规划代理（PlanAgent）。
    - **联盟式归因 (Coalitional-Style Attribution)**：将每个反馈组件视为博弈论中的“玩家”，将不同反馈组合下的代际性能视为联盟收益。
        - **边际贡献 (Banzhaf Value)**：使用 **Banzhaf值（$\phi_i$）** 量化单个反馈组件的平均边际贡献。选择Banzhaf而非Shapley值，是因为所有反馈是同时被规划器消费的，不考虑排列顺序。
        - **交互效应 (Interaction Term)**：使用 **成对交互项（$\sigma_{ij}$）** 来量化两个反馈组件之间的协同（$\sigma_{ij} > 0$）或冗余/竞争关系（$\sigma_{ij} < 0$）。

### 6. 论文的主要结论与发现
1.  **显式规划并非万能**：只有**当反馈信号与当前程序状态对齐时**，显式规划才有效。没有反馈的规划会损害性能，而反馈对齐的规划则能带来稳定改进。
2.  **有效规划源于多反馈交互**：规划的有效性并非由单个反馈主导，而是源于多个反馈组件的结构性互动（协同）。例如，早期的编译成功依赖静态分析，后期的性能提升依赖性能剖析器，而调试器主要在互动中起作用。
3.  **摘要辅助但不取代规划**：反馈摘要（summarization）能降低反馈复杂性，尤其对弱模型有益，但它无法替代显式规划。规划负责维护和跨代重用决策结构，而摘要只是一个表达形式。
4.  **强模型计划可迁移**：由强推理模型生成的高级计划可以部分迁移到弱模型，指导其生成更优的代码。这种迁移在**同模型族**（如DeepSeek-R1到DeepSeek-V3.2）内效果更好，表明共享的表示结构有利于计划的可解释性。
5.  **鲁棒性结论**：以上发现的“反馈-规划”结构在广泛的骨干模型、工作负载和参考归纳机制下保持稳定，表明这是一个与特定经验和搜索动力学解耦的、架构不变性的内在决策模式。

1.  **方法论的创新性**：提出“轨迹冻结+局部干预”的归因范式，从根本上解决了自进化代理评估中的核心难题（轨迹漂移），为后续研究提供了新的、更精确的分析工具。
2.  **分析的深度与粒度**：超越了传统的“最好结果”比较，深入到**生成级别**，量化了单个反馈组件及其交互的贡献，揭示了决策过程的动态演变。
3.  **结论的普适性**：通过在工作负载、骨干模型、进化算子等多个维度进行大规模的泛化性实验，证明了其结论的稳健性和普遍适用性，而非特定任务的“过拟合”。
4.  **理论到实践的闭环**：基于分析出的不变性规律（如反馈的阶段依赖、计划的可迁移性），设计了轻量级插件 **CugEdit**，并在KernelBench上实现SOTA性能，展示了理论发现对实践设计的指导价值。

### 8. 不足与局限
1.  **归因的边界（Scope of Attribution）**：论文明确指出，其归因是 **“信息性归因”（informational attribution）** 而非严格的“因果归因”。框架冻结了程序状态，只分析了反馈对当前规划决策的影响，但未能解释反馈如何通过跨代的程序状态演化来产生间接的、非本地效应。这仍然是一个开放难题。
2.  **状态表示的局限**：由于编译级别的CUDA内核状态元素（如寄存器分配、内存布局）的演化是一个复杂的“编译器研究挑战”（正如文中引用），论文的工作流仅冻结了 **“隐式记忆状态”（implicit memory states，即冻结的程序代码和参考）**，而无法深入到内核内部的语义演化进行归因。因此，它侧重于反馈->规划这一步，而非长期优化路径的归因。
3.  **资源与环境的限制**：
    - 虽然提出了计算效率更高的`IntervenePipe`，但构建和分析冻结轨迹仍然需要不小的计算资源和时间。
    - 当前验证主要在PolyBench-ACC套件上，虽然进行了泛化测试，但整个方法论的有效性仍需在更复杂、更大规模的真实应用（如大规模深度学习模型训练内核）中进一步验证。
4.  **实验的局限性**：
    - 主要实验环境基于单岛（single-island）进化，以避免种群随机性带来的混淆。虽然这有利于归因分析，但可能无法完全代表多岛、有迁移的复杂进化场景。
    - 模型队列虽然多样，但主要为英文语境的模型。结论对其他语言或非主流架构的通用性有待考证。
5.  **应用的局限**：该方法当前专门针对CUDA内核生成这一特定领域，尽管在CPUNumba上展示了跨域潜力，但其对更通用、更复杂的LLM代理任务（如网页浏览、工具使用）的适用性和有效性尚待探索。
