## 摘要
基于大语言模型的多智能体系统需要许多协调选择，这些选择难以先验固定：调用哪个技能协议，哪个智能体角色应执行子任务，为每个角色绑定哪个模型，角色间如何交互，何时使用检索或验证，以及何时完全省略一个步骤。这些选择与任务领域和操作约束相互作用，因此静态流水线和一次性模型比较只能提供设计空间的有限视角。本文介绍AgensFlow，一个开源框架，将多智能体协调视为部分可观测下的在线策略学习问题。该框架使协调决策可观测且可从重复轨迹中学习，而不是将技能、角色、模型、拓扑和评估选择视为固定的流水线设计。AgensFlow在两个语料库上进行评估：分布式系统事件任务和安全咨询任务。评估显示三个主要结果：在协调密集型类别上，学习到的路由比固定流水线基线达到更高质量的工作点；skip:X将拓扑压缩隔离为基板中有意义的部分；热启动的策略图可以在保持平稳期质量的同时降低探索成本。总体而言，结果支持学习到的、可审计的路由相比静态布线可以改进协调密集型多智能体工作流。

## Abstract
Multi-agent systems built on large language models (LLMs) require many coordination choices that are difficult to fix a priori: which skill protocol to invoke, which agent role should perform a subtask, which model to bind to each role, how roles should interact, when to use retrieval or verification, and when to omit a step entirely. These choices interact with task regime and operational constraints, so static pipelines and one-off model comparisons provide only a limited view of the design space. This paper introduces AgensFlow, an open-source framework that treats multi-agent coordination as an online policy-learning problem under partial observability. The framework makes coordination decisions observable and learnable from repeated trajectories, rather than treating skill, role, model, topology, and evaluation choices as fixed pipeline design.   AgensFlow is evaluated on two corpora: distributed-systems incident tasks and security-advisory tasks. The evaluation shows three main results: learned routing reaches a higher-quality operating point than a fixed pipeline baseline on coordination-heavy classes; skip:X isolates topology compression as a meaningful part of the substrate; and warm-started policy graphs can reduce exploration cost while preserving plateau quality. Overall, the results support that learned, auditable routing can improve coordination-heavy multi-agent workflows over static wiring.

## 2. 论文提出的方法论

- **核心思想**：构建一个**协调策略基板**，将任务抽象为“折叠签名”（regime label + handoff mask + bucketed 信念），在每次路由步骤选择合法动作（调用特定技能+模型，或跳过某步骤），根据轨迹级奖励（质量+成本+重试惩罚）更新策略图，最终形成可审计的路由策略。

- **关键技术细节**：
  - **状态抽象：折叠任务签名** (\(s_t\))  
    由规则型 regime 检测器（straightforward, evidence_heavy, ambiguous, contradictory, high_risk, exploratory）生成 regime 标签 \(ℓ_t\)，结合 7 维 handoff 掩码 \(z_t\) 和四个信念估计（正确性 \(\hat{c}_t\)、不确定性 \(\hat{u}_t\)、矛盾风险 \(\hat{r}_t\)、证据充分性 \(\hat{e}_t\)）组成。信念由代理角色（planner, memory, solver, critic, verifier, synthesizer）的行为更新，签名折叠使得相似任务共享价值估计。

  - **动作空间**：
    \[
    a_t \in \mathcal{A}(s_t) = \{\text{invoke}(k, m)\} \cup \{\text{skip}:X\} \cup \{\text{terminate}\}
    \]
    其中 invoke 指定技能协议 \(k\) 与模型 \(m\) 的组合（实验中使用 9 个求解器变体），skip:X 允许省略某一尚在计划中的步骤（如检索、验证），terminate 由预算耗尽或治理层触发，非策略直接选择。

  - **奖励函数**：
    \[
    r(t) = w_q Q(t) - w_c C(t) - w_\rho \rho(t)
    \]
    质量分 \(Q(t)\) 由 RelativeJudge 提供（相对排序、跨评委平均、置信度加权）；成本 \(C(t)\) 为归一化 token 消耗；\(\rho(t)\) 为重试或失败惩罚。

  - **策略学习与动作选择**：使用**可靠性感知的 UCB1** 变体：
    \[
    \text{score}(s,a) = \bar{r}(s,a) + c_s \sqrt{\frac{\log(N_s+1)}{N_{s,a}}} - \lambda f(s,a)
    \]
    其中 \(c_s\) 按签名访问量退火（初始 1.4，衰减至 0.5），\(f(s,a)\) 为失败率，\(\lambda=0.5\) 惩罚反复故障的动作。策略图以 (签名, 动作) 为键存储统计量，可持久化和审计。

  - **框架六大层**：Layer 0（搜索包装器）→Layer 1（Schema、regime、技能注册）→Layer 2（代理 + 传输 + 轨迹）→Layer 3（策略图 + 路由器 + LangGraph 集成）→Layer 4（奖励与 RelativeJudge）→Layer 5（预检、治理、报告），所有层运行时同时活跃。

- **奖励审计**：RelativeJudge 对同类任务轨迹进行相对排序，并引入跨模型家族的评委平均（如 Claude + GPT + Qwen），输出轴级得分（目标达成、证据支撑、协调、恢复），实现奖励信号的可观测性，降低单一评委偏差。

## 3. 实验设计

- **数据集/场景**：
  - **语料 1**：60 个分布式系统事件任务（Paxos/Raft、逻辑时钟、gossip、CRDT、一致性哈希等），用于生成热启动策略图。
  - **语料 2**：60 个合成安全咨询任务，涵盖 8 种场景类别：C1 程序性、C2 单文档、C3 跨文档多厂商推理、C4 综合、C5 语料外模糊问题、C6 程序性衍生、C7 缓解正确性、C8 跨厂商对。两类语料共享特征分类和动作界面，形成跨域迁移测试。

- **对比方法（4 个臂）**：
  1. **固定基线**：7 单元固定流水线，单次通过，无学习。
  2. **无跳过消融（no-skip ablation）**：8 个 epoch，启用变异池学习但强制关闭 skip:X。
  3. **主要（main）**：8 个 epoch，skip:X 开启，冷启动。
  4. **热启动（warm-start）**：8 个 epoch，skip:X 开启，用语料 1 学到的 443 节点策略图初始化。

- **评价指标**：RelativeJudge 质量分（单评委 RJ(sj) 与三评委 RJ(3j)），token 消耗，各类别质量提升量。

## 6. 论文的主要结论与发现

1. **学习型路由在协调密集型任务上显著优于静态流水线**：在跨文档多厂商推理（C3，+0.18）、跨厂商对分析（C8，+0.16）、缓解正确性（C7，+0.13）等类别上，质量提升明显；程序性类别仅有微微差距或持平。
2. **skip:X 机制有价值**：策略学会了按任务类别跳过某些步骤，从而压缩拓扑；学习曲线显示早期成本下降后质量微降再平衡，说明跳过决策受奖励反馈调节。
3. **热启动策略图能降低探索成本并保持平稳期质量**：热启动在 plateau 用更少 token（约 −10%）达成与冷启动可比的 audited 质量（+0.012），全期 token 节省约 21%；但单评委信号会夸大质量增益，凸显奖励审计的必要性。
4. **奖励审计必须内置**：不同评委给出不同结论，表明协调策略的评估本身脆弱，因此需要将奖励信号作为系统的一部分进行多评委审计，而不能当作中性测量。

## 7. 优点

- **问题视角转换**：将协调设计空间建模为可学习的策略问题，相对于固定流水线更灵活、适应性强。
- **框架全面性与可审计性**：技能、模型、拓扑的选择全部记录在策略图中，可以事后审查，满足生产系统的透明性需求。
- **skip:X 拓扑学习**：将步骤省略作为一等动作，使拓扑可学习而非硬编码，实验证明其对成本和质量平衡有积极作用。
- **奖励系统的严谨设计**：相对排序、跨模型评委平均、置信度加权和轴级分解，有效缓解了奖励黑客和单点评估偏差。
- **迁移学习验证**：跨域策略图热启动显示了减少探索开销的潜力，为复用协调知识提供了实证支持。
- **开源与可复现**：框架和实验均可端到端复现。

## 8. 不足与局限

- **拓扑结构单一**：评估仅覆盖了线性带跳过拓扑，未测试并行、分层分解、分支合并等更复杂的协调模式，不能代表所有可能的拓扑设计。
- **任务特征分类局限**：两个语料库共享同一套特征分类法，迁移实验未测试特征分类本身变化时的泛化能力。
- **缺乏与现有多智能体框架的对比**：仅与内部固定基线比较，未与 AutoGen、CAME等显式对话型多智能体系统进行横向对比，外部适用性参考有限。
- **奖励仍依赖大语言模型评委**：虽然引入了多评委审计，但评分本身仍由大语言模型生成，存在潜在偏差或 hallucination 风险，且未与人评估进行校准。
- **信念估计为启发式**：当前信念更新基于简易规则，非学习得到，可能影响签名抽象的信息保真度。
- **安全咨询任务为合成数据**：可能未完全反映真实场景的复杂性与噪声。
