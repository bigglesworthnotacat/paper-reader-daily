## 摘要
多智能体大语言模型系统通过组合多个智能体的输出来提升推理能力，但交互密集的方法可能引入错误传播和较高的通信开销。当智能体交换原始响应或推理轨迹时，错误的中间推理可能被采纳并放大，导致自信但错误的共识；多轮通信也增加了 token 消耗、延迟和推理成本。本文提出了一个名为 DarkForest 的受控通信协调框架。DarkForest 首先保持智能体独立，使每个智能体在未见其他智能体输出的情况下生成答案。然后，它将原始响应解析为结构化的候选记录，将语义等价的候选归为簇，并利用智能体可靠性、置信度、解析质量、支持模式可靠性和独立性校正，估计这些簇上的校准置信分布。协调器仅通过受控通信从此置信状态接收策略允许的证据。在六个推理基准上的实验表明，DarkForest 取得了领先的整体质量，在基准指标上将最强基线提高了多达 30.7%，且与通信密集的基线相比，token 消耗降低多达 6.5 倍。

## Abstract
Multi-agent LLM systems improve reasoning by combining outputs from multiple agents, but interaction-heavy methods can introduce error propagation and high communication overhead. When agents exchange raw responses or reasoning traces, incorrect intermediate reasoning may be adopted and amplified, leading to confident but wrong consensus; multi-round communication also increases token consumption, latency, and inference cost. In this paper, we propose a controlled-communication coordination framework named DarkForest. DarkForest first keeps agents independent, so each agent produces an answer without seeing the others' outputs. It then parses the raw responses into structured candidate records, groups semantically equivalent candidates into clusters, and estimates a calibrated belief distribution over these clusters using agent reliability, confidence, parse quality, support-pattern reliability, and independence corrections. A coordinator receives only policy-permitted evidence from this belief state with controlled communication. Experiments on six reasoning benchmarks show that DarkForest achieves leading overall quality, improves the strongest baseline by up to 30.7\% on benchmark metrics, and reduces token consumption by up to $6.5\times$ compared with communication-heavy baselines.

### 2. 论文提出的方法论

DarkForest 是一种测试时协调框架，其核心思想是“独立产生候选、校准聚合、受控通信”。整体流程如下：

- **独立候选生成**：所有智能体在彼此不暴露输出的情况下，独立为相同输入 `x` 生成响应 `r_i`，以保留尽可能多的条件独立性，避免运行时产生额外的依赖。
- **解析与规范化**：
  - 通过特定任务解析器 `P` 将原始输出 `r_i` 转换为结构化观察 `o_i = (a_i, z_i, c_i, v_i, q_i)`，包含候选答案、规范形、置信度、解析有效性和解析质量元数据。
  - **规范形**用于将语义等价的候选映射到同一表示，仅在紧凑的候选记录上进行协调。
- **候选聚类**：将兼容的解析候选分组为簇 `C_z = (z, S_z, π_z)`。其中 `S_z` 表示支持候选 `z` 的智能体集合，`π_z` 为支持模式（即记录是哪个或哪些智能体支持了该候选，而不仅仅是数量）。
- **构建校准信念分布**：
  - 为每个候选簇 `C_z` 分配一个校准证据得分：
    `s(z) = R_{π_z} · Σ_{i∈S_z} α_i · ρ_i · δ_i · ϕ(c_i)`
    - `α_i`：智能体 `i` 的校准可靠性；
    - `R_{π_z}`：支持模式 `π_z` 的校准可靠性（评估联合一致模式的可靠性，如互补智能体的一致比高度相关智能体的一致提供的证据更强）；
    - `ρ_i`：解析质量惩罚（如输出格式异常但可解析时降低权重）；
    - `δ_i`：独立性校正折扣因子（当支持智能体已知相关时，防止将其视为完全独立的证据源）；
    - `ϕ(c_i)`：置信度乘数 `0.5 + c_i`，将置信度作为弱调制信号。
  - 得分归一化为后验类信念分布 `P(z|O)`，并识别最高后验候选 `z*` 及后验边缘 `Δ`。
  - 校准阶段在留出数据上离线进行，估算可靠性项，不更新模型参数。
- **受控披露与协调**：
  - 定义披露算子 `D_Ω`，仅将策略 `Ω` 允许的紧凑证据 `E`（如候选标识符、规范候选、置信度、信念摘要，而非完整推理痕迹）暴露给协调器。
  - 协调器 `m_coord` 接收原始输入 `x` 和暴露证据 `E`，生成最终候选 `z_coord`。信念状态被视作先验而非证明。
- **确定性护栏**：当信念状态强烈支持一个与协调器输出冲突的候选时，护栏会介入并采用 `z*`。这确保语言模型推理与确定性证据检查是分离的。

- **质量与成本双优**：DarkForest 在六个基准上取得了领先或次优的任务质量，相比通信密集的基线，token 消耗最多降低 **6.5 倍**，相对最强基线在指标上提升高达 **30.7%**。

- **披露越多不一定越好**：向协调器暴露过多信息（如完整推理痕迹）反而可能降低质量并显著增加成本，凸显了紧凑、校准证据的价值。

### 8. 不足与局限

- **校准依赖留出数据**：校准过程需要特定任务的留出数据集来估算可靠性参数，这可能限制了其在新任务上的零样本泛化能力。

- **离线校准的静态性**：校准参数在测试时是冻结的，无法根据推理过程中的动态变化进行调整。
