## 摘要
基于大语言模型（LLM）的智能体越来越多地被用于需要规划、工具使用和与外部服务交互的复杂任务。它们对不可信外部内容的依赖使其暴露于间接提示注入（IPI）之下，其中嵌入在检索数据中的对抗性指令劫持了智能体的行为。现有攻击依赖于无法适应特定智能体防御的静态载荷；即使最近的适应性方法也缺乏结构化的反馈来指导优化。我们引入了 IterInject，一个反馈引导的迭代框架，它闭合了注入、诊断和优化之间的循环：一个基于规则的诊断器生成带有行为描述的结构化结果标签，一个基于 LLM 的优化器根据完整的优化历史来优化载荷。一个合成步骤从失败模式中生成新的伪装种子，使策略空间能够自我演化。在 AgentDojo 和 InjectAgent 上，IterInject 在四个受害者模型上显著优于静态基线和现有的适应性方法。在 Claude Code（一个带有分层防御的生产级编码智能体）上的扩展实验显示，优化后的载荷在 9 个目标中的 5 个上取得了完全成功；即使那些抵抗完全利用的目标也显示出迭代优化带来的可衡量的改进。我们进一步对 IPI 进行了机制分析，识别出中间到后期层中存在一种注意力介导的阈值机制；三种因果干预验证了这一发现，并指向了具体的防御方向。

## Abstract
LLM-based agents are increasingly deployed for complex tasks requiring planning, tool use, and interaction with external services. Their reliance on untrusted external content exposes them to indirect prompt injection (IPI), in which adversarial instructions embedded in retrieved data hijack agent behavior. Existing attacks rely on static payloads that cannot adapt to agent-specific defenses; even recent adaptive methods lack structured feedback to guide optimization. We introduce \oursys, a feedback-guided iterative framework that closes the loop between injection, diagnosis, and refinement: a rule-based diagnoser produces structured outcome labels with behavioral descriptions, and an LLM-based optimizer refines payloads conditioned on the full optimization history. A synthesis step generates new disguise seeds from failure patterns, enabling the strategy space to self-evolve. On AgentDojo and InjectAgent, \oursys substantially outperforms static baselines and existing adaptive methods across four victim models. Extension experiments on Claude Code, a production-grade coding agent with layered defenses, show that optimized payloads achieve full success on 5 of 9 targets; even those that resist full exploitation exhibit measurable improvement from iterative refinement. We further present a mechanistic analysis of IPI, identifying an attention-mediated threshold mechanism in mid-to-late layers; three causal interventions validate this finding and point to concrete defense directions.

### 2. 方法论

- **核心思想（闭环迭代）**：IterInject 框架由三个关键环节构成闭环：
  - **注入与诊断**：将当前载荷注入受害智能体的外部上下文，利用**基于规则的反馈诊断器**解析智能体的响应，输出**四等级的结构化标签**（`SUCCESS`、`PARTIAL`、`DETECTED`、`IGNORED`）和具体的行为描述。
  - **迭代优化**：利用一个**LLM驱动的载荷优化器**，以上一轮结果的全部优化历史（`H_i`）为上下文，生成下一轮精炼后的载荷（`p_{i+1}`），当达到成功、超过无改善忍耐上限或耗尽预算时终止内循环。
  - **种子合成与自进化**：每完成一定批次（`K`个）种子后，通过**规则选择**（如拼接、格式转换、伪装、升级、重复等策略）和**LLM合成器**，从失败模式中生成全新的伪装种子并加入种子库，使策略空间**持续自我扩大**。
- **关键技术细节与流程**：
  - **种子库（Seed Bank）**：从大规模公开红队提交中抽取伪装模板，并为每个种子维护累计得分 `S(d)`，以决定评估与应用优先级。
  - **载荷初始化**：`p_0 = d \oplus a`，即将伪装种子与攻击目标拼接。
  - **诊断反馈**：状态标签映射为数值（3/2/1/0），同时提供为何如此判断的自然语言说明。
  - **载荷精炼公式**：`p_{i+1} = \mathcal{M}(p_i, f_i, H_i)`。
  - **跨目标得分传播**：对所有攻击目标按序评估，尝试过的种子按平均数值得分更新，未尝试的种子赋予该目标上所有种子的均值 `\bar{v}_m`，做到**有效种子自然提前、未被试过的种子不被永久压底**。
  - **种子合成**：诊断器根据批次失败记录的多数标签决定策略（SPLICE/CAMOUFLAGE/FORMAT\_SHIFT/ESCALATE/REPEAT），然后由LLM合成器产生新种子。

### 6. 主要结论与发现

- **攻击有效性**：IterInject在AgentDojo上对所有4个受害模型的综合ASR均超过基线，其中在DeepSeek上提升最大（47.8% vs. 32.9%），且JSR最高。在InjectAgent上，将多数模型的ASR从接近零提升至33%–90%，总体优于AgentVigil。
- **组件协同性**：消融实验证实**种子初始化、迭代优化和上下文学习三者存在超加性（super‑additive）增益**。
- **对防御的鲁棒性**：即便开启AgentDojo的多种防御，IterInject仍保持最高ASR（如Delimit防御下18.8%）。
- **真实系统脆弱性**：Claude Code实验中，全框架在9个目标中使5个达到完全成功，其余4个均有可量化的改善，凸显生产级智能体依然易受迭代攻击。
- **IPI的内在机制**：
  - 识别出中后期层（如31–47层）为**注意力放大区**，成功注入在该区域对载荷token有显著更高注意力。
  - 存在**类似阈值的决策边界**：成功和失败案例均表现高置信度、低熵，但部分成功案例熵值显著升高；成功与失败案例在隐藏状态空间中高度相似，其区分主要由注意力重分配驱动。
- **因果干预验证**：在注意力放大区敲除载荷注意力可逆转32%的成功样本；注意力增强可大幅提升失败样本的成功率，且逆转比例随偏置大小单调增加，证实了注意力阈值的因果作用。

### 7. 优点

- **方法创新性**：将**结构化多级诊断反馈**与**LLM驱动优化**和**种子自我合成**相结合，形成闭环自进化框架，显著优于依赖粗糙反馈或固有种子的现有工作。
