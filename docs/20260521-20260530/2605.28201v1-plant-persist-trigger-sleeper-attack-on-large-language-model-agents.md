## 摘要
大语言模型（LLM）智能体仍易受来自外部环境的安全威胁，攻击者将对抗性内容注入外部观测（如工具返回数据、网页或MCP上下文），引发有害的智能行为，例如不安全操作或错误输出。现有研究通常关注单次交互攻击，即智能体在观察到对抗性内容后，立即在一次用户请求内表现出有害行为。然而，我们表明，对抗性内容也可以在同一智能体服务的多次交互间持续存在，使得此类威胁更难检测和缓解。具体而言，对抗性内容可能持久存在于智能体状态中，在交互间保持休眠，随后被良性用户查询激活。我们将这类安全威胁形式化为休眠攻击（Sleeper Attack）。为了对其进行评估，我们构建了一个包含1,896个实例的基准，涵盖六种现实世界有害结果、三种攻击策略和三种智能体状态目标：会话上下文、记忆和可重用技能。在七个强大的开源和闭源LLM上的实验表明，即使最先进的LLM智能体在单次交互基线下攻击成功率较低，它们仍然易受休眠攻击。我们的代码和数据可在https://anonymous.4open.science/r/skdvnfu23ihr9wdscnksf1asdffsaef获取。

## Abstract
Large Language Model (LLM) agents remain vulnerable to safety threats from the external environment, where attackers inject adversarial content into external observations such as tool-returned data, webpages, or MCP context, causing harmful agentic behaviors such as unsafe actions or incorrect outputs. Existing studies typically focus on single-interaction attacks, where the agent observes adversarial content and immediately exhibits harmful behavior within one user request. However, we show that adversarial content can also persist across interactions served by the same agent, making such threats harder to detect and mitigate. Specifically, adversarial content may persist in the agent state, remain dormant across interactions, and later be activated by a benign user query. We formalize this type of safety threat as Sleeper Attack. To evaluate it, we construct a benchmark with 1,896 instances covering six real-world harmful outcomes, three attack strategies, and three agent state targets: session context, memory, and reusable skills. Experiments on seven strong open-source and closed-source LLMs show that state-of-the-art LLM agents remain vulnerable to Sleeper Attack, even when they achieve low attack success rates under a single-interaction baseline. Our code and data are available at https://anonymous.4open.science/r/skdvnfu23ihr9wdscnksf1asdffsaef.

- **核心问题**：本文发现对抗内容可以**跨交互持久化**：它被植入智能体状态（如会话上下文、记忆、可重用技能），保持“休眠”，并在后续**良性用户请求**时被激活，执行有害操作。这种“休眠攻击 (Sleeper Attack)”更隐蔽、更难检测与缓解。

### 2. 方法论
- **核心思想**：将休眠攻击定义为 **植入 (Plant) → 持久 (Persist) → 触发 (Trigger)** 三阶段过程。攻击者利用一次交互将对抗信息写入智能体状态，该信息在后续多次交互中保持不活跃，直到某一无害查询将其激活。
- **数学形式化**：  
  - 将智能体系统抽象为由 LLM 策略 `πθ`、外部环境 `E` 与状态转移函数 `f` 构成的交互过程。  
  - 单次交互攻击中，对抗观测 `oadv_t,i` 立即导致当前交互历史 `hadv_t` 不安全 (`U(hadv_t)=1`)。  
  - 休眠攻击中，`oadv_t,i` 不立刻触发不安全行为，但存储在状态更新 `sadv_{t+1}=f(s_t, hadv_t)` 中，并在未来第 `t+k` 次交互中由良性查询 `x_{t+k}` 唤醒，导致 `U(hadv_{t+k})=1`。
- **攻击策略**：  
  - **延迟指令植入 (LIP)**：在植入的对抗内容中嵌入恶意指令，等待未来任务时执行。  
  - **主动信息引出 (PIE)**：植入一条缺少关键参数的恶意指令，迫使智能体在后续主动向用户询问敏感信息后再执行有害操作（增强版 LIP）。  
  - **持久信息腐败 (PIC)**：篡改智能体状态中的持久化事实（如默认收件地址、账户号），导致后续无害查询时引用错误信息而造成损害。
- **状态目标**：对抗内容可藏匿于 **会话上下文 (Session)**、**记忆模块 (Memory)** 或 **可重用技能 (Skill)** 三种抽象层级。
- **基准构建方法**：  
  - 使用 OpenAI Agents SDK 和 ToolEmu 模拟工具环境（含 330 种工具），由 DeepSeek‑v3.2 模拟工具输出。  
  - 覆盖六类现实有害后果：经济损失、账户/系统劫持、物理世界伤害、个人数据泄露、金融数据泄露、其他敏感信息泄露。  
  - 按给定危害标签和策略生成具体攻击目标、良性触发问题，并组合成两段式评测实例，同时支持多交互长度场景。  
  - 采用基于规则的确定性评估（非 LLM 评判），针对每条轨迹检验动作顺序与关键参数值是否满足预定义成功条件，所有规则经过三轮多模型校验。

### 3. 实验设计
- **基准与对比设置**：  
  - 自身基准包含 **1,896 个评估实例**，分别应用于三种攻击策略（LIP、PIE、PIC）× 四种评估表面：**直接单次攻击（Direct）**、**会话状态（Session）**、**技能状态（Skill）**、**记忆状态（Memory）**。  
  - 主要对比对象为当前顶尖 LLM：**Gemini‑3‑Flash、Gemini‑3.1‑Pro、DeepSeek‑R1、Qwen3.5‑Plus、Qwen3.5‑Flash、GPT‑5.4、Llama‑3.3‑70B**，共七个模型。  
  - 评估指标为**满足所有规则的轨迹占比，即攻击成功率 (ASR)**。
- **补充实验**：  
  - **新鲜会话重放**：检验已在原会话成功写入但同会话未攻击成功的持久状态，在全新会话中是否能被激活（仅针对 Memory 和 Skill）。  
  - **条件触发分类**：在 LIP 会话场景中，考察恶意指令能否被不同语义线索（时间、情绪、关键词、逻辑组合）触发。  
  - **长距离触发**：将 PIC 的触发交互延长至最多 20 轮，对比会话、记忆、技能状态的衰减趋势。  
  - **轻量防御测试**：在 Qwen3.5‑Plus 上测试两种训练无关的防御：基于规则的指令前缀和基于 guard 模型的工具返回过滤。  
  - **开源模型规模实验**：评估 Qwen3‑4B、8B、14B、32B 在三种状态上的平均 ASR 随模型大小的变化。

### 6. 主要结论与发现
- **普遍脆弱性**：所有被测模型在休眠攻击下均显著脆弱，即使它们在直接单次交互下的 ASR 很低（如 LIP 直接基线 ASR 为 11.1%，最强状态目标可达 39.9%）。
- **攻击策略与状态影响巨大**：  
  - **PIE** 从直接基线近乎 0% 上升至部分状态超过 70%（如 Gemini‑3.1‑Pro 在 Skill 达 74.1%）。  
  - **PIC** 整体威胁最大（平均 ASR 47.8%），某些模型在延迟触发后接近饱和（Gemini‑3.1‑Pro Skill 达 100%）。  
- **持久状态跨会话风险**：写入成功但当前会话未攻击成功的案例，在**新会话中仍可能被激活**（如 Qwen3.5‑Plus 的 PIC Memory 累计 ASR 高达 94.3%）。  
- **触发条件可多样化**：除工具调用外，时间、情绪等良性语义线索也可有效激活恶意指令。  
- **轻量防御不足**：基于规则或 guard 的实时过滤可降低部分攻击成功率，但无法根除风险。  
- **模型规模不保证安全**：在开源 Qwen3 系列中，更大模型在某些策略上（尤其是 PIC）反而呈现更高的 ASR。
