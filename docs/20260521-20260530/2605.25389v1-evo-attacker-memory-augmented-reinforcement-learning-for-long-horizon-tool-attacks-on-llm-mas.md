## 摘要
虽然基于大语言模型的多智能体系统（LLM-MAS）通过编排专用智能体和外部工具展现出解决复杂任务的卓越能力，但对工具输出的隐式信任构成了关键的攻击面。现有工具攻击受限于领域特异性或固定模板。为应对这些挑战，我们提出Evo-Attacker，它将工具攻击形式化为一个自我进化、记忆增强的强化学习过程。Evo-Attacker构建了动态攻击记忆，并运用审慎推理在关键时刻检索对抗模式并策划修改干预措施。此外，我们引入Attack-Flow GRPO，通过终端结果优化中间推理步骤，解决了长程信用分配挑战。全面实验表明，Evo-Attacker始终优于基线方法，凸显了其泛化与进化能力，以及防御性工具保护措施的迫切需求。

## Abstract
While Large Language Model-based Multi-Agent Systems (LLM-MAS) demonstrate remarkable capabilities in solving complex tasks by orchestrating specialized agents and external tools, the implicit trust in tool outputs creates a critical attack surface. Existing tool attacks are limited by domain specificity or fixed and static templates. To address these challenges, we propose Evo-Attacker, which formulates the tool attack as a self-evolving, memory-augmented reinforcement learning process. Evo-Attacker constructs a dynamic attack memory and employs deliberative reasoning to retrieve adversarial patterns and strategize modifying interventions at critical moments. Furthermore, we introduce Attack-Flow GRPO to optimize intermediate reasoning steps via terminal outcomes, addressing the long-horizon credit assignment challenge. Comprehensive experiments demonstrate that Evo-Attacker consistently outperforms baselines, highlighting its generalization and evolutionary capabilities and the urgent need for defensive tool safeguards.

## 2. 方法论

### 2.1 问题形式化与威胁模型

- **LLM-MAS形式化**：系统建模为动态有向图\(G=(A,E)\)，节点为智能体，边为通信信道。智能体在每一步\(t\)根据角色\(\rho_a\)、可用工具\(S_a^{(t)}\)产生工具调用序列\(C_t(a)=\{\tau_{t,1},\dots,\tau_{t,k_t}\}\)，每条调用\(\tau=(\text{id}, \text{args}, r)\)包含返回值\(r\)。目标任务\(T\)的执行迹产生最终结果\(o\)，攻击目标为最大化系统失败概率\(\max \mathbb{E}[J(o)]\)，其中\(J(o)=1\)表示失败。
- **威胁模型（灰盒）**：
  - **能力**：攻击者仅能劫持某一目标智能体\(a^*\)的工具信道，可监听并篡改其工具调用的返回值\(r \to r'\)，但不能访问智能体间的消息或内部状态。
  - **约束**：有攻击预算\(B\)，整个轨迹最多允许修改\(B\)次工具返回，迫使攻击者战略性地选择最关键的薄弱点。
  - **目标**：在预算内最大化任务失败概率，同时避免触发检测。

### 2.2 Evo-Attacker 框架

Evo-Attacker 将工具攻击建模为**记忆增强的强化学习过程**，包含三个阶段：

#### (1) 攻击记忆生成（Attack Memory Generation）
- 构建动态攻击记忆\(\mathcal{M}_A\)，初始为空，通过探索阶段积累成功攻击的经验。
- 当一条攻击轨迹以系统失败告终时，抽取结构化的记忆条目：
  \[
  m(e) = \langle X_{ctx}, T_{trace} \rangle
  \]
  其中\(X_{ctx}=(G,T,a^*)\)为任务上下文，\(T_{trace}=\{(\tau_t, \phi_t, r'_t)\}_{t=1}^K\)记录原始调用\(\tau_t\)、应用的修改指令\(\phi_t\)和篡改后的返回\(r'_t\)。
- 记忆库持续累积，形成对抗模式的“经验池”。

#### (2) 记忆增强攻击（Memory-Augmented Attack）
攻击者在每个回合\(t\)，基于状态\(x_t=\langle C_t(a^*), H_{t-1}\rangle\)（当前工具调用 + 历史记录），执行**审慎推理**（Deliberative Reasoning），相当于把记忆检索和修改决策变成一个可优化的推理链：

- **检索（Retrieve）**：根据\(X_{ctx}\)和\(x_t\)，策略\(\pi_\theta\)动态生成检索查询\(q_t\)，从\(\mathcal{M}_A\)中取回语义最相似的top-\(k\)记忆\(\mathcal{M}^{(k)}_t\)。该查询可捕获当前工具的功能签名和情境意图。
- **反思（Reflect）**：基于检索到的对抗模式，策略生成可行性分析\(c_t\)和控制决策\(d_t\)：
  \[
  c_t, d_t \sim \pi_\theta(\cdot | \mathcal{M}^{(k)}_t, X_{ctx}, x_t)
  \]
  决策\(d_t\)取：
  - `ATTACK`：当前信息充分且时机合适，准备实施修改；
  - `CONTINUE`：信息不足，利用\(c_t\)更新上下文并重新检索；
  - `NOOP`：当前交互不适于攻击（如低相关性、严格校验），保留预算。
- **修改（Modify）**：若\(d_t=\text{ATTACK}\)且预算未耗尽，策略选择目标工具索引\(i_t\)，生成具体修改指令\(\phi_{t,i_t}\)，由修改工具将原始返回\(r_{t,i_t}\)替换为对抗性返回\(r'_{t,i_t}\)，回传给目标智能体。

#### (3) Attack-Flow GRPO 优化
为解决长程信用分配（一个攻击序列中，最后的成功取决于多个早期决策），提出**Attack-Flow GRPO**：

- **轨迹定义**：一次攻击轨迹\(\zeta=[(s_1,y_1),\dots,(s_L,y_L)]\)，其中\(y_t\)为攻击者在检索、反思、修改阶段生成的所有结构化输出token，环境（LLM-MAS）被当做冻结环境。
- **奖励设计**：
  \[
  R(\zeta) = \mathbb{I}(J(o_{sys})=1) + \lambda \cdot R_{struct}(\zeta)
  \]
  即任务失败获得+1奖励，同时加入格式化惩罚（如预算超支、无效格式）。
- **优势计算**：对于同一任务上下文下并行采样的\(G\)条轨迹，计算组内相对优势：
  \[
  A_i = \frac{R(\zeta_i) - \text{mean}(\{R(\zeta_1),\dots,R(\zeta_G)\})}{\text{std}(\{R(\zeta_1),\dots,R(\zeta_G)\})}
  \]
- **策略优化目标**：只优化攻击者生成的token（索引集合\(I_{atk}\)），环境返回和受害智能体消息不参与梯度更新：
  \[
  \mathcal{L}(\theta) = \frac{1}{G}\sum_{i=1}^G \frac{1}{L_i}\sum_{t\in I_{atk}} \left[ \min(\rho_t A_i, \text{clip}(\rho_t, 1-\epsilon, 1+\epsilon)A_i) - \beta D_{KL}(\pi_\theta \|\pi_{ref})_t \right]
  \]
  其中\(\rho_t = \frac{\pi_\theta(y_t|y_{<t},s_t)}{\pi_{old}(y_t|y_{<t},s_t)}\)为重要性采样比，\(\beta\)为KL散度惩罚系数。
- **关键思想**：将单一的终端奖励**广播**给每个攻击者步骤，使早期战略性决策能基于长程贡献得到强化。

## 6. 主要结论与发现

- **统一框架的优越性**：Evo-Attacker在所有架构和基准上一致且大幅优于四种基线，尤其在需要长程规划的任务（如DeepResearch）上优势显著，说明记忆增强与审慎推理有效打破了固定模板的泛化瓶颈。
- **动态记忆与反思是关键**：消融表明记忆检索提供了可迁移的对抗先验，反思模块通过可行性评估分配预算到高价值漏洞，两者协同使攻击成功率远高于零样本方法。
- **Attack-Flow GRPO解决长程信用分配**：去除RL训练后性能大幅退化，证明了将终端奖励广播到中间步骤并只优化攻击者token的必要性。
- **隐蔽性与高适应性**：Evo-Attacker在多种LLM检测器下维持高绕过率，且可无缝适配不同闭源或开源攻击模型、受害模型，展现出极强的模型无关性和进化潜力。
- **安全警示**：实验结果深刻揭示出当前LLM-MAS对工具层攻击的脆弱性，强调了部署加密签名、返回验证等防御机制的迫切性。

## 7. 优点

- **方法创新性**：首次将工具攻击形式化为记忆增强的RL过程，融合检索、反思、修改的完整推理链，且通过Attack-Flow GRPO进行端到端优化，有效克服了长程依赖与信用分配难题。
- **动态进化能力**：攻击记忆库持续积累成功经验，使策略能从历史中自我进化，适应非静态的LLM-MAS环境。
- **通用性强**：设计上不依赖特定工具或任务，实验涵盖三类架构、六种数据集，展示了跨领域跨拓扑的优异迁移性。
- **实验全面可靠**：多维度消融、超参分析、隐蔽性检测、跨模型泛化实验，充分验证了各模块的有效性和方法的鲁棒性。
- **安全贡献**：不仅提出攻击框架，还通过开源模型优化达到闭源模型水平的实验结果，为社区提供了一种可复现的红队测试工具，促进防御技术发展。

## 8. 不足与局限

- **计算开销**：审慎推理和Attack-Flow GRPO优化引入了更大的推理时间和token消耗，相对于简单模板攻击效率偏低，不利于实时嵌入的轻量级部署。
- **防御评估有限**：隐蔽性测试仅针对LLM语义检测器，未考虑非语义防御机制（如密码学签名验证、工具参数白名单过滤），攻击者在更严格的非AI防御前可能被直接拦截。
- **观察信息受限**：灰盒设定下攻击者仅能观察目标智能体的工具调用，无法感知智能体间通信或系统全局状态，可能错失更优的攻击时机。
- **环境模拟**：所有实验均在仿真的WebArena、WebShop等基准中完成，未涉及真实在线服务或动态更新的生产系统，实际攻击效果有待进一步验证。
- **攻击预算依赖**：攻击成功率与预算正相关，预算极低时长程攻击效果会打折扣，攻击策略的预算效率仍有提升空间。
