## 摘要
基于大型语言模型（LLM）的多智能体系统越来越依赖中间通信来协调复杂任务。虽然大多数现有系统通过自然语言进行通信，但近期研究表明，潜在通信，特别是通过 Transformers 的键值（KV）缓存，能够提高效率并保留更丰富的任务相关信息。然而，KV 缓存也编码了上下文输入、中间推理状态以及智能体特定信息，从而创建了一个不透明的通道，敏感内容可能借此在智能体之间传播而无需显式的文本披露。为了解决这一问题，我们提出了 LCGuard（潜在通信防护），一个在多智能体 LLM 系统中实现安全 KV 潜在通信的框架。LCGuard 将共享的 KV 缓存视为潜在工作记忆，并在缓存内容传输给其他智能体之前学习表示层面的变换。我们通过重构从操作上形式化了表示层面的敏感信息泄漏：如果攻击者的解码器能够从共享的缓存中恢复出智能体特定的敏感输入，则该缓存是不安全的。这引出了一个对抗训练框架，其中攻击者学习重构敏感输入，而 LCGuard 学习既能保留任务相关语义、又能减少可重构信息的变换。在多个模型族和多智能体基准上的实验评估表明，与标准的 KV 共享基线相比，LCGuard 能够持续降低基于重构的泄漏和攻击成功率，同时保持有竞争力的任务性能。

## Abstract
Large language model (LLM)-based multi-agent systems increasingly rely on intermediate communication to coordinate complex tasks. While most existing systems communicate through natural language, recent work shows that latent communication, particularly through transformer key-value (KV) caches, can improve efficiency and preserve richer task-relevant information. However, KV caches also encode contextual inputs, intermediate reasoning states, and agent-specific information, creating an opaque channel through which sensitive content may propagate across agents without explicit textual disclosure. To address this, we introduce \textbf{LCGuard} (Latent Communication Guard), a framework for safe KV-based latent communication in multi-agent LLM systems. LCGuard treats shared KV caches as latent working memory and learns representation-level transformations before cache artifacts are transmitted across agents. We formalize representation-level sensitive information leakage operationally through reconstruction: a shared cache artifact is unsafe if an adversarial decoder can recover agent-specific sensitive inputs from it. This leads to an adversarial training formulation in which the adversary learns to reconstruct sensitive inputs, while LCGuard learns transformations that preserve task-relevant semantics and reduce reconstructable information. Empirical evaluations across multiple model families and multi-agent benchmarks show that LCGuard consistently reduces reconstruction-based leakage and attack success rates while maintaining competitive task performance compared to standard KV-sharing baselines.

*   **核心问题**：基于大型语言模型（LLM）的多智能体系统（MAS）正逐渐采用高效、高信息密度的 **KV 缓存（Key-Value Cache）** 潜在通信方式代替传统的自然语言通信。然而，KV 缓存作为一个语义稠密的内部表示，不仅编码了任务信息，还隐含了**智能体的上下文输入、中间推理状态和特定敏感信息**，形成一个不透明的高带宽信息通道。
*   **安全隐患**：当智能体交换 KV 缓存时，攻击者若通过（例如）监听通信链路、日志或控制部分智能体等方式获取这些表示，可以**不依赖显式文本输出，直接通过重构解码器恢复出原始敏感输入**，构成严重的推理时隐私泄漏。
*   **现实缺口**：现有的多智能体安全机制（如针对输出或工具动作的防护）和 KV 缓存安全研究（如服务端隔离）并未针对此种**旨在共享的潜在通信内部信息**提供防护，存在缺乏表征层面信息流控制的框架空白。

### 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想（LCGuard）**：将共享的 KV 缓存视为“潜在工作记忆”，在智能体传输前为其学习一个**表示层面的变换**，使变换后的表示仍能支持下游任务性能，但**最大限度地抑制攻击者从中重构敏感信息的可能性**。
*   **威胁模型与泄漏形式化**：将泄漏操作性定义为“可重构性”。攻击者目标是用解码器 \( D_i \) 从观察到的通信 artifact \( M_{\text{obs}} \) 中最小化重构损失 \( \mathcal{L}_{\text{rec}}^{(i)} \)。若重构损失远低于无任何通信时的先验损失 \( \mathcal{L}_{\text{prior}}^{(i)} \)，则泄漏严重。
*   **关键技术——对抗优化**：
    *   将学习目标建模为 **minimax 博弈**：通信变换（防御方）试图最大化重构损失（提高隐私），对抗解码器（攻击方）试图最小化重构损失。
    *   总目标函数：\( \min_{\{\phi_i\}} \max_{\{\psi_i\}} \left[ \beta \sum_{i=1}^{N} \mathcal{L}_{\text{rec}}^{(i)}(M_{\text{obs}}) + \mathcal{L}_{\text{task}}(M) \right] \)。其中 \( \beta \) 为隐私-效用平衡超参数。
    *   **训练流程**：交替更新。攻击者步骤（固定通信函数，强化解码器以最小化重构损失）；通信步骤（固定解码器，优化通信函数以最小化整体目标，即任务损失 + 泄漏惩罚）。
*   **通信变换架构**：采用**轻量级残差瓶颈变换**作用于 KV 张量，公式如 \( K_{\text{san}} = K + W_2^K \sigma(W_1^K \text{LN}(K)) \)，通过低维瓶颈层限制精细敏感信息的通过，同时残差连接保留任务语义。

### 6. 论文的主要结论与发现

*   **原始 KV 共享存在极高泄漏风险**：在所有设置下，Vanilla KV Sharing 在获得最高任务有用性的同时，攻击成功率（ASR）高达 80%-90%。
*   **LCGuard 实现了最优的隐私-效用权衡**：全系统 LCGuard 能够将 ASR 大幅降低至 **65%-75%**（如从 0.87 降至 0.22），同时维持了与原始 KV 共享接近的任务有用性。
*   **系统级防护优于局部防护**：Full-System LCGuard 在各项指标上一致优于 Per-Agent LCGuard，证明泄漏具有组合性，信息会通过多跳路径重新聚合，需要联合优化。
*   **现有方法存在根本性缺陷**：输出层方法（PrivAct）无法约束潜在表示的泄漏（ASR 仍高）；噪声方法（ADAPT）虽能降低泄漏，但会严重破坏任务有用性，不适用于需精确协作的场景。
*   **泄漏难易度随模型容量和拓扑复杂度增加**：更大模型和更复杂的图拓扑会加剧泄漏，但 LCGuard 在这些挑战下依然稳健。

### 7. 优点：方法或实验设计上的亮点

*   **新视角**：首次系统性地将**多智能体通信中的共享 KV 缓存视为攻击面**，并量化分析其表征层面的隐私泄漏。
*   **原理性方法**：通过 **“重构”操作性定义泄漏**和 **minimax 对抗优化**框架，为控制内部表征的信息流提供了清晰、自适应的解决思路。
*   **架构友好**：提出的**残差瓶颈变换**轻量、模块化，可在不同模型间插拔使用，不引入显著的推理开销（相较于文本通信仍保持 3.6 倍加速）。
*   **实验扎实**：在**多模型、多基准、多拓扑、多指标**以及丰富的消融研究（隐私-效用权衡 β、攻击者强度、泄漏传播过程）下，提供了极具说服力的证据，展现了方法的泛化性。

### 8. 不足与局限

*   **实验覆盖范围限制**：未在**异构多智能体系统**（智能体使用不同架构或表示空间不兼容）和**多模态智能体**（如处理视觉、音频）场景下进行验证。
*   **隐私保证的性质**：LCGuard **不提供差分隐私等形式化的理论隐私保证**，其防护效果依赖于实验中所用攻击解码器的强度，本质上是一种经验性防御。
*   **数据依赖性**：方法的训练依赖于**有标注的成对训练数据**（任务输入-敏感属性对），这可能限制其在现实世界无监督部署中的直接应用。
