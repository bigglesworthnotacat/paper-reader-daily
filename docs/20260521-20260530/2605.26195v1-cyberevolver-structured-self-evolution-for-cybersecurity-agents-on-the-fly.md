## 摘要
基于大语言模型的代理正越来越多地用于网络安全任务，但大多数现有系统依赖于固定的、人工设计的脚手架，难以适应多样化的目标和故障模式。我们提出了CyberEvolver，一个自演化的网络安全代理框架，它基于失败执行尝试的经验迭代地修改自身的脚手架。网络安全中的自演化具有挑战性，因为可能的脚手架变更空间在很大程度上是非结构化的，执行反馈稀疏且常被环境掩盖，而低多样性的更新可能导致错误在反复迭代中累积。CyberEvolver通过以下方式应对这些挑战：一个四层可演化代理架构，将脚手架优化分解为结构化组件；一种痕迹到诊断的机制，将嘈杂的执行日志转化为可操作的修订信号；以及一种基于种群的束搜索策略，在演化过程中保留多样化的代理变体。我们使用四个开源大语言模型，在CTF挑战、漏洞利用和渗透测试任务上评估了CyberEvolver。在这些设置中，CyberEvolver将种子代理的成功率平均提高了13.6%，并优于六个人工设计的网络安全代理以及两种从其他领域移植的自改进方法。这些结果表明，脚手架自演化是为安全测试构建自适应大语言模型代理的一个有前景的方向。

## Abstract
LLM-based agents are increasingly used for cybersecurity tasks, but most existing systems rely on fixed, human-designed scaffolds that struggle to adapt across diverse targets and failure modes. We introduce \textsc{CyberEvolver}, a self-evolving cybersecurity agent framework that iteratively revises its own scaffold based on experience from failed execution attempts. Self-evolution in cybersecurity is challenging because the space of possible scaffold changes is largely unstructured, execution feedback is sparse and often obscured by the environment, and low-diversity updates can cause errors to compound over repeated iterations. \textsc{CyberEvolver} addresses these challenges with a four-layer evolvable agent architecture that decomposes scaffold optimization into structured components, a trace-to-diagnosis mechanism that converts noisy execution logs into actionable revision signals, and a population-based beam search strategy that preserves diverse agent variants during evolution. We evaluate \textsc{CyberEvolver} on CTF challenges, vulnerability exploitation, and penetration-testing tasks using four open-source LLMs. Across these settings, \textsc{CyberEvolver} improves the seed agent's success rate by $13.6$\,\% on average, and outperforms six human-designed cybersecurity agents as well as two self-improvement methods adapted from other domains. These results suggest that scaffold self-evolution is a promising direction for building adaptive LLM agents for security testing.

### 方法论
*   **核心思想 (CyberEvolver框架)**：一个闭环的迭代自演化过程：**执行 → 总结 → 诊断 → 选择 → 变异**。代理针对特定目标（靶机）执行任务失败后，会分析执行轨迹，诊断失败原因，并对自身的脚手架进行分层、结构化的变异，以产生更优的“后代”代理变体。
*   **关键技术细节**：
    *   **四层可演化代理架构**：为了结构化变异空间，将代理的脚手架分解为四个独立的可演化层，每一层负责不同的功能并显示独特的失败模式：
        *   策略 (Strategy, $L_S$): 系统提示词，定义推理框架。
        *   环境接口 (Environment Interface, $L_I$): 实例提示词，定义与Shell等环境的可靠交互规则。
        *   感知 (Perception, $L_P$): 观察处理层，负责过滤和格式化原始命令输出。
        *   领域知识 (Domain Knowledge, $L_D$): 技能库，包含可随时加载的、针对特定漏洞的攻击手册。
    *   **痕迹到诊断机制**：将嘈杂的执行日志转化为可操作的变异信号。分两步：首先进行**痕迹压缩**以保留关键的利用痕迹；然后生成**结构化诊断报告**，内容包含失败归因、根本原因和进度评分。进度评分用于在同代代理变体中比较优劣，以指导选择。
    *   **基于种群的束搜索**：为确保变异多样性，采用束搜索策略。每一代保留分数最高的$k$个（如3个）代理作为父本，每个父本通过分层变异产生$m$个（如3个）子代，自然淘汰低分分支，避免单一路径演化陷入局部最优。
*   **演化算法流程 (Algorithm 1)**:
    1.  **执行与验证**: 当前种群中的每个代理在目标靶机上执行任务。若成功则停止；否则记录失败的执行轨迹。
    2.  **总结与诊断**: 对轨迹进行压缩和结构化诊断，生成包含失败原因和进度分数$s$的报告。
    3.  **选择**: 依据进度分数$s$对所有代理进行排序，选择前$k$个（Top-K）作为下一代的“父本”。
    4.  **分层变异**: 每个父本根据其诊断报告，并行生成$m$个“子代”变体。变异按层序贯进行（$L_S$, $L_I$, $L_D$, $L_P$），允许后续层的修改建立在前面层修改的基础上。修改后立即进行语法校验，确保有效性。
    5.  **迭代**: 最终形成的新种群进入下一代循环，直至达到最大代数$T$。

*   **Backbone Model**: 使用了四个前沿的开源大语言模型：**Kimi-K2.5, MiniMax-M2.5, DeepSeek-V3.1, Qwen3-235B-A35B-Instruct-2507**。
*   **对比基准**:
    *   **种子代理 (Seed Agent)**: 未演化的初始代理，报告其在16次独立运行（pass@16）下的表现，作为纯粹采样的能力上限。
    *   **基准特定的人类设计代理 (6种)**: 单智能体框架（NYUCTFAgent, AutoPenBench-Agent, CyAgent）和多智能体框架（DCipher, VulnBot, T-Agent）。
    *   **其他领域的自改进方法 (2种)**: ACE（通过自然语言手册优化）和HGM（通过代码编辑优化），被适配应用到网络安全场景。

### 资源与算力
*   **推理硬件**：模型推理部署在配备**8块 NVIDIA H200 GPU（每块141GB显存）**、160核CPU和1.8TB系统内存的专用服务器上。
*   **执行硬件**：代理执行、基准测试编排和目标机容器运行在另一台独立主机（Intel Xeon 6982P-C CPU, 64线程, 247GB内存）上，以确保模型推理资源不受影响。
*   **计算开销**：单个目标的演化过程使用最多16次总交互预算（rollout budget），相比无演化的重复采样，**消耗的总令牌数平均减少了 ~17.5%**。

### 实验数量与充分性
*   **实验量级**：实验覆盖了**4个Backbone模型 × 3个基准测试 × 多种方法（包括消融实验）**的广泛组合，合计在384个任务（192+33+40）上进行了多轮演化评估。
*   **消融实验**：通过移除分层变异、结构化诊断、束搜索等核心组件，验证了每个模块的贡献，其中**分层变异被证明是贡献最大的组件**。
*   **实验的充分性与公平性**：
    *   **充分性**：实验设计严谨，覆盖了不同任务类型和难度，并与三类、共8种不同的基准方法进行了对比，结果具有说服力。
    *   **客观公平性**：所有对比方法在统一的沙箱环境和交互步骤预算下进行评估。论文还通过**分析兄弟变体的源代码差异距离**，证明了进化过程产生了多样性而非近似的变异，排除了随机性效果。
    *   **额外分析**：提供了关于**解决方案泄露、资源消耗统计、HGM方法的失败原因分析**等补充材料，增加了实验的透明度。

### 主要结论与发现
1.  **突破采样上限**：CyberEvolver的表现远超种子代理的pass@16结果，平均提升**13.6%**。这证明自演化能解决固定脚手架无法触及的任务，并非依赖“碰运气”的解决。
2.  **优于现有方法**：在所有模型和基准测试组合中，CyberEvolver的解决率均**一致地超越**了6种人类设计的网络安全代理和2种移植的自改进方法（如ACE, HGM），验证了其核心设计的有效性。
3.  **结构化演化的必要性**：HGM分析表明，非结构化的代码变异会将72%的修改集中在无用的“工具包装器”上，揭示了网络安全自演化必须针对领域特点进行定制。
4.  **演化过程产生可解释的进步**：该机制能将失败的尝试转化为具体的跨代改进，例如发现隐藏的SQL盲注通道，并优化为高效的二分搜索提取方法。

### 优点
*   **问题与视角新颖性**：首次将“在线”脚手架自演化引入网络安全代理领域，并将其定义为一个具有独特挑战的、独立的问题，超越了简单应用现有自改进方法。
*   **方法论独创性**：创新性地提出了**四层代理分解架构**与**痕迹到诊断的反馈机制**，将非结构化的优化问题转化为结构化的、可归因的变异过程，为未来研究提供了清晰的框架。
*   **工程实现与可复现性**：发布了**统一的评估套件**，整合了三个异构的网络安全基准测试，支持大规模并行评估，极大降低了未来研究的复现和对比门槛。
*   **实验分析的深入性**：不仅提供了性能对比，还通过**消融实验、搜索树分析、变异多样性分析及失败案例深度解剖**，全面验证了各组件的有效性，并揭示了其他方法失败的根本原因。

### 不足与局限
*   **任务范围限制**：研究**仅限于进攻性安全测试**（CTF、渗透、漏洞利用），并未覆盖漏洞发现、防御性工作流或实际野外部署与披露流程。
*   **演化预算固定**：实验使用了比较保守的最大16节点演化预算。更大预算或不同调度策略下的改善、漂移、过度特化及收敛性有待探索。
*   **跨目标迁移未涉及**：当前的“在线”演化不研究跨目标的知识迁移，即如何将一个目标上学到的经验推广到其他相似漏洞上，而这是通用智能的更高追求。
*   **结果波动与通用性**：文中结果也显示，某些挑战（如`ezrop`）即使进化也未能解决。性能主要依赖推理模型（Refiner Backbone）的质量，不同模型间的变异多样性差异较大，表明方法效果存在波动。
*   **与现实应用的差距**：评估均基于受控的沙箱环境。在真实网络环境（带防火墙、IDS/IPS等）中，代理的执行和反馈信号可能更为复杂，自演化的有效性需进一步验证。
