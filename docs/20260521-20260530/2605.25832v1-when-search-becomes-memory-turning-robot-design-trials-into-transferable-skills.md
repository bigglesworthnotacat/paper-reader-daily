## 摘要
大型语言模型越来越多地被用作进化机器人设计的提案生成器，然而大多数循环仍然是无记忆的：模拟器结果塑造下一代种群，但并未保存为可复用的设计知识。我们提出了Auto-Robotist，一个自我进化的LLM智能体，它将形态搜索轨迹提炼成显式的自然语言技能库。每个技能存储了一种结构原型、基于证据的正负规则以及支持这些规则的设计评估，使设计记忆可审查而非隐含在种群中。在搜索过程中，智能体检索技能以调节对精英个体的LLM编辑，同时保留遗传算法变异路径进行探索；评估后，它通过添加、诊断和合并来更新技能库。在涵盖运动、穿越和物体交互的七项EvoGym任务中，Auto-Robotist改进了5x5的冷启动搜索，并将学习到的技能迁移到10x10设计空间，其中基于参考条件的迁移在每个任务上都优于遗传算法。这些结果表明，LLM智能体可以将昂贵的物理评估转化为可复用、可审计的设计原则。我们的代码将在接收后发布。

## Abstract
Large language models (LLMs) are increasingly used as proposal generators for evolutionary robot design, yet most loops remain memoryless: simulator results shape the next population but are not preserved as reusable design knowledge. We present Auto-Robotist, a self-evolving LLM agent that distills morphology-search traces into an explicit natural-language skill library. Each skill stores a structural archetype, evidence-grounded positive and negative rules, and the evaluated designs that support them, making design memory inspectable rather than implicit in a population. During search, the agent retrieves skills to condition LLM edits of elite bodies while retaining a Genetic Algorithm (GA) mutation path for exploration; after evaluation, it updates the library through Add, Diagnose, and Merge. Across seven EvoGym tasks spanning locomotion, traversal, and object interaction, Auto-Robotist improves cold-start 5x5 search and transfers learned skills to 10x10 design spaces, where reference-conditioned transfer outperforms GA on every task. These results suggest that LLM agents can convert expensive physical evaluations into reusable, auditable design principles. Our code will be released upon acceptance.

## 2. 方法论  
- **核心思想**：构建一个自进化的LLM智能体（Auto‑Robotist），将形态搜索轨迹转化为显式的自然语言**技能库**。每个技能存储结构原型（L1）、证据支持的正负设计规则（L2）以及支撑规则的评估观察（L3）。

- **搜索过程**：  
  1. 检索任务与尺度相关的技能，按公式 \(w_s = \frac{1+\sum \operatorname{clip}(g_{s,i}/\delta_{\max},0,1)}{2+n_s}\) 计算权重并采样一个技能。  
  2. LLM 根据抽取的技能编辑精英父代（Path A），同时保留 GA 变异路径（Path B）以维持探索。  
  3. 对有效子代进行 PPO 控制器训练和评估，获得适应度。  
- **技能库更新**：通过三个操作维护记忆——**ADD**（从未分配证据中创建新 L1 原型）、**DIAGNOSE**（为已有技能更新规则，利用增益和结构模式提炼正/负规则）、**MERGE**（合并语义冗余的技能）。

## 6. 主要结论与发现  
- **冷启动 5×5 搜索**：Auto‑Robotist 在所有七项任务上持平或超越 GA，平均收敛加速比约 1.47，平均领先比例约 70%，尤其在需要载荷路径、接触和协调力传导的任务（Carrier、Jumper、Pusher）上提升显著。  
- **跨尺度迁移**：带参考设计的 Auto‑Robotist 在 10×10 空间全面优于 GA，终点适应度平均提升 +1.55，领先比例在六项任务超过 90%。Carrier 任务增益最大（+5.71），说明负载与支撑技能成功迁移。  
- **技能库的作用**：去除参考示例后，纯技能库依然在 Walker、Carrier、Jumper 上超过 GA，证明学到的不是具体形态的复制，而是**可泛化的结构关系**（如支撑框架、柔性接触面）。  
- **消融实验**（以 Pusher 为例）表明：移除 Diagnose 或 Merge 会使性能下降 16–18%，纯 LLM（无 GA 变异）下降 28.6%，无 L2/L3 规则参考下降 23.0%，说明各组件均发挥实质作用。  

## 7. 优点与亮点  
- **显式、可审计的设计记忆**：用自然语言三层技能结构替代隐式的种群知识，每条规则均关联具体评估证据，便于追溯和审查。  
- **强迁移能力**：技能从 5×5 迁移到 10×10 只依靠结构关系，无需体素上采样，避免了直接缩放带来的功能退化。  
- **与 GA 互补**：LLM 提议引导搜索方向，GA 维持多样性，两者配合使搜索既高效又鲁棒。  
- **证据驱动的技能更新**：ADD、DIAGNOSE、MERGE 操作保证技能库不会膨胀为单个故事的堆砌，且能随着新证据自我修正。  
- **完备的消融与定性分析**：不仅报告数值，还展示技能库内容、设计演化轨迹和滚动播放行为，帮助理解迁移背后的物理原理。  

## 8. 不足与局限  
- **仅限 2D 体素仿真环境**：未在 3D 形态、真实物理平台或复杂动态环境中验证，技能库的泛化能力边界未知。  
- **评估预算固定，未考虑计算时间成本**：没有报告实际 GPU 时数或能量消耗，难以评估整体效率。  
- **单次运行统计**：部分对比曲线仅来自一次实验，缺少多种子结果和置信区间，结论的稳定性有待加强。  
- **技能库维护依赖固定 LLM 后端的质量**：gpt‑5.5 的输出可能包含幻觉或不合理规则，依赖仿真器反馈过滤，但对其他模型和组件字母表的可移植性未测试。  
- **安全与部署风险**：作者明确指出该方法适用于研究，若直接迁移至硬件可能过度拟合模拟器缺陷或产生脆性设计，需要额外的安全约束和验证。
