## 摘要
大型语言模型（LLM）智能体依赖可复用技能来解决复杂任务。然而，现有的技能创建方法将技能视为孤立和静态的制品，限制了其可复用性、可靠性和长期改进。我们提出了MUSE-Autoskill智能体（基于记忆利用的技能演化），这是一个以技能为中心的智能体框架，使智能体能够通过统一的生命周期（创建、记忆、管理、评估和精炼）持续提升其任务解决能力。我们的框架允许智能体按需创建技能，跨任务存储和复用技能，高效组织和选择技能，并通过单元测试和运行时反馈进行评估以实现持续精炼。我们进一步引入了技能层级记忆，为每个技能积累跨任务的经验，从而实现更有效的复用和长期适应。在SkillsBench上的实验初步证明，受生命周期管理的技能可以提高任务成功率、效率、复用性和跨智能体迁移能力，凸显了将技能视为长期存续、经验感知和可测试资产的重要性。

## Abstract
Large language model (LLM) agents rely on reusable skills to solve complex tasks. However, existing skill creation approaches treat skills as isolated and static artifacts, limiting their reusability, reliability, and long-term improvement. We propose MUSE-Autoskill Agent (Memory-Utilizing Skill Evolution), a skill-centric agent framework that lets agents continuously improve their task-solving capability by creating, reusing, and refining skills under a unified lifecycle (creation, memory, management, evaluation, and refinement). Our framework enables agents to create skills on demand, store and reuse them across tasks, organize and select them efficiently, and evaluate them through unit tests and runtime feedback for continuous refinement. We further introduce skill-level memory that accumulates experience for each skill across tasks, enabling more effective reuse and adaptation over time. Experiments on SkillsBench provide initial evidence that lifecycle-managed skills can improve task success, efficiency, reuse, and cross-agent transfer, highlighting the importance of treating skills as long-lived, experience-aware, and testable assets.

### 2. 论文提出的方法论

论文提出了 **MUSE-Autoskill (Memory-Utilizing Skill Evolution)** 框架，核心是围绕技能的统一五阶段生命周期。

*   **核心思想**
    *   将智能体的所有非内建功能都外化为结构化的“技能”。一个技能是一个包含`SKILL.md`（定义接口）、`scripts/`（可执行代码）、`tests/`（单元测试）等组件的标准目录。
    *   通过内置的`skill_create`工具，将技能创建与执行紧密结合，智能体可在推理（ReAct）循环中按需生成新技能。
    *   引入多层次记忆和自适应上下文管理，使技能具有经验积累和长期运行能力。

*   **关键技术细节**
    *   **技能创建 (Creation)**：当现有技能不足时，智能体调用`skill_create`。系统生成完整的技能包，并自动运行`tests/`中的单元测试。**只有通过所有测试的技能才会被注册到技能库**，否则会触发自动修复。这形成了一个**创建 → 评估 → 注册**的闭环。
    *   **技能记忆 (Memory)**：设计了三个层次的记忆。
        *   **技能级记忆 (Skill-level Memory)**：**核心创新**。每个技能拥有一个`.memory.md`文件，用于追加该技能在不同任务中的使用笔记、经验教训和已知失败模式。当技能被再次调用时，这些记忆会与接口文档一同呈现给智能体。
        *   **短期记忆**：维护当前任务上下文，并可进行自适应压缩。
        *   **长期记忆**：存储跨会话的持久性笔记和通用经验教训。
    *   **技能管理 (Management)**：通过SKILL.md的元数据为技能建索引。在每个任务开始时，智能体系统提示中只注入技能名称和描述的**目录**，智能体决定调用后，再去读取完整的SKILL.md内容，实现了渐进式信息披露，节约Token。
    *   **技能评估与精炼 (Evaluation & Refinement)**：通过执行技能包内的单元测试进行评估。测试失败或运行时产生错误，会自动触发`update_skill`工具对技能进行打补丁和再生，实现持续自我修正。
    *   **上下文管理 (Context Management)**：
        *   将对话历史组织为**有向无环图（DAG）**，支持跨会话状态恢复。
        *   **两级自适应压缩**：当上下文接近Token预算（`COMPRESS_TOKEN_THRESHOLD`）时触发。
            *   **L1(单节点压缩)**：对于Token超过阈值（`NODE_COMPRESS_TOKEN_THRESHOLD`）的单次交互内容，进行就地摘要并替换。
            *   **L2(链条压缩)**：如果L1后仍超预算，则将对话历史中间段的多个节点合并成一个摘要节点。为保证效果，**始终固定保留最早和最新的K轮对话**不被压缩。
