## 摘要
近期在长周期智能体任务上的进展主要由通过更强的模型、更好的工具和更有效的脚手架来扩展单个智能体所驱动。相比之下，对于向外扩展的理解要少得多：即多个对等智能体，都以同一任务为目标，能否成为能力的额外来源，而不依赖于显式的角色专业化或工作流编排。我们研究了这个问题，并提出AgentFugue，一个围绕共享推理中心构建的集体推理框架。当对等智能体并行探索同一任务时，该中心记录每个智能体已确认、尝试或排除的简洁笔记，并使每个智能体能够有选择地访问其他智能体以对其当前搜索有用的形式发现的内容。这种设计将原本孤立的轨迹转变为可重用中间推理的连接生态，无需集中规划。我们将该中心实例化为一个插件式通信层，通过监督微调和端到端强化学习进行训练。在我们研究的长周期挑战性场景中，AgentFugue相较于强基线方法有所提升。我们的结果表明，集体推理可以将向外扩展的对等智能体系统转变为一种独特的能力增益来源，而不仅仅是消耗更多计算资源的方式。

## Abstract
Recent progress on long-horizon agentic tasks has been driven largely by scaling up individual agents through stronger models, better tools, and more effective scaffolding. In contrast, much less is understood about scaling out: whether multiple peer agents, all targeting the same task, can become an additional source of capability without relying on explicit role specialization or workflow orchestration. We study this question and propose AgentFugue, a collective reasoning framework built around a shared reasoning hub. As peer agents explore the same task in parallel, the hub records concise notes on what each agent has established, attempted, or ruled out, and enables each agent to selectively access what other agents have discovered in a form useful for its current search. This design turns otherwise isolated trajectories into a connected ecology of reusable intermediate reasoning without requiring centralized planning. We instantiate the hub as a plug-in communication layer, trained with supervised fine-tuning and end-to-end reinforcement learning. Across the challenging long-horizon settings we study, AgentFugue improves over strong baselines. Our results suggest that collective reasoning can turn scaling out peer agent systems into a distinct source of capability gains, rather than merely a way of spending more compute.

## 一、论文的核心问题与整体含义

该文探讨了长周期智能体任务中**向外扩展（scaling out）**的潜力。现有工作主要通过更强的模型、更丰富的工具和更精巧的单智能体脚手架（即 scaling up）来提升能力，但对于多个**对等智能体**（peer agents）共同完成同一任务、不依赖角色分工或固定工作流的情形，能否产生新的能力增益，理解尚不充分。论文的核心假说是：只需让并行探索同一任务的多个智能体**共享中间推理片段**，便可将孤立的搜索轨迹连接为一个可重用的推理生态，从而将智能体数量的增加转变为实打实的能力提升，而非仅仅是计算资源的简单堆积。由此提出了 **AgentFugue** 框架。

## 二、方法论

### 1. 核心思想：共享推理中心（Shared Reasoning Hub）

- 多个对等智能体并行探索同一任务，但环境与推理路径各自独立。
- 一个**共享推理中心**（以下简称“中心”）位于团队一侧，负责将各智能体已完成的推理片段压缩成**可复用的笔记**，并支持后续有选择地读取这些笔记，供同伴在当前搜索中使用。
- 中心**不扮演中央规划者**角色，不分解任务或指派子目标，只负责“记录与检索”，以此实现 **fugue（赋格曲）式集体推理**——多个独立声部相互呼应、发展彼此的主题。

### 2. 关键机制

**片段划分与写入（Episode Writing）**

- 每个智能体的交互历史按**固定本地上下文预算**（如64K tokens）划分为片段（episode）。当一个片段被触发写入时，中心模型将其压缩为一则 **episode 笔记** \(z_{i,e}\)：
  \[
  z_{i,e} = M_{\text{write}}(\epsilon_{i,e})
  \]
  笔记会记录该片段中已确认的事实、尝试过的思路、已排除的路径等。
- 原始片段内容存入中心存储，智能体自身的工作上下文则被**替换为压缩笔记**，以释放上下文窗口。

**意图驱动的读取（Intent‐driven Reading）**

- 智能体根据当前上下文和对队友笔记的粗粒度了解，向中心发出**带意图的读取请求** \(q_{i,t}\) 以及要查阅的队友片段索引集 \(E_{i,t}\)。
- 中心根据意图检索原始片段内容并合成一段针对该需求的**阅读摘要** \(r_{i,t}\)：
  \[
  r_{i,t} = M_{\text{read}}\bigl(q_{i,t}, \{\epsilon_{j,e} : \epsilon_{j,e} \in E_{i,t}\}\bigr)
  \]
- 这种两级设计（粗粒度笔记浏览 + 按需深度查阅）避免了无通信与全广播两种极端。

### 3. Hub 的训练

- 中心模型从 **Qwen3.5‑9B** 初始化，分别构建 \(M_{\text{write}}\) 和 \(M_{\text{read}}\) 实例。
- **第一阶段：监督微调（SFT）**。用教师模型为书写笔记和阅读摘要生成参考输出，以标准语言模型损失联合训练。
- **第二阶段：团队级策略优化（GRPO）**。在整个多智能体循环中，冻结所有任务智能体，仅优化中心输出。对每个任务实例采样一组中心输出，根据下游任务成功与否及答案简洁程度给予奖励，再按组内相对优势计算策略梯度，同时引入 KL 散度防止偏离参考策略过远。

## 三、实验设计

### 1. 数据集与评测基准
实验在三个挑战性长周期基准上进行，以覆盖不同能力需求：
- **BrowseComp**：深度多跳网页搜索与跨文档证据聚合，需要产生简短事实性答案。评估 200 题随机子集，采用官方 LLM‑as‑a‑judge 协议。
- **WideSearch**：广度优先的证据收集，要求枚举并整合大量平行信息。使用完整测试集，采用结构化字段匹配评分。
- **HLE（Humanity’s Last Exam）**：跨学科多步推理题，较少依赖网页导航，强调推理深度。评估 200 题随机子集，同样使用官方评判协议。

### 2. 对比方法
论文比较了三组系统，所有多智能体系统共享相同的工具栈与交互预算（每查询 150 轮），以保证对比公平：
- **单智能体 ReAct**：Claude‑Opus‑4.5、Kimi‑K2.5、Qwen3.5‑35B‑A3B、GLM‑4.7、DeepSeek‑v4‑Flash 等最强基模型，以标准 ReAct 循环运行。
- **单智能体 DeepResearch**：具备扩展脚手架（搜索规划、摘要记忆、迭代精炼）的深度研究系统，如 WebThinker、WebSailor、AgentFold、Tongyi‑DeepResearch、OpenAI DeepResearch 等。
- **多智能体系统**：
  - **Naive‑Multi‑Agent**：元智能体分解任务，子智能体并行搜索，最后汇总。
  - **Swarm‑Multi‑Agent**：Kimi‑K2.5 的 swarm 设置，元智能体可动态创建并指派子智能体，但子智能体间仍通过元智能体沟通。
  - **AgentFugue**：本方法，用共享推理中心替代中央元智能体，实现平级智能体间的水平通信。

### 3. 扩展与消融研究
- **同构团队扩展**：使用相同 Qwen3.5‑35B‑A3B 骨干，团队大小 \(N=1,2,3,5,8\)，考察纯数量带来的能力变化。
- **异构团队扩展**：逐步在 Qwen 基础上加入 DeepSeek‑v4‑Flash、GLM‑4.7、Kimi‑K2.5，形成 \(N=1 \to 4\) 的异构团队，分析不同能力骨干互补的效果。
- **上下文窗口消融**：在 \(N=2\) 的 BrowseComp 子集上，将中心上下文预算从 16K 变化到 128K，观察其对各项聚合指标准确率的影响，并揭示了 32K 为最优而主实验所用 64K 并非最优。
- 还通过案例分析了共享记忆在成功与失败场景中的具体运作方式。

## 六、主要结论与发现

1.  **AgentFugue 在所有基准上均显著超越基线**。
    - 在主表 \(N=2\) 设定下，相较于 Swarm‑Multi‑Agent 和 Naive‑Multi‑Agent，同骨干下分别平均提升 5.4 和 7.7 个百分点；相较于最强单智能体 ReAct 及 DeepResearch 系统也有明显优势。
    - 提升在搜索密集型（BrowseComp）、广度密集型（WideSearch）和推理密集型（HLE）三类任务上均稳定出现。
2.  **同构扩展可将智能体数量转化为能力提升**。
    - 随着团队大小从 1 增加到 5，单智能体准确率和组平均准确率均持续上升；\(N=5\) 后趋于饱和。
    - 每个智能体的搜索与访问调用数随团队扩大而单调下降，而共享记忆调用数上升，说明智能体以更少独立探索换取了更多协调增益。
3.  **异构扩展中弱智能体获益最大，强智能体亦未被拖累**。
    - 加入更强骨干后，所有模型的个体准确率均高于单独运行时的基线，而 Qwen 等相对较弱的模型获得了最大幅度提升。
    - 异构团队的整体 Pass@k 等聚合指标随模型种类增加更快上升，且共识类聚合器（MV、WMV）与 BoN 的差距缩小，表明异构团队的答案更独立、共识更可靠。
4.  **答案聚合方式不影响“扩展有效”这一结论**。
    - 无论采用 BoN、MV、WMV、FewTool 还是 Pass，均呈现随 N 单调上升的趋势，说明增益来自底层轨迹质量，而非具体选择规则。
5.  **中心上下文窗口存在最优值**。
    - 在 32K 时各项指标达到最高；64K（主实验配置）并非最优，而过大或过小的窗口均会损害性能。
6.  **共享记忆的积极一面是提供“失败地图”和已排筛选证据，消极一面则可能引发确认偏误**。
    - 案例显示，当笔记清晰记录已尝试和已失败的路径时，后续智能体能跳过重复工作直击关键证据；但当某一独特线索被反复强调时，即便伴随硬性约束的违背，智能体仍可能被锚定在错误答案上。

## 七、优点

- **概念新颖且定位清晰**：明确区分了“scaling up”与“scaling out”，并将集体推理作为第三种能力扩展轴。
- **机制设计巧妙**：“片段化写入 + 意图驱动读取”的两级通信方案，在保持轨迹多样性的同时实现了信息的按需共享，不依赖中央规划。
- **训练与部署分离**：Hub 可独立训练，任务智能体冻结，降低了耦合度，也使 Hub 能作为插件式通信层嵌入不同智能体系统。
- **实验设计全面**：多基准、多基线、多骨干、同构与异构团队、多种聚合策略、消融实验与案例研究，构成完整的证据链。
- **公平的对比框架**：所有多智能体系统共享工具栈和交互预算，突出了通信机制本身的效果。
- **清晰的效率分析**：通过工具调用次数的变化（搜索下降、记忆上升）展示了协调开销与探索收益的权衡。

## 八、不足与局限

- **算力消耗未知**：未提供训练阶段的 GPU 型号、数量与时长，观测者无法估算复现成本。
- **覆盖范围有限**：
  - 仅测试了少量骨干模型（主要集中在 Qwen 和 DeepSeek 等几个模型）和一种规模的 Hub（9B）；在更大更强的模型上是否仍有效未知。
  - 任务类型局限于长周期问答与事实梳理，尚未在开放式报告撰写、持续软件工程、更丰富工具的现实交互等场景验证。
- **集体推理的潜在风险**：
  - 若 Hub 产出的笔记质量低下、信息不完整或过于自信，错误或偏差可能在整个团队中传播，导致过早收敛到错误方向。

- **最佳配置未采用**：主实验使用了 64K 写入触发，而消融表明 32K 明显更优，意味着报告中性能并非方法上限。
