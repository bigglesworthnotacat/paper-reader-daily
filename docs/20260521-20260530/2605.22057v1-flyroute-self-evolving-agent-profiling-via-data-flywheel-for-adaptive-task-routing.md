## 摘要
企业路由器将查询分配给专家智能体，但已部署的画像保持静态，而智能体却在不断进化（提示词、工具、模型），开发者很少更新描述或示例。我们提出了FlyRoute，一个自我进化的画像框架，它从真实流量中积累能力证据：分发候选者，通过质量把关将成功的配对存入每个智能体的成功存储库，定期将证据提炼为学习到的能力描述，并将这些描述与通过BM25检索到的成功案例一同注入到LLM路由器中。为了使这一飞轮高效利用数据，FlyRoute引入了一种定向探索策略，该策略结合了画像不确定性、BM25相关性和词汇新颖性，仅针对合理的查询优先处理画像不足的智能体，并避免冗余的证据收集。在我们专有的企业开发者支持数据集（包含真实路由查询）上进行的实验中，FlyRoute将同一骨干的零样本LLM路由器从72.57%提升到78.04%，每个智能体仅使用五个种子查询，表明画像检索已经加强了冷启动路由。在通过飞轮处理了7,211个带标签的训练查询后，准确率提升至89.83%（比零样本提高17.26个百分点；比冷启动提高11.79个百分点），在四个专家领域的标准路由准确率（基于单正确答案测试查询）上均有一致的提升。

## Abstract
Enterprise routers assign queries to expert agents, yet deployed profiles stay static while agents evolve (prompts, tools, models), and developers rarely keep descriptions or exemplars current. We present FlyRoute, a self-evolving profiling framework that grows capability evidence from real traffic: dispatch candidates, quality-gate successful pairs into each agent's success store, periodically distill evidence into learned capability descriptions, and inject those descriptions together with BM25-retrieved successes into an LLM router. To make this flywheel data-efficient, FlyRoute introduces a targeted exploration policy that combines profile uncertainty, BM25 relevance, and lexical novelty, prioritizing under-profiled agents only for plausible queries and avoiding redundant evidence collection. In experiments on our proprietary enterprise developer-support dataset of real routed queries, FlyRoute improves a same-backbone zero-shot LLM router from 72.57% to 78.04% with only five seed queries per agent, showing that profile retrieval already strengthens cold-start routing. After streaming 7,211 labeled training queries through the flywheel, accuracy rises to 89.83% (+17.26pp over zero-shot; +11.79pp over cold start), with consistent gains across four expert domains under standard routing accuracy on single-gold test queries.

### 2. 论文提出的方法论：核心思想、关键技术细节
FlyRoute 框架构建了一个从实际流量中学习和进化的闭环系统，其核心由三大模块组成：

#### 2.1 自适应智能体画像 (Adaptive Agent Profiling)
每个智能体的画像不再是固定的文本，而是一套动态结构：
*   **档案构成**：
    *   `种子描述 (d_seed)`：开发者提供的初始（可能不准确）描述，用于冷启动。
    *   `学习描述 (d_learn)`：从成功案例中自动提炼出的、反映其当前真实能力的最新描述。
    *   `成功案例库 (E_i)`：存储通过质量评估的`（查询，响应）`配对。
*   **质量把关与更新**：当一个查询被分配给智能体并得到响应后，一个由大模型（LLM）担任的“裁判”会对其进行评分。当评分 `s_i ≥ θ`（质量阈值）时，该配对被视为成功案例，录入 `E_i`。
*   **能力蒸馏**：当成功案例积累到一定数量（如 `M=20`）后，系统会提示大模型，基于种子描述和这些最新的成功案例，**生成或更新 `d_learn`**。这确保了画像能捕捉到智能体行为的最新变化，并固化成路由时可用的精简文本。

#### 2.2 不确定性驱动的探索 (Uncertainty-Driven Exploration)
为解决“利用-探索”困境（纯粹利用已知最强智能体会导致其他智能体失去收集证据的机会），FlyRoute 设计了一种高效的定向探索策略：
*   **画像不确定性 (U)**：由成功案例库的大小决定，公式为 `U(a_i) = 1 / (1 + |E_i|^α)`。案例越少，不确定性越高，探索需求越大。
*   **探索价值 (V_explore)**：对于待选查询 `q` 和智能体 `a_i`，其探索价值是 `U(a_i)` 与查询-智能体 BM25 相关性得分 `R(q, a_i)` 的乘积。这确保了**只将相关的查询用于探索，避免盲目分发**。
*   **新颖性加权**：为避免收集冗余证据，系统会惩罚与已有案例相似度高的查询，最终探索得分公式为 `V+_explore = U(a_i) · R(q, a_i) · [1 + β · (1 - R(q, a_i))]`。这优先考虑了对画像有增量价值的查询。

#### 2.3 画像感知的路由 (Profile-Aware Routing)
路由决策融合了全局总结与局部证据：
*   路由器（一个LLM）的提示词中会同时包含：
    1.  每个智能体的**全局画像总结**（ `d_learn` 或 `d_seed` ）。
    2.  通过 BM25 从各智能体成功案例库中检索出的、**与当前查询最相关的 Top-K 实例**。
*   LLM 结合这两类信息进行判断，即使早期检索结果较弱，画像总结也能提供粗粒度的能力边界信息。
