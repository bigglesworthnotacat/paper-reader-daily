## 摘要
大型语言模型（LLM）智能体在解决现实世界任务时积累丰富的情节性轨迹，但尚不明确此类经验能否提炼为可复用的过程性技能。我们推出SkillEvolBench，一个用于评估从经验复用到技能形成过程的诊断基准。它包含六个现实世界智能体环境下的180项任务，组织为角色条件化的任务家族，具有共享的潜在过程。智能体从习得任务中学习，利用压缩轨迹和验证器反馈更新外部技能库，然后面对冻结的部署任务，测试上下文迁移、对抗捷径和组合能力。通过将自生成和精选起点的技能演化与无技能及原始轨迹对照进行比较，SkillEvolBench将过程抽象与基础能力、精选先验知识及直接复用情节性痕迹分离开来。在十种模型配置和三种智能体框架下，我们发现当前智能体通常能进行局部适应，但很少形成稳健的可复用技能。基于技能的条件能改善习得或回放，且个别模型有时在特定部署维度上有所提升，但这些增益在冻结部署下不稳定。原始轨迹复用往往优于提炼后的技能，表明当前的抽象过程丢弃了对未来任务仍有用的情境和过程线索。容量与成本分析进一步表明，写入更多技能或更大的三级资源库并不足够：额外更新可能提升覆盖率，同时引入特定于情节的漂移和过程性杂乱。这些发现将SkillEvolBench定位为一个测试平台，用于衡量一次性经验何时成为持久的过程性知识而非任务局部记忆。

## Abstract
Large language model (LLM) agents accumulate rich episodic trajectories while solving real-world tasks, but it remains unclear whether such experience can be distilled into reusable procedural skills. We introduce SkillEvolBench, a diagnostic benchmark for evaluating this step from experience reuse to skill formation. It contains 180 tasks across six real-world agent environments, organized into role-conditioned task families with shared latent procedures. Agents learn from acquisition tasks, update an external skill library using compacted trajectories and verifier feedback, and then face frozen deployment tasks testing context shift, adversarial shortcuts, and composition. By comparing self-generated and curated-start skill evolution against no-skill and raw-trajectory controls, SkillEvolBench separates procedural abstraction from base capability, curated prior knowledge, and direct reuse of episodic traces. Across ten model configurations and three agent harnesses, we find that current agents often adapt locally but rarely form robust reusable skills. Skill-based conditions can improve acquisition or replay, and individual models sometimes gain on specific deployment axes, but these gains are unstable under frozen deployment. Raw-trajectory reuse frequently outperforms distilled skills, suggesting that current abstraction procedures discard contextual and procedural cues that remain useful for future tasks. Capacity and cost analyses further show that writing more skills or larger Tier-3 resource libraries is not sufficient: additional updates can improve coverage while introducing episode-specific drift and procedural clutter. These findings position SkillEvolBench as a testbed for measuring when one-off experience becomes durable procedural knowledge rather than task-local memory.

### 2. 论文提出的方法论：核心思想与关键技术细节

*   **核心思想**：SkillEvolBench设计了一套模拟“终生学习”（lifelong learning）片段的协议。智能体在解决基础任务（获取阶段）的过程中积累经验，然后由一个独立的“技能作者”（Skill Author）尝试将这些经验抽象为技能条目并存入外部技能库。在随后的冻结部署阶段，智能体仅能使用该技能库，而无法再修改它，以此来评测之前积累的经验是否已成功转化为可复用的知识。
*   **基准架构与流程**：
    *   **任务结构**：包含6个真实环境（如代码调试、API编排、数据处理）下的30个任务族，共180个任务。每个任务族共享一个潜在过程。
    *   **角色条件化任务弧**：每个任务族内的任务分为6个角色：**3个获取任务**（标准、增强、变体）用于学习和技能更新；**3个冻结部署任务**（上下文迁移、对抗、组合）用于评测技能的可迁移性、鲁棒性和模块化能力。
    *   **技能库与更新机制**：技能库是环境范围内外部存储。更新由“技能作者”（一个独立于任务执行智能体的LLM调用）负责。它接收压缩的任务轨迹和结构化验证器反馈（包含结果、过程和诊断），然后决定对技能库执行以下操作之一：**创建新技能、修订现有技能或不操作**。

### 6. 论文的主要结论与发现

1.  **局部适应而非技能形成**：当前LLM智能体普遍表现出“局部过程性适应”，即技能可以改善其在**原始任务上的表现（LSR）或回放成功率（RSR）**，但这些提升**无法稳定地迁移到冻结的、更难部署任务（ESR）** 上。
2.  **存在“有损抽象瓶颈”**：核心发现是，**原始轨迹（Raw-Trajectory）控件经常优于所有基于蒸馏技能的条件**。这说明当前的技能抽象过程丢弃了原始轨迹中对未来任务有用的上下文和过程性线索，是一个“有损”的过程。
3.  **更多不等于更好**：简单地增加技能库大小（如强制捆绑三级资源）或更频繁地更新技能（Always策略）并不能单调地提高性能。这可能导致“过程性杂乱”，引入情景特定的漂移，反而损害部署时的性能。关键在于**选择性地进行过程抽象**。
4.  **失效模式多样**：部署任务的不同维度（CSSR, ARSR, CompSR）揭示了不同的失效模式，如技能无法在上下文中被触发、对对抗性捷径的脆弱性、以及多技能组合失败。一个统一的成功率数字会掩盖这些关键区别。
5.  **环境与模型依赖性**：技能演化的成功与否高度依赖于任务结构、部署角色、基础模型能力和编写成本。例如，结构化环境（如代码修复）中的技能演化比开放性环境（如沟通调度）更可靠。

### 7. 优点：方法或实验设计的亮点

*   **问题导向的基准设计**：SkillEvolBench的设计精巧，直接对准“从经验到技能”这一核心科学问题，而非简单的任务成功度量。其角色条件化任务弧是模拟技能迁移与泛化的出色设计。

*   **巧妙的对照实验**：“原始轨迹”对照组的设置是本文一大亮点，直接且有力地揭示了“有损抽象”这一关键瓶颈，使得结论极具说服力。
*   **严格的实验协议**：冻结部署、独立的技能作者、回放机制和跨环境库重置等设计，确保了评估的客观性和实验的内部有效性，排除了许多混淆因素。

### 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等

*   **模型与框架覆盖范围**：实验虽覆盖10种大模型，但主要集中在头部商业闭源模型（Claude， GPT, Gemini）。对于开源模型和更广泛的智能体框架，其结论的普适性有待验证。
