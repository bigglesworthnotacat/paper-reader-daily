## 摘要
随着自主代理（如OpenClaw）越来越频繁地以深层系统权限执行复杂任务，它们带来了严重且未得到缓解的安全风险。当前的漏洞分析绝大多数聚焦于单轮、无状态行为，忽视了有状态、多轮交互和动态工具调用所固有的扩展攻击面。在本文中，我们针对基于大语言模型的代理系统，提出了一种新颖的多维规避框架。我们引入了三种隐蔽的攻击向量：（1）时间规避，将恶意有效载荷分散到连续的交互轮次中；（2）空间规避，将有效载荷隐藏在能够规避标准大语言模型解析机制的复杂外部产物中；（3）语义规避，将恶意意图隐藏在良性的上下文噪音之下。为了系统量化这些威胁，我们构建了A3S-Bench，一个包含2,254条真实世界代理执行轨迹的综合基准测试。通过将标准代理框架分别集成10种主流大语言模型骨干，并在20个实际威胁场景下进行评估，我们证明我们的规避框架将平均风险触发率从基线的28.3%提升至52.6%。这些发现揭示了当前自主代理系统中现有防御措施无法解决的系统性、架构层面漏洞，突显了针对这些独特威胁定制防御机制的迫切需求。

## Abstract
As autonomous agents (e.g., OpenClaw) increasingly operate with deep system-level privileges to execute complex tasks, they introduce severe, unmitigated security risks. Current vulnerability analyses overwhelmingly focus on single-turn, stateless behaviors, overlooking the expanded attack surface inherent in stateful, multi-turn interactions and dynamic tool invocations. In this paper, we propose a novel, multi-dimensional evasion framework targeting LLM-based agent systems. We introduce three stealthy attack vectors: (1) Temporal evasion, which fragments malicious payloads across sequential interaction turns; (2) Spatial evasion, which conceals payloads within complex external artifacts that evade standard LLM parsing mechanisms; and (3) Semantic evasion, which obscures malicious intents beneath benign contextual noise. To systematically quantify these threats, we construct A3S-Bench, a comprehensive benchmark comprising 2,254 real-world agent execution trajectories. Evaluating a standard agent framework separately integrated with 10 mainstream LLM backbones against 20 practical threat scenarios, we demonstrate that our evasion framework elevates the average risk trigger rate from a 28.3\% baseline to 52.6\%. These findings reveal systemic, architecture-level vulnerabilities in current autonomous agent systems that existing defenses fail to address, highlighting an urgent need for defense mechanisms tailored to the unique threats.

#### 2.1 威胁模型与风险分类法

- **威胁模型**：假设攻击者只能通过自然语言输入通道注入对抗内容，不能修改代理的系统提示、模型权重或编排框架；攻击者拥有黑盒访问权限。注入分为直接注入（控制用户消息）、间接注入（污染文件/网页/命令输出）和混合注入（两者结合）。
- **风险分类法**：从攻击目标出发，将风险划分为三大类十个细粒度类别：
  - **边界突破**：越狱攻击、沙箱逃逸、特权提升。
  - **持久状态污染**：恶意技能安装、记忆篡改、配置篡改。

#### 2.2 多维规避攻击策略

提出三大高级策略，分别从时间、空间、语义维度隐藏攻击意图：

- **跨轮碎片化 (Temporal Evasion)**：将恶意目标分解为多个看似无害的轮次片段，每个片段单独检查不触发警报，但组合后还原完整攻击。例如将数据外泄脚本的各个部分分散到多个连续的“运维脚本”轮次中。
- **检测范围逃避 (Spatial Evasion)**：将有效载荷注入到会话之外的工件（如文件、技能包、配置文件）中，当代理执行正常的工具调用来获取这些工件时，安全监控因未检查工件内容而失效。
- **良性上下文隐藏 (Semantic Evasion)**：将危险步骤编织进看似合理的多步工作流中，或构造含义模糊的请求使其最自然的执行路径恰好产生危害，利用代理对任务上下文的默认服从倾向绕过语义过滤器。

#### 2.3 A3S-Bench 数据合成流水线

采用自动化三阶段流水线生成基准数据集：

- **阶段1: 种子生成**：由多个种子LLM（Claude Opus 4.5、GPT-5.2、Kimi-K2.5）根据给定的使用场景和风险类别生成完全良性的多轮对话种子，同时生成工作空间初始化命令和预期工具响应锚点。生成时要求工作空间丰富、轮次间有依赖、用户消息信息密度高。
- **阶段2: 种子整理**：对种子进行可行性校验（Shell命令语法、文件路径、URL真实性），并通过语义相似度去重以保证多样性。
- **阶段3: 有效载荷注入**：注入LLM根据指定的攻击技术（共34种，分为基础级和高级两大类）在种子对话的选定轮次中修改用户消息或工具响应，实现直接、间接或混合注入，必要时增加新轮次以实现分步触发或延迟触发。

### 6. 论文的主要结论与发现

- **通用脆弱性**：使用任何 LLM 骨干的 OpenClaw 代理系统均表现出不可忽视的漏洞，在基础攻击下平均风险触发率已达 28.3%。
- **高级攻击显著提升风险**：三种高级策略将平均 RTR@1 提升至 52.6%（最高 77.7%），并且超过半数的模型攻击成功率翻倍以上。
- **关键风险类别**：沙箱逃逸和信息泄漏在所有模型中最易被利用；即使在专有安全模型上，记忆篡改、恶意技能注入等代理特有威胁的触发率仍可超过 30%。
- **多轮注入威力更大**：多轮注入成功率（58.6%）远超单轮直接注入（46.4%）和单轮间接注入（16.2%），表明现有代理缺乏跨轮次意图识别能力。
- **持久状态攻击链**：案例显示先通过污染记忆/文件等组件，可在后续正常请求中静默触发信息泄漏等危害，所有模型均未能检测这种跨组件传播。
- **安全与效用可兼得**：Sonnet 4.5 和 GPT-5.2 在保持高任务完成率（≈90%）的同时实现了较低风险触发率。
- **现有防御不足**：外部防护模型仅能捕获少数攻击，且大部分与代理自身拒绝重叠；平台升级只对安全对齐较好的模型有效，无法替代模型自身的安全能力。

### 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等

- **平台单一性**：仅在 OpenClaw 一个代理平台上评估。虽然攻击面具有架构共性，但其他类似代理（如 QClaw、Manus、Devin）可能有不同的安全机制（沙箱策略、权限粒度），结论的泛化性尚需验证。

- **攻击维度有限**：目前未覆盖多模态攻击（如图像、音频输入绕过文本安全过滤器），这在实际具有多模态交互能力的代理中是可能的威胁向量。
- **防御实验的深度**：仅测试了两种轻量级防护模型和一次平台升级，未探索更多样化的防御思路（如记忆溯源、工具调用意图校验、跨轮次异常检测等），因此不能断言“所有现有防御均无效”。
- **数据合成依赖 LLM**：种子和攻击案例由 LLM 生成，可能引入模型特有的偏见或对话风格，与现实多样化的用户行为仍有差距。
- **黑色盒攻击假设**：未考虑攻击者可能具有更强能力（如部分白盒访问或训练数据投毒），限制了威胁模型的覆盖广度。
