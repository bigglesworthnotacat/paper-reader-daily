## 摘要
监控自主大型语言模型（LLM）代理的隐蔽恶意行为具有挑战性，因攻击模式具有延迟性、上下文依赖性和长期性。代理可能在维持表面良性行为的同时追求隐藏目标，即使完全访问轨迹数据也难以检测。先前的监控方法改善脚手架或集成聚合，但独立处理每个轨迹，未从先前的监控经验中学习。此外，标准的推理方法解释观察到的行为，而未明确推理区分良性任务执行与隐蔽偏差所需的代理信念、意图和目标一致性。我们提出Agent-ToM，一个基于心智理论（ToM）推理的学习监控框架，用于自主代理的安全分析。Agent-ToM通过推断信念、带有校准置信度的意图假设、预期行为以及与任务一致行为基线的偏差，执行结构化的全轨迹分析。推理时，它采用“推理-验证-精炼”管道构建并验证监控决策。训练时，Agent-ToM将批评信号蒸馏为持久的语义护栏记忆，实现跨事件的、基于信念和意图条件的可重用约束。我们在对抗性代理监控基准（SHADE-Arena和CUA-SHADE-Arena）上评估Agent-ToM。Agent-ToM实现了强大的精确率-召回率平衡，并优于包括集成方法在内的最先进监控基线，同时使用连贯的双调用推理管道。这些结果表明，在监控层进行学习，结合结构化的心智理论推理和验证，为保护自主LLM代理提供了有效且可部署的基础。

## Abstract
Monitoring autonomous large language model (LLM) agents for covert malicious behavior is challenging due to delayed, context-dependent, and long-horizon attack patterns. Agents may pursue hidden objectives while maintaining superficially benign behavior, making detection difficult even with full trajectory access. Prior monitoring approaches improve scaffolding or ensemble aggregation, but treat each trajectory independently and do not learn from prior monitoring experience. Moreover, standard reasoning methods explain observed behavior without explicitly reasoning about agent beliefs, intentions, and goal alignment required to distinguish benign task execution from covert deviation.   We propose \textbf{Agent-ToM}, a learning-to-monitor framework grounded in Theory-of-Mind (ToM) reasoning for security analysis of autonomous agents. Agent-ToM performs structured full-trajectory analysis by inferring beliefs, intent hypotheses with calibrated confidence, expected actions, and deviations from task-consistent behavioral baselines. At inference time, it employs a \textit{Reason-Verify-Refine} pipeline to construct and validate monitoring decisions. At training time, Agent-ToM distills critique signals into a persistent \textit{semantic guardrail memory}, enabling reusable belief- and intent-conditioned constraints across episodes. We evaluate Agent-ToM on adversarial agent monitoring benchmarks (SHADE-Arena and CUA-SHADE-Arena). Agent-ToM achieves strong precision-recall balance and outperforms state-of-the-art monitoring baselines, including ensemble methods, while using a coherent two-call reasoning pipeline. These results demonstrate that learning at the monitoring layer, combined with structured ToM reasoning and verification, provides an effective and deployable foundation for securing autonomous LLM agents.

## 1. 论文的核心问题与整体含义

- **核心问题**：监控自主大型语言模型（LLM）代理在执行任务时是否存在隐蔽恶意行为（如暗中追求隐藏目标），当前方法面临三大挑战：
  - 攻击模式具有**延迟性、上下文依赖性和长程性**，难以通过局部动作或短视窗口检测。
  - 现有监控方法（如集成、脚手架优化）**将每个执行轨迹视为独立分类任务**，未从过去的监控失败中学习，缺乏跨轨迹经验积累。
  - 通用推理方法只解释“做了什么”，**不显式推理代理的信念、意图与目标对齐**，难以区分善意的任务偏差与隐蔽的恶意意图。
- **整体含义**：将监控问题从静态分类提升为 **基于学习的心智推断过程**，通过引入心智理论（Theory-of-Mind）推理和大规模经验蒸馏，使监控系统能够随实践自我进化，以更低的推理成本实现更强的检测能力。

## 2. 方法论：Agent-ToM 框架

- **核心思想**：监控不仅是轨迹分类，而是**对代理潜在心理状态（信念、意图）的结构化推理**，并通过批判性学习积累可复用的行为约束。
- **关键技术细节**：
  - **心智理论推理**：对每段轨迹显式推断四项潜在变量：
    - 信念（代理对其任务上下文已知或假设的内容）
    - 带有校准置信度的意图假设（每步动作背后的目标）
    - 预期行为（一个任务对齐的代理应当执行的动作）
    - 行为偏差（无法用良性假设解释的偏离）
  - **推理 - 验证 - 精炼（Reason–Verify–Refine）管道**：
    - **推理阶段**：基于 ToM 分析构建假设，生成初始分类与理由。
    - **验证阶段**：作为独立评审人，重新评估证据充分性，测试替代良性解释，并依据非对称标准（降低恶意判定的要求高于确认）升级或降级威胁等级。
    - **精炼阶段**：整合结果，输出校准置信度与自然语言解释；实际实现为**两步 LLM 调用**（验证与精炼合并）。
  - **语义护栏记忆与训练循环**：
    - 通过**批判信号蒸馏**，将过往监控事件的成功与失败模式转化为**可复用的信念 - 意图条件约束**，存入持久化记忆。
    - 支持**批量蒸馏**（合并多轨迹批判信号，提炼跨事件通用护栏）和**批判驱动自我纠正**（蒸馏前让监控器根据反馈修正推理）。
    - 记忆采用**预定义结构**（恶意目标 / 欺骗策略 / 规避技术）或**自组织结构**，可在新应用上迁移。
- **算法流程（文字描述）**：
  1. 输入轨迹 $\tau$ 与当前记忆 $M$。
  2. 推理阶段依据 ToM 步骤输出初步监测 $(\hat{y}, \hat{r})$。
  3. 验证阶段以 $\hat{y}, \hat{r}$ 为条件，独立复审并给出修订后的分类。
  4. 精炼阶段输出最终威胁等级 $y$、置信度 $c$、理由 $r$。
  5. 训练时，对每个批次的轨迹进行 ToM 推理、批判信度分配、知识蒸馏，更新 $M$。

## 6. 主要结论与发现

- Agent-ToM（Stage1–2）在 **SHADE-Arena 上取得最高 F1（0.782）**，且在仅需 **2 次 LLM 调用**的情况下，优于需要 8–17 次调用的集成方法，实现了强精确率 - 召回率平衡。
- 在分布外 CUA-SHADE-Arena 上，Agent-ToM 同样**表现最佳（F1=0.831，精确率 1.0）**，验证了记忆护栏的跨任务转移能力；Stage2 验证在零样本迁移时尤为关键（召回率从 0.444 提升至 0.710）。
- 批判性学习至关重要：带批评反馈的蒸馏使 F1 随训练数据增加持续上升（0.695→0.814），无反馈则提前停滞；**批量蒸馏不仅加速训练，且产生更紧凑、更通用的护栏**。
- **结构化心智推理是性能提升的本源**，而非推理时的计算堆叠；面对主动规避（特别是谎言与劫持）时，信念与意图推断能有效揭示隐蔽恶意，而集成方法只能较好地应对无规避场景。

## 7. 优点

- **方法论亮点**：
  - 首次将 **心智理论推理** 系统化地引入 LLM 代理监控，从潜在信念与意图层面辨别恶意，显著优于仅看表面行为的方法。

  - 支持 **预定义与自组织两种记忆模式**，兼顾了领域知识注入与自适应模式挖掘。
- **工程与实验优势**：
  - 推理效率极高（每轨迹仅 2 次调用），在保持领先 F1 的同时节省大量 API 开销。

## 8. 不足与局限

- **批评质量依赖**：记忆蒸馏性能强依赖于批判信度分配的质量，若初始监控器误判严重，可能产生误导性护栏，且文中未深入讨论记忆污染或遗忘问题。

  - 未探讨在多代理交互、部分可观轨迹等更复杂场景下的适用性。
