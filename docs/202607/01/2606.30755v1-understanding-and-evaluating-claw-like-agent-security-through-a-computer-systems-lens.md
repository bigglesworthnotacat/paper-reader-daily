## 摘要
类爪形AI智能体（例如OpenClaw）是始终在线的进程，持久拥有对凭证、文件、工具和外部服务的访问权限。它们承担系统级职责——安装软件包、维护状态、调度子任务和协调I/O——使得安全失败比其他智能体严重得多。然而，现有基准测试关注模型响应和工具调用，几乎未衡量跨组件失效模式。我们采用计算机系统类比：将类爪形智能体视为一个智能体计算机系统，其网关运行时扮演类似操作系统的中介角色，其技能（Skills）类似于用户安装的应用程序，其插件（Plugins）类似于具有运行时权限的可加载扩展。每个组件都有对应的经典组件，其保护机制——经过数十年网络安全研究完善——在智能体侧缺失。基于这一视角，我们开发了SafeClawArena基准测试，包含跨四个攻击面（技能供应链完整性、持久状态利用、跨边界数据流和间接提示注入）的406个对抗性任务，在真实智能体平台的容器化副本中执行，使用金丝雀标记凭证，并通过九个输出通道的自动污点追踪进行评估。我们评估了三个平台（OpenClaw、NemoClaw、SeClaw）和五个前沿大语言模型。最高攻击成功率可达70%；恶意插件在100%的情况下成功，无论使用何种大语言模型。SeClaw将GPT-5.4的攻击成功率从70%降低到22%，部分是通过效用-安全权衡而非主动防御实现的，而Claude-Opus-4.6在每个平台上已经接近22%的低点。这些结果暴露了当前防御的不足，并指出了未来加固的方向。代码和数据：https://github.com/sunblaze-ucb/SafeClawArena。

## Abstract
Claw-like AI agents (e.g., OpenClaw) are always-on processes with persistent access to credentials, files, tools, and external services. They take on system-level responsibilities -- installing packages, maintaining state, scheduling subtasks, and mediating I/O -- making security failures far more severe than in other agents. Yet existing benchmarks focus on model responses and tool calls, leaving cross-component failure modes largely unmeasured. We adopt a computer-system analogy: treating a Claw-like agent as an agentic computer system whose gateway runtime plays an OS-like mediation role, whose Skills resemble user-installed applications, and whose Plugins resemble loadable extensions with runtime privileges. Each component has a classical counterpart whose protection mechanisms -- refined over decades of cybersecurity research -- are absent on the agent side. From this perspective, we develop SafeClawArena, a benchmark of 406 adversarial tasks across four attack surfaces (Skill Supply-Chain Integrity, Persistent State Exploitation, Cross-Boundary Data Flow, and Indirect Prompt Injection), executed in containerized replicas of real agent platforms with canary-marked credentials and evaluated via automated taint tracking across nine output channels. We evaluate three platforms (OpenClaw, NemoClaw, SeClaw) and five frontier LLMs. The highest attack success rate reaches 70%; malicious Plugins succeed in 100% of cases regardless of the LLM. SeClaw cuts GPT-5.4's attack success rate from 70% to 22%, partly through utility-security tradeoffs rather than active defenses, while Claude-Opus-4.6 already sits near a 22% floor on every platform. These results expose the inadequacy of current defenses and suggest directions for future hardening. Code and data: https://github.com/sunblaze-ucb/SafeClawArena.

## 1. 核心问题与整体含义（研究动机和背景）

类爪形AI智能体（如OpenClaw）是一种**始终在线、持久运行**的进程，直接部署在用户环境中，拥有对凭证、文件系统、Shell、工具和外部服务的连续访问权限。它们承担系统级职责（安装软件包、维护持久状态、调度子任务、协调I/O），且拥有庞大的用户群。这使得其安全失效后果远比其他专门智能体严重——例如官方技能市场已发现超过1,184个恶意技能，2026年已曝出130多个CVE。然而，现有安全基准（如InjecAgent、AgentDojo等）仅关注模型层的响应和工具调用安全性，**完全忽略了跨组件（网关运行时、技能加载器、插件加载器、持久存储、审计日志）的失效模式**。论文的核心问题是：“这种类爪形智能体有多安全？”，并通过**计算机系统视角**系统化地揭示差距。

**整体含义**：当前类爪形智能体的安全防御严重不足，必须从纯模型对齐转向系统级防御，且不同模型对同一加固措施的反应截然不同，需要联合评估。

## 2. 方法论

### 核心思想：计算机系统类比
将类爪形智能体映射为经典计算机系统的组合：
- **网关运行时** → 操作系统，负责加载代码、中介工具调用、调度、持久化状态。
- **可安装的技能（Skills）** → 用户安装的应用程序。
- **进程内插件（Plugins，npm包）** → 以运行时权限可加载的扩展。
- **持久内存、配置文件、审计日志** → 文件系统/存储，但缺少访问控制、完整性验证和红action。

### 缺失的经典安全原则（5条）
| 原则 | 在智能体中的失效 |
|------|----------------|
| **I1 进程隔离** | 用户提示、技能指令、读取的文件内容共享同一LLM上下文缓冲区，无隔离 |
| **I2 最小特权** | 技能和插件加载后继承智能体的全部权限 |
| **I3 持久状态保护** | 内存、配置、审计日志无完整性验证、无访问控制、无敏感内容红action |
| **I4 跨边界中介** | 所有出站调用使用同一共享凭证集，无法区分来源 |
| **I5 数据-指令分离** | 文档内容与用户指令在上下文中具有相同权威性，可覆盖指令 |

### 四个攻击面（基准维度）
- **SSI**：技能供应链完整性（对应I1、I2）
- **PSE**：持久状态利用（对应I3）
- **CDF**：跨边界数据流（对应I4）
- **IPI**：间接提示注入（对应I5）

### SafeClawArena 实现
- 共**406个对抗性任务**，从上述原则自上而下生成。
- 每个任务在**容器化副本**中运行，使用生产级代理平台（OpenClaw、NemoClaw、SeClaw）而非自定义脚手架。
- 通过**金丝雀标记凭证**（Canary）和自动污点追踪评估9个输出通道（回复、出站消息、Google模拟调用、内存写、日志、配置、文件、Webhook、Cron）。

  - **NemoClaw**（继承OpenClaw，添加用户级进程隔离）

- **5个前沿LLM**：
  - GPT-5.1-Codex（GPT-5.1-C）

  - Gemini-3-Flash（G-Flash）

- 各维度特定安全分数（SSI/PSE/CDF/IPI），总体攻击成功率（Attack Success Rate, %）和安全分数。

## 6. 主要结论与发现

1. **恶意插件100%成功**（不受LLM影响）：因为插件作为原生代码运行在网关进程中，完全绕过LLM，模型层对齐无效。SeClaw通过直接移除该功能避免。
2. **最强模型不一定最安全**：例如在NemoClaw上，Opus-4.6在“不安全的捆绑脚本”类别中成功率更高（更强指令跟随反而有害）。
3. **平台加固效果高度模型依赖**：
   - SeClaw将GPT-5.4的攻击率从70%降至22%，但对G-Flash几乎无改进甚至恶化（PSE从60%升至61.7%，IPI从67%升至73%）。
   - Opus-4.6在所有平台上已接近22%低点，加固收益极微。
4. **加固一个表面可能将攻击转移到另一个表面**：SeClaw使用结构化工具调用，关闭了非结构化文本泄漏通道，但要求模型正确填充模式字段，导致一些模型（如G-Pro）反而表现更差。
5. **效用-安全权衡**：SeClaw约一半的安全优势来自**移除功能**（无技能捆绑插件加载器、更窄的配置模式），而非主动防御。
6. **防御的全面部署不等于实际效果**：例如内存完整性检查（D5）和输入净化（D10）在设计上针对特定攻击，但在测试中几乎无效果（−1pp和−6pp），而技能内容审计（D4）的大幅下降主要源于架构移除。
7. **需要深度防御和联合评估**：没有任何单一层能解决所有问题，必须联合权衡。

## 7. 优点

- **创新视角**：将计算机系统数十年的安全原则系统应用于智能体安全，提供结构化分析框架（组件映射、原则-攻击面-防御对应），非简单枚举攻击。
- **生产级测试**：使用真实代理平台（OpenClaw等）作为测试床，而非自定义Python脚手架，结果直接反映部署行为。
- **任务生成原理驱动**：406个任务自上而下从5条原则聚类产生，覆盖了此前基准全遗漏的SSI、PSE、CDF三大攻击面，且可扩展。
- **自动化评估**：基于金丝雀的污点追踪，确定性、可复现、无LLM评判偏差。
- **防御覆盖分析**：定义了11种系统级防御并评估其在三个平台上的部署状态，首次量化了理论覆盖与实际效果之间的差距（如D5、D10效果很差）。
- **暴露结构性漏洞**：如恶意插件100%绕过LLM、日志文件泄漏等，为平台开发者提供明确改进方向。

## 8. 不足与局限

- **平台覆盖有限**：仅测试类爪形家族（OpenClaw、NemoClaw、SeClaw），未覆盖AutoGen、LangChain、Cursor等其他框架。虽然原则通用，但实际测试结果不直接迁移。
- **防御消融实验缺失**：定义了11种防御，但未实际实现并逐一消融测量其贡献（仅列出部署状态并间接比较SeClaw与OpenClaw的差异，难以区分主动防御与架构移除的效果）。
- **计算资源未报告**：未提及GPU型号、总量、推理成本，降低了可复现性。
- **LLM版本漂移**：供应商管理的模型权重随时间变化，影响长期复现。
- **任务覆盖非穷举**：24个子类别基于当前代理架构，未来可能出现新攻击面（如多智能体协调、模型侧持久内存），需扩展。
- **IPI范围有限**：仅假设用户明确要求智能体读取文档，未涵盖间接入口（如工具调用触发读取）。
- **外部效度**：指标基于金丝雀凭证，真实世界中攻击者可能使用更复杂技巧，基准可能高估或低估某些威胁。
- **无真实网络通信**：Google服务使用模拟器，无法完全代表真实跨边界行为的细微差异。
