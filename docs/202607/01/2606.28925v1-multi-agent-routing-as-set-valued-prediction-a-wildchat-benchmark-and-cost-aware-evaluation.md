## 摘要
从自然语言提示中进行工具和智能体路由本质上是一个集值预测问题：单个查询可能需要多个智能体，而过度选择会增加执行成本。这里引入的基准源自WildChat，包含在固定12个智能体目录上的3000个提示，具有在固定模式下的AI辅助启发式标签和用于多标签评估的受控重平衡。评估协议结合了集合级指标（精确率、召回率、F1、Jaccard和精确匹配）、延迟、面向执行的能力覆盖模拟，以及基于序数智能体成本层级的约束加权路由设置。比较的方法包括最近邻匹配、线性多标签分类、依赖感知基线、微调编码器、通过加权智能体路由（WAR）的确定性加权后评分，以及零样本LLM基线。结果表明，监督路由显著优于最近邻和零样本LLM路由。微调编码器在无约束集合准确率上表现最强，而线性多标签模型提供了最强的实用基线。在约束设置下，加权路由层在强监督评分器之上应用时提高了效用，其中Encoder+WAR的增益最大。总体而言，该基准和评估协议支持在固定目录多智能体路由中精度-成本权衡的可重复研究。

## Abstract
Tool and agent routing from natural-language prompts is naturally a set-valued prediction problem: a single query may require multiple agents, while over-selection increases execution cost. The benchmark introduced here is derived from WildChat and contains 3,000 prompts over a fixed 12-agent catalog, with AI-assisted heuristic labels under a fixed schema and controlled rebalancing for multi-label evaluation. The evaluation protocol combines set-level metrics (Precision, Recall, F1, Jaccard, and Exact Match), latency, an execution-oriented capability-coverage simulation, and a constrained weighted-routing setting based on ordinal agent-cost tiers. Compared methods include nearest-neighbor matching, linear multilabel classification, dependency-aware baselines, a fine-tuned encoder, deterministic weighted post-scoring via Weighted Agent Routing (WAR), and a zero-shot LLM baseline. Results show that supervised routers substantially outperform nearest-neighbor and zero-shot LLM routing. The fine-tuned encoder achieves the strongest unconstrained set accuracy, while the linear multilabel model provides the strongest practical baseline. In the constrained setting, the weighted routing layer improves utility when applied on top of strong supervised scorers, with the largest gain observed for Encoder+WAR. Overall, the benchmark and evaluation protocol support reproducible study of accuracy-cost trade-offs in fixed-catalog multi-agent routing.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 将路由问题建模为：给定输入提示 \(x\) ，输出预测集合 \(\hat{S}(x) \subseteq A\)（\(A\)为固定12个智能体目录），目标是最小化与真实集合\(G(x)\)的差异并控制执行成本。
- 评估分为**无约束**（仅优化集合重叠）和**约束**（考虑智能体执行成本层级）两种场景。

### 关键技术细节
1. **通用嵌入与智能体表示**：使用sentence-transformer编码器（MPNet骨干）将提示和智能体配置文本映射到向量空间，采用余弦相似度。
2. **路由方法家族**：
   - **最近邻（KNN）**：基于提示与智能体配置的余弦相似度排序。
   - **线性多标签分类（ML）**：每个智能体独立训练一个线性SVM（one-vs-rest），输出分数后阈值化得到预测集合。
   - **依赖感知基线**：Classifier Chains（CC）和ML-kNN，显式建模标签共现。
   - **微调编码器（Encoder）**：端到端微调sentence-transformer加上线性分类头，直接输出多标签概率。
   - **零样本LLM**：使用GPT-4o在固定目录约束下生成集合。
3. **加权智能体路由（WAR）**：一种确定性后处理层，对每个智能体分数施加成本惩罚：\(\tilde{s}_i(x) = s_i(x) - \lambda c(a_i)\)，其中\(c(a_i) \in \{1,2,3\}\)为序数成本层级。然后按相同阈值选择集合。
4. **集合构造规则**：阈值化 + Top-1回退（如果为空）。
5. **评估协议**：
   - 集合指标：精确率、召回率、F1、Jaccard、精确匹配（Exact Match）。
   - 执行模拟：成功率（能力完全覆盖）、覆盖率、成本、额外智能体数、效用（覆盖 - 0.1×成本 - 0.05×额外）。

## 3. 实验设计

### 数据集与基准
- **来源**：WildChat公开提示，筛选出与12个智能体相关的3000条提示。
- **标签生成**：AI辅助启发式标注，固定目录下生成参考集合；通过重平衡使每个智能体出现约375次，集合大小分布为1/2/3个agent = 1800/900/300。
- **划分**：2400训练、300开发、300测试（按集合大小分层）。
- **一致性验证**：计算提示相似性与标签集Jaccard的Pearson相关系数为0.517，说明标签系统有可学结构。

### 对比方法
- **无监督/轻量**：Majority（最频繁agent）、KNN（余弦相似度）
- **监督多标签**：线性ML（线性SVM）、依赖感知（CC、ML-kNN）
- **深度模型**：微调Encoder（MPNet + 线性头）
- **零样本LLM**：GPT-4o
- **成本感知**：ML+WAR、Encoder+WAR

### 超参数设置
- 阈值 \(t\) 从0.4到0.9每0.1步长在开发集上搜索，最终统一用 \(t=0.6\)。
- WAR惩罚 \(\lambda\) 搜索集合：0, 0.02, 0.05, 0.10, 0.15。
- 微调Encoder：3 epoch，batch size 8，学习率2e-5，权重衰减1e-2。

## 6. 论文的主要结论与发现

1. **微调Encoder在无约束设置下最优**：F1为89.59±0.33，Jaccard 85.52，Exact Match 73.11，领先所有其他方法。
2. **线性ML是强实用基线**：F1 71.79，效率高、易于重训练，适合生产部署。
3. **零样本LLM表现差**：F1仅41.51，远低于监督方法，说明固定目录路由需针对性训练。
4. **WAR提升成本效用**：ML+WAR将效用从0.465提升到0.480，Encoder+WAR从0.654提升到0.670，且Encoder+WAR的成功率从74.33%提高到92.11%，实现了更好的能力覆盖。
5. **依赖感知基线（CC、ML-kNN）处于中间水平**，但未超越线性ML，说明语义表示质量比标签依赖建模更重要。
6. **阈值选择关键**：在\(t=0.6\)处达到最优，过低或过高都会显著降低F1。
