## 摘要
开源AI基础设施（从模型服务引擎、智能体平台到模型上下文协议（MCP）生态系统以及语言模型本身）的快速增长，已经超过了可用于防御的安全工具的发展。我们提出了AI-Infra-Guard，这是一个开源框架，它围绕一个单一的观察结果组织AI红队测试：AI智能体的攻击面跨多个层级（基础设施、协议/工具、智能体行为和模型）分层，并且没有单一的检测范式适用于所有层级。因此，该框架为每个层级匹配一种范式，从涵盖75+个AI组件和1400+个漏洞规则的确定性规则匹配，到对MCP服务器和智能体技能包的LLM驱动的智能体审计，以及多轮黑盒智能体红队测试，再到包含16个数据集中26+个攻击操作的越狱工具集。据我们所知，这是唯一一个涵盖所有这些方面的开源框架，包括对日益扩展AI智能体的智能体技能的供应链审计。我们将AI-Infra-Guard作为开源发布，以便“层级-范式匹配”能够作为智能体安全的实用基础，并成为社区共同构建的基础。

## Abstract
The fast growth of open-source AI infrastructure, from model serving engines and agent platforms to the Model Context Protocol (MCP) ecosystem and the language models themselves, has outpaced the security tooling available to defend it. We present AI-Infra-Guard, an open-source framework that organizes AI red teaming around a single observation: the attack surface of an AI agent is stratified across layers (infrastructure, protocol/tool, agent behavior, and model), and no single detection paradigm fits all of them. The framework therefore matches a paradigm to each layer, from deterministic rule matching over 75+ AI components and 1{,}400+ vulnerability rules, through LLM-driven agentic auditing of MCP servers and agent-skill packages and multi-turn black-box agent red teaming, to a jailbreak harness with 26+ attack operators over sixteen datasets. To our knowledge it is the only open-source framework to span all of these, including supply-chain auditing of the agent skills that increasingly extend AI agents. We release AI-Infra-Guard as open source so that \emph{layer-paradigm matching} can serve as a practical foundation for agent security and a shared base for the community to build on.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：开源AI基础设施（如模型服务引擎Ollama/vLLM、智能体平台Dify/LangFlow、MCP服务器等）发展迅速，但现有安全工具严重滞后。这些组件具有“新”（未收录于传统指纹库）、“版本不规则”（如`b7824`、`2.3.dev`）、“威胁模型变迁”（重点从注入转向未认证访问、凭证泄露、提示注入、工具滥用、对齐侵蚀）三大特性，传统扫描器无法有效应对。
- **整体含义**：AI系统的攻击面是**分层的**（基础设施、协议/工具、智能体行为、模型），每种层级需要的证据类型不同，因此不能使用单一检测范式。需要一个统一框架，为每层匹配最合适的检测范式，实现全面的安全评估。论文提出的AI-Infra-Guard正是这种理念的实践，并开源供社区使用。

## 2. 方法论：核心思想、关键技术细节
### 核心思想：层-范式匹配（Layer-Paradigm Matching）
- 将AI智能体攻击面分为四层：**基础设施**、**协议/工具**、**智能体行为**、**模型**。
- 每层根据所需证据类型匹配检测范式：
  - **基础设施层** → 确定性规则匹配（签名证据）
  - **协议/工具层** → LLM驱动的智能体审计（语义证据）
  - **智能体行为层** → 多轮黑盒红队测试（行为证据）
  - **模型层** → 大规模越狱攻击枚举+LLM判别（统计证据）

### 关键技术细节
- **基础设施扫描（M1）**：手写解释器执行YAML规则，包括指纹规则（107条，覆盖75组件）和漏洞规则（1443条）。使用版本规范化处理不规则版本字符串。发现分三级：已验证（verified）、基于版本（version-based）、推断（inferred）。
- **MCP服务器审计（M2）**：LLM驱动的agentic harneess，提供工具（文件读写、shell执行等），采用reason-act循环和上下文压缩。支持静态白盒（源码）和动态黑盒（元数据+调用结果）两种模式。引入“Prompt-as-Rule”范式，将检测知识编码为自然语言条件，并明确排除条件。重点防御扫描器被自身分析的工件（间接提示注入）。
- **智能体技能审计（M7）**：扩展M2范式，专门审计技能包（SKILL.md、安装脚本等）。采用预处理→静态风险线索→AI审计→工具环境→分类报告的流水线。集成腾讯云威胁情报API。发布SkillTrustBench基准（5520个案例，9类威胁）。
- **AI智能体红队测试（M8）**：黑盒多轮对话。四个并行技能工人（数据泄露、工具滥用、间接注入、授权绕过），使用升级阶梯（direct→encoded→jailbreak）和停止规则控制成本。利用canary令牌和注入标记实现客观验证。
- **LLM越狱评估（M9）**：三角色（模拟器、目标、判别器），三成分（漏洞、攻击算子、度量）。攻击算子库含26+算子（单轮编码/伪装、多轮crescendo/tree-of-attacks等），16个数据集（约7248条提示）。使用LLM-as-judge判定成功。

## 6. 主要结论与发现
- AI-Infra-Guard是首个统一覆盖基础设施、协议/工具、智能体行为、模型四层的开源AI安全评估框架。
- 层-范式匹配原则有效：每层使用最适合的检测范式，避免了“一把锤子敲所有钉子”。
- 技能审计模块在SkillTrustBench上达到高F1分数（>0.98），且表现随基础模型提升而提升，说明框架设计与模型能力解耦良好。
- “Prompt-as-Rule”范式成功将安全知识编码为自然语言条件，并需要明确排除条件以防止过度报告。
- 扫描器本身需要防御间接提示注入（通过对提示和结构分离、路径沙箱等）。这是LLM安全工具的普遍需求。

## 7. 优点
