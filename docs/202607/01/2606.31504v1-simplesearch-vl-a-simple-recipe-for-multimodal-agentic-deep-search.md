## 摘要
我们提出了SimpleSearch-VL，一个高效、可靠、实用的多模态智能搜索框架。其核心思想是改进智能体自身的搜索与验证过程，而非扩展数据、工具或辅助模型组件。在高效性方面，因子化自适应展开（FAR）通过形成信息量更丰富的训练组来提高采样效率，同时利用冗余样本缓解长尾延迟并暴露困难样本。在可靠性方面，SimpleSearch-VL执行证据验证推理，明确使用思维链验证来评估检索到的视觉和文本线索与原始上下文的相关性。在实用性方面，SimpleSearch-VL保持轻量级工具接口，并在智能体内部执行网页自摘要，无需额外外部依赖。仅使用5K监督工具穿插轨迹和2K强化学习数据，SimpleSearch-VL在Qwen3-VL智能体基线基础上，8B和30B-A3B变体分别提升了15.8和16.0平均分。SimpleSearch-VL-30B-A3B模型进一步实现了与智能体Gemini-3-Pro竞争的性能。

## Abstract
We present SimpleSearch-VL, an efficient, reliable, and practical framework for multimodal agentic search. Its core idea is to improve the agent's own search-and-verification process rather than scaling data, tools, or auxiliary model components. For efficiency, Factorized Adaptive Rollout (FAR) improves sampling efficiency by forming more informative training groups while using redundant samples to mitigate long-tail latency and expose hard samples. For reliability, SimpleSearch-VL performs evidence-verified reasoning, explicitly using chain-of-thought verification to assess the relevance of retrieved visual and textual cues to the original context. For practicality, SimpleSearch-VL keeps a lightweight tool interface and performs webpage self-summary within the agent, requiring no additional external dependencies. With only 5K supervised tool-interleaved trajectories and 2K RL data, SimpleSearch-VL improves Qwen3-VL agentic baselines by 15.8 and 16.0 average points for the 8B and 30B-A3B variants, respectively. The SimpleSearch-VL-30B-A3B model further achieves performance competitive with agentic Gemini-3-Pro.

# SimpleSearch-VL 论文详细总结

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：多模态智能体搜索（multimodal agentic search）旨在让大模型主动获取外部视觉和文本证据以回答知识密集型视觉问题。然而，现有方法在三个方面面临挑战：
  - **效率低下**：多轮工具调用导致长尾延迟，固定采样预算无法自适应终止或重分配。
  - **可靠性不足**：检索到的视觉证据（如反向图像搜索的缩略图、标题、URL）可能看似合理但实际不匹配，缺乏显式的验证机制。
  - **实用性受限**：许多方法依赖复杂工具编排或外部摘要模型，增加了部署成本和系统依赖性。
- **整体含义**：SimpleSearch-VL 提出通过改进智能体自身的搜索与验证过程（而非扩展数据、工具或辅助模型组件）来同时提升效率、可靠性和实用性，从而以极低的训练成本实现最先进的性能。

## 2. 方法论

### 核心思想
- 将多模态搜索建模为马尔可夫决策过程，智能体交替进行推理、工具调用、证据验证和答案生成。
- 核心创新在于**因子化自适应展开（FAR）**、**证据验证推理**和**网页自摘要**。

### 关键技术细节
1. **因子化自适应展开（FAR）**
   - 将固定采样预算分解为两个维度：**提示扩展（Prompt Expansion）** 增加候选提示组池；**展开分配（Rollout Allocation）** 在同一展开深度下横向为各组分配额外尝试次数。
   - **信号评估**：使用答案正确性指标判断组内是否具有有用奖励变异（即既有正确也有错误的样本）。
   - **展开过滤**：一旦某组出现一个正确答案，后续扩展槽位被屏蔽（“仅全错时扩展”规则），避免冗余。
   - **停止规则**：累积足够的有效组或达到总槽位数量的99%即停止，避免等待慢速轨迹。
   - **策略更新**：使用 RLOO 估计器，基于完整组的留一法基线计算优势，仅对策略生成令牌应用梯度。

2. **证据验证推理**
   - **视觉证据验证**：反向图像搜索返回匹配的缩略图、网页标题和URL，智能体在利用前先比较缩略图与查询区域是否匹配，实现显式视觉验证。
   - **文本证据验证**：搜索结果为文本片段、标题和URL，智能体可进行页面级验证。
   - 在 SFT 数据构建中，保留原始工具调用和观察，但重写助手推理以暴露验证决策。

3. **网页自摘要**
   - 使用策略模型自身对访问的网页进行目标导向的条件化摘要，无需外部摘要模型。
   - 训练和推理时均使用同一模型，使摘要行为与智能体策略共同适应。

### 公式与算法流程（文字说明）
- 状态：`st = (Q, I, Ht)`，包含问题、输入图像和历史交互。
- 动作：每步生成 `<thinking>` 后输出一个工具调用或最终答案。
- 奖励：`r = 0.5 * 格式正确性 + 1 * 答案正确性`。
- FAR 的展开过滤：`αk(x) = 1(所有已观察展开均为错误)`，当为0时屏蔽后续扩展。
- 更新子集选择：优先在组内保留不同奖励、不同推理轮数和不同工具调用次数的轨迹。
- 优势计算：`A_i = (K/(K-1)) * (r_i - μ_x)`，其中 μ_x 为组平均奖励。

## 3. 实验设计

### 数据集与场景
- 在六个多模态搜索基准上评估：
  - **MMSearch**：多模态网页搜索
  - **MMSearch+**：更难的MMSearch版本
  - **BrowseComp-VL**：浏览式视觉问答
  - **FVQA**：基于事实的视觉问答
  - **LiveVQA**：实时视觉知识获取
  - **SimpleVQA**：多模态事实性评估
- 训练数据：5K 监督工具穿插轨迹（来自 Qwen3-VL-235B-A22B 生成经 Gemini 审查重写）+ 2K RL 数据。

### 对比方法
- 直接答案基线：Qwen3-VL-8B/30B-A3B、Gemini-3-Pro、Claude-Opus-4.6、Kimi-K2.5

## 6. 主要结论与发现

- **SimpleSearch-VL 以极小的训练代价达到了最先进性能**，验证了优化搜索验证过程（而非放大数据或模型）的有效性。
- **FAR 显著提升了训练效率和质量**：相比固定采样，FAR 在相近时间成本下提升平均分 6.2 分，信号率更快下降（表示收敛加速），且能恢复更多困难样本（恢复率从 11.7% 提升至 17.7%）。
- **证据验证推理是关键**：去除视觉缩略图导致 SFT 和 RL 模型平均下降约 3 分，说明显式验证不可或缺。
- **自摘要优于外部摘要**：自摘要平均提升 1.8-2.6 分，且部署效率更高（2 个智能体节点比 1+1 节点快 28.5%）。
- **轻量工具接口有效**：零样本对比中，SimpleSearch-VL 工具框架优于 Vision-DeepResearch 和 OpenSearch-VL 的框架。
- **SFT 提供强先验**：SFT 初始化后，RL 搜索行为更稳定（轮次变化小）；直接 RL 则更积极扩展图像搜索。

## 7. 优点

- **数据与计算高效**：仅需 5K SFT + 2K RL 数据，单机 8×H200 一天内完成训练，可复现性强。
- **方法简洁实用**：无需外部摘要模型、复杂工具编排；工具接口仅三个（文本搜索、图像搜索、页面访问）。
- **可靠性提升**：通过缩略图验证实现显式证据可信度检查，减轻幻觉和错误匹配。
- **自适应采样**：FAR 信号感知分配，避免冗余，加速收敛，同时暴露困难样本。
- **广泛的实验验证**：在 6 个基准上对比 10+ 方法，消融实验覆盖关键设计，结果一致且显著。
- **开源贡献**：提供代码库和训练流程，促进后续研究。

## 8. 不足与局限

- **失败模式分析**：论文列出两类失败——实体选择错误（如将通用创作图片误认为 Taylor Swift）和细粒度证据提取错误（正确识别艺术家但选错重叠歌曲）。表明方法在弱视觉线索或强干扰下仍有改进空间。
- **多图像场景覆盖有限**：SFT 中仅 7% 多图像样本，RL 中 20.2%，且全部来自 LiveVQA，可能削弱跨图像推理能力。
- **基准覆盖范围**：主要聚焦于视觉事实型问答，未涉及更开放或合成推理任务（如多跳长文档、多模态报告生成）。虽然论文提到了 Vision-DeepResearch 等对比，但未在所有基准上与其他方法完全对齐。
- **工具调用延迟**：虽然 FAR 缓解了长尾等待，但未对工具调用本身进行优化（如异步或缓存策略仅针对区域级别图像搜索），在实际高频调用场景下仍可能成为瓶颈。
- **部署考虑**：自摘要增加推理计算开销（每次页面访问需生成摘要），虽然论文显示效率尚可，但大规模页面访问时可能累积成本。
- **泛化性**：基座模型固定为 Qwen3-VL，未验证在其他 MLLM 上的可迁移性。训练数据偏重英语，跨语言或跨领域搜索能力未知。
- **消融设置**：主要消融在 8B 模型和 100 步 RL 上进行，30B 结果的消融较少，存在规模泛化风险。
